# LeetCode 347: Top K Frequent Elements 記憶體分析

## 問題概述
給定整數陣列 `nums` 和整數 `k`，回傳出現頻率最高的 `k` 個元素。

**範例**: nums = [1,1,1,2,2,3], k = 2 → 輸出 = [1,2]

---

## 1. Struct 記憶體佈局

此解法不使用單一 struct，而是使用多個獨立陣列。以下展示各陣列的記憶體配置：

| 陣列名稱 | 類型 | 大小 | 起始地址 | 說明 |
|---------|------|------|---------|------|
| `frequencies` | int[20001] | 80004 bytes | 0xA000 | 頻率計數陣列，索引 = 值 + 10000 |
| `uniqueValues` | int[numsSize] | 4*numsSize | 0xB000 | 存儲不同的值 |
| `bucketSizes` | int[numsSize+1] | 4*(numsSize+1) | 0xC000 | 各頻率的值個數 |
| `buckets` | int*[numsSize+1] | 8*(numsSize+1) | 0xD000 | 指向各頻率桶陣列的指針 |
| `bucketWriteIndex` | int[numsSize+1] | 4*(numsSize+1) | 0xE000 | 各桶的寫入游標 |
| `answer` | int[k] | 4*k | 0xF000 | 結果陣列 |

## 3. 陣列索引版對應

### 3.1 frequencies 陣列索引對應

```
frequencies[i] = 值 v 在 nums 中出現的次數

索引對應：
  frequencies[i] ← count of value (i - 10000)

範例：
  i = 10001 → 值 1 的頻率
  i = 10002 → 值 2 的頻率
  i = 10003 → 值 3 的頻率

OFFSET = 10000 (用於處理負數)
```

### 3.2 buckets 與 bucketSizes 的對應

```
bucketSizes[freq] = 頻率為 freq 的不同值個數
buckets[freq]     = 指向頻率為 freq 的值陣列的指針

例：
  bucketSizes[1] = 2  (表示頻率=1的值有2個)
  buckets[1]     = 指向陣列 [1, 3, ...] (頻率=1的所有值)

  bucketSizes[2] = 1  (表示頻率=2的值有1個)
  buckets[2]     = 指向陣列 [2, ...] (頻率=2的所有值)

  bucketSizes[3] = 1  (表示頻率=3的值有1個)
  buckets[3]     = 指向陣列 [...] (頻率=3的所有值)
```

## 4. 搜尋／操作過程追蹤

### 範例：nums = [1,1,1,2,2,3], k = 2

#### **第1階段：頻率計數**

```
初始狀態：
frequencies 全部 = 0

步驟1：處理 nums[0]=1
  frequencies[10001]++ → 1
  狀態：frequencies[10001] = 1

步驟2：處理 nums[1]=1
  frequencies[10001]++ → 2
  狀態：frequencies[10001] = 2

步驟3：處理 nums[2]=1
  frequencies[10001]++ → 3
  狀態：frequencies[10001] = 3

步驟4：處理 nums[3]=2
  frequencies[10002]++ → 1
  狀態：frequencies[10001] = 3, frequencies[10002] = 1

步驟5：處理 nums[4]=2
  frequencies[10002]++ → 2
  狀態：frequencies[10001] = 3, frequencies[10002] = 2

步驟6：處理 nums[5]=3
  frequencies[10003]++ → 1
  狀態：frequencies[10001] = 3, frequencies[10002] = 2, frequencies[10003] = 1

頻率計數完成：
┌─────────────────────────────────┐
│ frequencies @ 0xA000            │
│ [10001:3] [10002:2] [10003:1]  │
└─────────────────────────────────┘
```

#### **第2階段：收集唯一值**

```
遍歷 frequencies，找出所有非零項：

步驟1：檢查 frequencies[10001]=3 → 非零
  uniqueValues[0] = 1
  偏移量計算：10001 - 10000 = 1 ✓

步驟2：檢查 frequencies[10002]=2 → 非零
  uniqueValues[1] = 2
  偏移量計算：10002 - 10000 = 2 ✓

步驟3：檢查 frequencies[10003]=1 → 非零
  uniqueValues[2] = 3
  偏移量計算：10003 - 10000 = 3 ✓

唯一值已收集，nuniqueCount = 3：
┌─────────────────────────────────┐
│ uniqueValues[6] @ 0xB000        │
│ [0:1] [1:2] [2:3] [3:?] ...    │
└─────────────────────────────────┘
```

#### **第3階段：計算桶大小**

```
遍歷 uniqueValues，計算每個頻率的值個數：

步驟1：uniqueValues[0]=1，頻率=frequencies[10001]=3
  bucketSizes[3]++ → 1

步驟2：uniqueValues[1]=2，頻率=frequencies[10002]=2
  bucketSizes[2]++ → 1

步驟3：uniqueValues[2]=3，頻率=frequencies[10003]=1
  bucketSizes[1]++ → 1

桶大小計算完成：
┌──────────────────────────────┐
│ bucketSizes[7] @ 0xC000      │
│ [0:0] [1:1] [2:1] [3:1]      │
│       [4:0] [5:0] [6:0]      │
└──────────────────────────────┘

說明：
  bucketSizes[1] = 1 → 頻率1有1個值(3)
  bucketSizes[2] = 1 → 頻率2有1個值(2)
  bucketSizes[3] = 1 → 頻率3有1個值(1)
```

#### **第4階段：分配桶陣列**

```
為每個非零的 bucketSizes[freq] 分配陣列：

步驟1：bucketSizes[1]=1 → malloc(1*sizeof(int))
  buckets[1] = 0x10000

步驟2：bucketSizes[2]=1 → malloc(1*sizeof(int))
  buckets[2] = 0x11000

步驟3：bucketSizes[3]=1 → malloc(1*sizeof(int))
  buckets[3] = 0x12000

記憶體分配狀態：
┌──────────────────────────────────────┐
│ buckets[7] @ 0xD000                  │
│ [0:NULL] [1:0x10000] [2:0x11000]     │
│          [3:0x12000] [4:NULL] ...    │
└──────────────────────────────────────┘

┌─────────────┬─────────────┬─────────────┐
│ @ 0x10000   │ @ 0x11000   │ @ 0x12000   │
│ freq=1桶    │ freq=2桶    │ freq=3桶    │
│ [0:0]       │ [0:0]       │ [0:0]       │
└─────────────┴─────────────┴─────────────┘
(均為空，待填充)
```

#### **第5階段：填充桶陣列**

```
遍歷 uniqueValues，根據頻率放入對應桶：

步驟1：uniqueValues[0]=1，頻率=3
  buckets[3][bucketWriteIndex[3]++] = 1
  buckets[3][0] = 1
  bucketWriteIndex[3] = 1

步驟2：uniqueValues[1]=2，頻率=2
  buckets[2][bucketWriteIndex[2]++] = 2
  buckets[2][0] = 2
  bucketWriteIndex[2] = 1

步驟3：uniqueValues[2]=3，頻率=1
  buckets[1][bucketWriteIndex[1]++] = 3
  buckets[1][0] = 3
  bucketWriteIndex[1] = 1

填充完成後：
┌─────────────┬─────────────┬─────────────┐
│ @ 0x10000   │ @ 0x11000   │ @ 0x12000   │
│ freq=1桶    │ freq=2桶    │ freq=3桶    │
│ [0:3]       │ [0:2]       │ [0:1]       │
└─────────────┴─────────────┴─────────────┘

writeIndex狀態：
┌────────────────────────────────┐
│ bucketWriteIndex[7] @ 0xE000   │
│ [0:0] [1:1] [2:1] [3:1] ...   │
└────────────────────────────────┘
```

#### **第6階段：從高頻率掃描收集答案**

```
倒序掃描 buckets，收集前 k=2 個值：

初始狀態：answer[2]，answerIndex=0，count=0

步驟1：freq=6 → bucketSizes[6]=0 → 跳過

...（bucketSizes[4,5,6,7]=0，全部跳過）

步驟2：freq=3 → bucketSizes[3]=1
  遍歷 buckets[3]：
    buckets[3][0] = 1
    answer[0] = 1
    answerIndex = 1
    count = 1

步驟3：freq=2 → bucketSizes[2]=1，count < k
  遍歷 buckets[2]：
    buckets[2][0] = 2
    answer[1] = 2
    answerIndex = 2
    count = 2

步驟4：count==k → 停止掃描

最終答案：
┌──────────────────────────────┐
│ answer[2] @ 0xF000           │
│ [0:1] [1:2]                  │
└──────────────────────────────┘

結果：[1, 2] ✓
```

## 6. 記憶體存取模式比較

### 6.1 Bucket Sort 方法（本解法）

```
記憶體存取模式：
  階段1：frequencies[]
    ├─ 順序寫：O(numsSize)  → L1 快取友善
    ├─ 密集存取：每個位置最多訪問一次
    └─ 空間開銷：RANGE*sizeof(int) = 80KB

  階段2：frequencies[] + uniqueValues[]
    ├─ 隨機讀：O(RANGE) → L2 快取失誤
    ├─ 順序寫：O(nuniqueCount)
    └─ 通過 OFFSET 轉換：CPU 計算開銷小

  階段3-5：buckets[] 和各桶
    ├─ buckets[] 是指針陣列：8字節對齊
    ├─ 各桶內順序訪問：L1 快取友善
    ├─ 寫入順序：由 bucketWriteIndex[] 控制
    └─ 掃描順序：倒序遍歷 frequencies

快取行為：
  ✓ Phase 1-2：主要是順序和半順序存取
  ✗ Phase 2 中 frequencies[] 的隨機讀（RANGE 很大）
  ✓ Phase 3-5：各桶的順序讀寫很快

空間複雜度：O(n + k + RANGE)
  - frequencies：80KB（常數）
  - uniqueValues：4*n
  - buckets + bucketWriteIndex + bucketSizes：4*(n+1) * 2
  - answer：4*k
```

### 6.2 最大堆方法

```
記憶體存取模式：
  構建堆：O(n) 插入，每次 O(log n) 調整
    ├─ 隨機訪問子節點和父節點
    ├─ 分支預測困難
    └─ L1 快取失誤率高

  彈出 k 個：O(k log n)
    ├─ 重複隨機訪問堆結構
    └─ 快取局部性差

快取行為：
  ✗ 樹結構的隨機訪問
  ✗ 分支預測失敗
  ✗ 偽共享（若多線程）

空間複雜度：O(n)
  - 堆陣列：4*n
  - 其他：常數

比較：
  時間：Bucket Sort O(n+k) vs Heap O(n log n)
  快取：Bucket Sort 更友善（相對）
```

### 6.3 快速選擇方法

```
記憶體存取模式：
  分割：O(n) → 隨機訪問和交換
    ├─ 原地操作，cache 局部性中等
    ├─ 指針運動：前後相向移動
    └─ 可預測性差

遞迴：O(log n) 深度
  ├─ 棧空間：O(log n)
  └─ 呼叫開銷

快取行為：
  ◐ 優於堆，劣於 Bucket Sort
  ◐ 原地操作減少記憶體分配

空間複雜度：O(log n)
  - 遞迴棧：O(log n)
  - 原地修改：不額外分配

比較：
  時間：均勻 O(n)，最差 O(n²)
  空間：最少 O(log n)
  快取：中等
```

### 6.4 方法對比表

| 特性 | Bucket Sort | 最大堆 | 快速選擇 |
|-----|-----------|--------|---------|
| 時間複雜度 | O(n+k) | O(n log n) | O(n) avg |
| 空間複雜度 | O(n+RANGE) | O(n) | O(log n) |
| 快取友善度 | ⭐⭐⭐⭐ | ⭐⭐ | ⭐⭐⭐ |
| 記憶體分配次數 | O(n) | O(n) | 0 |
| 分支預測 | 好 | 差 | 中 |
| 適合 k 很小時 | ⭐⭐⭐⭐ | ✗ | ⭐⭐⭐ |
| 適合 n 很大時 | ⭐⭐⭐ | ✗ | ⭐⭐⭐⭐ |
| 穩定性 | 穩定 | 不穩定 | 不穩定 |

### 6.5 實際存取追蹤（Bucket Sort）

```
L1/L2 快取行為 (假設 L1=32KB, L2=256KB, line=64B):

Phase 1 (頻率計數)：
  存取：frequencies[10001], frequencies[10002], ...
  模式：相鄰位置批量訪問 → L1 命中率高 ✓
  缺失：第一次存取 → 填充 L1
  計數：6 次讀寫 → 1-2 個快取行 → 95% 命中率

Phase 2 (掃描 frequencies 收集唯一值)：
  存取：frequencies[0..20000]
  模式：順序掃描 ✓
  缺失：每個快取行約 16 個值
  計數：20001 次讀 → 1250 個快取行 → 掃描友善

Phase 3-5 (桶操作)：
  存取：buckets[], bucketSizes[], bucketWriteIndex[]
  模式：隨機讀 buckets，順序寫到各桶
  缺失：buckets[] 指針解引用 → L2 到 L1
  計數：多次重定向 → 平均 50-70% 命中率

Phase 6 (倒序掃描)：
  存取：buckets[k..1]，各桶順序讀
  模式：順序讀 → L1 命中率高 ✓
  計數：k 次迭代 × 桶大小 → 85% 命中率
```

## 總結

**Bucket Sort + Frequency Counting** 算法在記憶體操作上的優勢：

1. **時間複雜度**：O(n+k)，在 k 遠小於 n 時最優
2. **快取友善**：Phase 1 和 6 的順序訪問命中率高
3. **預測性好**：迴圈控制簡單，分支預測準確率高
4. **記憶體足跡**：O(n + RANGE)，RANGE 為常數可接受
5. **實現簡潔**：無指針運動或複雜的樹操作

**劣勢**：
- RANGE 固定為 20001，空間開銷固定
- Phase 2 隨機掃描 frequencies 有快取失誤
- 不適用於動態範圍的值
