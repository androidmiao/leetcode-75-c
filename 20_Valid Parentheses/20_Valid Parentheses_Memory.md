# LeetCode 20 Valid Parentheses 記憶體分析

## 1. 堆疊記憶體佈局

### 記憶體區域配置

在 C 語言中，使用 `char* stack = malloc(len * sizeof(char))` 分配的堆疊具有以下記憶體佈局：

```
程式記憶體空間：

┌─────────────────────────────────────────────────────┐
│  棧區 (Stack)                                        │
├─────────────────────────────────────────────────────┤
│  [ESP-4]  top (int)                 ─────▶ 0xFFFC  │
│  [ESP-8]  stack (char*)             ─────▶ 0xFFF8  │
│  [ESP-12] s (char*)                 ─────▶ 0xFFF4  │
└─────────────────────────────────────────────────────┘
                       │
                       │ malloc() 返回指標
                       ▼
┌─────────────────────────────────────────────────────┐
│  堆區 (Heap)                                         │
├─────────────────────────────────────────────────────┤
│  stack[0]  ('(' or '{' etc.)        ─────▶ 0x2000  │
│  stack[1]  ('(' or '{' etc.)        ─────▶ 0x2001  │
│  stack[2]  ...                      ─────▶ 0x2002  │
│  ...                                                 │
│  stack[n-1]                         ─────▶ 0x2000+n-1
└─────────────────────────────────────────────────────┘
```

### 詳細變數位置

```
程式棧區段（假設 ESP = 0xFFFF）：

位址        │ 內容              │ 大小    │ 說明
────────────┼──────────────────┼─────────┼──────────────────────
0xFFF4      │ 0x3000 (char*)   │ 8 bytes │ s (輸入字串指標)
0xFFFC      │ 0x2000 (char*)   │ 8 bytes │ stack (堆疊指標)
0x10004     │ -1 (int)         │ 4 bytes │ top (棧頂索引)
────────────┴──────────────────┴─────────┴──────────────────────

堆記憶體區段（假設堆起點 = 0x2000）：

位址        │ 內容              │ 說明
────────────┼──────────────────┼──────────────────────────────
0x2000      │ '\0' (未初始化)  │ stack[0] - 容量 len 的陣列
0x2001      │ '\0'             │ stack[1]
0x2002      │ '\0'             │ stack[2]
...         │ ...              │ ...
0x2000+n-1  │ '\0'             │ stack[n-1]
```

### char 陣列在記憶體中的儲存方式

每個 `char` 占用 **1 個位元組**，連續存儲：

```
堆疊陣列記憶體分佈（輸入 s = "{[()]}" 時推送過程）：

初始狀態（top = -1）：
┌────────┐
│ 0x2000 │ ' ' (未使用)
├────────┤
│ 0x2001 │ ' ' (未使用)
├────────┤
│ 0x2002 │ ' ' (未使用)
├────────┤
│ 0x2003 │ ' ' (未使用)
├────────┤
│ 0x2004 │ ' ' (未使用)
├────────┤
│ 0x2005 │ ' ' (未使用)
└────────┘
  top=-1 (棧空)
```

## 3. 搜尋／操作過程追蹤

### 範例 1：s = "{[()]}" （有效，返回 true）

#### 詳細執行步驟

```
輸入字串：s = "{[()]}"
字串長度：len = 6
堆疊容量：len = 6 (分配 6 個 char)
初始 top = -1

─────────────────────────────────────────────────────────

步驟 1：處理 s[0] = '{'

  操作前狀態：
  ┌─ top = -1
  │
  └─ stack 為空

  當前字元：'{' (開括號)
  條件檢查：s[0] == '{' ─▶ 是開括號

  動作：push('{')
  ├─ top++  (top: -1 ─▶ 0)
  └─ stack[0] = '{' (記憶體位置 0x2000)

  操作後狀態：
  ┌─ top = 0
  └─ stack[0] = '{'  (位址 0x2000)

─────────────────────────────────────────────────────────

步驟 2：處理 s[1] = '['

  操作前狀態：
  ┌─ top = 0
  └─ stack[0] = '{'  (位址 0x2000)

  當前字元：'[' (開括號)
  條件檢查：s[1] == '[' ─▶ 是開括號

  動作：push('[')
  ├─ top++  (top: 0 ─▶ 1)
  └─ stack[1] = '[' (記憶體位置 0x2001)

  操作後狀態：
  ┌─ top = 1
  ├─ stack[0] = '{'  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)

─────────────────────────────────────────────────────────

步驟 3：處理 s[2] = '('

  操作前狀態：
  ┌─ top = 1
  ├─ stack[0] = '{'  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)

  當前字元：'(' (開括號)
  條件檢查：s[2] == '(' ─▶ 是開括號

  動作：push('(')
  ├─ top++  (top: 1 ─▶ 2)
  └─ stack[2] = '(' (記憶體位置 0x2002)

  操作後狀態：
  ┌─ top = 2
  ├─ stack[0] = '{'  (位址 0x2000)
  ├─ stack[1] = '['  (位址 0x2001)
  └─ stack[2] = '('  (位址 0x2002)

─────────────────────────────────────────────────────────

步驟 4：處理 s[3] = ')'

  操作前狀態：
  ┌─ top = 2
  ├─ stack[0] = '{'  (位址 0x2000)
  ├─ stack[1] = '['  (位址 0x2001)
  └─ stack[2] = '('  (位址 0x2002)

  當前字元：')' (閉括號)
  棧空檢查：top >= 0 ─▶ true (棧非空)

  條件檢查：s[3] == ')' ─▶ 是閉括號

  動作：檢查匹配
  ├─ 讀取 stack[top] = stack[2]
  │  讀取位置 0x2002 ─▶ '('
  │
  ├─ 比較：stack[2] == '(' 且 s[3] == ')'
  │  檢查結果：'(' ◄──▶ ')' ─▶ 配對成功 ✓
  │
  └─ pop()
     └─ top--  (top: 2 ─▶ 1)

  操作後狀態：
  ┌─ top = 1
  ├─ stack[0] = '{'  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)
     (stack[2] 已丟棄)

─────────────────────────────────────────────────────────

步驟 5：處理 s[4] = ']'

  操作前狀態：
  ┌─ top = 1
  ├─ stack[0] = '{'  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)

  當前字元：']' (閉括號)
  棧空檢查：top >= 0 ─▶ true (棧非空)

  條件檢查：s[4] == ']' ─▶ 是閉括號

  動作：檢查匹配
  ├─ 讀取 stack[top] = stack[1]
  │  讀取位置 0x2001 ─▶ '['
  │
  ├─ 比較：stack[1] == '[' 且 s[4] == ']'
  │  檢查結果：'[' ◄──▶ ']' ─▶ 配對成功 ✓
  │
  └─ pop()
     └─ top--  (top: 1 ─▶ 0)

  操作後狀態：
  ┌─ top = 0
  └─ stack[0] = '{'  (位址 0x2000)
     (stack[1] 已丟棄)

─────────────────────────────────────────────────────────

步驟 6：處理 s[5] = '}'

  操作前狀態：
  ┌─ top = 0
  └─ stack[0] = '{'  (位址 0x2000)

  當前字元：'}' (閉括號)
  棧空檢查：top >= 0 ─▶ true (棧非空)

  條件檢查：s[5] == '}' ─▶ 是閉括號

  動作：檢查匹配
  ├─ 讀取 stack[top] = stack[0]
  │  讀取位置 0x2000 ─▶ '{'
  │
  ├─ 比較：stack[0] == '{' 且 s[5] == '}'
  │  檢查結果：'{' ◄──▶ '}' ─▶ 配對成功 ✓
  │
  └─ pop()
     └─ top--  (top: 0 ─▶ -1)

  操作後狀態：
  ┌─ top = -1
  └─ stack 為空（所有元素已彈出）

─────────────────────────────────────────────────────────

步驟 7：最終檢查

  當前狀態：
  ├─ 已遍歷完所有字元 (i == 6)
  ├─ 棧為空 (top == -1)
  └─ 所有匹配都成功

  終止條件滿足

─────────────────────────────────────────────────────────

結論：返回 true ✓

理由：
  1. 所有開括號都找到了對應的閉括號
  2. 配對順序正確（最後開的最先關）
  3. 最終棧為空
```

### 範例 2：s = "([)]" （無效，返回 false）

#### 詳細執行步驟

```
輸入字串：s = "([)]"
字串長度：len = 4
堆疊容量：len = 4 (分配 4 個 char)
初始 top = -1

─────────────────────────────────────────────────────────

步驟 1：處理 s[0] = '('

  操作前狀態：
  ┌─ top = -1
  │
  └─ stack 為空

  當前字元：'(' (開括號)
  條件檢查：s[0] == '(' ─▶ 是開括號

  動作：push('(')
  ├─ top++  (top: -1 ─▶ 0)
  └─ stack[0] = '(' (記憶體位置 0x2000)

  操作後狀態：
  ┌─ top = 0
  └─ stack[0] = '('  (位址 0x2000)

─────────────────────────────────────────────────────────

步驟 2：處理 s[1] = '['

  操作前狀態：
  ┌─ top = 0
  └─ stack[0] = '('  (位址 0x2000)

  當前字元：'[' (開括號)
  條件檢查：s[1] == '[' ─▶ 是開括號

  動作：push('[')
  ├─ top++  (top: 0 ─▶ 1)
  └─ stack[1] = '[' (記憶體位置 0x2001)

  操作後狀態：
  ┌─ top = 1
  ├─ stack[0] = '('  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)

─────────────────────────────────────────────────────────

步驟 3：處理 s[2] = ')'

  操作前狀態：
  ┌─ top = 1
  ├─ stack[0] = '('  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)

  當前字元：')' (閉括號)
  棧空檢查：top >= 0 ─▶ true (棧非空)

  條件檢查：s[2] == ')' ─▶ 是閉括號

  動作：檢查匹配
  ├─ 讀取 stack[top] = stack[1]
  │  讀取位置 0x2001 ─▶ '['
  │
  ├─ 比較：stack[1] == '[' 且 s[2] == ')'
  │  檢查結果：'[' ◄──X──▶ ')'
  │                    不匹配！
  │
  └─ 配對失敗 ✗

  結論：立即返回 false ✗

  原因分析：
    棧頂元素是 '['，但當前字元是 ')'
    根據括號匹配規則：
      '[' 的配對字元應該是 ']'
      '(' 的配對字元應該是 ')'
    這裡出現了交叉配對 ([)] 的情況
    這在數學上被認為是無效的序列

─────────────────────────────────────────────────────────

最終狀態（未完成遍歷）：

  ┌─ top = 1
  ├─ stack[0] = '('  (位址 0x2000)
  └─ stack[1] = '['  (位址 0x2001)

  未處理的字元：s[3] = ']'

─────────────────────────────────────────────────────────

結論：返回 false ✗

理由：
  1. 在 s[2] = ')' 時檢測到配對錯誤
  2. 棧頂是 '['，但遇到 ')'
  3. 根據有效括號序列規則，這種交叉配對是無效的
  4. 算法立即返回 false，不再繼續處理後續字元
```

## 6. 堆疊記憶體對比

### 迭代方案（當前解決方案）

#### 迭代堆疊特性

```
實現方法：

bool isValid(char* s) {
    int len = strlen(s);
    char* stack = (char*)malloc(len * sizeof(char));
    int top = -1;

    for (int i = 0; i < len; i++) {
        char c = s[i];

        if (c == '(' || c == '[' || c == '{') {
            // push
            stack[++top] = c;
        } else {
            // pop and check
            if (top < 0) return false;
            char open = stack[top--];
            if (!isMatch(open, c)) return false;
        }
    }

    free(stack);
    return top == -1;
}
```

## 附錄：C 語言實現細節

### 完整程式碼註解版本

```c
#include <stdbool.h>
#include <string.h>
#include <stdlib.h>

// 檢查括號是否配對的輔助函數
static bool isMatch(char open, char close) {
    return (open == '(' && close == ')') ||
           (open == '[' && close == ']') ||
           (open == '{' && close == '}');
}

// 主函數
bool isValid(char* s) {
    // 步驟 1：計算字串長度並分配堆疊
    //         分配位置：堆記憶體 (0x2000)
    int len = strlen(s);                    // s 位址：0x3000
    char* stack = (char*)malloc(len * sizeof(char));  // 返回 0x2000
    int top = -1;                           // 棧頂索引：-1 (空)

    // 步驟 2：遍歷每個字元
    for (int i = 0; i < len; i++) {         // i 存儲位置：棧幀
        char c = s[i];                      // c 讀取來自：s[i] (0x3000 + i)

        // 步驟 3：開括號推送
        if (c == '(' || c == '[' || c == '{') {
            stack[++top] = c;               // 遞增 top，寫入 stack[top]
                                           // 寫位置：0x2000 + top
        } else {
            // 步驟 4：閉括號彈出並檢查
            if (top < 0) {                  // 棧空檢查
                free(stack);
                return false;               // 多餘的閉括號
            }

            char open = stack[top--];       // 讀 stack[top]，遞減 top
                                           // 讀位置：0x2000 + top

            if (!isMatch(open, c)) {        // 配對檢查
                free(stack);
                return false;               // 配對失敗
            }
        }
    }

    // 步驟 5：最終檢查棧是否為空
    bool result = (top == -1);              // 讀 top 位置：0x10004
    free(stack);                            // 釋放堆記憶體
    return result;                          // 返回檢查結果
}

// 記憶體存取序列統計：
// ├─ 字串讀取：len 次（順序存取，位址 0x3000 - 0x3000+len）
// ├─ top 讀取：len + 1 次（頻繁，位址 0x10004）
// ├─ top 寫入：len 次（頻繁，位址 0x10004）
// ├─ stack 讀取：~len/2 次（隨機，位址 0x2000 - 0x2000+len）
// ├─ stack 寫入：~len/2 次（隨機，位址 0x2000 - 0x2000+len）
// └─ 總計：約 4*len 次記憶體操作
```

### 記憶體複雜度證明

```
空間複雜度分析：

1. 輸入字串：O(n) - 由呼叫者提供（不計入）

2. 堆疊陣列：O(n)
   ├─ 分配：char* stack = malloc(n * sizeof(char))
   ├─ 大小：n bytes (最壞情況：所有開括號)
   └─ 位置：0x2000

3. 局部變數：O(1)
   ├─ stack (char*):  8 bytes
   ├─ top (int):      4 bytes
   ├─ len (int):      4 bytes
   ├─ i (int):        4 bytes
   ├─ c (char):       1 byte + 3 padding
   └─ 小計：          ~24 bytes

4. 總空間：O(n) + O(1) = O(n)

主導項：O(n) 來自堆疊陣列分配

最壞情況輸入：
├─ "(((((...(((("  (n 個開括號)
├─ 堆疊需要存儲所有 n 個開括號
└─ 空間複雜度達到 O(n) 上界

最優情況輸入：
├─ "()"  (n=2)
├─ "()()()..."  (交替配對)
├─ 堆疊最多存儲 1 個元素
└─ 實際空間：O(2) = O(1)（但漸進仍為 O(n)）

平均情況輸入：
├─ 隨機混合的括號
├─ 堆疊深度：O(sqrt(n)) 至 O(n)
└─ 漸進仍為 O(n)（由於最壞情況支配）
```
