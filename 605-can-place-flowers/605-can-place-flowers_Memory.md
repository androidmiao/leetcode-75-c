# LeetCode 605 - Can Place Flowers 記憶體分析

## 1. 結構記憶體佈局

LeetCode 605 使用平面陣列 `int* flowerbed`，不涉及 struct。

**基本型別:**
- `int` 大小：4 bytes（32 位元）
- 陣列元素：連續排列在記憶體中

**記憶體佈局原理:**
```
int flowerbed[n] ┌─────────────────────────┐
                 │ flowerbed[0] │ flowerbed[1] │ ... │ flowerbed[n-1] │
                 └─────────────────────────┘
                 ◄─ 每個元素 4 bytes ──▶
```

---

## 2. 完整資料結構記憶體配置

**使用範例:** `flowerbed = [1, 0, 0, 0, 1]`（5 個元素）

**假設基底位址:** `0xA000`（假設地址）

### 元素-地址映射表

| 索引 | 值  | 地址    | 16進制位移 |
|------|-----|---------|-----------|
| 0    | 1   | 0xA000  | +0        |
| 1    | 0   | 0xA004  | +4        |
| 2    | 0   | 0xA008  | +8        |
| 3    | 0   | 0xA00C  | +12       |
| 4    | 1   | 0xA010  | +16       |


**通用地址公式:**

## 4. 搜尋／操作過程追蹤

### 演算法邏輯

```c
bool canPlaceFlowers(int* flowerbed, int flowerbedSize, int n) {
    for (int i = 0; i < flowerbedSize; i++) {
        if (*(flowerbed + i) == 0) {  // 當前位置為空
            int emptyLeft = (i == 0) || (*(flowerbed + i - 1) == 0);
            int emptyRight = (i == flowerbedSize - 1) || (*(flowerbed + i + 1) == 0);

            if (emptyLeft && emptyRight) {
                *(flowerbed + i) = 1;  // 種植花卉
                n--;
            }
        }
    }
    return n <= 0;
}
```

### 例1: `flowerbed = [1,0,0,0,1], n = 1` → **true**

**初始狀態:**
```
┌────┬────┬────┬────┬────┐
│ 1  │ 0  │ 0  │ 0  │ 1  │
└────┴────┴────┴────┴────┘
  i=0 i=1 i=2 i=3 i=4
```

**Step 1: i = 0**
```
目前指標位置：
     ▼
┌────┬────┬────┬────┬────┐
│ 1  │ 0  │ 0  │ 0  │ 1  │
└────┴────┴────┴────┴────┘

prev = NULL (i == 0)
curr = *(0xA000) = 1
next = *(0xA004) = 0

判斷: *(flowerbed + 0) == 0? → 1 == 0? → False
動作: 跳過，不處理
n = 1 (不變)
```

**Step 2: i = 1**
```
           ▼
┌────┬────┬────┬────┬────┐
│ 1  │ 0  │ 0  │ 0  │ 1  │
└────┴────┴────┴────┴────┘

prev = *(0xA000) = 1
curr = *(0xA004) = 0
next = *(0xA008) = 0

判斷: *(flowerbed + 1) == 0? → 0 == 0? → True
emptyLeft = (i == 0) || (*(flowerbed + 0) == 0)? → False || (1 == 0)? → False
emptyRight = (i == 4) || (*(flowerbed + 2) == 0)? → False || (0 == 0)? → True

條件: emptyLeft && emptyRight? → False && True? → False
動作: 不種植
n = 1 (不變)
```

**Step 3: i = 2**
```
                ▼
┌────┬────┬────┬────┬────┐
│ 1  │ 0  │ 0  │ 0  │ 1  │
└────┴────┴────┴────┴────┘

prev = *(0xA008 - 4) = *(0xA004) = 0
curr = *(0xA008) = 0
next = *(0xA00C) = 0

判斷: *(flowerbed + 2) == 0? → 0 == 0? → True
emptyLeft = (i == 0) || (*(flowerbed + 1) == 0)? → False || (0 == 0)? → True
emptyRight = (i == 4) || (*(flowerbed + 3) == 0)? → False || (0 == 0)? → True

條件: emptyLeft && emptyRight? → True && True? → True
動作: 種植花卉！*(flowerbed + 2) = 1
陣列變為: [1, 0, 1, 0, 1]
n-- → n = 0
```

**Step 4: i = 3**
```
                     ▼
┌────┬────┬────┬────┬────┐
│ 1  │ 0  │ 1  │ 0  │ 1  │ (已更新)
└────┴────┴────┴────┴────┘

prev = *(0xA00C - 4) = *(0xA008) = 1 (已更新)
curr = *(0xA00C) = 0
next = *(0xA010) = 1

判斷: *(flowerbed + 3) == 0? → 0 == 0? → True
emptyLeft = (i == 0) || (*(flowerbed + 2) == 0)? → False || (1 == 0)? → False
emptyRight = (i == 4) || (*(flowerbed + 4) == 0)? → False || (1 == 0)? → False

條件: emptyLeft && emptyRight? → False && False? → False
動作: 不種植
n = 0 (不變)
```

**Step 5: i = 4**
```
                          ▼
┌────┬────┬────┬────┬────┐
│ 1  │ 0  │ 1  │ 0  │ 1  │
└────┴────┴────┴────┴────┘

curr = *(0xA010) = 1

判斷: *(flowerbed + 4) == 0? → 1 == 0? → False
動作: 跳過
n = 0 (不變)
```

**返回:**
```
return n <= 0? → 0 <= 0? → True ✓
```

## 6. 記憶體存取模式比較

### 直接索引 vs 指標存取

| 面向 | 直接索引存取 | 指標存取 |
|------|-----------|--------|
| **語法** | `flowerbed[i]` | `*(flowerbed + i)` |
| **編譯時計算** | 索引地址：`base + i * 4` | 運行時指標運算 |
| **CPU 暫存器** | 直接載入 base + offset | 先計算 base + i*4，再解參考 |
| **快取友善** | 高（連續存取） | 高（相同陣列） |
| **分支預測** | 易於預測迴圈存取 | 相同（相同模式） |
| **指令數** | 較少 | 稍多（指標運算）|
| **程式碼可讀性** | 更清晰 | 較複雜 |

### 本問題的存取模式

```
每個迭代 (i):
  - 1次讀：當前位置 *(flowerbed + i)
  - 1-2次讀：左鄰居 *(flowerbed + i - 1) 或邊界檢查
  - 1-2次讀：右鄰居 *(flowerbed + i + 1) 或邊界檢查
  - 0-1次寫：可能種植 *(flowerbed + i) = 1

總計: O(n) 讀取，最多 O(n/3) 寫入（花卉密度上限）
```

### 快取行為分析

**假設:** 64 bytes 快取行，int 4 bytes → 16 ints/快取行

**例: flowerbed = [1,0,0,0,1]（全在一快取行內）**
```
快取行：[int0 int1 int2 int3 int4 ... int15]
        ◄────────────── 64 bytes ──────────────▶

第1次命中: 讀 i=0 → 整行載入快取
第2-5次命中: i=1,2,3,4 → 快取命中（無額外延遲）

快取命中率: ~100%（陣列小）
```

**最壞情況:** 大陣列（> 1MB）
```
快取遺失增加，但仍為順序存取（CPU 預取單位友善）
預測性預取: 有效降低延遲
```

## 7. 記憶體安全性考量

**指標越界風險:**
```c
// 危險: 未檢查邊界
prev = flowerbed + i - 1;  // 當 i=0 時，前往 flowerbed[-1] (非法!)
```

**安全修復:**
```c
// 正確做法
int emptyLeft = (i == 0) || (*(flowerbed + i - 1) == 0);
// ◄─ 先檢查 i == 0，短路求值防止解參考
```

**本解決方案:** ✓ 安全（包含邊界檢查）

---

## 8. 最壞情況分析

**情景:** 全為 0 的陣列 `flowerbed = [0,0,0,0,0,...,0]`（n 很大）

```
迭代軌跡:
i=0: 種植 → [1,0,0,0,0,...]
i=1: 左鄰 1，無法種
i=2: 可種 → [1,0,1,0,0,...]
i=3: 左鄰 1，無法種
i=4: 可種 → [1,0,1,0,1,...]
...

種植密度: ~1/2 (每 2 位置 1 花)
時間複雜度: O(n)
空間複雜度: O(1) (就地修改)
```

**記憶體寫入峰值:** 最多 `n/2` 次 `*(flowerbed + i) = 1`
