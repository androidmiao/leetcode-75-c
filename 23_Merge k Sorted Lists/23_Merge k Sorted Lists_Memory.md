# 23. Merge k Sorted Lists — 記憶體位址與指標追蹤

> 以下所有記憶體位址皆為假設值，僅供說明用途。

## 1. Struct 記憶體佈局

```c
struct ListNode {
    int val;                // 4 bytes (offset 0)
    struct ListNode *next;  // 8 bytes (offset 8, 64-bit 對齊)
};
// sizeof(struct ListNode) = 16 bytes (含 4 bytes padding)
```

## 3. 分治合併過程追蹤

### 3.1 遞迴呼叫樹

```
mergeKListsHelper(lists, 0, 2)
├── mergeKListsHelper(lists, 0, 1)        // 左半
│   ├── mergeKListsHelper(lists, 0, 0)    // 回傳 lists[0] = 0xA000
│   ├── mergeKListsHelper(lists, 1, 1)    // 回傳 lists[1] = 0xB000
│   └── mergeTwoLists(0xA000, 0xB000)     // 合併結果 → Phase 1
├── mergeKListsHelper(lists, 2, 2)        // 右半，回傳 lists[2] = 0xC000
└── mergeTwoLists(Phase1結果, 0xC000)      // 最終合併 → Phase 2
```

### 3.2 Phase 1: mergeTwoLists(0xA000, 0xB000)

合併 `1→4→5` 與 `1→3→4`。

`dummy` 位於堆疊上（假設位址 0xE000）。

| 步驟 | l1 位址 | l1->val | l2 位址 | l2->val | 比較結果 | current->next 設為 | current 移至 |
|------|---------|---------|---------|---------|----------|-------------------|-------------|
| 初始 | 0xA000 | 1 | 0xB000 | 1 | — | — | 0xE000 (dummy) |
| 1 | 0xA000 | 1 | 0xB000 | 1 | 1 < 1? 否 → 取 l2 | 0xB000 | 0xB000 |
| 2 | 0xA000 | 1 | 0xB010 | 3 | 1 < 3? 是 → 取 l1 | 0xA000 | 0xA000 |
| 3 | 0xA010 | 4 | 0xB010 | 3 | 4 < 3? 否 → 取 l2 | 0xB010 | 0xB010 |
| 4 | 0xA010 | 4 | 0xB020 | 4 | 4 < 4? 否 → 取 l2 | 0xB020 | 0xB020 |
| 5 | 0xA010 | 4 | NULL | — | l2 為空 → 接上 l1 | 0xA010 | — |

指標改寫追蹤：

```
步驟 1: dummy(0xE000).next = 0xB000    l2: 0xB000 → 0xB010
步驟 2: node_1a(0xB000).next = 0xA000  l1: 0xA000 → 0xA010
步驟 3: node_0a(0xA000).next = 0xB010  l2: 0xB010 → 0xB020
步驟 4: node_1b(0xB010).next = 0xB020  l2: 0xB020 → NULL
步驟 5: node_1c(0xB020).next = 0xA010  (接上 l1 剩餘)
```

Phase 1 結果（回傳 dummy.next = 0xB000）：

```
  0xB000       0xA000       0xB010       0xB020       0xA010       0xA020
  ┌───┬────┐   ┌───┬────┐   ┌───┬────┐   ┌───┬────┐   ┌───┬────┐   ┌───┬────┐
  │ 1 │next┼──▶│ 1 │next┼──▶│ 3 │next┼──▶│ 4 │next┼──▶│ 4 │next┼──▶│ 5 │NULL│
  └───┴────┘   └───┴────┘   └───┴────┘   └───┴────┘   └───┴────┘   └───┴────┘
```

| 初始 | 0xB000 | 1 | 0xC000 | 2 | — | — | 0xF000 (dummy) |

| 2 | 0xA000 | 1 | 0xC000 | 2 | 1 < 2? 是 → 取 l1 | 0xA000 | 0xA000 |
| 3 | 0xB010 | 3 | 0xC000 | 2 | 3 < 2? 否 → 取 l2 | 0xC000 | 0xC000 |

## 5. 堆疊記憶體對比

### 5.1 遞迴版堆疊使用（目前實作）

```
呼叫堆疊（k=3, depth = ⌈log₂ 3⌉ = 2 層）：

┌──────────────────────────────────────────┐  ◄── 堆疊頂
│ mergeTwoLists(0xA000, 0xB000)            │
│   dummy: 16 bytes (堆疊上)               │
│   current: 8 bytes                       │
│   框架大小: ~48 bytes                     │
├──────────────────────────────────────────┤
│ mergeKListsHelper(lists, 0, 1)           │
│   mid, l1, l2: 各 4/8 bytes             │
│   框架大小: ~32 bytes                     │
├──────────────────────────────────────────┤
│ mergeKListsHelper(lists, 0, 2)           │
│   mid, l1, l2: 各 4/8 bytes             │
│   框架大小: ~32 bytes                     │
├──────────────────────────────────────────┤
│ mergeKLists(lists, 3)                    │
│   框架大小: ~16 bytes                     │
└──────────────────────────────────────────┘  ◄── 堆疊底
```

最大堆疊深度 = O(log k)。k = 10⁴ 時，最多 14 層，約 14 × 48 = 672 bytes。

### 5.2 迭代版（無遞迴開銷）

若改為迭代式 bottom-up 合併，只需一個迴圈和臨時指標：

```
int interval = 1;
while (interval < listsSize) {
    for (int i = 0; i + interval < listsSize; i += interval * 2) {
        lists[i] = mergeTwoLists(lists[i], lists[i + interval]);
    }
    interval *= 2;
}
```

堆疊只需一層 `mergeTwoLists` 的框架（~48 bytes），額外空間 O(1)。

## 6. 記憶體存取模式分析

### 原地改寫 (In-place Rewiring) 特性

此分治合併演算法的一大優勢是**完全原地改寫指標**，不分配新節點：

| 維度 | 分治合併（本實作） | 暴力法（方法 1） |
|------|------------------|-----------------|
| 新 malloc 次數 | 0 | N（建立全新串列） |
| 記憶體分配器壓力 | 無 | 高 |
| Cache 行為 | 存取現有節點（可能分散） | 寫入新連續記憶體（較佳局部性） |
| 總記憶體使用 | 8N bytes（原始節點） | 16N bytes（原始 + 新） |

### 指標改寫的 Cache 影響

鏈結串列節點通常在堆上分散分配。合併過程中的 `current->next = l1` 或 `current->next = l2` 寫入操作會觸發：

1. **讀取** l1/l2 指向的節點（可能 cache miss）
2. **寫入** current 的 next 欄位（通常在 cache 中，因為剛被存取過）

最壞情況下，每步可能觸發一次 cache miss（64-byte cache line 可容納 4 個 ListNode），但由於節點在堆上的分布取決於 malloc 實作，實際行為變化很大。

## 7. 退化情況分析

### 7.1 所有節點集中在一個串列

若 `lists = [[1,2,3,...,N], [], [], ..., []]`（k 個串列，只有第一個非空）：

- 遞迴仍會拆分成 O(log k) 層
- 但每次 `mergeTwoLists` 的一個參數為 NULL，立即回傳
- 實際工作量 = O(N)（最後一次非空合併遍歷整個串列）
- 堆疊深度仍為 O(log k)

### 7.2 每個串列只有一個節點

若 `lists = [[a₁], [a₂], ..., [aₖ]]`（k 個串列各一個節點）：

- 每次 `mergeTwoLists` 只需 1 次比較 + 接上剩餘
- 第一層：k/2 次合併，每次 1 比較 → k/2 次比較
- 第二層：k/4 次合併，每次最多 2 比較 → k/2 次比較
- 總比較次數 ≈ (k/2) × log₂ k = O(k log k) = O(N log k)（因 N = k）

### 7.3 k = 1

- 直接回傳 `lists[0]`，無任何合併
- 時間 O(1)，空間 O(1)

### 7.4 k = 10⁴, N = 10⁴（約束上限）

- 堆疊深度 = ⌈log₂ 10000⌉ = 14 層
- 每層合併總工作 = O(N) = O(10⁴)
- 總工作 = 14 × 10⁴ = 1.4 × 10⁵ 次操作，極為高效
