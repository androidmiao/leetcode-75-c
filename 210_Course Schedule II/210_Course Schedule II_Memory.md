# LeetCode 210. Course Schedule II — 記憶體與陣列索引圖解

> 所有記憶體位址皆為假設值，用於教學示意。

---

## 1. 資料結構記憶體佈局

本題使用 **pointer-to-pointer 鄰接表**（`int** adj`），而非 struct node。核心變數：

| 變數          | 型別          | 用途                                    |
|---------------|---------------|-----------------------------------------|
| `adj`         | `int**`       | 鄰接表：adj[i] 指向課程 i 的先修課陣列    |
| `adj_sizes`   | `int*`        | adj_sizes[i] = adj[i] 中已填入的元素數    |
| `state`       | `int*`        | 三色標記陣列（0=WHITE, 1=GRAY, 2=BLACK） |
| `result`      | `int*`        | 拓撲排序結果陣列                          |
| `idx`         | `int`         | result 的寫入游標                         |
| `has_cycle`   | `static bool` | 環偵測全域旗標                            |

---

## 2. 完整資料結構記憶體配置

### 範例：numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]

### 2.1 配置表

| 變數             | 假設位址   | 大小              | 內容                              |
|------------------|-----------|-------------------|-----------------------------------|
| `adj`            | 0xA000    | 4 × 8 = 32 bytes | 4 個 int* 指標                     |
| `adj[0]`（陣列） | 0xB000    | 4 × 4 = 16 bytes | `[]`（空）                         |
| `adj[1]`（陣列） | 0xB100    | 4 × 4 = 16 bytes | `[0]`                             |
| `adj[2]`（陣列） | 0xB200    | 4 × 4 = 16 bytes | `[0]`                             |
| `adj[3]`（陣列） | 0xB300    | 4 × 4 = 16 bytes | `[1, 2]`                          |
| `adj_sizes`      | 0xC000    | 4 × 4 = 16 bytes | `[0, 1, 1, 2]`                    |
| `state`          | 0xD000    | 4 × 4 = 16 bytes | 初始 `[0, 0, 0, 0]`               |
| `result`         | 0xE000    | 4 × 4 = 16 bytes | 結果陣列                           |

### 2.2 鄰接表記憶體圖

```
adj (0xA000)                          各 adj[i] 指向的陣列
┌──────────┐
│ adj[0]   │──────────────────────▶ 0xB000: ┌───┬───┬───┬───┐
│ 0xB000   │                                │   │   │   │   │  (空，adj_sizes[0]=0)
├──────────┤                                └───┴───┴───┴───┘
│ adj[1]   │──────────────────────▶ 0xB100: ┌───┬───┬───┬───┐
│ 0xB100   │                                │ 0 │   │   │   │  adj_sizes[1]=1
├──────────┤                                └───┴───┴───┴───┘
│ adj[2]   │──────────────────────▶ 0xB200: ┌───┬───┬───┬───┐
│ 0xB200   │                                │ 0 │   │   │   │  adj_sizes[2]=1
├──────────┤                                └───┴───┴───┴───┘
│ adj[3]   │──────────────────────▶ 0xB300: ┌───┬───┬───┬───┐
│ 0xB300   │                                │ 1 │ 2 │   │   │  adj_sizes[3]=2
└──────────┘                                └───┴───┴───┴───┘
```

### 2.3 邏輯圖（有向圖，course → prereq）

```
        3
       / \
      v   v
      1   2
       \ /
        v
        0
```

箭頭方向：3→1, 3→2, 1→0, 2→0（依賴方向）

## 3. 陣列索引版對應

由於本題是通用有向圖（非樹），沒有 `2*i+1` 這類隱式索引公式。但可以用 **CSR（Compressed Sparse Row）** 格式將鄰接表壓縮為兩個一維陣列：

### CSR 格式

| 陣列       | 索引 | 0 | 1 | 2 | 3 | 4  |
|------------|------|---|---|---|---|----|
| `offset[]` |      | 0 | 0 | 1 | 2 | 4  |
| `edges[]`  |      | 0 | 0 | 1 | 2 |    |

規則：課程 `i` 的先修課列表 = `edges[offset[i] .. offset[i+1]-1]`

- 課程 0：`edges[0..0)` = 空
- 課程 1：`edges[0..1)` = `[0]`
- 課程 2：`edges[1..2)` = `[0]`
- 課程 3：`edges[2..4)` = `[1, 2]`

CSR 記憶體佈局：

```
offset (0xF000):  ┌───┬───┬───┬───┬───┐
                  │ 0 │ 0 │ 1 │ 2 │ 4 │   (numCourses+1 個元素)
                  └───┴───┴───┴───┴───┘

edges  (0xF100):  ┌───┬───┬───┬───┐
                  │ 0 │ 0 │ 1 │ 2 │       (共 4 條邊)
                  └───┴───┴───┴───┘
```

CSR 優勢：連續記憶體、cache-friendly、O(V+E) 空間而非 O(V²)。

## 4. DFS 拓撲排序過程追蹤

### 範例 1：numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]

初始狀態：

```
state  (0xD000):  ┌───┬───┬───┬───┐
                  │ 0 │ 0 │ 0 │ 0 │   全部 WHITE
                  └───┴───┴───┴───┘

result (0xE000):  ┌───┬───┬───┬───┐
                  │   │   │   │   │   空
                  └───┴───┴───┴───┘
                    ^
                    idx=0
```

#### 步驟 1：主迴圈 i=0 → `state[0]==0` → 呼叫 `dfs(0)`

```
dfs(0): state[0] = 1 (GRAY)
        adj_sizes[0] = 0 → 無鄰居 → 迴圈跳過
        state[0] = 2 (BLACK)
        result[0] = 0, idx = 1

state:  ┌───┬───┬───┬───┐
        │ 2 │ 0 │ 0 │ 0 │
        └───┴───┴───┴───┘
         BLK WHT WHT WHT

result: ┌───┬───┬───┬───┐
        │ 0 │   │   │   │
        └───┴───┴───┴───┘
              ^
              idx=1
```

#### 步驟 2：主迴圈 i=1 → `state[1]==0` → 呼叫 `dfs(1)`

```
dfs(1): state[1] = 1 (GRAY)
        鄰居 prereq = adj[1][0] = 0
          state[0] == 2 (BLACK) → 跳過
        state[1] = 2 (BLACK)
        result[1] = 1, idx = 2

state:  ┌───┬───┬───┬───┐
        │ 2 │ 2 │ 0 │ 0 │
        └───┴───┴───┴───┘
         BLK BLK WHT WHT

result: ┌───┬───┬───┬───┐
        │ 0 │ 1 │   │   │
        └───┴───┴───┴───┘
                  ^
                  idx=2
```

#### 步驟 3：主迴圈 i=2 → `state[2]==0` → 呼叫 `dfs(2)`

```
dfs(2): state[2] = 1 (GRAY)
        鄰居 prereq = adj[2][0] = 0
          state[0] == 2 (BLACK) → 跳過
        state[2] = 2 (BLACK)
        result[2] = 2, idx = 3

state:  ┌───┬───┬───┬───┐
        │ 2 │ 2 │ 2 │ 0 │
        └───┴───┴───┴───┘
         BLK BLK BLK WHT

result: ┌───┬───┬───┬───┐
        │ 0 │ 1 │ 2 │   │
        └───┴───┴───┴───┘
                      ^
                      idx=3
```

#### 步驟 4：主迴圈 i=3 → `state[3]==0` → 呼叫 `dfs(3)`

```
dfs(3): state[3] = 1 (GRAY)
        鄰居 prereq = adj[3][0] = 1
          state[1] == 2 (BLACK) → 跳過
        鄰居 prereq = adj[3][1] = 2
          state[2] == 2 (BLACK) → 跳過
        state[3] = 2 (BLACK)
        result[3] = 3, idx = 4

state:  ┌───┬───┬───┬───┐
        │ 2 │ 2 │ 2 │ 2 │
        └───┴───┴───┴───┘
         BLK BLK BLK BLK

result: ┌───┬───┬───┬───┐
        │ 0 │ 1 │ 2 │ 3 │  ← 合法拓撲排序 ✓
        └───┴───┴───┴───┘
                          ^
                          idx=4
```

最終結果：`[0, 1, 2, 3]`

### 範例 2（有環）：numCourses = 2, prerequisites = [[1,0],[0,1]]

鄰接表：

```
adj[0] = [1]   adj_sizes[0] = 1
adj[1] = [0]   adj_sizes[1] = 1
```

初始狀態：

```
state:  ┌───┬───┐       result: ┌───┬───┐
        │ 0 │ 0 │               │   │   │
        └───┴───┘               └───┴───┘
         WHT WHT                  ^  idx=0
```

#### 步驟 1：主迴圈 i=0 → 呼叫 `dfs(0)`

```
dfs(0): state[0] = 1 (GRAY)
  │
  ├─ 鄰居 prereq = adj[0][0] = 1
  │    state[1] == 0 (WHITE) → 遞迴進入 dfs(1)
  │
  │  dfs(1): state[1] = 1 (GRAY)
  │    │
  │    ├─ 鄰居 prereq = adj[1][0] = 0
  │    │    state[0] == 1 (GRAY) ← 回邊！
  │    │    has_cycle = true ← 設定旗標
  │    │    return
  │    │
  │    └─ (has_cycle → return)
  │
  └─ (has_cycle → return)

state:  ┌───┬───┐
        │ 1 │ 1 │   兩個都卡在 GRAY（遞迴未完成）
        └───┴───┘
         GRY GRY
```

`has_cycle = true` → `*returnSize = 0` → 返回空陣列

### 範例 3：numCourses = 2, prerequisites = [[1,0]]

```
adj[0] = []    adj_sizes[0] = 0
adj[1] = [0]   adj_sizes[1] = 1
```

#### 步驟 1：dfs(0)

```
dfs(0): state[0]=GRAY → 無鄰居 → state[0]=BLACK → result[0]=0, idx=1
```

#### 步驟 2：dfs(1)

```
dfs(1): state[1]=GRAY → 鄰居 0 已 BLACK → 跳過 → state[1]=BLACK → result[1]=1, idx=2
```

結果：`[0, 1]` ✓

## 5. 指標變數追蹤總表

### 範例 1：numCourses = 4, prerequisites = [[1,0],[2,0],[3,1],[3,2]]

| 步驟 | 呼叫       | course | state 變化      | 鄰居檢查               | 動作              | idx 變化 |
|------|-----------|--------|-----------------|------------------------|-------------------|---------|
| 1    | dfs(0)    | 0      | 0→1→2           | 無鄰居                  | result[0]=0       | 0→1     |
| 2    | dfs(1)    | 1      | 0→1→2           | 0: BLACK→跳過           | result[1]=1       | 1→2     |
| 3    | dfs(2)    | 2      | 0→1→2           | 0: BLACK→跳過           | result[2]=2       | 2→3     |
| 4    | dfs(3)    | 3      | 0→1→2           | 1: BLACK→跳過, 2: BLACK→跳過 | result[3]=3  | 3→4     |

統計：DFS 呼叫 4 次，鄰居檢查 4 次（= 邊數 E=4），所有節點各訪問 1 次。

### 範例 2（有環）：numCourses = 2, prerequisites = [[1,0],[0,1]]

| 步驟 | 呼叫       | course | state 變化      | 鄰居檢查               | 動作               | idx 變化 |
|------|-----------|--------|-----------------|------------------------|--------------------|---------|
| 1    | dfs(0)    | 0      | 0→1（卡住）      | 1: WHITE→遞迴           | 進入 dfs(1)        | 不變     |
| 2    | dfs(1)    | 1      | 0→1（卡住）      | 0: GRAY→回邊！          | has_cycle=true     | 不變     |

統計：DFS 呼叫 2 次，鄰居檢查 2 次，提前終止。

## 6. 記憶體存取模式比較

### Pointer-to-Pointer 鄰接表（目前實作）vs. CSR 格式

| 比較項目          | `int** adj`（目前）             | CSR（offset[] + edges[]）      |
|-------------------|---------------------------------|-------------------------------|
| 記憶體總量        | O(V² + V)（最壞每列配 V 元素）   | O(V + E)（精確配置）            |
| Cache 行為        | 每個 adj[i] 不同 malloc 位址，   | edges[] 連續存放，              |
|                   | 隨機跳躍 → cache miss 多         | 順序掃描 → cache-friendly       |
| 存取 adj[i][j]    | 兩次間接指標解參考               | offset[i]+j 一次算術 + 一次存取  |
| 插入邊（動態）    | O(1) 攤銷（若有空間）            | 困難（需重建）                   |
| 適用場景          | LeetCode（簡單、V≤2000）         | 生產環境、大規模圖               |

### 記憶體存取流程圖

```
Pointer-to-pointer:
  adj ──▶ [ptr0][ptr1][ptr2][ptr3]
            │     │     │     │
            ▼     ▼     ▼     ▼
          [...]  [0]   [0]   [1,2]    ← 各自獨立的 malloc 區塊

CSR:
  offset: [0, 0, 1, 2, 4]    ← 一個連續陣列
  edges:  [0, 0, 1, 2]       ← 一個連續陣列
```

## 7. 堆疊記憶體對比

### 遞迴式 DFS（目前實作）

最壞情況：線性鏈 0→1→2→...→(V-1)，DFS 遞迴深度 = V。

```
呼叫堆疊（V=4 線性鏈時）：

┌─────────────────────────────┐  ← 堆疊頂端
│ dfs(3)                      │  frame: course=3, i, prereq
│   局部變數 ≈ 24 bytes       │
├─────────────────────────────┤
│ dfs(2)                      │  frame: course=2, i, prereq
│   局部變數 ≈ 24 bytes       │
├─────────────────────────────┤
│ dfs(1)                      │  frame: course=1, i, prereq
│   局部變數 ≈ 24 bytes       │
├─────────────────────────────┤
│ dfs(0)                      │  frame: course=0, i, prereq
│   局部變數 ≈ 24 bytes       │
├─────────────────────────────┤
│ findOrder()                 │  adj, state, result, idx...
│   局部變數 ≈ 48 bytes       │
└─────────────────────────────┘  ← 堆疊底端

每個 dfs frame ≈ 24 bytes（含 return address、saved registers）
最壞堆疊使用：24 × V + 48 ≈ 24 × 2000 ≈ 48 KB
```

### 迭代式 Kahn's（堆積記憶體上的佇列）

```
堆疊使用：僅 findOrder() 一層 ≈ 固定大小
堆積使用：queue[] 最多 V 個元素 = 4 × V bytes

┌─────────────────────────────┐
│ findOrder()                 │  queue, indegree, result...
│   局部變數 ≈ 48 bytes       │  ← 堆疊深度固定！
└─────────────────────────────┘

queue (heap): ┌───┬───┬───┬───┐
              │ 0 │   │   │   │  front=0, rear=1
              └───┴───┴───┴───┘
```

| 比較項目       | 遞迴 DFS              | 迭代 Kahn's            |
|---------------|----------------------|------------------------|
| 堆疊深度      | O(V)                  | O(1)                   |
| 堆積使用      | O(V+E)（adj+state）   | O(V+E)（adj+indegree+queue） |
| V=2000 堆疊   | ≈ 48 KB               | ≈ 48 bytes             |
| V=100000 堆疊 | ≈ 2.4 MB（可能溢出）   | ≈ 48 bytes             |
| Linux kernel  | 8KB stack → ≈333 節點上限 | 無限制              |

## 8. 退化情況分析

### 退化圖形：線性鏈

```
prerequisites: [[1,0],[2,1],[3,2]]

adj[0] = []     adj[1] = [0]     adj[2] = [1]     adj[3] = [2]

邏輯圖：
  3 → 2 → 1 → 0
```

DFS 從 i=0 開始：dfs(0) 立即完成。dfs(1) 遞迴一層。dfs(2) 遞迴兩層。dfs(3) 遞迴三層。最大遞迴深度 = V-1。

### 退化圖形：星狀（一門課依賴所有其他）

```
prerequisites: [[3,0],[3,1],[3,2]]

adj[3] = [0, 1, 2]    （其餘皆空）

邏輯圖：
      3
     /|\
    v v v
    0 1 2
```

DFS 從 i=0, 1, 2 各自完成（深度 1）。dfs(3) 遞迴深度僅 1（鄰居都已 BLACK）。最大遞迴深度 = 2。此形狀對遞迴很友好。

### 退化圖形：完全 DAG

V 個節點，每個節點 i 依賴所有 j < i。邊數 E = V(V-1)/2。此時鄰接表 O(V²)，遍歷所有邊需 O(V²)。不影響正確性，但常數倍增大。
