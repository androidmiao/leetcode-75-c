# LeetCode 17 - 電話號碼的字母組合 記憶體分析

## 1. Struct 記憶體佈局

### PHONE_MAP 靜態陣列結構

PHONE_MAP 是一個包含 10 個 char* 指標的靜態常數陣列，映射數字 0-9 到對應的字母字符串。

```
靜態記憶體區域 (假設地址起點 0xA000):

┌─────────────────────────────────────────────────────────────────┐
│ PHONE_MAP[0..9] - 10 個指標，每個 8 bytes (64-bit)              │
├─────────────────────────────────────────────────────────────────┤
│ Index │ 地址      │ 值 (指向字符串)  │ 指向的字符串             │
├───────┼───────────┼──────────────────┼──────────────────────────┤
│  [0]  │ 0xA000    │ → 0xB000         │ "" (空字符串)            │
│  [1]  │ 0xA008    │ → 0xB001         │ "" (空字符串)            │
│  [2]  │ 0xA010    │ → 0xB002         │ "abc"                    │
│  [3]  │ 0xA018    │ → 0xB006         │ "def"                    │
│  [4]  │ 0xA020    │ → 0xB00A         │ "ghi"                    │
│  [5]  │ 0xA028    │ → 0xB00E         │ "jkl"                    │
│  [6]  │ 0xA030    │ → 0xB012         │ "mno"                    │
│  [7]  │ 0xA038    │ → 0xB016         │ "pqrs"                   │
│  [8]  │ 0xA040    │ → 0xB01B         │ "tuv"                    │
│  [9]  │ 0xA048    │ → 0xB020         │ "wxyz"                   │
└───────┴───────────┴──────────────────┴──────────────────────────┘

字符串字面量記憶體 (唯讀資料段):

┌──────────────────────────────────────────────────────────┐
│ 地址     │ 內容                  │ 備註                 │
├──────────┼───────────────────────┼──────────────────────┤
│ 0xB000   │ '\0'                  │ PHONE_MAP[0]         │
│ 0xB001   │ '\0'                  │ PHONE_MAP[1]         │
│ 0xB002   │ 'a' 'b' 'c' '\0'      │ PHONE_MAP[2]="abc"   │
│ 0xB006   │ 'd' 'e' 'f' '\0'      │ PHONE_MAP[3]="def"   │
│ 0xB00A   │ 'g' 'h' 'i' '\0'      │ PHONE_MAP[4]="ghi"   │
│ 0xB00E   │ 'j' 'k' 'l' '\0'      │ PHONE_MAP[5]="jkl"   │
│ 0xB012   │ 'm' 'n' 'o' '\0'      │ PHONE_MAP[6]="mno"   │
│ 0xB016   │ 'p' 'q' 'r' 's' '\0'  │ PHONE_MAP[7]="pqrs"  │
│ 0xB01B   │ 't' 'u' 'v' '\0'      │ PHONE_MAP[8]="tuv"   │
│ 0xB020   │ 'w' 'x' 'y' 'z' '\0'  │ PHONE_MAP[9]="wxyz"  │
└──────────┴───────────────────────┴──────────────────────┘
```

**指標大小**: 每個 PHONE_MAP[i] 是 8 bytes (64-bit 系統)
**總計**: PHONE_MAP 陣列 = 10 × 8 = 80 bytes
**字符串字面量**: 約 40 bytes (包括空終止符)

## 3. 陣列索引版對應

### 索引映射公式

```
輸入: digits="23"
     digits[0]='2', digits[1]='3'

步驟1: 數字到字符串的映射
────────────────────────────────────────

digits[0]='2' (ASCII 50)
  ↓
轉換為數字: 2
  ↓
PHONE_MAP[2] ────▶ "abc" (3個字母)
  ├─ letters[0]='a'
  ├─ letters[1]='b'
  └─ letters[2]='c'


digits[1]='3' (ASCII 51)
  ↓
轉換為數字: 3
  ↓
PHONE_MAP[3] ────▶ "def" (3個字母)
  ├─ letters[0]='d'
  ├─ letters[1]='e'
  └─ letters[2]='f'

步驟2: 組合位置計算公式
────────────────────────────────────────

對於digits="23"的第一個數字選擇索引 i，第二個數字選擇索引 j:

結果陣列索引 = i × len(PHONE_MAP[digits[1]]) + j
             = i × 3 + j

組合對應表:
┌───────────────┬───────────────┬─────────────────────┬────────┐
│ 第1個數字索引 │ 第2個數字索引 │ 組合 (path內容)      │ result │
│ (i)           │ (j)           │                     │ 索引   │
├───────────────┼───────────────┼─────────────────────┼────────┤
│   0           │   0           │ path="ad"           │   0    │
│   0           │   1           │ path="ae"           │   1    │
│   0           │   2           │ path="af"           │   2    │
│   1           │   0           │ path="bd"           │   3    │
│   1           │   1           │ path="be"           │   4    │
│   1           │   2           │ path="bf"           │   5    │
│   2           │   0           │ path="cd"           │   6    │
│   2           │   1           │ path="ce"           │   7    │
│   2           │   2           │ path="cf"           │   8    │
└───────────────┴───────────────┴─────────────────────┴────────┘

公式驗證:
┌─ i=0, j=0: 0×3+0=0   ✓ "ad"  → result[0]
├─ i=0, j=1: 0×3+1=1   ✓ "ae"  → result[1]
├─ i=0, j=2: 0×3+2=2   ✓ "af"  → result[2]
├─ i=1, j=0: 1×3+0=3   ✓ "bd"  → result[3]
├─ i=1, j=1: 1×3+1=4   ✓ "be"  → result[4]
├─ i=1, j=2: 1×3+2=5   ✓ "bf"  → result[5]
├─ i=2, j=0: 2×3+0=6   ✓ "cd"  → result[6]
├─ i=2, j=1: 2×3+1=7   ✓ "ce"  → result[7]
└─ i=2, j=2: 2×3+2=8   ✓ "cf"  → result[8]

步驟3: 字符串指標複製
────────────────────────────────────────

每個組合完成後，path中的字符被複製到堆記憶體:

path[] (棧)  ──複製──▶ result[idx] (堆)
"ad"                    0xC100
"ae"                    0xC104
... (依此類推)
```

## 4. 搜尋/操作過程追蹤

### 示例1: digits="23"

#### 遞迴過程圖解

```
recur(digits, 0, path, result, returnSize)
│
├─ index=0, path="" (0xD000 →['\0'][\?][\?])
│  │ PHONE_MAP[2]="abc"
│  │
│  ├─ 選擇 'a' → path="a" (0xD000 →['a'][\0][\?])
│  │  │
│  │  └─ recur(digits, 1, "a", result, returnSize)
│  │     │ index=1, path="a" (0xD000 →['a'][\0][\?])
│  │     │ PHONE_MAP[3]="def"
│  │     │
│  │     ├─ 選擇 'd' → path="ad" (0xD000 →['a']['d'][\0])
│  │     │  │
│  │     │  ├─ index==digitsSize (2==2) → 複製 "ad"
│  │     │  │  strcpy(result[0], "ad")
│  │     │  │  result[0] = 0xC100 ──▶ "ad"
│  │     │  │  returnSize[0]++ → 1
│  │     │  │  回溯: path[1]='d' ───▶ path[1]='?' (待重寫)
│  │     │  │
│  │     │  ├─ 選擇 'e' → path="ae" (0xD000 →['a']['e'][\0])
│  │     │  │  複製 "ae" → result[1] = 0xC104
│  │     │  │  returnSize[0]++ → 2
│  │     │  │
│  │     │  └─ 選擇 'f' → path="af" (0xD000 →['a']['f'][\0])
│  │     │     複製 "af" → result[2] = 0xC108
│  │     │     returnSize[0]++ → 3
│  │     │
│  │     └─ 回溯至 index=0
│  │
│  ├─ 選擇 'b' → path="b" (0xD000 →['b'][\0][\?])
│  │  │
│  │  └─ recur(digits, 1, "b", result, returnSize)
│  │     ├─ 選擇 'd' → path="bd" (複製 → result[3])
│  │     ├─ 選擇 'e' → path="be" (複製 → result[4])
│  │     └─ 選擇 'f' → path="bf" (複製 → result[5])
│  │
│  └─ 選擇 'c' → path="c" (0xD000 →['c'][\0][\?])
│     │
│     └─ recur(digits, 1, "c", result, returnSize)
│        ├─ 選擇 'd' → path="cd" (複製 → result[6])
│        ├─ 選擇 'e' → path="ce" (複製 → result[7])
│        └─ 選擇 'f' → path="cf" (複製 → result[8])
│
└─ 完成，returnSize=9
```

#### 詳細路徑記憶體狀態追蹤

```
───────────────────────────────────────────────────────────────
步驟1: 初始狀態
───────────────────────────────────────────────────────────────
recur(digits="23", index=0, path="", result, returnSize=0)

path[] 記憶體 (0xD000):
┌─────┬─────┬─────┐
│ \0  │ ?   │ ?   │
└─────┴─────┴─────┘
  ^
  path[0]指標

遞迴深度: 0
當前決策點: digits[0]='2' → PHONE_MAP[2]="abc"


───────────────────────────────────────────────────────────────
步驟2: 選擇第一個字母 'a'
───────────────────────────────────────────────────────────────
path[0]='a', index=0

path[] 記憶體 (0xD000):
┌─────┬─────┬─────┐
│ 'a' │ \0  │ ?   │
└─────┴─────┴─────┘
  ^
  當前指標位置 (path[0])

遞迴深度: 1
進入: recur(digits="23", index=1, ...)


───────────────────────────────────────────────────────────────
步驟3: 在 index=1 選擇 'd'
───────────────────────────────────────────────────────────────
path[1]='d', index=1

path[] 記憶體 (0xD000):
┌─────┬─────┬─────┐
│ 'a' │ 'd' │ \0  │
└─────┴─────┴─────┘
          ^
          當前指標位置 (path[1])

條件: index==digitsSize (1==2) ✗ 繼續遞迴
不，等等 —— digitsSize=2，當 index=1 時應該繼續


實際上讓我重新追蹤: digits="23" 意味著有2個數字
index 應該從 0 開始，遞迴到 index=2 時停止

重新開始:
digitsSize = 2
初始調用: recur(digits, 0, path, result, returnSize)

條件判斷: if (index == digitsSize) ← 當 index=2 時完成


───────────────────────────────────────────────────────────────
步驟1: 初始 (index=0)
───────────────────────────────────────────────────────────────

if (0 == 2)? NO → 繼續

digit = digits[0] - '0' = '2' - '0' = 2
letters = PHONE_MAP[2] = "abc"

迴圈遍歷 j:
  j=0: letters[0]='a'
    path[0] = 'a'
    path[] 記憶體: ['a'][?][?]
    呼叫 recur(digits, 1, path, result, returnSize)
        │
        ├─ index=1
        │
        ├─ if (1 == 2)? NO
        │
        ├─ digit = digits[1] - '0' = '3' - '0' = 3
        │ letters = PHONE_MAP[3] = "def"
        │
        └─ 迴圈遍歷 j:
              j=0: letters[0]='d'
                path[1] = 'd'
                path[] 記憶體: ['a']['d'][?]
                呼叫 recur(digits, 2, path, result, returnSize)
                    │
                    ├─ index=2
                    │
                    └─ if (2 == 2)? YES ← 複製組合
                        strcpy(result[returnSize[0]], path)
                        結果: result[0] ──▶ "ad" (0xC100)
                        returnSize[0]++ → 1
                        回傳 (回溯)
                        path[1] 會被下一次迭代覆寫
```

### 示例2: digits="2"

#### 遞迴過程圖解

```
recur(digits="2", index=0, path="", result, returnSize=0)

digitsSize = 1

步驟1: index=0
if (0 == 1)? NO

digit = digits[0] - '0' = '2' - '0' = 2
letters = PHONE_MAP[2] = "abc"

迴圈 j=0,1,2:
  j=0: path[0]='a'
       recur(digits, 1, "a", result, returnSize)
         index=1
         if (1 == 1)? YES ← 複製 "a"
         strcpy(result[0], "a")
         result[0] = 0xC200 ──▶ "a"
         returnSize[0]++ → 1

## 6. 記憶體存取模式比較

### 靜態陣列映射 vs 雜湊表映射

#### 方式1: 靜態陣列映射 (本解法採用)

```
PHONE_MAP[0..9] - 靜態常數陣列

記憶體佈局:
┌──────────────────────────────────────────┐
│ 靜態資料段 (唯讀，位置固定)               │
├──────────────────────────────────────────┤
│ PHONE_MAP[]:                             │
│  [0] → 0xB000: ""                        │
│  [1] → 0xB001: ""                        │
│  [2] → 0xB002: "abc"                     │
│  [3] → 0xB006: "def"                     │
│  [4] → 0xB00A: "ghi"                     │
│  [5] → 0xB00E: "jkl"                     │
│  [6] → 0xB012: "mno"                     │
│  [7] → 0xB016: "pqrs"                    │
│  [8] → 0xB01B: "tuv"                     │
│  [9] → 0xB020: "wxyz"                    │
└──────────────────────────────────────────┘

訪問過程 (O(1)):
digit = digits[i] - '0'  ← ASCII轉換 (1 次減法)
letters = PHONE_MAP[digit]  ← 直接索引 (1 次記憶體讀取)
```

**優點:**
- 查找時間: O(1) 常數時間
- 快取局部性好: PHONE_MAP 是連續陣列，所有指標存儲在一起
- 分支預測友好: 無雜湊碰撞，無條件判斷
- 記憶體佈局確定: 編譯時已知大小和位置

**缺點:**
- 固定大小: 只能映射 10 個數字 0-9
- 空間開銷: 存儲 0、1 的空字符串指標 (不過僅 16 bytes)

**快取分析:**
```
L1 快取行 (通常 64 bytes):
┌─────────────────────────────────────────────────┐
│ PHONE_MAP[0..7] 幾乎完全被包含在一個快取行中      │
│ (8 個指標 = 64 bytes)                           │
└─────────────────────────────────────────────────┘

訪問模式:
- 每次回溯呼叫中 PHONE_MAP[digit] 的訪問
- 順序性強，預取友好
- 未來訪問有高度可預測性
```

#### 方式2: 雜湊表映射 (假想替代)

```
假設使用簡單雜湊表:

typedef struct {
    int key;      // 數字 0-9
    const char* value;  // 字母字符串
} MapEntry;

// 雜湊表
MapEntry map[] = {
    {2, "abc"},
    {3, "def"},
    {4, "ghi"},
    {5, "jkl"},
    {6, "mno"},
    {7, "pqrs"},
    {8, "tuv"},
    {9, "wxyz"}
};

訪問過程 (O(1) 平均，O(n) 最壞):
1. 計算雜湊值: h = digit % 8
2. 檢查雜湊衝突: 迴圈尋找 map[h].key == digit
3. 返回 map[h].value
```

**缺點:**
- 雜湊計算開銷: 需要取模運算或位運算
- 衝撞處理: 最壞情況 O(n)，需要條件判斷
- 快取局部性差: 雜湊表位置分散，預取困難
- 分支預測失敗: 衝撞檢查導致分支預測困難

**快取分析:**
```
L1 快取行:
┌─────────────────────────────────────────────────┐
│ 雜湊表可能分散在多個快取行中                     │
│ (衝撞鏈的跳躍)                                   │
└─────────────────────────────────────────────────┘

訪問模式:
- 非順序訪問 (由雜湊值決定)
- 預取效果差
- 衝撞概率高時效率下降
```

│  index=1, letter_idx=0              │
│  path[1] = PHONE_MAP[3][0] = 'd'    │
│  index==digitsSize → 複製 "ad"     │
│  letter_idx++ → 1                   │
│  stack[1] = {index: 1, letter_idx: 1}
│                                     │
│ 步驟 3: 重新處理 stack[1]           │
│  index=1, letter_idx=1              │
│  path[1] = PHONE_MAP[3][1] = 'e'    │
│  複製 "ae"                          │
│  letter_idx++ → 2                   │
│  ...                                │
│                                     │
│ (迴圈直到所有組合完成)              │
└─────────────────────────────────────┘
```

#### 迭代 vs 遞迴 棧記憶體比較

```
┌────────────────────┬──────────────────┬──────────────────┐
│ 指標               │ 遞迴棧           │ 迭代棧           │
├────────────────────┼──────────────────┼──────────────────┤
│ 棧深度             │ digitsSize       │ digitsSize       │
│ 每幀大小           │ 60 bytes         │ 16 bytes         │
│ 總棧使用           │ 60×digitsSize    │ 16×digitsSize    │
├────────────────────┼──────────────────┼──────────────────┤
│ digits="23"        │ 120 bytes        │ 32 bytes         │
│ digits="2"         │ 60 bytes         │ 16 bytes         │
│ digits="7979"      │ 240 bytes        │ 64 bytes         │
│ digits="7777"      │ 240 bytes        │ 64 bytes         │
├────────────────────┼──────────────────┼──────────────────┤
│ 返回地址開銷       │ 有 (8 bytes/幀)  │ 無               │
│ 函式調用開銷       │ 有               │ 無               │
│ CPU 快取友好       │ 差 (幀數據分散)  │ 好 (棧連續)      │
│ 棧溢出風險         │ 中 (digitsSize≤4)│ 低               │
│ 代碼複雜度         │ 簡潔             │ 複雜             │
│ 調試難度           │ 簡單             │ 困難             │
└────────────────────┴──────────────────┴──────────────────┘

棧記憶體節省: (60-16) × digitsSize = 44 × digitsSize bytes

例如 digitsSize=4: 節省 176 bytes ≈ 27%
```

## 8. 退化情況分析

### 最壞情況: 所有 7 或 9 (每個映射 4 個字母)

#### 情況描述

```
最壞輸入: digits="7777"

每個數字的映射:
digits[0]='7' → PHONE_MAP[7]="pqrs"  (4 個字母)
digits[1]='7' → PHONE_MAP[7]="pqrs"  (4 個字母)
digits[2]='7' → PHONE_MAP[7]="pqrs"  (4 個字母)
digits[3]='7' → PHONE_MAP[7]="pqrs"  (4 個字母)

組合數: 4^4 = 256 個組合

示例組合:
"pppp", "ppqp", "ppqr", "ppqs",
"pppp", "ppqp", "ppqr", "ppqs",
...
"szyz", ... (256 個)
```

#### 組合公式

```
組合總數 = ∏(len(PHONE_MAP[digits[i]])) for i in [0, digitsSize)

digits="7777":
  4 × 4 × 4 × 4 = 4^4 = 256 組合

digits="9999":
  4 × 4 × 4 × 4 = 4^4 = 256 組合

digits="23456789":  (所有數字混合)
  3 × 3 × 3 × 3 × 3 × 3 × 3 × 3 = 3^8 = 6561 組合
  (8位全是 2,3,4,5,6,8 等各 3 個字母的數字)
```

#### 記憶體開銷詳細分析

```
最壞情況: digits="7777" (256 個組合, 每個長度 5 including \0)

1. result[] 陣列:
   ┌─────────────────────────────────────┐
   │ result[0] → 0xC000                  │
   │ result[1] → 0xC004                  │
   │ ...                                 │
   │ result[255] → 0xC000 + 255×8        │
   │ = 256 × 8 bytes = 2048 bytes        │
   └─────────────────────────────────────┘

2. 組合字符串:
   ┌─────────────────────────────────────┐
   │ 每個組合: 4 字符 + '\0' = 5 bytes    │
   │ 256 個組合: 256 × 5 = 1280 bytes    │
   │                                     │
   │ 示例:                               │
   │ result[0] → "pppp" (5 bytes)        │
   │ result[1] → "ppqp" (5 bytes)        │
   │ ...                                 │
   │ result[255] → "szyz" (5 bytes)      │
   └─────────────────────────────────────┘

3. 總堆記憶體:
   result[] 指標陣列: 2048 bytes
   字符串數據: 1280 bytes
   ─────────────────────────
   總計: 3328 bytes

4. 棧記憶體 (path):
   path[] 大小: digitsSize + 1 = 4 + 1 = 5 bytes

5. 返回值:
   returnSize (int): 4 bytes

總記憶體: 3328 + 5 + 4 = 3337 bytes ≈ 3.3 KB
```

#### 時間複雜度分析

```
遞迴樹深度: digitsSize = 4
分支因數: 4 (對於 '7' 和 '9')

遞迴調用總數: 4 + 4^2 + 4^3 + 4^4 = 4 + 16 + 64 + 256 = 340 次

複製操作數: 256 次 (一次一個組合)
每次複製: strlen(path) 次位元組複製 + memcpy overhead

總時間複雜度: O(4^digitsSize × digitsSize)
            = O(組合數 × 平均字符串長度)

digits="7777" 時:
  時間 ≈ 256 × 4 = 1024 字符複製操作
```

#### 最壞情況下的遞迴樹圖

```
digits="7777" 的遞迴樹結構 (簡化):

                   recur(0)
                  /  /  \ \
                 p  q  r  s  (16 個分支)
                /│  ├─ q────┐
               / │  │       ├──r────┐
              p  q  r  s        s   (64 個分支)
             ... (256 個葉子節點，每個都執行複製)

            節點總數: 1 (深度0)
                     + 4 (深度1)
                     + 16 (深度2)
                     + 64 (深度3)
                     + 256 (深度4，葉子)
                     = 341 個節點

複製發生在所有葉子節點 (256 個)
```

#### 空間最佳化建議

```
對於超大輸入 (如 digits="79999999" 可能超 6 位):

1. 流式處理: 不存儲所有組合，而是逐個產生和處理
   記憶體: O(digitsSize) 而非 O(組合數)

2. 外部棧: 改用迭代 + 顯式棧
   減少遞迴棧幀的重複數據 (返回地址等)

3. 字符串池: 共享重複的子字符串
   (通常不適用，因為大多數組合不同)

4. 壓縮編碼: 每個組合編碼為整數索引
   記憶體: O(組合數) 個整數而非字符串
   當需要實際字符串時再解碼
```

## 9. 總結

### 關鍵記憶體特性

| 項目 | 詳細 |
|------|------|
| **靜態數據** | PHONE_MAP[10] = 80 bytes (指標) + ~40 bytes (字符串字面量) |
| **動態堆記憶體** | result[] + 組合字符串，取決於組合數 |
| **棧記憶體** | path[] (digitsSize+1 bytes) + 遞迴幀 (~60 bytes/幀) |
| **最大遞迴深度** | digitsSize (通常 ≤ 4) |
| **最壞情況** | digits="7777": 3328 bytes 堆 + 340 次遞迴調用 |
| **訪問模式** | 靜態數組：O(1) 直接索引，快取友好 |
| **優化策略** | 利用靜態數組常數時間映射，避免雜湊開銷 |
