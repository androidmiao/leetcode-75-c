# LeetCode 334: 遞增三元子序列 (Increasing Triplet Subsequence) 記憶體分析

## 1. 變數記憶體佈局

### 函式堆疊框架 (Stack Frame Layout)

```
棧頂 (Higher Address)
┌─────────────────────────────────┐
│     nums[i] 迴圈變數 (int)      │  0xF010  (4 bytes)
├─────────────────────────────────┤
│     second (int)                │  0xF00C  (4 bytes)
├─────────────────────────────────┤
│     first (int)                 │  0xF008  (4 bytes)
├─────────────────────────────────┤
│     nums_size (int)             │  0xF004  (4 bytes)
├─────────────────────────────────┤
│     nums (指標, 8 bytes)        │  0xF000  (8 bytes)
└─────────────────────────────────┘
棧底 (Lower Address)

記憶體配置:
- nums (0xF000 - 0xF007):        指向堆記憶體的指標 ──▶ 0xA000
- nums_size (0xF004 - 0xF007):   陣列大小
- first (0xF008 - 0xF00B):       追蹤最小的第一個數值
- second (0xF00C - 0xF00F):      追蹤最小的第二個數值
- i (0xF010 - 0xF013):          迴圈計數器

初始值:
  first = INT32_MAX (0x7FFFFFFF)
  second = INT32_MAX (0x7FFFFFFF)
  i = 0
```

## 2. 完整資料結構記憶體配置

### 堆記憶體中的陣列配置 (Example 3: [2,1,5,0,4,6])

```
堆記憶體 (Heap Memory)

┌────────────────────────────────────────────────────────────┐
│                  陣列基底地址: 0xA000                      │
├────────────────────────────────────────────────────────────┤
│ Index │ 記憶體位址 │ 值     │ 二進制表示 (32-bit)         │
├──────┼──────────┼──────┼─────────────────────────────────┤
│  [0] │ 0xA000   │  2   │ 0x00000002 ████ 32-bit int    │
├──────┼──────────┼──────┼─────────────────────────────────┤
│  [1] │ 0xA004   │  1   │ 0x00000001 ████ 32-bit int    │
├──────┼──────────┼──────┼─────────────────────────────────┤
│  [2] │ 0xA008   │  5   │ 0x00000005 ████ 32-bit int    │
├──────┼──────────┼──────┼─────────────────────────────────┤
│  [3] │ 0xA00C   │  0   │ 0x00000000 ████ 32-bit int    │
├──────┼──────────┼──────┼─────────────────────────────────┤
│  [4] │ 0xA010   │  4   │ 0x00000004 ████ 32-bit int    │
├──────┼──────────┼──────┼─────────────────────────────────┤
│  [5] │ 0xA014   │  6   │ 0x00000006 ████ 32-bit int    │
└──────┴──────────┴──────┴─────────────────────────────────┘

陣列元素計算公式:
  nums[i] 所在的記憶體位址 = 0xA000 + (i × 4)

  例如:
    nums[0] = 0xA000 + (0 × 4) = 0xA000
    nums[1] = 0xA000 + (1 × 4) = 0xA004
    nums[2] = 0xA000 + (2 × 4) = 0xA008
    nums[3] = 0xA000 + (3 × 4) = 0xA00C
    nums[4] = 0xA000 + (4 × 4) = 0xA010
    nums[5] = 0xA000 + (5 × 4) = 0xA014

指標指向關係:
  堆棧記憶體            堆記憶體
  ┌──────────┐
  │ nums     │
  │ 0xF000   │ ───▶ ┌────────────────┐
  └──────────┘      │ 0xA000: 2      │
                    │ 0xA004: 1      │
                    │ 0xA008: 5      │
                    │ 0xA00C: 0      │
                    │ 0xA010: 4      │
                    │ 0xA014: 6      │
                    └────────────────┘
```

## 5. 記憶體存取模式分析

### 快取友善性分析 (Cache-Friendly Analysis)

#### 實際方案 (優化版本)

```
┌─────────────────────────────────────────────────────────┐
│         時間複雜度: O(n)                                 │
│         空間複雜度: O(1)                                 │
└─────────────────────────────────────────────────────────┘

堆記憶體存取:
  ┌──────────────────────────────────┐
  │ 順序存取模式 (Sequential Access) │
  ├──────────────────────────────────┤
  │ i=0: 讀取 0xA000 ┐               │
  │ i=1: 讀取 0xA004 │ 連續地址      │
  │ i=2: 讀取 0xA008 │ 相差4字節    │
  │ i=3: 讀取 0xA00C │ 完美快取對齊 │
  │ i=4: 讀取 0xA010 │               │
  │ i=5: 讀取 0xA014 ┘               │
  └──────────────────────────────────┘

快取行為 (L1/L2/L3):
  • 典型快取行大小: 64 字節
  • 單個快取行可容納: 16 個 int (4 字節 × 16)
  • 存取模式: 純順序，無跳躍
  • 預取成功率: 100% (順序存取完全可預測)
  • 快取未命中: 最小化 (僅在快取行邊界處)

堆棧記憶體存取:
  ┌──────────────────────────────────┐
  │ 堆棧變數 (Stack Variables)        │
  ├──────────────────────────────────┤
  │ 只有 2 個額外變數被維護:          │
  │   • first  (0xF008)              │
  │   • second (0xF00C)              │
  ├──────────────────────────────────┤
  │ 迴圈變數 (Loop Variable)         │
  │   • i (0xF010)                   │
  ├──────────────────────────────────┤
  │ 這些變數位於同一快取行!           │
  │ 存取成本極低                      │
  └──────────────────────────────────┘

總體堆記憶體配置:
  ┌─────────────────────────────────────┐
  │ 堆棧 (Stack)      - 固定 20 字節   │
  │   無額外分配                        │
  ├─────────────────────────────────────┤
  │ 堆 (Heap)         - 24 字節(6×int) │
  │   輸入陣列指標傳入                  │
  └─────────────────────────────────────┘
  總計: 零額外堆分配 ✓
```

#### 壞方案 (低效版本)

```
┌─────────────────────────────────────────────────────────┐
│  假設方案: 使用兩個大小為 3 的陣列                      │
│     int candidates[3];  // 儲存候選值                   │
│     int positions[3];   // 儲存索引                      │
└─────────────────────────────────────────────────────────┘

堆棧記憶體浪費:
  ┌──────────────────────────────────────┐
  │ candidates[3]:  12 字節 (3 × 4 bytes)│
  ├──────────────────────────────────────┤
  │ positions[3]:   12 字節 (3 × 4 bytes)│
  ├──────────────────────────────────────┤
  │ 總額外堆棧:     24 字節              │
  └──────────────────────────────────────┘

堆記憶體存取複雜性:
  • 迴圈內記憶體寫入: 多個陣列元素更新
  • 快取污染: 兩個分離的陣列，快取行浪費
  • 指令數增加: 陣列索引計算，邊界檢查

  範例代碼:
    for (int i = 0; i < n; i++) {
        if (candidates[0] > nums[i]) {
            candidates[0] = nums[i];
            positions[0] = i;
        } else if (candidates[1] > nums[i]) {
            candidates[1] = nums[i];
            positions[1] = i;
        } else if (candidates[2] > nums[i]) {
            // ... 多個分支
        }
    }

    缺點:
      ✗ 每個檢查需要 2 個陣列查詢
      ✗ 非連續記憶體存取
```

### 迭代方案 - O(1) 堆疊

```
┌────────────────────────────────────────────────────────┐
│           當前方案: 迭代實現                           │
├────────────────────────────────────────────────────────┤
│  bool increasingTriplet(int* nums, int nums_size)     │
└────────────────────────────────────────────────────────┘

堆疊框架結構:

  ┌──────────────────────────────────────┐
  │  返回地址 (Return Address)            │  (8 bytes)
  │  之前的基指標 (Previous Base Pointer) │  (8 bytes)
  ├──────────────────────────────────────┤
  │  參數:                               │
  │    nums        (指標, 8 bytes) ──▶ 0xA000
  │    nums_size   (int, 4 bytes)   = 6
  ├──────────────────────────────────────┤
  │  本地變數:                           │
  │    first       (int, 4 bytes)   = 1
  │    second      (int, 4 bytes)   = 5
  │    i           (int, 4 bytes)   = 0
  ├──────────────────────────────────────┤
  │  堆棧使用總計: 48 字節               │
  │  其中本地變數: 12 字節               │
  └──────────────────────────────────────┘

迴圈執行時堆棧狀態:

  迭代 1:  ┌─────────────────┐
           │ first=INT32_MAX │
           │ second=INT32_MAX│
           │ i=0             │
           └─────────────────┘

  迭代 2:  ┌─────────────────┐
           │ first=1         │  ◄── first 更新
           │ second=INT32_MAX│
           │ i=1             │
           └─────────────────┘

  迭代 3:  ┌─────────────────┐
           │ first=1         │
           │ second=5        │  ◄── second 更新
           │ i=2             │
           └─────────────────┘

  迭代 4-6:
           ┌─────────────────┐
           │ first 或 second │  繼續更新或返回
           │ 可能更新        │
           │ i++             │
           └─────────────────┘

特點:
  ✓ 固定堆棧大小 (常數空間)
  ✓ 無遞歸呼叫
  ✓ 無迴圈堆棧增長
  ✓ 單層函式堆棧框架
```
