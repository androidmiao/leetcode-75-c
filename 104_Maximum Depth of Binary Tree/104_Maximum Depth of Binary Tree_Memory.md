## 3. 陣列索引版對應

### 完整陣列與索引映射

當樹以陣列形式表示 `[3,9,20,null,null,15,7]` 時：

| 索引 i | 節點值 | parent索引 | left子索引 | right子索引 | 說明          |
|-------|-------|-----------|----------|-----------|---------------|
| 0     | 3     | -         | 2*0+1=1  | 2*0+2=2   | root          |
| 1     | 9     | (i-1)/2=0 | 2*1+1=3  | 2*1+2=4   | left of root  |
| 2     | 20    | (i-1)/2=0 | 2*2+1=5  | 2*2+2=6   | right of root |
| 3     | null  | 1         | 2*3+1=7  | 2*3+2=8   | left of 9     |
| 4     | null  | 1         | 2*4+1=9  | 2*4+2=10  | right of 9    |
| 5     | 15    | (i-1)/2=2 | 2*5+1=11 | 2*5+2=12  | left of 20    |
| 6     | 7     | (i-1)/2=2 | 2*6+1=13 | 2*6+2=14  | right of 20   |

### 索引公式

- **左子節點索引：** `2*i + 1`
- **右子節點索引：** `2*i + 2`
- **父節點索引：** `(i - 1) / 2` (整數除法)

## 4. 搜尋／操作過程追蹤

### 範例 1: root = [3,9,20,null,null,15,7], 答案 = 3

```c
int maxDepth(struct TreeNode* root) {  // root @ 0xA000, val=3
    if (root == NULL) return 0;        // ✗ not NULL
    int leftDepth = maxDepth(root->left);    // 遞迴呼叫 0xB000
    int rightDepth = maxDepth(root->right);  // 遞迴呼叫 0xC000
    return (leftDepth > rightDepth ? leftDepth : rightDepth) + 1;
}
```

## 6. 記憶體存取模式比較

### 指標式樹 vs 陣列式樹

| 特性              | 指標式（Pointer）       | 陣列式（Array Index）  |
|------------------|----------------------|----------------------|
| **記憶體位址**    | 動態分配(heap)        | 連續(stack/heap)      |
| **節點存取延遲**  | 指標追蹤(cache miss)  | 直接索引(cache hit)   |
| **空間開銷**      | 指標額度: 2*8=16bytes | NULL節點: 4-8 bytes   |
| **插入/刪除**     | O(1)指標修改          | O(n)陣列搬移          |
| **遍歷順序**      | 任意(遞迴/迭代)       | 隱含層序              |
| **不平衡樹**      | 空間未浪費            | 多個NULL槽位          |
| **快取局部性**    | 差(指標跳轉)          | 優(連續索引)          |

### 快取行為分析

**指標式樹（Example 1）：**
```
訪問模式：
0xA000 (父) ──────────────▶ 0xB000 (子1)  [cache miss]
                              ▼
                           NULL (grand-child1) [miss]
                           NULL (grand-child2) [miss]
0xA000 (父) ──────────────▶ 0xC000 (子2)  [miss]
                              ▼
                           0xD000 (grand-child1) [miss]
                           0xE000 (grand-child2) [miss]

預期快取未命中：5~7次 (非連續記憶體)
```

**陣列式樹（Example 1）：**
```
訪問模式：
array[0] ──▶ array[1], array[2] (連續索引)
            ▼
         array[3..6] (連續索引區間)

預期快取命中：高 (連續記憶體)
```

- 總 stack 用量：3 × 24 = 72 bytes

### 迭代版本（Iterative） - 明確堆疊

```c
int maxDepth_iterative(struct TreeNode* root) {
    if (root == NULL) return 0;

    int depth = 0;
    struct TreeNode** stack[1000];  // explicit stack
    int top = 0;
    stack[top++] = root;

    while (top > 0) {
        int size = top;
        depth++;
        for (int i = 0; i < size; i++) {
            struct TreeNode* node = stack[--top];
            if (node->left) stack[top++] = node->left;
            if (node->right) stack[top++] = node->right;
        }
    }
    return depth;
}
```

**堆記憶體配置：**
```
Stack 陣列 @ 0x2000 (大小: 1000 * 8 bytes = 8KB)

┌─────────────────────────────────────┐
│ stack[0]: 0xA000 (root)             │
├─────────────────────────────────────┤
│ stack[1]: 0xB000 (1st left child)   │
├─────────────────────────────────────┤
│ stack[2]: 0xC000 (1st right child)  │
├─────────────────────────────────────┤
│ stack[3]: 0xD000 (2nd level left)   │
├─────────────────────────────────────┤
│ ...                                 │
└─────────────────────────────────────┘
```

### 遞迴 vs 迭代比較表

| 項目              | 遞迴（遞迴堆疊）      | 迭代（明確堆疊）    |
|------------------|------------------|-----------------|
| **堆疊使用**      | 系統 call stack  | 使用者 heap 堆疊 |
| **最壞情況**      | O(h) 深度h      | O(n/2) 葉節點數  |
| **額外空間**      | O(h) 遞迴深度    | O(w) 最大寬度    |
| **代碼複雜度**    | 簡潔易懂         | 較複雜           |
| **堆疊溢出風險**  | 高(極深樹)       | 低(heap充足)     |
| **快取效率**      | 差(堆疊跳轉)     | 可控(預分配)     |

**具體數字（Example 1，深度3寬度2）：**
- 遞迴：3 × 24B = 72B
- 迭代：最多儲存 2 節點指標 = 2 × 8B = 16B

```
索引: i  2i+1  2i+2  i-1

浪費槽位：[2, 4, 5, 7, 8] = 5個NULL
有效節點：[0, 1, 3, 6, 9] = 5個（散布在陣列中）
```

### 複雜度影響分析

| 指標    | 平衡二叉樹（AVL）  | 退化樹（全斜）    | 差異       |
|--------|-----------------|-----------------|-----------|
| 深度h  | log₂(n)         | n               | n倍       |
| 時間複雜度 | O(log n)      | O(n)            | n倍       |
| 空間複雜度 | O(log n)      | O(n)            | n倍       |
| 快取未命中 | ~log n 次     | ~n 次           | n倍       |
| 陣列浪費槽 | ~0%           | ~50%            | 浪費嚴重  |
| maxDepth呼叫 | 2n-1      | 2n-1            | 相同      |

### Example 1 退化變體追蹤

**全左傾版：[1,2,3,4,null,null,null]**

```c
maxDepth(0xA000) val=1
├─ maxDepth(0xB000) val=2
│  ├─ maxDepth(0xC000) val=3
│  │  ├─ maxDepth(0xD000) val=4
│  │  │  ├─ maxDepth(NULL) → return 0
│  │  │  └─ maxDepth(NULL) → return 0
│  │  │  return 0 + 1 = 1
│  │  └─ maxDepth(NULL) → return 0
│  │  return 1 + 1 = 2
│  └─ maxDepth(NULL) → return 0
│  return 2 + 1 = 3
└─ maxDepth(NULL) → return 0
return 3 + 1 = 4

遞迴深度：4 層（全線性）
Call stack 最大寬度：4 frame
```

## 總結

### 核心概念

1. **struct TreeNode** 占用 24 bytes (64位)：
   - val (int): 4 bytes
   - padding: 4 bytes
   - left/right (pointer): 各 8 bytes

2. **DFS 遞迴遍歷**：
   - 時間複雜度：O(n) - 訪問所有節點
   - 空間複雜度：O(h) - 遞迴深度
   - 最壞情況：O(n) 退化樹

3. **指標導航**：
   - 每次走訪讀取 24 bytes 節點
   - 指標跳轉導致 cache miss 增加
   - NULL 檢查占總呼叫數的比例

4. **陣列表示**：
   - 索引公式：左=2i+1，右=2i+2
   - 優點：連續記憶體、快速存取
   - 缺點：不平衡樹時浪費槽位 (~50%)

5. **堆疊使用**：
   - 遞迴：系統 call stack (風險: overflow)
   - 迭代：自訂 heap stack (控制性佳)
