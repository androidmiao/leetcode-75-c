# 122. Best Time to Buy and Sell Stock II — 記憶體與陣列索引圖解

> 以下所有記憶體位址均為假設值，用於教學說明。

## 1. 陣列記憶體佈局

`int prices[]` 在記憶體中為連續的 4-byte 整數。

以 `prices = [7, 1, 5, 3, 6, 4]`（pricesSize = 6）為例：

```
假設基底位址: 0xA000

位址        索引    值     說明
┌──────────┬──────┬──────┬──────────────────────┐
│ 0xA000   │ [0]  │  7   │ prices[0] = 7        │
├──────────┼──────┼──────┼──────────────────────┤
│ 0xA004   │ [1]  │  1   │ prices[1] = 1        │
├──────────┼──────┼──────┼──────────────────────┤
│ 0xA008   │ [2]  │  5   │ prices[2] = 5        │
├──────────┼──────┼──────┼──────────────────────┤
│ 0xA00C   │ [3]  │  3   │ prices[3] = 3        │
├──────────┼──────┼──────┼──────────────────────┤
│ 0xA010   │ [4]  │  6   │ prices[4] = 6        │
├──────────┼──────┼──────┼──────────────────────┤
│ 0xA014   │ [5]  │  4   │ prices[5] = 4        │
└──────────┴──────┴──────┴──────────────────────┘

位址計算公式: &prices[i] = base + i * sizeof(int) = 0xA000 + i * 4
```

### 快取行分析

```
快取行大小：64 bytes = 16 個 int

┌─── 快取行 0（0xA000 ~ 0xA03F）─────────────────────┐
│ prices[0..5] 全部在同一條快取行內                      │
│ [7][1][5][3][6][4][?][?][?][?][?][?][?][?][?][?]   │
└──────────────────────────────────────────────────────┘

→ 整個陣列只需 1 次快取未命中（cold miss），後續全部命中
→ 順序存取 = 完美的空間局部性，硬體 prefetcher 可預測
```

## 2. 搜尋／操作過程追蹤

### 範例 1：`prices = [7, 1, 5, 3, 6, 4]`

#### 變數初始狀態

```
max_profit = 0
i = 1（從第 1 個元素開始）
```

#### 步驟追蹤

```
步驟 1: i=1
  指標位置: &prices[1]=0xA004, &prices[0]=0xA000
  ┌──────┬──────┬──────┬──────┬──────┬──────┐
  │  7   │ [1]  │  5   │  3   │  6   │  4   │
  └──────┴──▲───┴──────┴──────┴──────┴──────┘
         prices[i]
  比較: prices[1](1) > prices[0](7)? → NO
  動作: 跳過（下跌段）
  max_profit = 0

步驟 2: i=2
  指標位置: &prices[2]=0xA008, &prices[1]=0xA004
  ┌──────┬──────┬──────┬──────┬──────┬──────┐
  │  7   │  1   │ [5]  │  3   │  6   │  4   │
  └──────┴──────┴──▲───┴──────┴──────┴──────┘
              prices[i]
  比較: prices[2](5) > prices[1](1)? → YES
  動作: max_profit += 5 - 1 = 4
  max_profit = 4

步驟 3: i=3
  指標位置: &prices[3]=0xA00C, &prices[2]=0xA008
  ┌──────┬──────┬──────┬──────┬──────┬──────┐
  │  7   │  1   │  5   │ [3]  │  6   │  4   │
  └──────┴──────┴──────┴──▲───┴──────┴──────┘
                     prices[i]
  比較: prices[3](3) > prices[2](5)? → NO
  動作: 跳過（下跌段）
  max_profit = 4

步驟 4: i=4
  指標位置: &prices[4]=0xA010, &prices[3]=0xA00C
  ┌──────┬──────┬──────┬──────┬──────┬──────┐
  │  7   │  1   │  5   │  3   │ [6]  │  4   │
  └──────┴──────┴──────┴──────┴──▲───┴──────┘
                            prices[i]
  比較: prices[4](6) > prices[3](3)? → YES
  動作: max_profit += 6 - 3 = 3
  max_profit = 7

步驟 5: i=5
  指標位置: &prices[5]=0xA014, &prices[4]=0xA010
  ┌──────┬──────┬──────┬──────┬──────┬──────┐
  │  7   │  1   │  5   │  3   │  6   │ [4]  │
  └──────┴──────┴──────┴──────┴──────┴──▲───┘
                                   prices[i]
  比較: prices[5](4) > prices[4](6)? → NO
  動作: 跳過（下跌段）
  max_profit = 7
```

#### 指標移動摘要

```
┌───────────────────────────────────────┐
│ i: 1 → 2 → 3 → 4 → 5 → (結束)       │
│ 位址: 0xA004 → 0xA008 → 0xA00C       │
│       → 0xA010 → 0xA014              │
│ 每步偏移: +4 bytes (sizeof(int))      │
└───────────────────────────────────────┘
```

### 範例 2：`prices = [1, 2, 3, 4, 5]`

假設基底位址: 0xB000

```
位址        索引    值
┌──────────┬──────┬──────┐
│ 0xB000   │ [0]  │  1   │
│ 0xB004   │ [1]  │  2   │
│ 0xB008   │ [2]  │  3   │
│ 0xB00C   │ [3]  │  4   │
│ 0xB010   │ [4]  │  5   │
└──────────┴──────┴──────┘
```

#### 步驟追蹤

```
步驟 1: i=1, prices[1](2) > prices[0](1)? YES → +1, profit=1
步驟 2: i=2, prices[2](3) > prices[1](2)? YES → +1, profit=2
步驟 3: i=3, prices[3](4) > prices[2](3)? YES → +1, profit=3
步驟 4: i=4, prices[4](5) > prices[3](4)? YES → +1, profit=4

→ 每步都累加，最終 profit = 4  ✓
```

### 範例 3：`prices = [7, 6, 4, 3, 1]`

假設基底位址: 0xC000

```
步驟 1: i=1, prices[1](6) > prices[0](7)? NO  → skip
步驟 2: i=2, prices[2](4) > prices[1](6)? NO  → skip
步驟 3: i=3, prices[3](3) > prices[2](4)? NO  → skip
步驟 4: i=4, prices[4](1) > prices[3](3)? NO  → skip

→ 沒有任何正差價，最終 profit = 0  ✓
```

## 3. 指標變數追蹤總表

### 範例 1：`prices = [7, 1, 5, 3, 6, 4]`

| 步驟 | i | &prices[i] | prices[i] | prices[i-1] | 差值 | 動作 | max_profit |
|------|---|------------|-----------|-------------|------|------|------------|
| 1 | 1 | 0xA004 | 1 | 7 | -6 | skip | 0 |
| 2 | 2 | 0xA008 | 5 | 1 | +4 | add | 4 |
| 3 | 3 | 0xA00C | 3 | 5 | -2 | skip | 4 |
| 4 | 4 | 0xA010 | 6 | 3 | +3 | add | 7 |
| 5 | 5 | 0xA014 | 4 | 6 | -2 | skip | 7 |

**統計：** 5 次比較，2 次累加，5 次記憶體讀取（每步讀 prices[i]，prices[i-1] 在上一步已讀取可快取重用）

### 範例 2：`prices = [1, 2, 3, 4, 5]`

| 步驟 | i | &prices[i] | prices[i] | prices[i-1] | 差值 | 動作 | max_profit |
|------|---|------------|-----------|-------------|------|------|------------|
| 1 | 1 | 0xB004 | 2 | 1 | +1 | add | 1 |
| 2 | 2 | 0xB008 | 3 | 2 | +1 | add | 2 |
| 3 | 3 | 0xB00C | 4 | 3 | +1 | add | 3 |
| 4 | 4 | 0xB010 | 5 | 4 | +1 | add | 4 |

**統計：** 4 次比較，4 次累加（全部上漲 — 分支預測器可完美預測）

### 範例 3：`prices = [7, 6, 4, 3, 1]`

| 步驟 | i | &prices[i] | prices[i] | prices[i-1] | 差值 | 動作 | max_profit |
|------|---|------------|-----------|-------------|------|------|------------|
| 1 | 1 | 0xC004 | 6 | 7 | -1 | skip | 0 |
| 2 | 2 | 0xC008 | 4 | 6 | -2 | skip | 0 |
| 3 | 3 | 0xC00C | 3 | 4 | -1 | skip | 0 |
| 4 | 4 | 0xC010 | 1 | 3 | -2 | skip | 0 |

**統計：** 4 次比較，0 次累加（全部下跌 — 分支預測器可完美預測）

## 4. 記憶體存取模式比較

本題只使用陣列（無指標結構），因此不存在「指標版 vs. 陣列版」的對比。但可以比較不同陣列存取模式：

| 存取模式 | 說明 | 快取行為 |
|---|---|---|
| 順序存取（本解法） | `prices[0], prices[1], ..., prices[n-1]` | 最佳：完美空間局部性，prefetcher 可預測 |
| 隨機存取（若需要） | 按交易對跳躍存取 | 較差：可能跨快取行 |
| 兩次遍歷 | 先找波谷再找波峰（峰谷法） | 相同快取行為但增加一層迴圈 |

### 暫存器使用分析

```
演算法只需要以下暫存器：
┌─────────────┬──────────────────────────────┐
│ 暫存器      │ 用途                          │
├─────────────┼──────────────────────────────┤
│ R1          │ i（迴圈計數器）               │
│ R2          │ max_profit（累計利潤）         │
│ R3          │ pricesSize（陣列大小）         │
│ R4          │ prices（基底位址指標）         │
│ R5          │ prices[i]（暫存當前值）        │
│ R6          │ prices[i-1]（暫存前一值）      │
│ R7          │ 差值暫存                      │
└─────────────┴──────────────────────────────┘

→ 7 個暫存器，遠低於通用 CPU 的可用暫存器數量
→ 不會產生暫存器溢出（register spill）到堆疊
```

## 5. 堆疊記憶體分析

本解法為純迭代（無遞迴），堆疊使用量固定：

```
呼叫 maxProfit() 時的堆疊框架：

高位址
┌──────────────────────────────┐
│ 返回位址 (8 bytes)            │  ← 呼叫者的下一條指令
├──────────────────────────────┤
│ 舊 frame pointer (8 bytes)   │
├──────────────────────────────┤
│ pricesSize (4 bytes)         │  ← 參數（可能在暫存器）
├──────────────────────────────┤
│ *prices (8 bytes)            │  ← 參數（可能在暫存器）
├──────────────────────────────┤
│ max_profit (4 bytes)         │  ← 區域變數
├──────────────────────────────┤
│ i (4 bytes)                  │  ← 區域變數
├──────────────────────────────┤
│ padding (4 bytes)            │  ← 對齊
└──────────────────────────────┘
低位址

堆疊總用量: ~40 bytes（固定，不隨 n 變化）
→ O(1) 空間複雜度 ✓
```

### 與暴力遞迴法的堆疊對比

```
暴力法（Approach 1）的遞迴深度可達 n：

高位址
┌────────────────────┐
│ calculate(prices,0) │  ← 第 0 層
├────────────────────┤
│ calculate(prices,1) │  ← 第 1 層
├────────────────────┤
│ calculate(prices,2) │  ← 第 2 層
├────────────────────┤
│       ...          │
├────────────────────┤
│ calculate(prices,n) │  ← 第 n 層
└────────────────────┘

堆疊總用量: ~40n bytes → O(n) 空間
當 n = 30000（最大限制）時: ~1.2 MB
→ 可能導致 Stack Overflow！
```
