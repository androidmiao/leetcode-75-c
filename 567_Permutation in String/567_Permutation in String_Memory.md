## 1. 區域變數記憶體佈局

### 棧幀結構 (假設基址 0x7FFE0000)

```
棧成長方向: ▼

0x7FFE0000  ┌─────────────────────────────────┐
            │     l1 (int, 4 bytes)           │  偏移: 0x00
            ├─────────────────────────────────┤
0x7FFE0004  │     l2 (int, 4 bytes)           │  偏移: 0x04
            ├─────────────────────────────────┤
0x7FFE0008  │     count1[26] (26×int=104B)    │  偏移: 0x08～0x6F
            │     ├─ count1[0]:  0x7FFE0008   │
            │     ├─ count1[1]:  0x7FFE000C   │
            │     ├─ count1[25]: 0x7FFE006C   │
            ├─────────────────────────────────┤
0x7FFE0070  │     count2[26] (26×int=104B)    │  偏移: 0x70～0xD7
            │     ├─ count2[0]:  0x7FFE0070   │
            │     ├─ count2[1]:  0x7FFE0074   │
            │     ├─ count2[25]: 0x7FFE00D4   │
            ├─────────────────────────────────┤
0x7FFE00D8  │     matches (int, 4 bytes)      │  偏移: 0xD8
            ├─────────────────────────────────┤
0x7FFE00DC  │     r (int, 4 bytes)            │  偏移: 0xDC
            ├─────────────────────────────────┤
0x7FFE00E0  │     in (int, 4 bytes)           │  偏移: 0xE0
            ├─────────────────────────────────┤
0x7FFE00E4  │     out (int, 4 bytes)          │  偏移: 0xE4
            ├─────────────────────────────────┤
0x7FFE00E8  │     返回地址 / 棧頂                │
            └─────────────────────────────────┘

總棧幀大小: 232 bytes (0xE8)
```

### count1[26] 和 count2[26] 記憶體佈局

```
字元對應: 'a'=0, 'b'=1, 'c'=2, ..., 'z'=25

count1 陣列 (相對基址 0x7FFE0008):
┌──────┬──────┬──────┬──────┬─ ... ─┬──────┬──────┐
│ [0]  │ [1]  │ [2]  │ [3]  │       │ [24] │ [25] │  ('a'~'z')
├──────┼──────┼──────┼──────┼─ ... ─┼──────┼──────┤
│0x08  │0x0C  │0x10  │0x14  │ ...   │0xC8  │0xCC  │  (4字節間隔)
└──────┴──────┴──────┴──────┴─ ... ─┴──────┴──────┘
 'a'    'b'    'c'    'd'    ...    'y'    'z'

count2 陣列 (相對基址 0x7FFE0070):
┌──────┬──────┬──────┬──────┬─ ... ─┬──────┬──────┐
│ [0]  │ [1]  │ [2]  │ [3]  │       │ [24] │ [25] │  ('a'~'z')
├──────┼──────┼──────┼──────┼─ ... ─┼──────┼──────┤
│0x70  │0x74  │0x78  │0x7C  │ ...   │0x130 │0x134 │  (4字節間隔)
└──────┴──────┴──────┴──────┴─ ... ─┴──────┴──────┘
 'a'    'b'    'c'    'd'    ...    'y'    'z'
```

### Example 1 初始化: s1="ab", s2="eidbaooo"

**初始化後的陣列狀態:**

```
s1 = "ab" (長度 2)
s2 = "eidbaooo" (長度 8)

初始化迴圈後 (i = 0,1):
i=0: s1[0]='a' ───▶ count1[0]++  (count1[0]=1)
     s2[0]='e' ───▶ count2[4]++  (count2[4]=1)
i=1: s1[1]='b' ───▶ count1[1]++  (count1[1]=1)
     s2[1]='i' ───▶ count2[8]++  (count2[8]=1)

count1[26] 初始狀態:
┌────┬────┬────┬────┬────┬────┬─ ... ─┬────┐
│ 1  │ 1  │ 0  │ 0  │ 0  │ 0  │ ...   │ 0  │
└────┴────┴────┴────┴────┴────┴─ ... ─┴────┘
 [0] [1] [2] [3] [4] [5]        [25]
 'a'  'b'  'c'  'd'  'e'  'f'    'z'

count2[26] 初始狀態:
┌────┬────┬────┬────┬────┬────┬─ ... ─┬────┐
│ 0  │ 0  │ 0  │ 0  │ 1  │ 0  │ ...   │ 0  │  (count2[4]=1 for 'e')
├────┼────┼────┼────┼────┼────┼─ ... ─┼────┤
│ 0  │ 0  │ 0  │ 0  │ 1  │ 0  │ ...   │ 0  │  (+ count2[8]=1 for 'i')
└────┴────┴────┴────┴────┴────┴─ ... ─┴────┘
 [0] [1] [2] [3] [4] [5]        [25]
 'a'  'b'  'c'  'd'  'e'  'f'    'z'
```

**matches 計算:**
```
迴圈 i=0..25: 只有 count1[0]==count2[0]? (1==0? 否)
             count1[1]==count2[1]? (1==0? 否)
             其餘都相等 (0==0)

matches = 24 (因為位置 0 和 1 不匹配)
```

## 4. 滑動視窗過程追蹤 (Example 1)

### Example 1: s1="ab", s2="eidbaooo"

```
l1 = 2, l2 = 8
檢查: l1 > l2? (2 > 8? 否)

═══════════════════════════════════════════════════════

【初始化階段】
迴圈 i=0..1 (窗口初始化):
  i=0: s1[0]='a'(0x61) ───▶ count1[0]++  → count1[0]=1
       s2[0]='e'(0x65) ───▶ count2[4]++  → count2[4]=1
  i=1: s1[1]='b'(0x62) ───▶ count1[1]++  → count1[1]=1
       s2[1]='i'(0x69) ───▶ count2[8]++  → count2[8]=1

count1: [1,1,0,0,0,...]  (只有 [0]=1 for 'a', [1]=1 for 'b')
count2: [0,0,0,0,1,0,0,0,1,0,...] (count2[4]=1 for 'e', count2[8]=1 for 'i')

matches 計算:
┌─────────────────────────────────────────┐
│ 迴圈 i=0..25:                           │
│   i=0: count1[0]==count2[0]? 1==0? ✗   │
│   i=1: count1[1]==count2[1]? 1==0? ✗   │
│   i=2..25: count1[i]==count2[i]? 0==0? ✓ (24個)│
├─────────────────────────────────────────┤
│ matches = 24                            │
└─────────────────────────────────────────┘

═══════════════════════════════════════════════════════

【滑動視窗迴圈】

────────────────────────────────────────────────────────
步驟 1: r=2
────────────────────────────────────────────────────────

s2 視窗狀態:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│'e'  │'i'  │'d' ◀▶ 'b' │'a' │'o' │'o' │'o' │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
[0]   [1]   [2]   [3]   [4]   [5]   [6]   [7]
           ▲
      進入此位置

檢查: matches == 26? (24 == 26? 否) ──▶ 繼續

進入字元操作:
  in = s2[2] - 'a' = 'd' - 'a' = 3
  out = s2[2-2] - 'a' = s2[0] - 'a' = 'e' - 'a' = 4

count2[3]++ (進入 'd'):
  count2[3] 從 0 → 1
  count2[3] == count1[3]? (1 == 0? 否)
  count2[3] == count1[3]+1? (1 == 1? ✓) ──▶ matches--
  matches: 24 → 23

count2[4]-- (離開 'e'):
  count2[4] 從 1 → 0
  count2[4] == count1[4]? (0 == 0? ✓) ──▶ matches++
  matches: 23 → 24

狀態: count2[3]=1, count2[4]=0
      matches=24, 窗口="id"

────────────────────────────────────────────────────────
步驟 2: r=3
────────────────────────────────────────────────────────

s2 視窗狀態:
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│'e'  │'i'  │'d'  │'b' ◀▶'a' │'o' │'o' │'o' │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
[0]   [1]   [2]   [3]   [4]   [5]   [6]   [7]
                  ▲
             進入此位置

檢查: matches == 26? (24 == 26? 否) ──▶ 繼續

進入字元操作:
  in = s2[3] - 'a' = 'b' - 'a' = 1
  out = s2[3-2] - 'a' = s2[1] - 'a' = 'i' - 'a' = 8

count2[1]++ (進入 'b'):
  count2[1] 從 0 → 1
  count2[1] == count1[1]? (1 == 1? ✓) ──▶ matches++
  matches: 24 → 25

count2[8]-- (離開 'i'):
  count2[8] 從 1 → 0
  count2[8] == count1[8]? (0 == 0? ✓) ──▶ matches++
  matches: 25 → 26

狀態: count2[1]=1, count2[8]=0
      matches=26 ✓✓✓ 發現排列!
      窗口="db"

【返回 true】

════════════════════════════════════════════════════════
結論: s1="ab" 的排列 "ba" 在 s2 的位置 [3..4] 找到
════════════════════════════════════════════════════════
```

## 7. 記憶體存取模式分析

### 快取行為分析

#### 頻率陣列的快取特性

```
count1[26] 和 count2[26] 大小:
  26 個 int × 4 bytes/int = 104 bytes

典型 L1 快取行大小: 64 bytes
所以兩個陣列各自跨越約 2 個快取行:
  count1: 快取行 1 (0x7FFE0008～0x7FFE0047) + 快取行 2 (0x7FFE0048～0x7FFE006F)
  count2: 快取行 3 (0x7FFE0070～0x7FFE00AF) + 快取行 4 (0x7FFE00B0～0x7FFE00D7)
```

#### 空間局部性 (Spatial Locality)

```
【良好的空間局部性】
─────────────────────────────────────────
迴圈中的陣列訪問模式:

初始化迴圈:
  for i in 0..l1-1:
    count1[s1[i]-'a']++  ◄── 頻率陣列順序訪問
    count2[s2[i]-'a']++  ◄──

滑動視窗迴圈:
  count2[in]++   ◄── 隨機訪問 (依賴 s2[r] 的值)
  count2[out]--  ◄── 隨機訪問 (依賴 s2[r-l1] 的值)

【字串訪問的時間局部性】
順序訪問 s2[r] 和 s2[r-l1]:
  每次迭代都訪問相近的記憶體位置
  ▼
  進入快取後，下一次迭代也會命中快取
  (假設字串長度不超過快取大小)
```

#### 快取命中率估計

```
假設: L1 快取容量 32 KB, 字串 s2 長度 8

【初始化階段】
count1[0..25] 首次訪問: 26 次快取未命中
count2[0..25] 首次訪問: 26 次快取未命中 (同一快取行，命中率高)
總未命中: ~52

【滑動視窗階段】
主迴圈執行 l2-l1 = 8-2 = 6 次

每次迭代:
  count2[in] 訪問: 已在快取中 ──▶ 快取命中
  count2[out] 訪問: 已在快取中 ──▶ 快取命中
  s2[r] 訪問: 順序存取 ──▶ 快取命中
  s2[r-l1] 訪問: 順序存取 ──▶ 快取命中

滑動視窗階段命中率: ~100% (6 次迭代全部命中)

總命中率: (52 + 6) / (52 + 6) 很高 ✓
```

#### 記憶體帶寬利用

```
【數據流】
讀取:
  ├─ s1 字串: 順序讀, l1 字節
  ├─ s2 字串: 順序讀, l2 字節
  └─ count1[26] 陣列: 隨機讀, 最多 26 次

寫入:
  ├─ count1[26] 陣列: 順序寫, 26 次 (初始化)
  ├─ count2[26] 陣列: 順序寫, l1 次 (初始化) + 隨機寫, 2(l2-l1) 次 (滑動)
  └─ matches 計數器: 少量寫, ~20 次

【帶寬效率】
初始化階段: 高效 (順序訪問)
滑動階段: 高效 (快取駐留 + 少量訪問)
整體: 帶寬利用率 中等偏高 (主要瓶頸是 count1/count2 的隨機訪問)
```

## 8. 棧記憶體使用

### 完整棧幀大小

```
組成部分:
┌────────────────────────────────────────────┐
│ l1 (4B) + l2 (4B) = 8B                     │ 區域整數變數
├────────────────────────────────────────────┤
│ count1[26] (104B) + count2[26] (104B) = 208B│ 頻率陣列
├────────────────────────────────────────────┤
│ matches (4B) + r (4B) + in (4B) + out (4B)  │ 迴圈變數
│ = 16B                                      │
└────────────────────────────────────────────┘

總棧幀大小: 8 + 208 + 16 = 232 bytes (0xE8)
```

### 與遞迴方案對比

```
【此題為何採用迭代而非遞迴】
─────────────────────────────────────────

該問題天然適合迭代的原因:
┌─────────────────────────────────────────────┐
│ 1. 問題無子問題結構                         │
│    ├─ 不存在"求更小規模的排列"的概念      │
│    └─ 滑動視窗是線性轉移，不是遞迴分解   │
├─────────────────────────────────────────────┤
│ 2. 狀態轉移簡單 (O(1) 時間)                 │
│    ├─ 只需更新 count2 和 matches            │
│    ├─ 無需回溯 (backtrack)                  │
│    └─ 無需跨函數調用恢復狀態                │
├─────────────────────────────────────────────┤
│ 3. 記憶體效率                               │
│    ├─ 迭代: 單一棧幀 232B                   │
│    └─ 遞迴: 每層棧幀 232B × l2 層 = 兆位元組│
│           (若採用遞迴求解 s2 的每個子串)  │
├─────────────────────────────────────────────┤
│ 4. 無深度問題                               │
│    ├─ 迭代層數 = l2-l1 (通常 < 10,000)     │
│    └─ 遞迴深度若為 l2，則易棧溢位          │
└─────────────────────────────────────────────┘
```

### 假設遞迴實現的棧成本

```
假想遞迴簽名:
bool checkInclusion_recursive(char* s1, char* s2, int pos,
                               int count1[26], int count2[26])

每層調用需要:
  ├─ 參數: s1(指標), s2(指標), pos(int), count1(指標), count2(指標) = ~20B
  ├─ 返回地址: ~8B
  ├─ 本地變數: l1, l2, in, out, matches = ~20B
  └─ 對齐填充: ~8B
  = 每層約 56B (在 x64 上)

若遞迴深度 = l2 - l1:
  Example: l2=8, l1=2 ──▶ 深度 6 ──▶ 棧使用 56×6 = 336B (可接受)
  但若 l2=1,000,000, l1=2 ──▶ 深度 999,998 ──▶ 棧使用 ~56MB ──▶ 棧溢位!

結論: 迭代設計對此問題是必需的，不是選項。
```

## 總結

此演算法展示了:

1. **固定大小滑動視窗** - 窗口大小始終為 l1
2. **頻率匹配追蹤** - 用 matches 計數器優化匹配檢測 (O(1) vs O(26))
3. **增量更新邏輯** - 只更新進入/離開元素的計數
4. **記憶體局部性** - 頻率陣列的快取友好性

**時間複雜度**: O(l2) - 單次遍歷 s2
**空間複雜度**: O(1) - 固定 52 個整數 + 幾個基本變數
**棧使用**: 232 bytes - 小且可預測
