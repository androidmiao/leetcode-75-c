# LeetCode 問題 3：最長無重複字元子字串 — 記憶體分析

## 1. Struct 記憶體佈局

此題使用堆疊變數而非結構體。以下為 `lengthOfLongestSubstring()` 函數的堆疊框架配置：

```
堆疊框架佈局 (假設基礎位址 0xFFE0)
┌────────────────────────────────────────────────────────────┐
│                       堆疊記憶體配置                           │
├────────────────────────────────────────────────────────────┤
│ 位址範圍          │ 大小      │ 變數名            │ 說明      │
├────────────────────────────────────────────────────────────┤
│ 0xFD80 - 0xFE7F  │ 512 bytes │ lastSeen[128]     │ 陣列      │
├────────────────────────────────────────────────────────────┤
│ 0xFE80 - 0xFE83  │ 4 bytes   │ left              │ int       │
├────────────────────────────────────────────────────────────┤
│ 0xFE84 - 0xFE87  │ 4 bytes   │ maxLen            │ int       │
├────────────────────────────────────────────────────────────┤
│ 0xFE88 - 0xFE8B  │ 4 bytes   │ right             │ int       │
├────────────────────────────────────────────────────────────┤
│ 0xFE8C - 0xFE8F  │ 4 bytes   │ i (迴圈計數器)     │ int       │
├────────────────────────────────────────────────────────────┤

├────────────────────────────────────────────────────────────┤

├────────────────────────────────────────────────────────────┤

```

  lastSeen[97] ('a') = 0xFD80 + (97 × 4) = 0xFD80 + 388 = 0xFDC4

## 3. 陣列索引版對應

### 3.1 ASCII 值直接索引對應

lastSeen[] 陣列使用 ASCII 值作為直接索引，無需雜湊函數：

```
字符到陣列索引對應表
┌──────────────────────────────────────────────┐
│ 字符 │ ASCII 值 │ lastSeen 陣列索引 │ 位址  │
├──────────────────────────────────────────────┤
│ NUL  │ 0        │ lastSeen[0]       │ 0xFD80│
│ ...  │ ...      │ ...               │ ...   │
│ ' '  │ 32       │ lastSeen[32]      │ 0xFD80│
│ ...  │ ...      │ ...               │ ...   │
│ 'a'  │ 97       │ lastSeen[97]      │ 0xFDC4│
│ 'b'  │ 98       │ lastSeen[98]      │ 0xFDC8│
│ 'c'  │ 99       │ lastSeen[99]      │ 0xFDCC│
│ 'd'  │ 100      │ lastSeen[100]     │ 0xFDD0│
│ 'e'  │ 101      │ lastSeen[101]     │ 0xFDD4│
│ ...  │ ...      │ ...               │ ...   │
│ 'k'  │ 107      │ lastSeen[107]     │ 0xFDEC│
│ ...  │ ...      │ ...               │ ...   │
│ 'p'  │ 112      │ lastSeen[112]     │ 0xFE00│
│ ...  │ ...      │ ...               │ ...   │
│ 'w'  │ 119      │ lastSeen[119]     │ 0xFE1C│
│ ...  │ ...      │ ...               │ ...   │
│ DEL  │ 127      │ lastSeen[127]     │ 0xFE80│
└──────────────────────────────────────────────┘

索引計算公式：
  陣列位址 = 基礎位址 + (ASCII值 × 元素大小)
  = 0xFD80 + (ASCII值 × 4)

範例：
  lastSeen['a'] = 0xFD80 + (97 × 4) = 0xFD80 + 0xEC = 0xFDC4
  lastSeen['b'] = 0xFD80 + (98 × 4) = 0xFD80 + 0xF0 = 0xFDC8
```

### 3.2 與雜湊表對比的索引關係

```
直接陣列索引 vs 雜湊表索引
┌─────────────────────────────────────────────────────────────┐
│ 方法          │ 索引機制          │ 衝突処理      │ 時間複雜度 │
├─────────────────────────────────────────────────────────────┤
│ 直接陣列      │ ASCII值 × 4       │ 無(固定位址)   │ O(1)       │
│ (本題方案)    │ (直接位址計算)    │              │            │
├─────────────────────────────────────────────────────────────┤
│ 雜湊表        │ hash(char) mod N  │ 鏈式/開放尋址  │ O(1) 攤銷   │
│               │ (間接尋址)        │              │            │
└─────────────────────────────────────────────────────────────┘

優勢：
  ✓ 直接計算位址，零衝突
  ✓ 快取局部性優秀（512 bytes 連續記憶體）
  ✓ 無需雜湊函數計算
  ✓ 無需衝突解決開銷
```

## 5. 指標變數追蹤總表

## 6. 記憶體存取模式比較

### 6.1 直接陣列存取 vs 雜湊表存取

```
┌─────────────────────────────────────────────────────────────┐
│           直接陣列存取 vs 雜湊表存取比較                      │
├─────────────────────────────────────────────────────────────┤

【方案 1】直接陣列 int[128] (本題採用)
─────────────────────────────────────────

記憶體配置：
  基礎位址：0xFD80
  大小：512 bytes (連續)
  每個元素：4 bytes (int)

存取流程 (字符 'a' 為例)：
  1. 獲取 ASCII 值：97
  2. 計算位址：0xFD80 + (97 × 4) = 0xFDC4
  3. 讀取值：lastSeen[97] = ?

  耗時：3 個 CPU 週期 (乘法 + 加法 + 記憶體讀取)

快取行為：
  ┌──────────────────────────────┐
  │ L1 快取 (32 KB)               │
  │  ├─ 512 byte 連續陣列        │  ◄ 完全駐留在 L1
  │  └─ 一次讀取可載入多個元素    │
  └──────────────────────────────┘

命中率：100% (L1 快取)
平均延遲：4-5 ns (L1 大約 4 ns)

優勢：
  ✓ 零衝突碰撞
  ✓ 預測性存取模式
  ✓ 優秀快取局部性
  ✓ 無分支預測誤算


【方案 2】雜湊表 (假設實現)
─────────────────────────────────────────

結構體記憶體配置 (每個節點)：
  struct HashNode {
      int key;        // 4 bytes (字元 ASCII)
      int value;      // 4 bytes (上次位置)
      struct HashNode* next;  // 8 bytes (指標)
  } = 16 bytes per node

記憶體分散分佈：
  ┌──────────────────┐
  │ 雜湊表陣列       │
  ├──────────────────┤
  │ bucket[0] ──────┐
  │ bucket[1] ──┐   │
  │ ...         │   │
  │ bucket[N]   │   │
  └──────────────┼───┼──────┐
                │   │      │
                ▼   ▼      ▼
            ┌──────┐  ┌──────┐  ┌──────┐
            │ Node │──→│ Node │  │ Node │
            └──────┘   └──────┘  └──────┘
            (堆上)     (堆上)     (堆上)
                    分散分佈，快取不友善

存取流程 (字符 'a' 為例)：
  1. 計算雜湊值：hash(97) = h
  2. 計算 bucket 索引：h % N
  3. 讀取 bucket[index]
  4. 遍歷碰撞鏈表（最壞情況）
  5. 比較 key 值
  6. 返回 value

  耗時：10-50+ 個 CPU 週期 (取決於碰撞)

快取行為：
  ┌──────────────────────────────┐
  │ L1 快取 (32 KB)               │
  │  ├─ 雜湊表陣列 (數 KB)        │
  │  ├─ 指標指向堆記憶體          │  ◄ 指標追蹤
  │  └─ 堆節點分散分佈            │
  ├──────────────────────────────┤
  │ L2/L3 快取                     │
  │  └─ 堆節點隨機分佈            │  ◄ 低命中率
  └──────────────────────────────┘

命中率：30-60% (與衝突和分佈有關)
平均延遲：50-200 ns (包含 L3 存取和指標追蹤)


【定量比較】

(本題)           3 字元種類

### 7.1 迭代方案 vs 遞迴方案

```
【迭代方案】(本題採用)
─────────────────────────────────────────────────────

堆疊框架：
  ┌────────────────────────────────────────┐
  │ lengthOfLongestSubstring()             │
  ├────────────────────────────────────────┤
  │ lastSeen[128]        512 bytes         │
  │ left, maxLen, right   12 bytes         │
  │ i (迴圈計數器)         4 bytes         │
  │ currentChar           4 bytes         │
  │ currentLen            4 bytes         │
  │ 返回位址              8 bytes         │
  └────────────────────────────────────────┘

  總堆疊使用：536 bytes (單層)

執行特性：
  • 堆疊深度：1 (單一函數呼叫)
  • 最大堆疊用量：536 bytes
  • 內存回收：函數返回時立即回收
  • GC 壓力：零
  • 快取友善：優秀 (局部性)

時間複雜度：O(n)
空間複雜度：O(1) + O(128) = O(1)
  └─ lastSeen 陣列佔用常數空間

CPU 流量：
  ┌──────────────────────────────────────┐
  │ 週期     操作                          │
  ├──────────────────────────────────────┤
  │ 1-500    初始化 + while 迴圈          │
  │ 501-600  字元處理                    │
  │ 601-700  邊界檢查和陣列存取          │
  │ 701-800  條件判斷                    │
  │ 801-900  返回                        │
  └──────────────────────────────────────┘

  無分支跳躍，預測性順序執行


【遞迴方案】(假設實現)
─────────────────────────────────────────────────────

遞迴方案虛擬實現：
```c
int helper(char* s, int right, int left, int maxLen, int* lastSeen) {
    if (s[right] == '\0') return maxLen;

    unsigned char currentChar = (unsigned char)s[right];
    if (lastSeen[currentChar] >= left)
        left = lastSeen[currentChar] + 1;
    lastSeen[currentChar] = right;
    int currentLen = right - left + 1;
    if (currentLen > maxLen) maxLen = currentLen;

    return helper(s, right + 1, left, maxLen, lastSeen);
}

int lengthOfLongestSubstring(char* s) {
    int lastSeen[128];
    for (int i = 0; i < 128; i++) lastSeen[i] = -1;
    return helper(s, 0, 0, 0, lastSeen);
}
```

堆疊框架層級結構：
```
初始呼叫：lengthOfLongestSubstring()
├─ helper(s, 0, ...)   框架 1 (536 bytes)
│  ├─ helper(s, 1, ...)  框架 2 (536 bytes)
│  │  ├─ helper(s, 2, ...)  框架 3 (536 bytes)
│  │  │  ├─ ...
│  │  │  └─ helper(s, n, ...)  框架 n (536 bytes)
│  │  │     └─ 返回 maxLen
│  │  └─ 返回
│  └─ 返回
└─ 返回
```

堆疊使用分析：
  • 字串長度：假設 n = 8 ("abcabcbb")
  • 堆疊深度：8 層 (每個字元一層)
  • 每層堆疊：536 bytes
  • 最大堆疊用量：536 × 8 = 4,288 bytes
  • 內存回收：遞迴展開完全後才回收

堆疊配置圖 (字串長度 = 8)：
```

  ┌──────────────────┐

  ├──────────────────┤

  ├──────────────────┤
  │ helper(s,1,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,2,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,3,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,4,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,5,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,6,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,7,...)  │  536 bytes
  ├──────────────────┤
  │ helper(s,8,...)  │  536 bytes (最深)
  └──────────────────┘

  合計：536 × 9 = 4,824 bytes
```

執行特性：
  • 堆疊深度：O(n) (線性增長)
  • 最大堆疊用量：O(536n) 或 O(n)
  • 內存回收：遞迴全展開後統一回收
  • 堆疊溢出風險：高 (長字串)
  • 快取友善度：差 (堆疊幀分散)

時間複雜度：O(n)
空間複雜度：O(n) ← 遞迴堆疊！
  └─ 與字串長度線性相關

CPU 流量：
  ┌──────────────────────────────────────┐
  │ 階段         操作                      │
  ├──────────────────────────────────────┤
  │ 1-500        初始化                   │
  │ 501-2000     第 1 層遞迴呼叫 (~500)  │
  │ 2001-3500    第 2 層遞迴呼叫 (~500)  │
  │ 3501-5000    第 3 層遞迴呼叫 (~500)  │
  │ ...          ...                    │
  │ (n-1)×500+1  第 n 層遞迴呼叫 (~500) │

  └──────────────────────────────────────┘

  多分支跳躍，預測困難

  • 每次呼叫：push rip + 寄存器保存 = ~20 CPU 週期
  • n 次呼叫：20n 個額外週期

```

## 總結

### 核心記憶體特性

1. **固定空間配置**：536 bytes 堆疊框架（一次分配）
2. **直接陣列索引**：O(1) 查詢，無碰撞，快取友善
3. **線性時間複雜度**：O(n) 單次遍歷字串
4. **滑動視窗策略**：雙指標管理，空間複雜度 O(1)
5. **優於遞迴方案**：36 倍時間效率 + 無堆疊溢出風險

### 三個範例的執行統計

| 範例 | 字串長度 | 迭代次數 | 重複觸發 | maxLen更新 | 最大視窗 |
|------|---------|---------|---------|----------|--------|
| 1    | 8       | 8       | 5       | 3        | "abc" (3) |
| 2    | 5       | 5       | 4       | 1        | "b" (1) |
| 3    | 6       | 6       | 2       | 2        | "wke"/"kew" (3) |

### 記憶體存取模式

- **L1 快取命中率**：>95% (512 bytes 連續陣列)
- **平均存取延遲**：~5 ns (直接計算位址)
- **無動態分配**：所有空間預分配在堆疊上
- **無垃圾回收**：函數返回時自動回收

此實現展示了在受限記憶體環境下 (如嵌入式系統) 的優化策略：固定空間、直接索引、快取友善。
