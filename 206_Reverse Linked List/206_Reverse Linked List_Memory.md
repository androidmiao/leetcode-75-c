# 206 Reverse Linked List - 記憶體分析

## 1. Struct 記憶體佈局

### ListNode 結構定義
```c
struct ListNode {
    int val;                    // 4 bytes
    struct ListNode *next;      // 8 bytes (64-bit system)
};
```

### 記憶體配置詳情

```

│   (4 bytes)     │   (4 bytes)     │   (8 bytes)     │
```

## 3. 陣列索引版對應

### 陣列表示
```c
int arr[5] = {1, 2, 3, 4, 5};

arr[0] = 1    (地址: 0xF000)
arr[1] = 2    (地址: 0xF004)
arr[2] = 3    (地址: 0xF008)
arr[3] = 4    (地址: 0xF00C)
arr[4] = 5    (地址: 0xF010)
```

### 遍歷比較

```
鏈表遍歷 (指標追蹤):
┌────────────┐
│ curr = 0xA000
│ curr->val = 1
│ curr = curr->next (0xB000)
│ curr->val = 2
│ curr = curr->next (0xC000)
│ ... (每次跳躍記憶體地址不連續)
└────────────┘

陣列遍歷 (順序存取):
┌────────────┐
│ arr[0] = 1 (0xF000)
│ arr[1] = 2 (0xF004) ◄── 相鄰地址
│ arr[2] = 3 (0xF008) ◄── 連續記憶體
│ arr[3] = 4 (0xF00C) ◄── CPU緩存友好
│ arr[4] = 5 (0xF010)
└────────────┘
```

### 關鍵差異
- **鏈表**: 指標遍歷，記憶體地址隨機分佈，每個節點都是獨立記憶體塊
- **陣列**: 連續記憶體，順序遍歷，CPU預取最佳化可用

## 6. 記憶體存取模式比較

### 鏈表 vs 陣列: 記憶體特性

#### 鏈表 (Linked List)

```
記憶體存取模式:

node1(0xA000) ────▶ node2(0xB000) ────▶ node3(0xC000)
  │                    │                    │
  └──相隔4096字節      └──相隔4096字節      │

特點:
  • 指標追蹤 (Pointer Chasing)
  • 隨機記憶體存取模式
  • CPU L1 Cache Miss 率高
  • 預取 (Prefetch) 無效
  • 每次跳躍都是獨立的記憶體訪問

Cache 狀態:
  L1 Cache [node1] ──(失效)──▶ 從記憶體讀取 node2 ──(失效)──▶ 從記憶體讀取 node3
  延遲: ~100-200 cycles per node
```

#### 陣列 (Array)

```
記憶體存取模式:

arr[0]     arr[1]     arr[2]     arr[3]     arr[4]
(0xF000) - (0xF004) - (0xF008) - (0xF00C) - (0xF010)
  │          │          │          │          │
  └──相鄰 4 bytes (連續佈局)───────────────────┘

特點:
  • 順序記憶體存取
  • 連續佈局
  • CPU L1 Cache Hit 率高 (>95%)
  • 硬體預取自動工作
  • 一次載入整個 Cache Line (64 bytes)

Cache 狀態:
  Cache Line [0-63 bytes]: arr[0..15] ◄── 一次性載入
  後續訪問直接在 Cache 中命中 (Hits)
  延遲: ~3-5 cycles per element
```

### 性能對比表

| 項目 | 鏈表 | 陣列 |
|------|------|------|
| **記憶體佈局** | 離散 (Scattered) | 連續 (Sequential) |
| **訪問模式** | 隨機指標追蹤 | 順序存取 |
| **Cache 命中率** | 低 (~10-20%) | 高 (>95%) |
| **CPU 預取效果** | 無效 | 最佳 |
| **每節點延遲** | ~100-200 cycles | ~3-5 cycles |
| **插入/刪除** | O(1) (已知位置) | O(n) (需移動) |
| **查詢 k-th** | O(n) | O(1) |
| **記憶體開銷** | +8 bytes per node | 無 |

### 記憶體開銷計算

#### 鏈表 (5 個節點)
```
結構体大小:        16 bytes/node × 5 = 80 bytes
指標開銷:          8 bytes/node × 5 = 40 bytes (已含在結構体中)
隱含成本:          記憶體碎片化 (外部碎片)

總有效容量: 5 × (4 bytes data + 8 bytes pointer) = 60 bytes
額外開銷: 16 bytes (每個物件的分配元數據)

實際記憶體使用: 60 + 16×5 = 140 bytes (包括分配器開銷)
```

#### 陣列 (5 個元素)
```
陣列大小:          4 bytes/int × 5 = 20 bytes
指標開銷:          0 bytes
分配器開銷:        ~16 bytes

總記憶體使用: 20 + 16 = 36 bytes
節省空間: (140 - 36) / 140 = 74% ✓
```

## 7. 堆疊記憶體對比

### 迭代方案 vs 遞迴方案

#### 迭代實現 (本題使用)

```c
struct ListNode* reverseList(struct ListNode* head) {
    struct ListNode* prev = NULL;    // 8 bytes
    struct ListNode* curr = head;    // 8 bytes
    while (curr) {
        struct ListNode* next = curr->next;  // 8 bytes
        curr->next = prev;
        prev = curr;
        curr = next;
    }
    return prev;
}
```

**堆疊使用:**
```
函數 reverseList 堆疊幀:
┌─────────────────────────────────────┐
│ 返回地址          (8 bytes)         │ ◄── 0xSP+24
├─────────────────────────────────────┤
│ 保存的 RBP/RDI    (8 bytes)         │ ◄── 0xSP+16
├─────────────────────────────────────┤
│ prev (ListNode*)  (8 bytes)         │ ◄── 0xSP+8
├─────────────────────────────────────┤
│ curr (ListNode*)  (8 bytes)         │ ◄── 0xSP+0
│ next (ListNode*)  (待分配, 8 bytes) │
└─────────────────────────────────────┘

總堆疊使用: 24-32 bytes (固定)
```

**時間複雜度:** O(n)
**空間複雜度:** O(1)

---

#### 遞迴實現 (參考)

```c
struct ListNode* reverseListRecursive(struct ListNode* head) {
    if (!head || !head->next) return head;

    struct ListNode* newHead = reverseListRecursive(head->next);
    head->next->next = head;
    head->next = NULL;
    return newHead;
}
```

**堆疊使用 (鏈表長度 = 5):**
```
第 5 層: reverseListRecursive(NULL)
├─ 返回地址 (8B) + 保存寄存器 (8B) + head (8B) + newHead (8B) = 32 bytes
│
第 4 層: reverseListRecursive(Node 5)
├─ 32 bytes
│
第 3 層: reverseListRecursive(Node 4)
├─ 32 bytes
│
第 2 層: reverseListRecursive(Node 3)
├─ 32 bytes
│
第 1 層: reverseListRecursive(Node 2)
├─ 32 bytes
│
第 0 層: reverseListRecursive(Node 1)  ◄── 呼叫棧根
├─ 32 bytes

        ┌──────────────────┐
        │ 第 5 層 (32 B)  │ ◄── 棧頂 (最深)
        ├──────────────────┤
        │ 第 4 層 (32 B)  │
        ├──────────────────┤
        │ 第 3 層 (32 B)  │
        ├──────────────────┤
        │ 第 2 層 (32 B)  │
        ├──────────────────┤
        │ 第 1 層 (32 B)  │
        ├──────────────────┤
        │ 第 0 層 (32 B)  │ ◄── 棧底 (初始呼叫)
        └──────────────────┘
```

**堆疊使用統計:**
```
深度:                5 層
每層大小:            ~32 bytes
總使用:              5 × 32 = 160 bytes
```

**時間複雜度:** O(n)
**空間複雜度:** O(n) ← 遞迴棧

### 大規模案例: n = 5000 節點

#### 迭代方案

```
堆疊使用:  固定 ~24 bytes
記憶體使用: O(1) - 不因輸入大小增加而增加

安全性: ✓ 不會棧溢出
性能:   ✓ 最優 (無函數呼叫開銷)
```

#### 遞迴方案

```
遞迴深度:           5000 層
每層大小:           ~32 bytes
總堆疊使用:         5000 × 32 = 160,000 bytes = 160 KB

棧空間限制 (典型):  ~8 MB (Linux 線程)
危險閾值:           >2 MB 使用 → 棧溢出風險

棧幀圖 (部分):
         ┌──────────────────────┐
         │ 第 5000 層 (32 B)   │ ◄── 險些溢出!
         ├──────────────────────┤
         │ ... (第 2-4999 層)  │
         ├──────────────────────┤
         │ 第 1 層 (32 B)       │
         ├──────────────────────┤
         │ 主程序幀 (64 B)      │
         └──────────────────────┘
         總: ~160 KB

風險: ✗ 高風險棧溢出 (n > 10000 時肯定溢出)
性能: ✗ 遞迴呼叫開銷 ~5000 次
```

## 總結

### 核心要點

1. **迭代反轉**: 使用三個指標 (prev, curr, next) 逐個反轉指標方向
   - 時間: O(n), 空間: O(1)
   - 絕對最優方案

2. **記憶體操作**:
   - 5 個節點 = 5次迭代 = 15 次指標賦值
   - 無額外結構體分配，只改變指標

3. **Cache 效能**:
   - 鏈表: 隨機訪問，Cache Miss 率高 (~90%)
   - 但算法本身極簡 (每節點 3 次操作)

4. **空間對比**:
   - 迭代: 固定 24 bytes 堆疊
   - 遞迴: 5000 nodes = 160 KB 堆疊 (危險!)

5. **實務考慮**:
   - 大鏈表 (>10K 節點) 必須使用迭代
   - 迭代版本絕不棧溢出
   - 記憶體佈局離散無法優化，但代碼簡潔高效
