# 876. 鏈表的中間節點 - 記憶體分析

## 問題概述
給定單向鏈表，找到鏈表的中間節點。使用快慢指標演算法：慢指標每次移動 1 步，快指標每次移動 2 步，直到快指標到達末尾。

**資料結構定義：**
```c
struct ListNode {
    int val;
    struct ListNode *next;
};
```

---

## 1. 單一節點的 Struct 記憶體佈局

### 1.1 記憶體配置圖

```
假設節點地址為 0xA000 (虛擬地址，僅用於示例)

┌────────────────────────────────┐
│      struct ListNode           │
│      起始地址: 0xA000           │
├────────────────────────────────┤
│ val (int, 4 bytes)             │  [0xA000 - 0xA003]
│ ┌──────────────────┐           │
│ │    0x00000003    │           │  值: 3
│ └──────────────────┘           │
├────────────────────────────────┤
│ 對齐填充 (padding, 4 bytes)     │  [0xA004 - 0xA007]
│ ┌──────────────────┐           │  (在 64 位系統中)
│ │  0x00000000      │           │
│ └──────────────────┘           │
├────────────────────────────────┤
│ *next (pointer, 8 bytes)       │  [0xA008 - 0xA00F]
│ ┌──────────────────────┐       │
│ │    0x0000A010       │───┐   │  指向下一個節點
│ └──────────────────────┘   │   │
├────────────────────────────────┤
│ 總大小: 16 bytes (4 + 4 + 8)   │
└────────────────────────────────┘
```

### 1.2 記憶體佈局詳細說明

| 欄位 | 位址範圍 | 大小 | 內容 | 說明 |
|------|---------|------|------|------|
| `val` | 0xA000-0xA003 | 4 bytes | 0x03 | 整數值 |
| 對齊補充 | 0xA004-0xA007 | 4 bytes | 0x00 | 64 位系統上的填充 |
| `next` | 0xA008-0xA00F | 8 bytes | 0x0000A010 | 指向下一個節點的指標 |

**記憶體欄配置 (Little-Endian)：**
```
地址     : 0xA000  0xA004  0xA008  0xA00F
內容     : [val ] [pad ] [next-ptr    ]
對齊模式 : 4字節   4字節   8字節
```

## 3. 陣列索引版對應

### 將鏈表對應到陣列

```c
// 虛擬陣列表示 (用於對比分析)
int A[] = {1, 2, 3, 4, 5};  // 容量: 5

陣列索引對應：
┌─────┬─────┬─────┬─────┬─────┐
│ A[0]│ A[1]│ A[2]│ A[3]│ A[4]│
├─────┼─────┼─────┼─────┼─────┤
│  1  │  2  │  3  │  4  │  5  │
└─────┴─────┴─────┴─────┴─────┘
  ^     ^     ^     ^     ^
  |     |     |     |     |
node1  node2 node3 node4 node5
0xA000 0xA010 0xA020 0xA030 0xA040
```

### 計算中間位置

**鏈表長度** = 5

**陣列索引中間位置** = `length / 2` = `5 / 2` = `2` (整數除法)

**中間值** = `A[2]` = `3` (node3)

**快慢指標對應**：
- 慢指標在陣列中對應索引：0 → 1 → 2 (停止)
- 快指標在陣列中對應索引：0 → 2 → 4 → NULL (停止)
- 結果：slow 指向 A[2]，值為 3

## 4. 搜尋／操作過程追蹤

### 4.1 示例 1: `[1, 2, 3, 4, 5]`

#### 演算法邏輯
```c
struct ListNode* middleOfList(struct ListNode* head) {
    struct ListNode *slow = head;
    struct ListNode *fast = head;

    while (fast && fast->next) {
        slow = slow->next;        // 慢指標移動 1 步
        fast = fast->next->next;  // 快指標移動 2 步
    }

    return slow;  // 返回中間節點
}
```

#### 逐步追蹤表

| 步驟 | slow 地址 | slow 值 | fast 地址 | fast 值 | 操作 | fast->next 存在？ |
|------|----------|--------|----------|--------|------|------------------|
| 0 | 0xA000 | 1 | 0xA000 | 1 | 初始化 (都指向 head) | ✓ |
| 1 | 0xA010 | 2 | 0xA020 | 3 | slow ← node2, fast ← node3 | ✓ |
| 2 | 0xA020 | 3 | 0xA040 | 5 | slow ← node3, fast ← node5 | ✗ |
| 結束 | 0xA020 | 3 | 0xA040 | 5 | fast->next = NULL，停止 | - |

**結果**：返回 slow (0xA020)，值為 3

#### 記憶體狀態詳細追蹤

```

[1]───▶[2]───▶[3]───▶[4]───▶[5]───▶NULL
 ▲
 │
slow, fast (都指向 node1)

條件檢查: fast && fast->next?
  fast = 0xA000 ✓
  fast->next = 0xA010 ✓
  執行迴圈


第 1 次迭代後 (步驟 1):
[1]───▶[2]───▶[3]───▶[4]───▶[5]───▶NULL
       ▲             ▲
       │             │
      slow          fast
     0xA010        0xA020
     (node2)       (node3)

指標移動記錄：
  slow: 0xA000 ─▶ 0xA010 (移動 1 步)
  fast: 0xA000 ─▶ 0xA020 (移動 2 步)

條件檢查: fast && fast->next?
  fast = 0xA020 ✓
  fast->next = 0xA030 ✓
  執行迴圈


第 2 次迭代後 (步驟 2):
[1]───▶[2]───▶[3]───▶[4]───▶[5]───▶NULL
                     ▲             ▲
                     │             │
                    slow          fast
                   0xA020        0xA040
                   (node3)       (node5)

指標移動記錄：
  slow: 0xA010 ─▶ 0xA020 (移動 1 步)
  fast: 0xA020 ─▶ 0xA040 (移動 2 步)

條件檢查: fast && fast->next?
  fast = 0xA040 ✓
  fast->next = NULL ✗
  停止迴圈


終止狀態:
slow 指向 node3 (0xA020)，值為 3
返回 slow = 0xA020
```

#### 陣列索引版追蹤

```
陣列: A = [1, 2, 3, 4, 5]
索引:     0  1  2  3  4

初始:
  slow_idx = 0
  fast_idx = 0

第 1 次迭代:
  slow_idx: 0 → 1
  fast_idx: 0 → 2

第 2 次迭代:
  slow_idx: 1 → 2
  fast_idx: 2 → 4

迴圈停止條件:
  fast_idx = 4, fast_idx + 1 = 5 (超出邊界)
  或 fast_idx = 4 時，fast_idx + 2 = 6 (超出邊界)

```

## 6. 記憶體存取模式比較

### 6.1 指標型 (鏈表) vs. 陣列型存取

#### 指標型存取 (目前實現)

```c
// 快慢指標鏈表遍歷
struct ListNode *slow = head;      // 棧上存儲指標 (8 bytes)
struct ListNode *fast = head;      // 棧上存儲指標 (8 bytes)

while (fast && fast->next) {
    slow = slow->next;             // 逐指標訪問，解引用成本
    fast = fast->next->next;       // 雙指標訪問，解引用成本
}
```

**記憶體存取特性:**

| 特性 | 細節 |
|------|------|
| 空間開銷 | 每節點 16 bytes (4 bytes val + 4 bytes padding + 8 bytes *next) |
| 總空間 (5 個節點) | 80 bytes |
| 快取行為 | 不連續記憶體位置，難以預測；L1/L2 快取命中率低 |
| 記憶體訪問 | 指標解引用 (dereference)，CPU 管線可能阻塞 |
| 預取效率 | 無法進行有效的硬體預取 (指標追蹤困難) |
| 存取延遲 | ~100-200 CPU 週期 (記憶體訪問) |

#### 陣列型存取 (對比)

```c
// 陣列式查找
int A[5] = {1, 2, 3, 4, 5};
int mid = A[5 / 2];  // 直接計算索引，一次訪問
```

**記憶體存取特性:**

| 特性 | 細節 |
|------|------|
| 空間開銷 | 連續記憶體，每元素 4 bytes (或 8 bytes for pointers) |
| 總空間 (5 個元素) | 20 bytes (整數) or 40 bytes (指標) |
| 快取行為 | 連續記憶體位置，理想的空間位置 |
| 記憶體訪問 | O(1) 隨機存取，預測性強 |
| 預取效率 | 硬體預取器可有效利用 (64 byte 快取行) |
| 存取延遲 | ~3-4 CPU 週期 (快取命中) |

### 6.2 快取行為分析

#### 指標型快取模型

```
記憶體佈局 (不連續):
┌─────────────┐       ┌─────────────┐       ┌─────────────┐
│  node1      │  ...  │  node2      │  ...  │  node3      │
│ 0xA000-     │       │ 0xA010-     │       │ 0xA020-     │
│ 0xA00F      │       │ 0xA01F      │       │ 0xA02F      │
└─────────────┘       └─────────────┘       └─────────────┘
 ▲                     ▲                     ▲
 │ 快取行 1             │ 快取行 2             │ 快取行 3
 │ (64 bytes)          │ (64 bytes)          │ (64 bytes)

快取失誤 (cache miss) 風險高：
- 每存取 next 指標後，需重新加載快取行
- 指標可能指向不同的快取行
- 無法進行順序預取
```

#### 陣列型快取模型

```
記憶體佈局 (連續):
┌───────────────────────────────────────────────┐
│ A[0]  A[1]  A[2]  A[3]  A[4]  ...             │
│ 0xB000                           0xB050        │
└───────────────────────────────────────────────┘
 ▲ 快取行 1 (64 bytes 可容納 8-16 個整數)

快取友好性 (cache friendly)：
- 數據緊湊排列
- 預取器可預測下一個訪問位置
- L1/L2 快取命中率 > 90%
- 順序訪問效率高
```

### 6.3 插入/刪除成本對比

| 操作 | 鏈表 | 陣列 |
|------|------|------|
| **在中間插入** | O(1) 時間 (已有指標) + O(1) 記憶體分配 | O(n) 時間 (移動元素) + O(1) 記憶體 |
| **在中間刪除** | O(1) 時間 (指標修改) | O(n) 時間 (移動元素) |
| **查找** | O(n) (必須遍歷) | O(1) 隨機存取，或 O(log n) 二分搜尋 |
| **記憶體利用** | 額外 8 bytes/節點 (指標) | 無額外開銷 |
| **記憶體碎片** | 高風險 (分散分配) | 無碎片 (連續) |

## 7. 堆疊記憶體對比

### 7.1 當前實現 (迭代法)

```c
struct ListNode* middleOfList(struct ListNode* head) {
    struct ListNode *slow = head;     // 堆疊: 8 bytes (指標 slow)
    struct ListNode *fast = head;     // 堆疊: 8 bytes (指標 fast)

    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
    }

    return slow;
}
```

**堆疊使用：**

| 變數 | 類型 | 大小 | 位址示例 |
|------|------|------|---------|
| `slow` | struct ListNode* | 8 bytes | rsp + 0 |
| `fast` | struct ListNode* | 8 bytes | rsp + 8 |
| **合計堆疊開銷** | | **16 bytes** | |

**O(1) 空間複雜度確認:**
- 不論鏈表長度多少，堆疊用量固定
- 無遞迴調用，無堆疊深度增長
- 僅儲存 2 個指標變數

### 7.2 假設遞迴實現 (對比)

```c
// 假設遞迴方法 (不推薦)
int findMiddleRecursive(struct ListNode* node, int depth) {
    if (node == NULL) return 0;

    int rest = findMiddleRecursive(node->next, depth + 1);  // 遞迴呼叫

    if (rest == (n / 2)) {  // n 需全域或傳遞
        // 找到中間節點
    }

    return rest + 1;
}
```

**遞迴版堆疊使用：**

| 呼叫層級 | 局部變數 | 返回地址 | 參數 | 堆疊每層 |
|---------|--------|--------|------|---------|
| depth=0 | rest (8) | 8 | node, depth (16) | 32 bytes |
| depth=1 | rest (8) | 8 | node, depth (16) | 32 bytes |
| ... | ... | ... | ... | ... |
| depth=5 | rest (8) | 8 | node, depth (16) | 32 bytes |

**遞迴版堆疊複雜度：**
- **時間複雜度**: 仍為 O(n/2) 比較操作
- **空間複雜度**: **O(n)** (呼叫堆疊深度)
- **鏈表長度 5**: 堆疊深度 5 層 × 32 bytes ≈ 160 bytes
- **鏈表長度 1000000**: 堆疊深度 1000000 層 = **堆疊溢出風險！**

### 7.3 迭代 vs. 遞迴效能對比

```
迭代方案:
┌─────────────────────────────┐
│ 執行時堆疊 (固定 16 bytes)  │
│ slow: 0xA000                │
│ fast: 0xA000                │
├─────────────────────────────┤
│ 堆積記憶體 (80 bytes 節點)  │
│ node1-5: 0xA000-0xA050      │
└─────────────────────────────┘
時間: O(n/2)
空間: O(1) 堆疊 + O(n) 堆積

遞迴方案 (假設):
┌─────────────────────────────┐
│ 呼叫棧幀 5 層                │
│ 第 5 層: depth=5, slow, fast │
│ 第 4 層: depth=4, slow, fast │
│ 第 3 層: depth=3, slow, fast │
│ 第 2 層: depth=2, slow, fast │
│ 第 1 層: depth=1, slow, fast │
│ 第 0 層: depth=0, slow, fast │
│ (共 160 bytes，可能溢出)    │
├─────────────────────────────┤
│ 堆積記憶體 (80 bytes 節點)  │
│ node1-5: 0xA000-0xA050      │
└─────────────────────────────┘
時間: O(n/2)
空間: O(n) 堆疊 + O(n) 堆積 ❌
```

**結論:**
- **迭代法優越**：固定 O(1) 堆疊空間，適合大規模鏈表
- **遞迴法劣勢**：O(n) 堆疊空間，長鏈表易堆疊溢出

## 8. 退化情況分析

### 8.1 單節點鏈表

**輸入**: `[5]` (只有一個節點)

**初始狀態:**
```
┌──────────┐
│ node1: 5 │  0xA000
│ next: NULL
└──────────┘
    ▲
    │
  slow
  fast
  (都指向 node1)
```

**迴圈條件檢查:**
```c
while (fast && fast->next) {
    // fast = 0xA000 ✓ (非 NULL)
    // fast->next = NULL ✗ (為 NULL)
    // 條件 false，不進入迴圈
}
```

**結果:**
- 迴圈不執行
- 直接返回 slow = head = 0xA000
- 返回值: node1 (值 5)

**驗證:**
- 長度 1，中間節點應為自己
- 返回正確 ✓

### 8.2 雙節點鏈表

**輸入**: `[1, 2]`

**初始:**
```
[1] ───▶ [2] ───▶ NULL
0xA000  0xA010
 ▲
slow, fast
```

**第 1 次迭代:**
```
條件: fast && fast->next?
  fast = 0xA000 ✓
  fast->next = 0xA010 ✓

執行:
  slow = slow->next = 0xA010
  fast = fast->next->next = NULL

[1] ───▶ [2] ───▶ NULL
0xA000  0xA010
       ▲   ▲
      slow fast
      (fast 現已為 NULL)
```

**終止:**
```
條件: fast && fast->next?
  fast = NULL ✗
  不執行迴圈

返回 slow = 0xA010 (node2, 值 2)
```

**驗證:**
- 長度 2，偶數
- 中間應返回第二個節點 (0 based: A[1])
- 返回正確 ✓

### 8.3 三節點鏈表

**輸入**: `[7, 8, 9]`

**初始:**
```
[7] ───▶ [8] ───▶ [9] ───▶ NULL
0xA000  0xA010  0xA020
 ▲
slow, fast
```

**第 1 次迭代:**
```
條件: fast && fast->next? ✓
執行:
  slow = 0xA010
  fast = 0xA020

[7] ───▶ [8] ───▶ [9] ───▶ NULL
0xA000  0xA010  0xA020
       ▲             ▲
      slow          fast
```

**第 2 次迭代:**
```
條件: fast && fast->next? (fast=0xA020, fast->next=NULL) ✗
停止迴圈

返回 slow = 0xA010 (node2, 值 8)
```

**驗證:**
- 長度 3，奇數
- 中間應為 A[3/2] = A[1] (0-based)
- 返回正確 ✓

### 8.4 複雜度確認

**關鍵性質：**
無論鏈表結構如何，該演算法的行為固定：

```
鏈表長度   │ 迭代次數  │ slow 移動步數 │ fast 移動步數
───────────┼──────────┼──────────────┼─────────────
1          │ 0        │ 0            │ 0
2          │ 1        │ 1            │ 2
3          │ 1        │ 1            │ 2
4          │ 2        │ 2            │ 4
5          │ 2        │ 2            │ 4
6          │ 3        │ 3            │ 6
n (奇數)   │ (n-1)/2  │ (n-1)/2      │ n-1
n (偶數)   │ n/2      │ n/2          │ n
───────────┴──────────┴──────────────┴─────────────

時間複雜度: O(n/2) = O(n)
空間複雜度: O(1) (固定 2 個指標)
```

**結論：**
- 算法始終為 O(n) 線性時間掃描
- 空間使用始終為 O(1)
- 無特殊退化情況導致複雜度變化

## 9. 總結

### 9.1 演算法核心特點

| 方面 | 說明 |
|------|------|
| **基本思想** | 快慢雙指標，快指標速度為慢指標的 2 倍 |
| **記憶體訪問** | 順序遍歷，指標操作，不連續記憶體存取 |
| **時間複雜度** | O(n/2) ≈ O(n) |
| **空間複雜度** | O(1) (僅 2 個棧變數) |
| **快取性能** | 較差 (指標跳躍，不連續) |
| **適用場景** | 單向鏈表，找中點/起點的 1/3, 1/4 等位置 |

### 9.2 記憶體佈局要點

- 每個 `struct ListNode` 佔 16 bytes (4 + 4 + 8)
- 指標型存取具有較高記憶體開銷 (每節點 8 bytes 指標)
- 非連續記憶體分布降低快取效率
- 堆疊開銷最小化 (僅 16 bytes)

### 9.3 效能建議

1. **小規模鏈表** (< 1000 節點)：快慢指標法可接受
2. **大規模鏈表** (> 100K 節點)：考慮快取友好的替代方案
3. **不適合遞迴** ：會導致 O(n) 堆疊開銷，長鏈表易溢出
4. **最優解** ：迭代快慢指標法，O(1) 空間，O(n) 時間
