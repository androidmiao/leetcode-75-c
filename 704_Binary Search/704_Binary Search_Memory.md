# LeetCode 704 二元搜尋 - 記憶體分析

## 1. 陣列記憶體佈局

`int* nums` 是一個連續的整數陣列，每個元素佔用 `sizeof(int) = 4` 位元組。

**假設基礎位址：`0xA000`**

對於 `nums = [-1, 0, 3, 5, 9, 12]`：

**位址計算公式：**
```
&nums[i] = base_address + i × sizeof(int)
&nums[i] = 0xA000 + i × 4
```

**逐個元素位址：**
| 索引 | 元素值 | 位址計算 | 十六進制位址 |
|------|-------|---------|------------|
| 0 | -1 | 0xA000 + 0 × 4 | 0xA000 |
| 1 | 0 | 0xA000 + 1 × 4 | 0xA004 |
| 2 | 3 | 0xA000 + 2 × 4 | 0xA008 |
| 3 | 5 | 0xA000 + 3 × 4 | 0xA00C |
| 4 | 9 | 0xA000 + 4 × 4 | 0xA010 |
| 5 | 12 | 0xA000 + 5 × 4 | 0xA014 |

## 2. 完整資料結構記憶體配置

### 記憶體配置表

| 位址 | 內容 | 大小 | 說明 |
|------|------|------|------|
| 0xA000 | `-1` (0xFFFFFFFF) | 4 bytes | nums[0] |
| 0xA004 | `0` (0x00000000) | 4 bytes | nums[1] |
| 0xA008 | `3` (0x00000003) | 4 bytes | nums[2] |
| 0xA00C | `5` (0x00000005) | 4 bytes | nums[3] |
| 0xA010 | `9` (0x00000009) | 4 bytes | nums[4] |
| 0xA014 | `12` (0x0000000C) | 4 bytes | nums[5] |

### ASCII 記憶體圖表

```
陣列 nums 在記憶體中的配置：

     +------- nums 指標 (指向 0xA000)
     |
     ▼
┌────────┬────────┬────────┬────────┬────────┬────────┐
│   -1   │   0    │   3    │   5    │   9    │   12   │
├────────┼────────┼────────┼────────┼────────┼────────┤

└────────┴────────┴────────┴────────┴────────┴────────┘

首位址（基礎）：0xA000
```

## 3. 陣列索引版對應

### 索引直接存取模式

```c
int search(int* nums, int numsSize, int target) {
    int left = 0;           // 左邊界索引
    int right = numsSize - 1; // 右邊界索引

    while (left <= right) {
        // 中點計算（避免整數溢位）
        int mid = left + (right - left) / 2;

        // 實際記憶體存取：
        // 位址：&nums[mid] = 0xA000 + mid × 4
        // 反解參考值：nums[mid] = *(0xA000 + mid × 4)

        if (nums[mid] == target) return mid;
        if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

### 中點計算安全性

**中點計算公式：**
```
mid = left + (right - left) / 2
```

**優點：避免溢位**
- 錯誤方式：`mid = (left + right) / 2` → 可能整數溢位
- 正確方式：`mid = left + (right - left) / 2` → 等價但安全

**位元級別說明：**
```
假設 left = 0, right = 5:
  right - left = 5 - 0 = 5
  (right - left) / 2 = 5 / 2 = 2
  mid = 0 + 2 = 2

此時存取位址：&nums[2] = 0xA000 + 2 × 4 = 0xA008
反解值：nums[2] = 3
```

## 6. 記憶體存取模式分析

### 二元搜尋的快取行為

#### 非連續存取模式

```
陣列索引：[0][1][2][3][4][5]
             ├─ mid = 2
             ├─ mid = 4 (範例 1)
             └─ mid = 0 (範例 2)

存取順序（範例 1）：
  第 1 次：索引 2 → 位址 0xA008
  第 2 次：索引 4 → 位址 0xA010

  位址間隔：0xA010 - 0xA008 = 8 bytes = 2 × sizeof(int)

存取順序（範例 2）：
  第 1 次：索引 2 → 位址 0xA008 (跳過了 [0][1])
  第 2 次：索引 0 → 位址 0xA000 (向後跳躍)
  第 3 次：索引 1 → 位址 0xA004 (順序存取)

  位址順序：0xA008 → 0xA000 → 0xA004 (非單調)
```

#### 快取效能分析

**假設 L1 快取行大小：64 bytes（典型 x86-64 架構）**

```
快取行分配（基於位址 0xA000）：
  ┌──────────────────────────────────────────────────────────────┐
  │ 快取行 1：0xA000 ~ 0xA03F (包含 16 個 int)                   │
  │ [0][1][2][3][4][5]......[15]                                  │
  │  ─  ─  ✓  ✓  ✓  ✓       ✓   (✓ 快取命中, ─ 未使用)          │
  └──────────────────────────────────────────────────────────────┘
```

**快取命中率（範例 1）：**
```
  存取 1：nums[2] @ 0xA008 → 快取未命中 (載入快取行)
  存取 2：nums[4] @ 0xA010 → 快取命中 (在同一快取行)

  快取命中率：1/2 = 50%
```

**快取命中率（範例 2）：**
```
  存取 1：nums[2] @ 0xA008 → 快取未命中 (載入快取行)
  存取 2：nums[0] @ 0xA000 → 快取命中 (同一快取行)
  存取 3：nums[1] @ 0xA004 → 快取命中 (同一快取行)

  快取命中率：2/3 ≈ 66.67%
```

#### 與線性搜尋的對比

**線性搜尋（順序存取）：**
```
存取順序：[0] → [1] → [2] → [3] → [4] → [5]
位址順序：0xA000 → 0xA004 → 0xA008 → 0xA00C → 0xA010 → 0xA014

快取行利用：
  快取行載入一次後，後續存取都在同一快取行內
  快取命中率：≈ 100%（預取機制發揮最大效果）
  記憶體頻寬利用：最優
```

**二元搜尋（跳躍存取）：**
```
存取順序：[2] → [4] (或 [2] → [0] → [1])
位址順序：非單調增長

快取行利用：
  雖然通常在同一快取行內，但無法充分利用預取機制
  快取命中率：通常 > 50%
  記憶體頻寬利用：中等
```

**結論：**
```
時間複雜度：
  ├─ 線性搜尋：O(n) - 最多 n 次記憶體存取
  └─ 二元搜尋：O(log n) - 最多 log₂(n) 次記憶體存取

記憶體存取次數（n = 1,000,000）：
  ├─ 線性搜尋：最壞情況 1,000,000 次存取
  └─ 二元搜尋：最壞情況 ~20 次存取

快取效率權衡：
  ├─ 線性搜尋：高快取命中率，但總存取次數多
  └─ 二元搜尋：低快取命中率，但總存取次數少

在大陣列上，二元搜尋的優勢遠超快取劣勢。
```

## 7. 堆疊記憶體對比：迭代 vs 遞迴

### 迭代實現（當前）

```c
int search(int* nums, int numsSize, int target) {
    int left = 0;
    int right = numsSize - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) return mid;
        if (nums[mid] < target) left = mid + 1;
        else right = mid - 1;
    }
    return -1;
}
```

**堆疊配置（迭代）：**
```
堆疊記憶體佈局（函式框架）：

  ┌─────────────────────────┐ ◄── ESP (堆疊指標) - 最高位址
  │    區域變數              │
  │  ├─ int left (4 bytes)   │ [0xBFFFFFF0]
  │  ├─ int right (4 bytes)  │ [0xBFFFFFF4]
  │  └─ int mid (4 bytes)    │ [0xBFFFFFF8]
  ├─────────────────────────┤
  │  參數（通過暫存器傳遞）   │
  │  ├─ int* nums            │ (rdi 暫存器)
  │  ├─ int numsSize         │ (rsi 暫存器)
  │  └─ int target           │ (rdx 暫存器)
  ├─────────────────────────┤
  │  返回位址                 │ [0xBFFFFFF8]
  └─────────────────────────┘ ◄── RBP (堆疊基指標)

堆疊使用量：
  └─ 3 × sizeof(int) = 12 bytes
  └─ 恆定，不隨遞迴深度改變
```

**記憶體存取序列（迭代，範例 1）：**
```
初始化：
  1. left = 0          → 寫入堆疊 [0xBFFFFFF0]
  2. right = 5         → 寫入堆疊 [0xBFFFFFF4]

迭代 1：
  3. mid = 2           → 寫入堆疊 [0xBFFFFFF8]
  4. 讀取 nums[2]      → 讀取記憶體 0xA008
  5. 更新 left = 3     → 寫入堆疊 [0xBFFFFFF0]

迭代 2：
  6. mid = 4           → 寫入堆疊 [0xBFFFFFF8]
  7. 讀取 nums[4]      → 讀取記憶體 0xA010
  8. 返回 4            → 函式結束

總堆疊操作：8 次
```

### 遞迴實現

```c
int searchRecursive(int* nums, int target, int left, int right) {
    if (left > right) return -1;

    int mid = left + (right - left) / 2;
    if (nums[mid] == target) return mid;

    if (nums[mid] < target)
        return searchRecursive(nums, target, mid + 1, right);
    else
        return searchRecursive(nums, target, left, mid - 1);
}
```

**堆疊配置（遞迴，深度 = 2）：**
```
堆疊記憶體佈局（多層框架）：

  ┌─────────────────────────┐ ◄── ESP (當前堆疊指標)
  │ 第 2 層框架 (mid = 4)    │
  │  ├─ int left (4 bytes)   │ [0xBFFFFEE0] = 3
  │  ├─ int right (4 bytes)  │ [0xBFFFFEE4] = 5
  │  ├─ int mid (4 bytes)    │ [0xBFFFFEE8] = 4
  │  └─ 返回位址             │ [0xBFFFFFEC]
  ├─────────────────────────┤
  │ 第 1 層框架 (mid = 2)    │
  │  ├─ int left (4 bytes)   │ [0xBFFFFEF0] = 0
  │  ├─ int right (4 bytes)  │ [0xBFFFFEF4] = 5
  │  ├─ int mid (4 bytes)    │ [0xBFFFFEF8] = 2
  │  └─ 返回位址             │ [0xBFFFFEFC]
  ├─────────────────────────┤
  │ main() 的框架            │
  │  ...                      │
  └─────────────────────────┘ ◄── RBP (堆疊基指標)

堆疊使用量（深度 d）：
  └─ d × (3 × sizeof(int) + sizeof(return_addr))
  └─ d × (12 + 8) = d × 20 bytes (假設 64 位系統)
```

**最壞情況堆疊深度：**
```
陣列大小 n：        最壞堆疊深度
├─ n = 6            depth = ceil(log₂(6)) = 3
├─ n = 1,000        depth = ceil(log₂(1000)) ≈ 10
└─ n = 1,000,000    depth = ceil(log₂(1000000)) ≈ 20

最壞堆疊使用量（n = 1,000,000）：
  遞迴：20 × 20 = 400 bytes
  迭代：12 bytes
  比例：400 / 12 ≈ 33 倍

堆疊溢位風險：
  └─ 遞迴實現在大陣列上仍安全（幾百 bytes），
     但對於非常深的遞迴（如樹遍歷），堆疊溢位可能發生
```

### 執行流程對比

#### 迭代執行流程（範例 1，target = 9）

```
[開始]
  │
  ├─ 初始化：left = 0, right = 5
  │
  ├─ 循環 1：mid = 2, nums[2] = 3, 3 < 9 → left = 3
  │
  ├─ 循環 2：mid = 4, nums[4] = 9, 9 == 9 → 返回 4
  │
  ├─ [結束]
```

**堆疊深度：1（單層框架始終）**
**堆疊峰值：12 bytes**

#### 遞迴執行流程（範例 1，target = 9）

```
[開始：searchRecursive(nums, 9, 0, 5)]
  │
  ├─ left = 0, right = 5, mid = 2
  ├─ nums[2] = 3 < 9 → 遞迴調用
  │
  ├─ [堆疊層 1]
  │  └─ searchRecursive(nums, 9, 3, 5)
  │     │
  │     ├─ left = 3, right = 5, mid = 4
  │     ├─ nums[4] = 9 == 9 → 返回 4
  │     │
  │     └─ [返回值傳回層 1]
  │
  ├─ [堆疊層 0] 返回 4
  │
  ├─ [結束]
```

**堆疊深度：2（層 0 → 層 1 → 返回）**
**堆疊峰值：20 bytes × 2 = 40 bytes**

| **堆疊溢位風險** | 無 | 極低（但理論存在）|
| **函式調用開銷** | 無 | 20 × log n 次調用 |
| **L1 快取局部性** | 中等 | 相同 |
| **代碼複雜性** | 簡單 | 複雜 |
| **推薦用途** | **✓ 生產環境** | 教學示例 |

## 總結

**關鍵記憶體洞察：**

1. **位址計算**：`&nums[i] = base + i × sizeof(int)` - 陣列在記憶體中是連續的
2. **存取模式**：二元搜尋進行跳躍存取，非連續，但總存取次數少 O(log n)
3. **快取表現**：雖然快取命中率不如線性搜尋，但絕對存取次數決定了整體優勢
4. **堆疊成本**：迭代版本恆定 O(1) 堆疊空間，遞迴版本 O(log n) 堆疊空間
5. **安全計算**：使用 `mid = left + (right - left) / 2` 避免整數溢位

**實戰建議：**
- 使用迭代實現用於生產環境（低開銷、無堆疊溢位風險）
- 理解遞迴版本用於學習分治演算法概念
- 在大陣列上，二元搜尋的O(log n)複雜度遠優於線性O(n)搜尋
