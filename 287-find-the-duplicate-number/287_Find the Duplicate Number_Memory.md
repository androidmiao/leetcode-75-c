# 287. Find the Duplicate Number — 記憶體位址 / 陣列索引圖解

> 以下所有記憶體位址皆為假設值，僅供說明用途。

## 1. 陣列記憶體佈局

### `int nums[]` 的記憶體配置

每個 `int` 佔 4 bytes。假設陣列起始位址為 `0xA000`。

以 `nums = [1, 3, 4, 2, 2]`（n=4, numsSize=5）為例：

```
位址        索引    值    指向索引（隱式鏈表 next）
┌──────────┬──────┬─────┬──────────────────────────┐
│ 0xA000   │  [0] │  1  │  ───▶ 索引 1 (0xA004)    │
├──────────┼──────┼─────┼──────────────────────────┤
│ 0xA004   │  [1] │  3  │  ───▶ 索引 3 (0xA00C)    │
├──────────┼──────┼─────┼──────────────────────────┤
│ 0xA008   │  [2] │  4  │  ───▶ 索引 4 (0xA010)    │
├──────────┼──────┼─────┼──────────────────────────┤
│ 0xA00C   │  [3] │  2  │  ───▶ 索引 2 (0xA008)    │
├──────────┼──────┼─────┼──────────────────────────┤
│ 0xA010   │  [4] │  2  │  ───▶ 索引 2 (0xA008)    │
└──────────┴──────┴─────┴──────────────────────────┘
```

### 隱式鏈表結構（以記憶體位址標注）

```
0xA000[0]=1 ───▶ 0xA004[1]=3 ───▶ 0xA00C[3]=2 ───▶ 0xA008[2]=4 ───▶ 0xA010[4]=2
                                                      ▲                         │
                                                      └─────────────────────────┘
                                                       循環入口 = 索引 2, 值 = 2
```

## 3. 陣列索引版對應

Floyd 演算法中的「指標跳轉」直接對應陣列索引操作：

```
指標操作              陣列索引操作
──────────────────    ──────────────────
ptr = ptr->next       idx = nums[idx]
ptr = ptr->next->next idx = nums[nums[idx]]
```

**索引跳轉表（Example 1: [1,3,4,2,2]）：**

| 當前索引 i | nums[i]（下一索引） | nums[nums[i]]（跳兩步） |
|-----------|---------------------|------------------------|
| 0         | 1                   | 3                      |
| 1         | 3                   | 2                      |
| 2         | 4                   | 2                      |
| 3         | 2                   | 4                      |
| 4         | 2                   | 4                      |

## 4. 搜尋過程追蹤

### Example 1: `nums = [1, 3, 4, 2, 2]`，答案 = 2

#### Phase 1: 尋找相遇點

**初始化：**
```
tortoise = nums[nums[0]] = nums[1] = 3    （位址 0xA004 → 讀取值 3）
hare     = nums[nums[nums[0]]] = nums[nums[1]] = nums[3] = 2  （位址 0xA00C → 讀取值 2）
```

**迭代追蹤：**

| 步驟 | tortoise 位址 | tortoise 值 | hare 位址 | hare 值 | 比較結果 | 動作 |
|------|--------------|-------------|-----------|---------|----------|------|
| init | 0xA00C       | 3           | 0xA008    | 2       | 3 ≠ 2   | 繼續 |
| 1    | 0xA008 (nums[3]) | 2       | 0xA010→0xA008 (nums[nums[2]]=nums[4]) | 2 | 2 == 2 | 相遇！ |

```
Phase 1 指標位置追蹤：
  [0]=1    [1]=3    [2]=4    [3]=2    [4]=2
  0xA000   0xA004   0xA008   0xA00C   0xA010

  初始化後:
    T(龜) 在值 3 → 邏輯位置 = 索引 3 的值
    H(兔) 在值 2 → 邏輯位置 = 索引 2 的值（但透過索引 3→2 跳了兩步）

  步驟 1:
    T: nums[tortoise] = nums[3] = 2    → 龜移到值 2
    H: nums[nums[hare]] = nums[nums[2]] = nums[4] = 2  → 兔移到值 2
    T == H == 2 ✓ 相遇！
```

**指標移動摘要框：**
```
┌─────────────────────────────────┐
│ Phase 1 指標移動摘要            │
│ tortoise: 3 → 2                │
│ hare:     2 → 2                │
│ 相遇值: 2                      │
│ 總步數: 1 次迭代               │
└─────────────────────────────────┘
```

#### Phase 2: 尋找循環入口

**重置：**
```
tortoise = 2 （保持相遇點的值）
hare = nums[0] = 1 （回到起點第一步）
```

**迭代追蹤：**

| 步驟 | tortoise 位址 | tortoise 值 | hare 位址 | hare 值 | 比較結果 | 動作 |
|------|--------------|-------------|-----------|---------|----------|------|
| init | —            | 2           | —         | 1       | 2 ≠ 1   | 繼續 |
| 1    | 0xA008 (nums[2]) | 4       | 0xA004 (nums[1]) | 3 | 4 ≠ 3 | 繼續 |
| 2    | 0xA010 (nums[4]) | 2       | 0xA00C (nums[3]) | 2 | 2 == 2 | 找到！ |

```
Phase 2 指標位置追蹤：

  步驟 1:
    T: nums[2] = 4    H: nums[1] = 3    (4 ≠ 3, 繼續)
  步驟 2:
    T: nums[4] = 2    H: nums[3] = 2    (2 == 2, 找到！)
```

**指標移動摘要框：**
```
┌─────────────────────────────────┐
│ Phase 2 指標移動摘要            │
│ tortoise: 2 → 4 → 2            │
│ hare:     1 → 3 → 2            │
│ 循環入口值: 2（重複數字）        │
│ 總步數: 2 次迭代               │
└─────────────────────────────────┘
```

### Example 2: `nums = [3, 1, 3, 4, 2]`，答案 = 3

#### Phase 1: 尋找相遇點

**初始化：**
```
tortoise = nums[nums[0]] = nums[3] = 4
hare     = nums[nums[nums[0]]] = nums[nums[3]] = nums[4] = 2
```

| 步驟 | tortoise | hare | 比較 | 動作 |
|------|----------|------|------|------|
| init | 4        | 2    | ≠    | 繼續 |
| 1    | nums[4]=2 | nums[nums[2]]=nums[3]=4 | 2≠4 | 繼續 |
| 2    | nums[2]=3 | nums[nums[4]]=nums[nums[2]]=nums[3]=4... |  |  |

讓我重新仔細追蹤：

```
init:  T = nums[nums[0]] = nums[3] = 4
       H = nums[nums[nums[0]]] = nums[nums[3]] = nums[4] = 2
       T(4) ≠ H(2)

步驟1: T = nums[4] = 2
       H = nums[nums[2]] = nums[3] = 4
       T(2) ≠ H(4)

步驟2: T = nums[2] = 3
       H = nums[nums[4]] = nums[nums[2]]...
       H: hare=4, nums[4]=2, nums[2]=3 → H=3
       T(3) == H(3) ✓ 相遇！
```

#### Phase 2: 尋找循環入口

```
重置: T=3 (保持), H=nums[0]=3

T(3) == H(3) ✓ 立即相遇！循環入口值 = 3
```

**指標移動摘要框：**
```
┌─────────────────────────────────┐
│ Example 2 完整摘要              │
│ Phase 1: T: 4→2→3  H: 2→4→3  │
│ 相遇值: 3                      │
│ Phase 2: T=3, H=3, 立即相遇    │
│ 重複數字: 3                     │
└─────────────────────────────────┘
```

### Example 3: `nums = [3, 3, 3, 3, 3]`，答案 = 3

```
init:  T = nums[nums[0]] = nums[3] = 3
       H = nums[nums[nums[0]]] = nums[nums[3]] = nums[3] = 3
       T(3) == H(3) ✓ 立即相遇（自迴圈）

Phase 2:
  T = 3 (保持), H = nums[0] = 3
  T(3) == H(3) ✓ 立即相遇！

重複數字: 3
```

## 6. 記憶體存取模式比較

| 維度 | 陣列隱式鏈表（本題方法） | 真實鏈表（顯式指標） |
|------|--------------------------|----------------------|
| 資料佈局 | 連續記憶體，`int` 陣列 | 堆分配的分散節點 |
| 單次存取成本 | `nums[i]`：基底位址 + i×4，一次記憶體讀取 | `node->next`：解引用指標，一次記憶體讀取 |
| 快取行為 | 陣列本身連續（好），但跳轉目標隨機（差） | 節點分散在堆上（差），跳轉也隨機（差） |
| 空間開銷 | 4n bytes（僅資料） | 每節點至少 8-16 bytes（資料+指標+對齊填充） |
| 插入/刪除 | O(n) 移動元素 | O(1) 修改指標 |
| Floyd 適用性 | 直接適用——`nums[i]` 就是 next 指標 | 直接適用——`node->next` |

**關鍵觀察：** 本題的精妙之處在於將陣列「重新解讀」為鏈表。陣列在記憶體中是連續的，但 Floyd 的存取模式是由資料值決定的跳躍式存取，因此實際的 cache 行為接近散亂的鏈表存取，而非典型的陣列線性掃描。

## 7. 堆疊記憶體對比

Floyd 演算法是純迭代的，不使用遞迴，因此堆疊使用量恆定：

```
iterative Floyd 堆疊幀:
┌───────────────────────┐
│ findDuplicate()       │
│   int tortoise  (4B)  │
│   int hare      (4B)  │
│   int* nums     (8B)  │
│   int n         (4B)  │
│   合計: ~20 bytes     │
└───────────────────────┘
```

**對比：遞迴方法（方法 4.1）的堆疊使用：**

```
最壞情況: nums = [2,3,4,...,n,1,1]
遞迴深度 = n

第 n 層:  store(nums, 1)     ←─ 棧頂
第 n-1 層: store(nums, n)
  ...
第 2 層:  store(nums, 3)
第 1 層:  store(nums, 2)
第 0 層:  findDuplicate()    ←─ 棧底

每層 ~16-24 bytes → 總計 O(n) 堆疊空間
當 n = 10^5 時約需 1.6-2.4 MB 堆疊空間
```

## 8. 退化情況分析

### 最壞情況：最長的 Phase 1

當循環入口在陣列末端，且尾巴很長時，Phase 1 需要更多步數。

**範例：** `nums = [2, 3, 4, 5, 6, 7, 8, 1, 1]`（n=8）

```
鏈表結構:
[0]=2 → [2]=4 → [4]=6 → [6]=8 → [8]=1 → [1]=3 → [3]=5 → [5]=7 → [7]=1
                                                                      │
                                   ┌──────────────────────────────────┘
                                   ▼
                                  [1]=3 → [3]=5 → [5]=7 → [7]=1 → [1] (循環)

尾巴長度 F = 1（從 [0] 到循環入口 [8]→[1]）
循環長度 C = 4（1→3→5→7→1）
```

### 最壞情況：自迴圈（Example 3 就是此情況）

```
nums = [k, k, k, ..., k]（所有元素相同）

[0]=k → [k]=k → [k]=k → ...  (自迴圈, C=1)

Phase 1: 立即相遇（0 次迭代）
Phase 2: 立即相遇（0 次迭代）
這其實是最快的情況，不是最壞的。
```

### 真正的最壞情況複雜度

Floyd 演算法的最壞情況仍為 O(n)：Phase 1 最多 O(n) 步（龜最多走 n 步就會進入循環並被兔追上），Phase 2 最多 O(n) 步（F ≤ n）。總步數 ≤ 3n = O(n)。
