# LeetCode 933 - Number of Recent Calls 記憶體分析

## 1. Struct 記憶體佈局

### 資料結構定義
```c
typedef struct {
    int data[10000];  // 儲存 ping 時間戳記
    int front;        // 第一個有效元素的索引
    int rear;         // 下一個寫入位置的索引
} RecentCounter;
```

### 欄位大小與位移

| 欄位名稱 | 資料型態 | 大小 | 位移 (Offset) | 記憶體範圍 |
|---------|---------|------|--------------|-----------|
| data | int[10000] | 40,000 bytes | 0x00 | 0xA000 ~ 0xA9C3F |
| front | int | 4 bytes | 0x9C40 | 0xA9C40 ~ 0xA9C43 |
| rear | int | 4 bytes | 0x9C44 | 0xA9C44 ~ 0xA9C47 |
| **總計** | - | **40,008 bytes** | - | **0xA000 ~ 0xA9C47** |

#### 初始狀態 (RecentCounter 建立後)

```
├─────────────────────────────────────────────────────────────┤

└─────────────────────────────────────────────────────────────┘
  ↑                           (未初始化)

狀態：空隊列，無有效元素
```

---

#### Step 1: ping(1)

```
操作前：t = 1, valid range = [1-3000, 1] = [-2999, 1]

┌─────────────────────────────────────────────────────────────┐
│ data[] 陣列                 | front: 0 | rear: 1             │
├─────────────────────────────────────────────────────────────┤
│ [1]  [?]  [?]  [?]  [?] ...                                  │
│  ↑                                      ↑                    │
│0xA000                                0xA004(rear)            │
└─────────────────────────────────────────────────────────────┘

記憶體寫入：
  • data[0] = 1          @ 0xA000
  • rear = 1             @ 0xA9C44

while 迴圈檢查：
  • front < rear? → 0 < 1 ✓
  • data[0] = 1 < (1-3000) = -2999? → FALSE
  • 停止移動 front

狀態：front = 0, rear = 1
有效元素：[data[0]=1]
返回值：1
```

#### Step 2: ping(100)

```
操作前：t = 100, valid range = [100-3000, 100] = [-2900, 100]

┌─────────────────────────────────────────────────────────────┐
│ data[] 陣列                 | front: 0 | rear: 2             │
├─────────────────────────────────────────────────────────────┤
│ [1]  [100][?]  [?]  [?] ...                                  │
│  ↑                    ↑                                      │
│0xA000               0xA008(rear)                             │
└─────────────────────────────────────────────────────────────┘

記憶體寫入：
  • data[1] = 100        @ 0xA004
  • rear = 2             @ 0xA9C44

while 迴圈檢查：
  • front < rear? → 0 < 2 ✓
  • data[0] = 1 < (100-3000) = -2900? → FALSE
  • 停止移動 front

狀態：front = 0, rear = 2
有效元素：[data[0]=1, data[1]=100]
返回值：2
```

#### Step 3: ping(3001)

```
操作前：t = 3001, valid range = [3001-3000, 3001] = [1, 3001]

┌─────────────────────────────────────────────────────────────┐
│ data[] 陣列                 | front: 0 | rear: 3             │
├─────────────────────────────────────────────────────────────┤
│ [1]  [100][3001][?]  [?] ...                                 │
│  ↑              ↑                                            │
│0xA000         0xA00C(rear)                                   │
└─────────────────────────────────────────────────────────────┘

記憶體寫入：
  • data[2] = 3001       @ 0xA008
  • rear = 3             @ 0xA9C44

while 迴圈檢查：
  • front < rear? → 0 < 3 ✓
  • data[0] = 1 < (3001-3000) = 1? → FALSE (邊界情況，1 不小於 1)
  • 停止移動 front

狀態：front = 0, rear = 3
有效元素：[data[0]=1, data[1]=100, data[2]=3001]
返回值：3
```

---

#### Step 4: ping(3002)

```
操作前：t = 3002, valid range = [3002-3000, 3002] = [2, 3002]

記憶體寫入前：
  • data[3] = 3002       @ 0xA00C
  • rear = 4             @ 0xA9C44

while 迴圈檢查（移除過期元素）：

  【迭代 1】
    • front < rear? → 0 < 4 ✓
    • data[0] = 1 < (3002-3000) = 2? → TRUE ✓ （1 已過期）
    • front++ → front = 1

  【迭代 2】
    • front < rear? → 1 < 4 ✓
    • data[1] = 100 < 2? → FALSE （100 仍在有效範圍 [2, 3002]）
    • 停止迴圈

最終狀態：
┌─────────────────────────────────────────────────────────────┐
│ data[] 陣列                 | front: 1 | rear: 4             │
├─────────────────────────────────────────────────────────────┤
│ [1]  [100][3001][3002][?] ...                                │
│       ↑              ↑                                       │
│    0xA004         0xA010(rear)                               │
│   (front)                                                    │
└─────────────────────────────────────────────────────────────┘

狀態：front = 1, rear = 4
有效元素：[data[1]=100, data[2]=3001, data[3]=3002]
返回值：3
```

## 3. 搜尋／操作過程追蹤

### 完整示例：[ping(1), ping(100), ping(3001), ping(3002)]

#### ping(1) - 詳細追蹤

```
【輸入】
  • t = 1
  • 時間範圍（有效視窗）：[t-3000, t] = [-2999, 1]

【記憶體操作】
步驟 1：將時間戳記寫入隊列尾部
  data[rear] = t
  → data[0] = 1  @ 記憶體位址 0xA000

步驟 2：移除過期的時間戳記（while 迴圈）
  while (front < rear && data[front] < t - 3000)

  【迭代檢查】
    • front = 0, rear = 0 (寫入前)
    • front < rear? → 0 < 0? FALSE
    • while 迴圈不執行

步驟 3：更新 rear 指標
  rear++
  → rear = 1  @ 記憶體位址 0xA9C44

【指標變動】
┌────────────────────┬──────────────┬──────────────┐
│ 指標變數           │ 舊值          │ 新值         │
├────────────────────┼──────────────┼──────────────┤
│ front              │ 0            │ 0 (不變)     │
│ rear               │ 0            │ 1            │
│ data[0]            │ ? (未初化)    │ 1            │
└────────────────────┴──────────────┴──────────────┘

【記憶體狀態】
  ┌─────────────────────────────┐
  │ data[0]=1  data[1]...      │
  │ ↑                           │
  │ 0xA000    front=0, rear=1   │
  └─────────────────────────────┘

【返回值】rear - front = 1 - 0 = 1
```

---

#### ping(100) - 詳細追蹤

```
【輸入】
  • t = 100
  • 時間範圍：[100-3000, 100] = [-2900, 100]
  • 目前記憶體：front=0, rear=1, data[0]=1

【記憶體操作】
步驟 1：寫入新時間戳記
  data[rear] = t
  → data[1] = 100  @ 記憶體位址 0xA004

步驟 2：移除過期元素 (while 迴圈)
  【迭代 1】
    • front = 0, rear = 1 (寫入前)
    • front < rear? → 0 < 1 ✓
    • data[0] = 1 < (100 - 3000)? → 1 < -2900? FALSE
    • 條件不滿足，迴圈終止

步驟 3：更新 rear
  rear++
  → rear = 2  @ 0xA9C44

【指標變動】
┌────────────────────┬──────────────┬──────────────┐
│ 指標變數           │ 舊值          │ 新值         │
├────────────────────┼──────────────┼──────────────┤
│ front              │ 0            │ 0 (不變)     │
│ rear               │ 1            │ 2            │
│ data[1]            │ ? (未初化)    │ 100          │
└────────────────────┴──────────────┴──────────────┘

【記憶體狀態】
  ┌──────────────────────────────────┐
  │ data[0]=1  data[1]=100  data[2]..│
  │ ↑                                 │
  │ 0xA000    front=0, rear=2         │
  └──────────────────────────────────┘

【返回值】rear - front = 2 - 0 = 2
```

---

#### ping(3001) - 詳細追蹤

```
【輸入】
  • t = 3001
  • 時間範圍：[3001-3000, 3001] = [1, 3001]
  • 目前記憶體：front=0, rear=2, data=[1,100,?]

【記憶體操作】
步驟 1：寫入新時間戳記
  data[rear] = t
  → data[2] = 3001  @ 記憶體位址 0xA008

步驟 2：移除過期元素
  【迭代 1】
    • front = 0, rear = 2
    • front < rear? → 0 < 2 ✓
    • data[0] = 1 < (3001 - 3000)? → 1 < 1? FALSE
    • 1 恰好等於下界 1，不滿足 < 條件
    • 迴圈終止

步驟 3：更新 rear
  rear++
  → rear = 3

【指標變動】
┌────────────────────┬──────────────┬──────────────┐
│ 指標變數           │ 舊值          │ 新值         │
├────────────────────┼──────────────┼──────────────┤
│ front              │ 0            │ 0 (不變)     │
│ rear               │ 2            │ 3            │
│ data[2]            │ ? (未初化)    │ 3001         │
└────────────────────┴──────────────┴──────────────┘

【記憶體狀態】
  ┌────────────────────────────────────────┐
  │ data[0]=1  data[1]=100  data[2]=3001   │
  │ ↑                                       │
  │ 0xA000    front=0, rear=3               │
  └────────────────────────────────────────┘

【返回值】rear - front = 3 - 0 = 3
```

---

#### ping(3002) - 詳細追蹤（關鍵步驟）

```
【輸入】
  • t = 3002
  • 時間範圍：[3002-3000, 3002] = [2, 3002]
  • 目前記憶體：front=0, rear=3, data=[1,100,3001,?]
  • 臨界點：data[0]=1 現在 < 2 (下界)，需要被移除

【記憶體操作】
步驟 1：寫入新時間戳記
  data[rear] = t
  → data[3] = 3002  @ 記憶體位址 0xA00C

步驟 2：移除過期元素（關鍵 while 迴圈）
  while (front < rear && data[front] < t - 3000)
  即：while (front < 3 && data[front] < 2)

  【迭代 1】
    • front = 0, rear = 3
    • front < rear? → 0 < 3 ✓
    • data[0] = 1 < 2? → TRUE ✓
    • data[0] 已過期，執行 front++
    • front = 1

  【迭代 2】
    • front = 1, rear = 3
    • front < rear? → 1 < 3 ✓
    • data[1] = 100 < 2? → FALSE
    • 100 在有效範圍內，停止迴圈

步驟 3：更新 rear
  rear++
  → rear = 4

【指標變動 - 舊值 → 新值】
┌────────────────────┬──────────────┬──────────────┐
│ 指標變數           │ 舊值          │ 新值         │
├────────────────────┼──────────────┼──────────────┤
│ front              │ 0            │ 1            │
│ rear               │ 3            │ 4            │
│ data[3]            │ ? (未初化)    │ 3002         │
└────────────────────┴──────────────┴──────────────┘

【記憶體狀態 - 過期元素跳過】
  ┌──────────────────────────────────────────────────┐
  │ [已過期] [100] [3001] [3002] [?]                 │
  │ data[0]  data[1] data[2] data[3]                │
  │                  ↑                               │
  │         前沒有指向這裡，front=1                  │
  │         0xA004  front指向這裡                    │
  └──────────────────────────────────────────────────┘

  實際記憶體：
  ┌──────────────────────────────────────────────────┐
  │ 位址      │ 內容      │ 有效性                    │
  ├──────────────────────────────────────────────────┤
  │ 0xA000   │ data[0]=1 │ ✗ 過期 (front=1 已跳過)  │
  │ 0xA004   │ data[1]=100 │ ✓ 有效                 │
  │ 0xA008   │ data[2]=3001 │ ✓ 有效                │
  │ 0xA00C   │ data[3]=3002 │ ✓ 有效                │
  │ 0xA9C40  │ front=1   │ 指向有效元素開始       │
  │ 0xA9C44  │ rear=4    │ 指向下一個寫入位置     │
  └──────────────────────────────────────────────────┘

【返回值】rear - front = 4 - 1 = 3
  有效元素：data[1]=100, data[2]=3001, data[3]=3002
```

## 4. 指標變數追蹤總表

### 按步驟的完整表格

```
┌──────┬─────┬───────┬────────────────┬──────────────┬──────────┬───────────┐
│ Step │ t   │ rear  │ front          │ data written │ expired  │ return    │
│      │     │       │ (後 → 前)      │ @address     │ check    │ value     │
├──────┼─────┼───────┼────────────────┼──────────────┼──────────┼───────────┤
│ 1    │ 1   │ 0→1   │ 0→0 (不變)     │ data[0]=1    │ 無       │ 1         │
│      │     │       │                │ @0xA000      │ 過期     │           │
├──────┼─────┼───────┼────────────────┼──────────────┼──────────┼───────────┤
│ 2    │ 100 │ 1→2   │ 0→0 (不變)     │ data[1]=100  │ data[0]  │ 2         │
│      │     │       │                │ @0xA004      │ ✓ 有效   │           │
├──────┼─────┼───────┼────────────────┼──────────────┼──────────┼───────────┤
│ 3    │3001 │ 2→3   │ 0→0 (不變)     │ data[2]=3001 │ data[0]  │ 3         │
│      │     │       │                │ @0xA008      │ ✓ 有效   │           │
├──────┼─────┼───────┼────────────────┼──────────────┼──────────┼───────────┤
│ 4    │3002 │ 3→4   │ 0→1 (推進)    │ data[3]=3002 │ data[0]  │ 3         │
│      │     │       │                │ @0xA00C      │ ✗ 過期   │           │
│      │     │       │                │              │ front++  │           │
│      │     │       │                │              │ data[1]  │           │
│      │     │       │                │              │ ✓ 有效   │           │
└──────┴─────┴───────┴────────────────┴──────────────┴──────────┴───────────┘

說明：
  • rear 從 0 開始，每次 ping 遞增
  • front 在第 4 次 ping 時推進（data[0]=1 過期）
  • 有效元素個數 = rear - front
  • 每次寫入 data[rear] 前遞增 rear（檢查後）
```

## 5. 記憶體存取模式比較

### 陣列型隊列 (Array-based Queue) vs 鏈結列表型隊列 (Linked-List Queue)

#### 5.1 記憶體佈局對比

##### 陣列型隊列 - 連續記憶體

```
重現象：在同一連續記憶體塊中

基址 0xA000
┌─────────────┬─────────────┬─────────────┬─────────────┬─────────────┐
│ data[0]     │ data[1]     │ data[2]     │ data[3]     │ data[4]     │
│ = 1         │ = 100       │ = 3001      │ = 3002      │ = ?         │
│ @ 0xA000    │ @ 0xA004    │ @ 0xA008    │ @ 0xA00C    │ @ 0xA010    │
├─────────────┼─────────────┼─────────────┼─────────────┼─────────────┤
│    4 bytes  │    4 bytes  │    4 bytes  │    4 bytes  │    4 bytes  │
└─────────────┴─────────────┴─────────────┴─────────────┴─────────────┘
  ↑ front=0                              ↑ rear=4

  /* 邏輯視圖 */
  data[] = [1, 100, 3001, 3002, ...]
           ↑           ↑
           0           3 (索引)
```

##### 鏈結列表型隊列 - 分散記憶體

```
各節點散佈在堆上，需要指標追蹤

  節點 1              節點 2              節點 3              節點 4
  @ 0x2F400          @ 0x4C800          @ 0x7D200          @ 0x9E600
  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
  │ val: 1      │    │ val: 100    │    │ val: 3001   │    │ val: 3002   │
```

#### 5.3 ping() 操作的記憶體存取對比

##### 陣列型隊列 - ping(3002) 的記憶體存取序列

```
1. 讀取指標（極快，快取在 0xA9C40/0xA9C44）
   ├─ 讀 front @ 0xA9C40 → 0
   └─ 讀 rear @ 0xA9C44 → 3

2. while 迴圈 - 順序讀取
   ├─ 讀 data[0] @ 0xA000 → 1 ✓ 快取命中
   ├─ 讀 data[1] @ 0xA004 → 100 ✓ 快取命中（相鄰字節）
   └─ front++ → 1 (停止)

3. 寫入新元素
   └─ 寫 data[3] @ 0xA00C

4. 更新 rear
   └─ 寫 rear @ 0xA9C44 → 4

【記憶體存取模式】
  0xA000 → 0xA004 → (迴圈停止) → 0xA00C → 0xA9C44
  |        |                       |        |
  讀       讀                       寫       寫
  相鄰    相鄰
  命中    命中

  快取行為（假設 64 字節快取行，0xA000~0xA03F）
  ┌────────────────────────────────────────────┐
  │ L1 快取行 (0xA000~0xA03F)                  │
  │ ┌──────┬──────┬──────┬──────┬──────┬─────┐ │
  │ │data[0]│data[1]│data[2]│data[3]│...  │   │ │
  │ │ 0xA00 │ 0xA04│ 0xA08 │ 0xA0C │      │   │ │
  │ └──────┴──────┴──────┴──────┴──────┴─────┘ │
  │   ✓      ✓      ✓      ✓                   │
  │   命中   命中   預載   預載                   │
  └────────────────────────────────────────────┘

  效果：幾乎無快取缺失
```

##### 鏈結列表型隊列 - ping(3002) 的記憶體存取序列

```
1. 讀取 head 指標
   └─ 讀 head @ 0x2F000 → 節點 @ 0x2F400

2. while 迴圈 - 指標追蹤（隨機跳躍）
   ├─ 讀節點 @ 0x2F400
   │   ├─ 讀 val @ 0x2F400 → 1 ✗ 快取缺失（新位址）
   │   └─ 讀 next @ 0x2F408 → 0x4C800
   │
   ├─ 讀節點 @ 0x4C800
   │   ├─ 讀 val @ 0x4C800 → 100 ✗ 快取缺失（距離遠）
   │   └─ 讀 next @ 0x4C808 → 0x7D200
   │
   └─ next != NULL，停止（訪問了 tail，但未達末尾）

3. 動態分配新節點
   ├─ malloc(新節點) → 0x9E600 (高成本)
   ├─ 寫 val @ 0x9E600 → 3002
   ├─ 寫 next @ 0x9E608 → NULL
   └─ 寫 tail.next @ 0x7D208 → 0x9E600

【記憶體存取模式】
  0x2F000 → 0x2F400 → 0x2F408 → 0x4C800 → 0x4C808 → ... → 0x9E600
  |        |          |          |          |               |
  讀       讀          讀          讀          讀               新分配
  指標     值          next指標     值          next指標         (malloc)

  快取行為（多個 L1 缺失）
  ┌─ 位址 0x2F400 │ 快取缺失 1
  ├─ 位址 0x4C800 │ 快取缺失 2 （距離 1.6MB，肯定缺失）
  ├─ 位址 0x7D200 │ 快取缺失 3
  └─ 位址 0x9E600 │ 快取缺失 4 + malloc 成本

  效果：多次快取缺失，加上動態分配開銷
```

### 情況 2：ping 分散，視窗持續滑動

```
【場景】

  （每次間隔 ~3100ms，超過視窗 3000ms）

【時間軸與視窗】
  時刻 1:     視窗 [-2999, 1]      → ping(1) → [1]
  時刻 3100:  視窗 [100, 3100]     → ping(3100) → [3100] （1 過期）
  時刻 6200:  視窗 [3200, 6200]    → ping(6200) → [6200] （3100 過期）
  時刻 9300:  視窗 [6300, 9300]    → ping(9300) → [9300] （6200 過期）
  ...

【記憶體行為】
  ping(1):    data[0]=1, front: 0, rear: 1    [1個元素]

  ping(3100): data[1]=3100, front→1, rear: 2  [1個元素]
              • while 檢查：data[0]=1 < 100? YES → front++
              • front: 0→1

  ping(6200): data[2]=6200, front→2, rear: 3  [1個元素]
              • while 檢查：data[1]=3100 < 3200? YES → front++
              • front: 1→2

  ping(9300): data[3]=9300, front→3, rear: 4  [1個元素]
              • while 檢查：data[2]=6200 < 6300? YES → front++
              • front: 2→3

  持續...

【記憶體佈局隨時間變化】
  第 1 步：data[0]=1,     front=0, rear=1 → 隊列=1
  第 2 步：data[1]=3100,  front=1, rear=2 → 隊列=1 (0→1 被跳過)
  第 3 步：data[2]=6200,  front=2, rear=3 → 隊列=1 (0→2 被跳過)
  第 4 步：data[3]=9300,  front=3, rear=4 → 隊列=1 (0→3 被跳過)

  經過 N 步（N > 10000）：會發生什麼？

  【陣列溢出風險分析】

  當 rear 達到 10000：
  ┌────────────────────────────────────────┐
  │ rear = 10000, front = 9999             │
  │ 隊列大小 = rear - front = 1            │
  │                                        │
  │ 下一個 ping 需要寫 data[10000]         │
  │ → 數組越界！ (資料[0..9999] 合法)      │
  │                                        │
  │ 解決方案 1：循環隊列 (環形緩衝)        │
  │   rear = (rear + 1) % 10000            │
  │   front = (front + 1) % 10000          │
  │                                        │
  │ 解決方案 2：重置隊列 (當 front 足夠大) │
  │   if (front > 5000) {                  │
  │     重新整理，將有效元素移到前面       │
  │   }                                    │
  │                                        │
  │ LeetCode 933 預設假設不會溢出           │
  │ (ping 次數合理，時間跨度有限)         │
  └────────────────────────────────────────┘

【性能分析】
  ✓ 每次 ping() 快速 (O(1)，front 只移動 1 次)
  ✓ 隊列大小受控 (始終 ≤ 1 元素)
  ✓ 空間使用相對高效 (front 逐漸推進，舊元素被跳過)
```

```
【記憶體狀態】
  ┌──────────────────────────────────────────┐
  │ data[0]=1000  data[1]=1000  data[2]=1000 │
  │ front=0       rear=3                     │
  │ 隊列大小 = 3                              │
  │                                          │
  │ 相同時間戳的多個 ping 都被保留           │
  │ （不存在過期檢查失敗的情況）             │
  └──────────────────────────────────────────┘

【潛在問題】
  ✗ 重複時間戳會導致隊列大小增長
  ✓ 但陣列型隊列仍能高效處理（O(1) 寫入）
  ✗ 如果連續 10000 個相同時間戳，隊列滿溢
```

### 情況 5：退化的快取行為

```
【場景】使用鏈結列表版本，且 malloc 碎片化嚴重

【記憶體佈局退化】
  ┌─────────────────────────────────────────┐
  │ malloc 分配位置（模擬碎片化）:          │
  │                                         │
  │ 節點 1 @ 0x10000 (malloc 返回)         │
  │ 節點 2 @ 0x50000 (碎片化，遠距)        │
  │ 節點 3 @ 0x20000 (更遠)                 │
  │ 節點 4 @ 0x90000 (極遠)                 │
  │ 節點 5 @ 0x30000                        │
  │ ...                                     │
  │                                         │
  │ L1 快取行（64 字節）：                  │
  │ ┌─────────────┐                         │
  │ │ 64-byte     │                         │
  │ │ 快取行      │                         │
  │ └─────────────┘                         │
  │                                         │
  │ 每次節點訪問：快取未命中 (MISS)        │
  │ 多 CPU 週期浪費在記憶體延遲             │
  └─────────────────────────────────────────┘

【while 迴圈的快取行為】
  node = head;
  while (node && node->val < threshold) {
    node = node->next;  // ← 隨機 JUMP，快取缺失
  }

  每次迭代：
    • 讀 node->val 可能快取缺失
    • 讀 node->next（指向下一隨機位置）
    • 下一迭代讀新位置→又是快取缺失

  實測CPU週期（模擬）：
  ┌─────────────┬──────────────┐
  │ 操作        │ CPU 週期     │
  ├─────────────┼──────────────┤
  │ 讀 node@0x10K (命中) │ ~3 │
  │ 讀 next 指標 @ 0x50K (缺失) │ ~200 │
  │ 讀 node@0x50K (缺失) │ ~200 │
  │ 讀 next 指標 @ 0x20K (缺失) │ ~200 │
  │ ...                   │ ... │
  └─────────────┴──────────────┘

  N 個節點的 while 迴圈：
    • 陣列版：~3N 週期 (快取命中)
    • 鏈表版：~200N 週期 (快取缺失)
    • 差異：~67 倍速度下降！
```

```
┌─────────────────────┬──────────────────────┬────────────┬─────────────┐
│ 退化情況            │ 記憶體行為           │ 性能影響   │ 陣列 vs 鏈表│

│ 1. 視窗滿載          │ rear 遞增，front 不動│ ✗ 空間浪費 │ 陣列 更差   │
│    (全部有效)        │ 隊列逐漸填滿         │ (40K 固定) │ (鏈表好)    │
├─────────────────────┼──────────────────────┼────────────┼─────────────┤
│ 2. 視窗滑動          │ front 逐漸推進       │ ✓ 高效     │ 陣列 更好   │

├─────────────────────┼──────────────────────┼────────────┼─────────────┤
```

### 最優方案選擇

```
場景 A：LeetCode 受限環境
  → 陣列型隊列 ✓ 最優
  • 固定大小足夠 (10000 > 預期 ping 數)
  • 簡潔高效，無 malloc 開銷
  • 快取友好，順序訪問

場景 B：實時系統 (long-lived)
  → 循環陣列 (circular buffer)
  • 記憶體受限，需要重複使用
  • 改進：rear = (rear+1) % 10000
  • front = (front+1) % 10000

場景 C：動態負載，容量未知
  → 鏈結列表 或 動態陣列
  • 不知道最大 ping 數
  • 優先考慮動態陣列 (vector/ArrayList)
  • 次選鏈表 (malloc 成本可接受)
```
