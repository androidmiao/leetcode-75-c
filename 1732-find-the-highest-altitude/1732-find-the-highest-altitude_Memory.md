# 1732. Find the Highest Altitude — 記憶體位址 / 陣列索引圖解

> 以下所有位址皆為假設值，僅供說明用途。

## Struct 記憶體佈局

本題不使用自訂 struct，資料結構是純 `int` 陣列。函式簽名：

```c
int largestAltitude(int* gain, int gain_size);
```

### 函式參數與局部變數

假設 `int` = 4 bytes，stack 起始位址 0x7FFE_EA00：

```
┌──────────────────────────────────────┐
│          Stack Frame                 │
├──────────────┬───────┬───────────────┤
│ 變數         │ 位址   │ 大小          │
├──────────────┼───────┼───────────────┤
│ gain (ptr)   │ EA00  │ 8 bytes (指標) │
│ gain_size    │ EA08  │ 4 bytes       │
│ current      │ EA0C  │ 4 bytes       │
│ highest      │ EA10  │ 4 bytes       │
│ i            │ EA14  │ 4 bytes       │
└──────────────┴───────┴───────────────┘
```

## 完整資料結構記憶體配置

### 範例 1：`gain = [-5, 1, 5, 0, -7]`

假設 `gain` 陣列起始位址 = `0x2000`：

| 索引 | 位址     | 值   |
|------|----------|------|
| 0    | `0x2000` | -5   |
| 1    | `0x2004` |  1   |
| 2    | `0x2008` |  5   |
| 3    | `0x200C` |  0   |
| 4    | `0x2010` | -7   |

```
記憶體佈局（連續 int 陣列）：

位址:  0x2000    0x2004    0x2008    0x200C    0x2010
      ┌────────┬────────┬────────┬────────┬────────┐
值:   │   -5   │    1   │    5   │    0   │   -7   │
      └────────┴────────┴────────┴────────┴────────┘
索引:     [0]      [1]      [2]      [3]      [4]
```

### 邏輯結構：海拔折線

```
海拔:   0    -5    -4     1     1    -6
        ●                ●─────●
        │\               │     │
        │ \              │     │
        │  \     ●───┐   │     │
        │   \        │   │     │
        │    ●───────●   │     │
        │                │     ●
       ─┼────────────────────────── 索引
        0    1    2    3    4    5
```

## 陣列索引版對應

海拔陣列 `altitude[]`（概念上，程式中不需要實際建立）：

```
altitude[k] = gain[0] + gain[1] + ... + gain[k-1]   (k >= 1)
altitude[0] = 0
```

| 索引 k | altitude[k] 計算式                | 值   |
|--------|----------------------------------|------|
| 0      | 0（起點）                         |  0   |
| 1      | 0 + (-5)                         | -5   |
| 2      | 0 + (-5) + 1                     | -4   |
| 3      | 0 + (-5) + 1 + 5                 |  1   |
| 4      | 0 + (-5) + 1 + 5 + 0            |  1   |
| 5      | 0 + (-5) + 1 + 5 + 0 + (-7)     | -6   |

公式：`altitude[k] = Σ gain[i] for i = 0 to k-1`（即 `gain[]` 的前綴和）。

## 搜尋／操作過程追蹤

### 範例 1：`gain = [-5, 1, 5, 0, -7]`，預期輸出 `1`

```
初始狀態：
  gain 指標 ───▶ 0x2000
  current = 0
  highest = 0

Step 0: i=0
  讀取 *(gain + 0) = *(0x2000) = -5
  current = 0 + (-5) = -5
  highest = max(0, -5) = 0     （不更新）
  ┌─────────────────────────────────────┐
  │ current: 0 → -5  │ highest: 0 → 0  │
  └─────────────────────────────────────┘

Step 1: i=1
  讀取 *(gain + 1) = *(0x2004) = 1
  current = -5 + 1 = -4
  highest = max(0, -4) = 0     （不更新）
  ┌──────────────────────────────────────┐
  │ current: -5 → -4  │ highest: 0 → 0  │
  └──────────────────────────────────────┘

Step 2: i=2
  讀取 *(gain + 2) = *(0x2008) = 5
  current = -4 + 5 = 1
  highest = max(0, 1) = 1      （更新!）
  ┌──────────────────────────────────────┐
  │ current: -4 → 1   │ highest: 0 → 1  │
  └──────────────────────────────────────┘

Step 3: i=3
  讀取 *(gain + 3) = *(0x200C) = 0
  current = 1 + 0 = 1
  highest = max(1, 1) = 1      （不更新）
  ┌──────────────────────────────────────┐
  │ current: 1 → 1    │ highest: 1 → 1  │
  └──────────────────────────────────────┘

Step 4: i=4
  讀取 *(gain + 4) = *(0x2010) = -7
  current = 1 + (-7) = -6
  highest = max(1, -6) = 1     （不更新）
  ┌──────────────────────────────────────┐
  │ current: 1 → -6   │ highest: 1 → 1  │
  └──────────────────────────────────────┘

迴圈結束 → return highest = 1 ✓
```

### 範例 2：`gain = [-4, -3, -2, -1, 4, 3, 2]`，預期輸出 `0`

假設 `gain` 陣列起始位址 = `0x3000`：

| 索引 | 位址     | 值   |
|------|----------|------|
| 0    | `0x3000` | -4   |
| 1    | `0x3004` | -3   |
| 2    | `0x3008` | -2   |
| 3    | `0x300C` | -1   |
| 4    | `0x3010` |  4   |
| 5    | `0x3014` |  3   |
| 6    | `0x3018` |  2   |

```
初始狀態：
  gain 指標 ───▶ 0x3000
  current = 0
  highest = 0

Step 0: *(0x3000) = -4 → current = -4,  highest = 0
Step 1: *(0x3004) = -3 → current = -7,  highest = 0
Step 2: *(0x3008) = -2 → current = -9,  highest = 0
Step 3: *(0x300C) = -1 → current = -10, highest = 0
Step 4: *(0x3010) =  4 → current = -6,  highest = 0
Step 5: *(0x3014) =  3 → current = -3,  highest = 0
Step 6: *(0x3018) =  2 → current = -1,  highest = 0

迴圈結束 → return highest = 0 ✓
（海拔從未超過起點 0，所以最高海拔就是 0）
```

## 指標變數追蹤總表

### 範例 1：`gain = [-5, 1, 5, 0, -7]`

| Step | 位址     | gain[i] | current (舊→新) | highest (舊→新) | 動作     |
|------|----------|---------|-----------------|-----------------|----------|
| 0    | `0x2000` | -5      | 0 → -5         | 0 → 0          | 不更新   |
| 1    | `0x2004` |  1      | -5 → -4        | 0 → 0          | 不更新   |
| 2    | `0x2008` |  5      | -4 → 1         | 0 → 1          | **更新** |
| 3    | `0x200C` |  0      | 1 → 1          | 1 → 1          | 不更新   |
| 4    | `0x2010` | -7      | 1 → -6         | 1 → 1          | 不更新   |

統計：存取 5 個元素、highest 更新 1 次、最終回傳 1。

### 範例 2：`gain = [-4, -3, -2, -1, 4, 3, 2]`

| Step | 位址     | gain[i] | current (舊→新) | highest (舊→新) | 動作   |
|------|----------|---------|-----------------|-----------------|--------|
| 0    | `0x3000` | -4      | 0 → -4         | 0 → 0          | 不更新 |
| 1    | `0x3004` | -3      | -4 → -7        | 0 → 0          | 不更新 |
| 2    | `0x3008` | -2      | -7 → -9        | 0 → 0          | 不更新 |
| 3    | `0x300C` | -1      | -9 → -10       | 0 → 0          | 不更新 |
| 4    | `0x3010` |  4      | -10 → -6       | 0 → 0          | 不更新 |
| 5    | `0x3014` |  3      | -6 → -3        | 0 → 0          | 不更新 |
| 6    | `0x3018` |  2      | -3 → -1        | 0 → 0          | 不更新 |

統計：存取 7 個元素、highest 更新 0 次、最終回傳 0。

## 記憶體存取模式比較

| 面向 | 本題（int 陣列循序存取） | 若用鏈結串列 |
|---|---|---|
| 存取模式 | 連續位址、sequential scan | 隨機位址、pointer chasing |
| Cache 行為 | 極佳：hardware prefetcher 有效，每個 cache line (64B) 涵蓋 16 個 int | 差：每個節點可能在不同 cache line |
| 空間額外開銷 | 0（純資料陣列） | 每個節點多 8 bytes（next 指標） |
| 隨機存取 | O(1) 透過 `gain[i]` | O(n) 需走訪 |

本題因為只做一次從頭到尾的掃描，int 陣列是最理想的資料結構，cache hit rate 接近 100%。

## 堆疊記憶體對比

本題只有迭代版，無遞迴。Stack frame 固定大小：

```
┌─────────────────────────────────┐
│ largestAltitude() stack frame   │
├─────────────────────────────────┤
│ return address    (8 bytes)     │
│ gain (pointer)    (8 bytes)     │
│ gain_size         (4 bytes)     │
│ current           (4 bytes)     │
│ highest           (4 bytes)     │
│ i                 (4 bytes)     │
├─────────────────────────────────┤
│ 總計 ≈ 32 bytes（含對齊）       │
└─────────────────────────────────┘
```

堆疊使用量 O(1)，與輸入大小無關。若改用遞迴寫法（每次處理 `gain[i]` 遞迴到 `gain[i+1]`），堆疊深度為 O(n)，每層約 32 bytes，對 n=100 約 3.2 KB — 不會溢出但完全沒必要。

## 退化情況分析

本題沒有退化情況對複雜度的影響，因為：

- 時間一定是 O(n)：必須看完每個 `gain[i]`
- 空間一定是 O(1)：只用固定變數

唯一的「邊界 case」是所有 `gain[i]` 都為負數（或零），此時 `highest` 永遠不被更新，回傳初始值 0 — 這正確反映了起點海拔 0 就是最高點。
