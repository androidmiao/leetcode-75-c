# 160. Intersection of Two Linked Lists — 記憶體位址圖解

> 以下所有記憶體位址皆為假設值，用於說明指標操作過程。

---

## 1. Struct 記憶體佈局

```c
struct ListNode {
    int val;                /* 4 bytes (offset 0x00) */
    struct ListNode *next;  /* 8 bytes (offset 0x08, 64-bit 系統) */
};
/* sizeof(struct ListNode) = 16 bytes（含 4 bytes padding） */
```

單一節點記憶體配置（假設位址 0xA000）：

```
        ┌───────────────────────────────────┐
0xA000  │  val (int)      │  padding        │
        │  4 bytes        │  4 bytes        │
        ├───────────────────────────────────┤
0xA008  │  next (struct ListNode *)         │
        │  8 bytes                          │
        └───────────────────────────────────┘
        總大小：16 bytes (0x10)
```

## 3. 搜尋過程追蹤

### Example 1 追蹤：intersectVal = 8

初始狀態：`pA = 0xA000 (headA)`，`pB = 0xB000 (headB)`

| 步驟 | pA 位址 | pA->val | pB 位址 | pB->val | pA == pB? | 動作 |
|------|---------|---------|---------|---------|-----------|------|
| 0 | 0xA000 | 4 | 0xB000 | 5 | ✗ | 繼續 |
| 1 | 0xA010 | 1 | 0xB010 | 6 | ✗ | 繼續 |
| 2 | 0xC000 | 8 | 0xB020 | 1 | ✗ | 繼續 |
| 3 | 0xC010 | 4 | 0xC000 | 8 | ✗ | 繼續 |
| 4 | 0xC020 | 5 | 0xC010 | 4 | ✗ | 繼續 |
| 5 | NULL | — | 0xC020 | 5 | ✗ | pA 跳至 headB |
| 6 | 0xB000 | 5 | NULL | — | ✗ | pB 跳至 headA |
| 7 | 0xB010 | 6 | 0xA000 | 4 | ✗ | 繼續 |
| 8 | 0xB020 | 1 | 0xA010 | 1 | ✗ | 繼續（值相同但位址不同！）|
| 9 | **0xC000** | **8** | **0xC000** | **8** | **✓** | **回傳 0xC000** |

#### 指標移動追蹤圖

```
步驟 0: pA ──▶ [4]     [1]     [8]     [4]     [5]     NULL
              0xA000  0xA010  0xC000  0xC010  0xC020

         pB ──▶ [5]     [6]     [1]     [8]     [4]     [5]     NULL
              0xB000  0xB010  0xB020  0xC000  0xC010  0xC020

步驟 5: pA 到達 NULL → 跳至 headB (0xB000)
步驟 6: pB 到達 NULL → 跳至 headA (0xA000)

步驟 9: pA ──▶ [8]  ◄── pB    （同一個節點 0xC000）
              0xC000
              ▲ 找到交叉點！
```

#### 指標移動摘要

```
pA 路徑：0xA000 → 0xA010 → 0xC000 → 0xC010 → 0xC020 → NULL
         → 0xB000 → 0xB010 → 0xB020 → [0xC000]  ← 相遇！
         共走 a + c + b = 2 + 3 + 3 = 8 步（+ 1 步 NULL→headB 跳轉）

pB 路徑：0xB000 → 0xB010 → 0xB020 → 0xC000 → 0xC010 → 0xC020 → NULL
         → 0xA000 → 0xA010 → [0xC000]  ← 相遇！
         共走 b + c + a = 3 + 3 + 2 = 8 步（+ 1 步 NULL→headA 跳轉）
```

### Example 3 追蹤：無交叉

```
listA = [2, 6, 4],  listB = [1, 5]
```

**節點位址對照表：**

| 節點 | val | 假設位址 | next |
|------|-----|----------|------|
| A0 | 2 | 0xA000 | 0xA010 |
| A1 | 6 | 0xA010 | 0xA020 |
| A2 | 4 | 0xA020 | NULL |
| B0 | 1 | 0xB000 | 0xB010 |
| B1 | 5 | 0xB010 | NULL |

| 步驟 | pA 位址 | pB 位址 | pA == pB? | 動作 |
|------|---------|---------|-----------|------|
| 0 | 0xA000 | 0xB000 | ✗ | 繼續 |
| 1 | 0xA010 | 0xB010 | ✗ | 繼續 |
| 2 | 0xA020 | NULL | ✗ | pB 跳至 headA |
| 3 | NULL | 0xA000 | ✗ | pA 跳至 headB |
| 4 | 0xB000 | 0xA010 | ✗ | 繼續 |
| 5 | 0xB010 | 0xA020 | ✗ | 繼續 |
| 6 | **NULL** | **NULL** | **✓** | **回傳 NULL** |

```
pA 路徑：A0 → A1 → A2 → NULL → B0 → B1 → [NULL]  (a + b = 3 + 2 = 5 步)
pB 路徑：B0 → B1 → NULL → A0 → A1 → A2 → [NULL]  (b + a = 2 + 3 = 5 步)
兩者同時到達 NULL → 無交叉
```

## 4. 指標變數追蹤總表

### Example 1 統計

| 指標 | 總移動步數 | 跳轉次數 | 到達交叉步數 |
|------|-----------|----------|-------------|
| pA | 9 步 | 1 次（NULL→headB） | 第 9 步 |

## 5. 記憶體存取模式比較

| 維度 | 指標鏈結串列 | 陣列模擬 |
|------|-------------|---------|
| **存取方式** | 間接存取：`pA->next` 需 dereference | 直接索引：`arr[i]` 隨機存取 |
| **Cache 行為** | 差——節點散布在 heap 各處，幾乎每次 `->next` 都是 cache miss | 優——連續記憶體，prefetch 有效 |
| **空間額外開銷** | 每節點 8 bytes（64-bit 指標）+ padding | 無指標開銷，但需預先配置固定大小陣列 |
| **插入/刪除** | O(1)——只需修改指標 | O(N)——需搬移元素 |
| **交叉串列可行性** | 自然支援——兩條串列的 next 指向同一節點 | 不自然——需額外映射表或 index 間接層 |

### 為什麼鏈結串列適合此題

此題的核心是兩條串列**共享相同的物理節點**（指標相等，非值相等）。這在陣列中無法自然表達——陣列元素的「相同」只能靠 index 相等或值相等來判斷，無法表達兩個不同起點最終匯聚到同一塊記憶體的概念。

## 6. 堆疊記憶體對比

此題的 Two Pointers 解法為純迭代，不涉及遞迴，因此堆疊使用量為常數：

```
迭代版本（本解法）：
┌─────────────────┐
│ stack frame      │
│  pA: 8 bytes     │
│  pB: 8 bytes     │
│  headA: 8 bytes  │  ← 函式參數
│  headB: 8 bytes  │  ← 函式參數
│  return addr     │
└─────────────────┘
總計：~40 bytes，O(1) 固定
```

若改用遞迴（學術上可行但此題不建議）：

```
假設遞迴版本——每步遞迴：
┌─────────────────┐ ← 第 N 層
│  pA, pB, ret    │
├─────────────────┤ ← 第 N-1 層
│  pA, pB, ret    │
├─────────────────┤
│       ...       │
├─────────────────┤ ← 第 1 層
│  pA, pB, ret    │
└─────────────────┘
最大深度：a + b + c 層
堆疊用量：O(M + N) — 完全失去空間優勢
```

在 Linux kernel 中，kernel stack 通常只有 8KB（x86-64）或 16KB，遞迴方案在長串列上很容易 stack overflow，因此迭代版本是唯一合理選擇。

## 7. 退化情況分析

### 最差情況：一條極長、一條極短、無交叉

```
A: [1] → [2] → ... → [30000] → NULL    (M = 30000)
B: [1] → NULL                           (N = 1)
```

- pA 走：30000 步到 NULL → 跳到 B → 1 步到 NULL → 跳到 A → 走到 NULL（但 pB 也同步到 NULL）
- pB 走：1 步到 NULL → 跳到 A → 30000 步到 NULL
- 總步數：M + N = 30001
- 每個指標各做 1 次跳轉
- 結果：pA == pB == NULL → 回傳 NULL

### 最差 cache 行為

當兩條串列的私有段很長且節點在記憶體中交錯配置時：

```
記憶體配置（最差情況）：
A 節點：0x1000, 0x9000, 0x3000, ...  ← 不連續
B 節點：0x5000, 0x2000, 0x7000, ...  ← 不連續

指標存取模式：
pA: 0x1000 → 0x9000 → 0x3000 → ...   ← 每次跳躍數 KB，cache miss
pB: 0x5000 → 0x2000 → 0x7000 → ...   ← 同樣大量 cache miss
```

交叉走法的第二遍（跳到對方串列後）理論上可以利用第一遍留在 cache 中的數據，但若串列長度超過 L1/L2 cache 容量（通常 256KB ~ 數 MB），第二遍仍會全部 cache miss。
