## 2. 完整資料結構記憶體配置

### 操作序列範例

```
heights = [2, 1, 5, 6, 2, 3]
new_heights = [0, 2, 1, 5, 6, 2, 3, 0]
```

### 分步驟記憶體狀態

#### 步驟 1：i=0, new_heights[0]=0, push 0

```
new_heights: [0, 2, 1, 5, 6, 2, 3, 0]
              ↑i

stack:  [0]        top=0
         ↑
       0xC000
       值: 0

max_area = 0
```

#### 步驟 2：i=1, new_heights[1]=2, 2>0 → push 1

```
new_heights: [0, 2, 1, 5, 6, 2, 3, 0]
                 ↑i

stack:  [0, 1]     top=1
         ↑  ↑
       0xC000 0xC004
       值: 0  值: 1

max_area = 0
```

#### 步驟 3：i=2, new_heights[2]=1, 1<2 → pop 1, 計算面積

```
pop: h_index=1, h=new_heights[1]=2
     w = i - stack[top] - 1 = 2 - 0 - 1 = 1
     area = 2 × 1 = 2    ← max_area 更新為 2

new_heights: [0, 2, 1, 5, 6, 2, 3, 0]
                    ↑i

1≥0? new_heights[0]=0 ≤ 1, 停止 pop
push 2:

stack:  [0, 2]     top=1
         ↑  ↑
       0xC000 0xC004
       值: 0  值: 2

max_area = 2
```

#### 步驟 4：i=3, new_heights[3]=5, 5>1 → push 3

```
stack:  [0, 2, 3]     top=2

max_area = 2
```

#### 步驟 5：i=4, new_heights[4]=6, 6>5 → push 4

```
stack:  [0, 2, 3, 4]     top=3

max_area = 2
```

#### 步驟 6：i=5, new_heights[5]=2 — 連續 pop！

```
pop 4: h=new_heights[4]=6, w=5-3-1=1, area=6   → max_area=6
pop 3: h=new_heights[3]=5, w=5-2-1=2, area=10  → max_area=10 ★

new_heights: [0, 2, 1, 5, 6, 2, 3, 0]
                          ↑i

2 ≥ new_heights[2]=1? 停止 pop
push 5:

stack:  [0, 2, 5]     top=2
         ↑  ↑  ↑
       0xC000 0xC004 0xC008
       值: 0  值: 2  值: 5

max_area = 10
```

這是關鍵步驟！面積 10 的矩形對應：
```
        ┌───┐
   6    │   │
   5  ┌─┤   │
   4  │ │   │
   3  │ │   │
   2  │ │   │
   1  │ │   │
     ─┼─┼───┼─
     [5] [6]
      3   4    (new_heights 索引)

高度 = 5, 寬度 = i - stack[top] - 1 = 5 - 2 - 1 = 2
面積 = 5 × 2 = 10
```

#### 步驟 7：i=6, new_heights[6]=3, 3>2 → push 6

```
stack:  [0, 2, 5, 6]     top=3

max_area = 10
```

#### 步驟 8：i=7, new_heights[7]=0 (右哨兵) — 清空棧

```
pop 6: h=3, w=7-5-1=1, area=3     → max_area 不變
pop 5: h=2, w=7-2-1=4, area=8     → max_area 不變
pop 2: h=1, w=7-0-1=6, area=6     → max_area 不變
push 7:

stack:  [0, 7]     top=1

max_area = 10  （最終結果）
```

## 3. 陣列索引版對應

### 原始索引 vs. 哨兵索引的對映

```
原始陣列:    heights[0]  heights[1]  heights[2]  heights[3]  heights[4]  heights[5]
                 ↓            ↓            ↓            ↓            ↓            ↓
哨兵陣列:  new_h[0]  new_h[1]  new_h[2]  new_h[3]  new_h[4]  new_h[5]  new_h[6]  new_h[7]
            左哨兵                                                               右哨兵
```

**索引轉換公式：**
- `new_heights[i] = heights[i - 1]`（當 1 ≤ i ≤ n）
- `new_heights[0] = 0`（左哨兵）
- `new_heights[n + 1] = 0`（右哨兵）

### 堆疊索引含義

棧中儲存的是 `new_heights` 的索引。當計算寬度時：

```
寬度 = i - stack[top] - 1
```

這表示：被 pop 柱子能延伸的範圍是從 `stack[top] + 1` 到 `i - 1`（不含左右邊界）。

## 4. 搜尋／操作過程追蹤

### 範例 1：heights = [2,1,5,6,2,3]，預期輸出 10

#### 指標版追蹤

| 步驟 | i | 位址(new_h) | new_h[i] | 動作 | 棧狀態(索引) | 棧狀態(高度) | max_area |
|-----|---|------------|----------|------|------------|------------|----------|
| 1 | 0 | 0xB000 | 0 | push 0 | [0] | [0] | 0 |
| 2 | 1 | 0xB004 | 2 | push 1 | [0,1] | [0,2] | 0 |
| 3 | 2 | 0xB008 | 1 | pop 1→area=2, push 2 | [0,2] | [0,1] | 2 |
| 4 | 3 | 0xB00C | 5 | push 3 | [0,2,3] | [0,1,5] | 2 |
| 5 | 4 | 0xB010 | 6 | push 4 | [0,2,3,4] | [0,1,5,6] | 2 |
| 6 | 5 | 0xB014 | 2 | pop 4→area=6, pop 3→area=10, push 5 | [0,2,5] | [0,1,2] | **10** |
| 7 | 6 | 0xB018 | 3 | push 6 | [0,2,5,6] | [0,1,2,3] | 10 |
| 8 | 7 | 0xB01C | 0 | pop 6→3, pop 5→8, pop 2→6, push 7 | [0,7] | [0,0] | 10 |

#### 指標移動摘要

```
┌──────────────────────────────────────────────┐
│ i 指標移動：                                    │
│ 0xB000 → 0xB004 → 0xB008 → 0xB00C →          │
│ 0xB010 → 0xB014 → 0xB018 → 0xB01C            │
│                                               │
│ 棧頂指標 (top) 變化：                            │
│ -1 → 0 → 1 → 1 → 2 → 3 → 2 → 3 → 1          │
│                                               │
│ 總 push 次數：8                                 │
│ 總 pop 次數：6                                  │
│ 總操作：14 = 2 × (n+2) — 驗證 O(n) 攤提         │
└──────────────────────────────────────────────┘
```

### 範例 2：heights = [2,4]，預期輸出 4

new_heights = [0, 2, 4, 0]

| 步驟 | i | new_h[i] | 動作 | 棧狀態(索引) | max_area |
|-----|---|----------|------|------------|----------|
| 1 | 0 | 0 | push 0 | [0] | 0 |
| 2 | 1 | 2 | push 1 | [0,1] | 0 |
| 3 | 2 | 4 | push 2 | [0,1,2] | 0 |
| 4 | 3 | 0 | pop 2→area=4, pop 1→area=4, push 3 | [0,3] | **4** |

```
pop 2: h=4, w=3-1-1=1, area=4 → max_area=4
pop 1: h=2, w=3-0-1=2, area=4 → max_area=4（相等，不更新）
```

## 5. 指標變數追蹤總表

### 範例 1 詳細 Pop 追蹤

| Pop 序號 | 被 pop 索引 | 高度 h | 左邊界(stack[top]) | 右邊界(i) | 寬度 w | 面積 | 是否更新 max |
|---------|-----------|-------|-------------------|----------|-------|------|------------|
| 1 | 1 | 2 | 0 | 2 | 1 | 2 | 0→2 |
| 2 | 4 | 6 | 3 | 5 | 1 | 6 | 2→6 |
| 3 | 3 | 5 | 2 | 5 | 2 | 10 | 6→**10** |
| 4 | 6 | 3 | 5 | 7 | 1 | 3 | 不變 |
| 5 | 5 | 2 | 2 | 7 | 4 | 8 | 不變 |
| 6 | 2 | 1 | 0 | 7 | 6 | 6 | 不變 |

**統計：**
- 棧操作總數：8 push + 6 pop = 14 次
- new_heights 存取次數：每次比較和 pop 時各一次
- max_area 更新次數：3 次（2 → 6 → 10）

## 6. 記憶體存取模式比較

### 單調棧法 vs. 暴力法

| 面向 | 單調棧法（本實作） | 暴力法 O(n²) |
|-----|----------------|-------------|
| **陣列存取** | 線性掃描 new_heights，順序存取 | 雙層迴圈，外層順序、內層也順序 |
| **Cache 行為** | 極佳 — new_heights 順序存取，stack 頂部熱點 | 良好 — 但重複掃描同一段資料 |
| **記憶體使用** | heights(n) + new_heights(n+2) + stack(n+2) ≈ 3n | heights(n) + 常數變數 ≈ n |
| **Cache miss** | 極少 — 連續存取 + 棧深度受限 | 多 — O(n²) 次存取導致更多冷存取 |
| **分支預測** | pop 條件不規則但攤提少 | 內層迴圈可預測（向右遞增） |

### 單調棧法 vs. 分治法

| 面向 | 單調棧法 | 分治法 O(n log n) |
|-----|---------|-----------------|
| **空間開銷** | O(n) 堆疊 + O(n) 陣列 | O(n) 遞迴棧幀 |
| **存取模式** | 連續線性 | 遞迴分割，存取跳躍 |
| **Cache 效率** | 高 — 線性掃描 | 中 — 遞迴深入時子陣列變小 |
| **實作風險** | 低 — 單迴圈 | 高 — 遞迴 + 找最小值 |

```
│ heightsSize (int)      4 bytes  │  ← 6

│ top (int)              4 bytes  │

├─────────────────────────────────┤
│ 棧幀總計 ≈ 68 bytes             │
└─────────────────────────────────┘

堆積 (heap) 配置:
┌─────────────────────────┐
│ new_heights: 32 bytes   │  ← 0xB000

├─────────────────────────┤
│ 堆積總計 ≈ 64 bytes      │
└─────────────────────────┘
```

### 假設改用遞迴分治法

```
最壞情況（已排序輸入 [1,2,3,4,5,6]）：
遞迴深度 = n = 6

棧幀 #6  ┌─ calculateArea(h, 5, 5) ─────────── ~40 bytes ─┐
棧幀 #5  ├─ calculateArea(h, 4, 5) ─────────── ~40 bytes ─┤
棧幀 #4  ├─ calculateArea(h, 3, 5) ─────────── ~40 bytes ─┤
棧幀 #3  ├─ calculateArea(h, 2, 5) ─────────── ~40 bytes ─┤
棧幀 #2  ├─ calculateArea(h, 1, 5) ─────────── ~40 bytes ─┤
棧幀 #1  ├─ calculateArea(h, 0, 5) ─────────── ~40 bytes ─┤
         └────────────────────────────────────────────────┘
         最壞棧使用: 6 × 40 = 240 bytes

一般情況（平衡分割）：
遞迴深度 = log₂(n) ≈ 3
棧使用: 3 × 40 = 120 bytes
```

**比較：** 迭代棧法使用固定的函數棧幀（~68 bytes）+ 堆積陣列（~64 bytes），總計 ~132 bytes。分治法最壞情況需 ~240 bytes 棧空間且有溢位風險。

## 8. 退化情況分析

### 退化情況一：完全遞增 [1, 2, 3, 4, 5, 6]

```
new_heights = [0, 1, 2, 3, 4, 5, 6, 0]

所有 push 在前，所有 pop 在最後（右哨兵觸發）：

棧增長過程:
i=0: [0]
i=1: [0,1]
i=2: [0,1,2]
i=3: [0,1,2,3]
i=4: [0,1,2,3,4]
i=5: [0,1,2,3,4,5]
i=6: [0,1,2,3,4,5,6]    ← 棧最大深度 = 7 = n+1
i=7: 連續 pop 6 次       ← 所有計算集中在此

棧深度最大值 = n + 1（含左哨兵）
```

### 退化情況二：完全遞減 [6, 5, 4, 3, 2, 1]

```
new_heights = [0, 6, 5, 4, 3, 2, 1, 0]

每一步都觸發 pop：

i=0: push 0          [0]
i=1: push 1          [0,1]
i=2: pop 1, push 2   [0,2]     ← 棧深度始終 ≤ 2
i=3: pop 2, push 3   [0,3]
i=4: pop 3, push 4   [0,4]
...

棧深度最大值 = 2（左哨兵 + 當前元素）
記憶體使用最小
```

### 退化情況三：全部相同 [3, 3, 3, 3, 3, 3]

```
new_heights = [0, 3, 3, 3, 3, 3, 3, 0]

由於 while 條件是 >（嚴格大於），相等高度不觸發 pop：
所有元素被 push，行為類似遞增情況。

棧深度最大值 = n + 1
最大面積 = 3 × 6 = 18（最後一個 pop 計算出）
```
