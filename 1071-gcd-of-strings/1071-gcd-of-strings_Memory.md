## 3. 陣列索引版對應

### 指標地址公式

對於字串 `s` 起始地址為 `base_addr`：

**公式:**
```
&s[index] = base_addr + index    （char 大小為 1 byte）
```

### 字元存取範例

```
str1:
  *(str1 + 0) = *(0xA000 + 0) = *(0xA000) = 'A' (0x41)
  *(str1 + 1) = *(0xA000 + 1) = *(0xA001) = 'B' (0x42)
  *(str1 + 5) = *(0xA000 + 5) = *(0xA005) = 'C' (0x43)

ret:
  *(ret + 0) = *(0xC000 + 0) = *(0xC000) = 'A' (0x41)
  *(ret + 1) = *(0xC000 + 1) = *(0xC001) = 'B' (0x42)
  *(ret + 2) = *(0xC000 + 2) = *(0xC002) = 'C' (0x43)

驗證公式：
  str1[pass * len + i] = str1[pass * 3 + i]
    pass=0, i=0: str1[0] = 'A'  vs  ret[0] = 'A'  ✓
    pass=0, i=1: str1[1] = 'B'  vs  ret[1] = 'B'  ✓
    pass=0, i=2: str1[2] = 'C'  vs  ret[2] = 'C'  ✓
    pass=1, i=0: str1[3] = 'A'  vs  ret[0] = 'A'  ✓
    pass=1, i=1: str1[4] = 'B'  vs  ret[1] = 'B'  ✓
    pass=1, i=2: str1[5] = 'C'  vs  ret[2] = 'C'  ✓
```

## 4. 搜尋／操作過程追蹤

### 例 1：str1 = "ABCABC", str2 = "ABC"

#### 逐步追蹤

```
【初始化】
str1len = 6, str2len = 3
str1len >= str2len → 不需交換
ret = malloc(4) → strcpy(ret, "ABC")
len = 3

【While 迴圈：檢查長度整除性】
len=3: 6%3=0 且 3%3=0 → ✓ 跳出迴圈

【第一輪驗證：ret[0..2] 覆蓋 str1】

  str1:  0xA000  0xA001  0xA002  0xA003  0xA004  0xA005
           │       │       │       │       │       │
           ▼       ▼       ▼       ▼       ▼       ▼
         ┌─────┬─────┬─────┬─────┬─────┬─────┐
         │  A  │  B  │  C  │  A  │  B  │  C  │
         └─────┴─────┴─────┴─────┴─────┴─────┘
         ◄───── pass 0 ────▶◄──── pass 1 ────▶

  ret:   0xC000  0xC001  0xC002
           │       │       │
           ▼       ▼       ▼
         ┌─────┬─────┬─────┐
         │  A  │  B  │  C  │
         └─────┴─────┴─────┘

  pass 0:
    i=0: ret[0]='A' vs str1[0*3+0]='A' → ✓ 匹配
    i=1: ret[1]='B' vs str1[0*3+1]='B' → ✓ 匹配
    i=2: ret[2]='C' vs str1[0*3+2]='C' → ✓ 匹配
    pass++  → pass=1

  pass 1:
    i=0: ret[0]='A' vs str1[1*3+0]='A' → ✓ 匹配
    i=1: ret[1]='B' vs str1[1*3+1]='B' → ✓ 匹配
    i=2: ret[2]='C' vs str1[1*3+2]='C' → ✓ 匹配
    pass++  → pass=2
    2 >= 6/3=2 → 跳出迴圈

【第二輪驗證：ret[0..2] 覆蓋 str2】

  str2:  0xB000  0xB001  0xB002
           │       │       │
           ▼       ▼       ▼
         ┌─────┬─────┬─────┐
         │  A  │  B  │  C  │
         └─────┴─────┴─────┘
         ◄───── pass 0 ────▶

  pass 0:
    i=0: ret[0]='A' vs str2[0*3+0]='A' → ✓ 匹配
    i=1: ret[1]='B' vs str2[0*3+1]='B' → ✓ 匹配
    i=2: ret[2]='C' vs str2[0*3+2]='C' → ✓ 匹配
    pass++  → pass=1
    1 >= 3/3=1 → 跳出迴圈

【收尾】
ret = realloc(ret, 3+1=4) → 不變
ret[3] = '\0'
回傳 "ABC"
```

## 6. 記憶體存取模式比較

### C 字串 vs. 長度前綴字串

```
C 字串（本題使用）：
┌─────────────────────────────────────────┐
│  C 字串記憶體布局                         │
│                                          │
│  ┌───┬───┬───┬───┬───┬───┬───┐          │
│  │ A │ B │ C │ A │ B │ C │\0 │          │
│  └───┴───┴───┴───┴───┴───┴───┘          │
│  0xA0 0xA1 0xA2 0xA3 0xA4 0xA5 0xA6     │
│                                          │
│  特點：                                   │
│  • strlen() 需要 O(n) 遍歷找到 '\0'       │
│  • 連續記憶體，快取友善                    │
│  • 無額外元資料開銷                        │
└──────────────────────────────────────────┘

長度前綴字串（如 Pascal string）：
┌─────────────────────────────────────────┐
│  長度前綴字串布局                          │
│                                          │
│  ┌──────┬───┬───┬───┬───┬───┬───┐       │
│  │ len=6│ A │ B │ C │ A │ B │ C │       │
│  └──────┴───┴───┴───┴───┴───┴───┘       │
│  0xA0   0xA4 0xA5 0xA6 0xA7 0xA8 0xA9   │
│                                          │
│  特點：                                   │
│  • 長度查詢 O(1)                           │
│  • 額外 4/8 bytes 開銷（size_t）           │
│  • 本題多次使用 strlen → 長度前綴更高效     │
└──────────────────────────────────────────┘
```

### 快取行為分析

```
本題的記憶體存取模式：

  ret[0..len-1] 被重複讀取（每個 pass 都讀一次）：
  ┌─────────────────────────────────────┐
  │  快取行 (64 bytes)                    │
  │  ┌───┬───┬───┐                       │
  │  │ A │ B │ C │ ← ret 通常在一行內     │
  │  └───┴───┴───┘                       │
  │  重複存取 → L1 快取命中率接近 100%      │
  └─────────────────────────────────────┘

  str1 被順序讀取：
  ┌─────────────────────────────────────┐
  │  快取行 (64 bytes)                    │
  │  ┌───┬───┬───┬───┬───┬───┐          │
  │  │ A │ B │ C │ A │ B │ C │          │
  │  └───┴───┴───┴───┴───┴───┘          │
  │  順序存取 → 預取機制有效               │
  └─────────────────────────────────────┘

總結：
  • ret 長度通常很短（≤ 較短字串長度），完全在 L1 快取中
  • str1/str2 為順序存取，CPU 預取器能有效預取
  • 整體快取命中率高，記憶體延遲影響小
```

## 7. 堆記憶體管理追蹤

### malloc / realloc / 回傳值的生命週期

```
時間線（例 1）：

  t0: ret = malloc(4)
      堆：┌───┬───┬───┬───┐
          │ ? │ ? │ ? │ ? │  ← 未初始化
          └───┴───┴───┴───┘
          0xC000

  t1: strcpy(ret, str2)
      堆：┌───┬───┬───┬───┐
          │ A │ B │ C │\0 │  ← 從 str2 複製
          └───┴───┴───┴───┘
          0xC000

  t2: 驗證通過, len=3

  t3: ret = realloc(ret, 4)
      堆：┌───┬───┬───┬───┐
          │ A │ B │ C │\0 │  ← 大小不變，可能原地返回
          └───┴───┴───┴───┘
          0xC000（或新地址）

  t4: ret[3] = '\0'  ← 已經是 '\0'，多餘但安全

  t5: return ret
      ※ 呼叫者負責 free(ret)
      ※ 若忘記 free → 記憶體洩漏

時間線（例 2, 縮小情況）：

  t0: ret = malloc(5)
      堆：┌───┬───┬───┬───┬───┐
          │ A │ B │ A │ B │\0 │
          └───┴───┴───┴───┴───┘
          0xC000  (5 bytes)

  t3: ret = realloc(ret, 3)
      堆：┌───┬───┬───┐
          │ A │ B │\0 │  ← 縮小到 3 bytes
          └───┴───┴───┘
          0xC000（通常原地縮小）

  ※ realloc 縮小時：
    - glibc: 通常原地返回，不移動資料
    - 若原 block 在 slab 中，可能返回新地址
    - 標準不保證縮小一定成功（但實務上幾乎總是成功）
```

## 總結

### 演算法特性

| 特性 | 說明 |
|------|------|
| **時間複雜度** | O(min(m,n)·(m+n)) 最差，O(m+n) 最佳 |
| **空間複雜度** | O(n)，堆上分配候選字串 |
| **快取友善度** | 高，短字串重複讀取 + 長字串順序讀取 |
| **記憶體分配** | 1 次 malloc + 1 次 realloc |
| **潛在風險** | realloc 返回 NULL 未檢查、呼叫者需 free |

### 核心啟示

```
字串比對的記憶體模式：

  ret（候選）: 短字串，重複讀取 → L1 快取常駐
       ┌───┐
       │ABC│ ← 反覆讀取，快取命中
       └───┘

  str1/str2: 順序掃描 → 預取友善
       ┌───────────────────────┐
       │A B C A B C│ ← 順序存取
       └───────────────────────┘
       ──────────────────────▶ 掃描方向

  malloc/realloc: 堆管理開銷
       ┌─────────┐         ┌───────┐
       │ 5 bytes │ ──▶     │3 bytes│  ← realloc 縮小
       └─────────┘         └───────┘
```
