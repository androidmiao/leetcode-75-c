# LeetCode 35: Search Insert Position — 記憶體分析

## 問題概述

**Function Signature:**
```c
int searchInsert(int* nums, int numsSize, int target)
```

**演算法:** 二分搜尋 (Binary Search)
**時間複雜度:** O(log n)
**空間複雜度:** O(1) — 迭代實現，無遞迴

---

## 1. 陣列記憶體佈局

### 範例: nums = [1, 3, 5, 6]

**假設基址 (Base Address):** 0xA000
**每個元素大小:** 4 bytes (int)

### 記憶體圖示

```
┌─────────────────────────────────────────────────┐
│  C Int 陣列在記憶體中的線性佈局                  │
└─────────────────────────────────────────────────┘

16 bytes total (4 elements × 4 bytes each)

Address        Data        Index
┌──────┬─────────────┐
│0xA000│      1      │  nums[0]
├──────┼─────────────┤
│0xA004│      3      │  nums[1]
├──────┼─────────────┤
│0xA008│      5      │  nums[2]
├──────┼─────────────┤
│0xA00C│      6      │  nums[3]
└──────┴─────────────┘

記憶體區間: 0xA000 - 0xA00F (16 bytes)
```

### 十六進制位址計算

| 索引 | 十進制偏移 | 十六進制地址 | 內容 |
|------|----------|------------|------|
| 0    | 0        | 0xA000     | 1    |
| 1    | 4        | 0xA004     | 3    |
| 2    | 8        | 0xA008     | 5    |
| 3    | 12       | 0xA00C     | 6    |

---

## 2. 指標與索引對應

### 地址計算公式

```c
memory_address = base_address + (index × sizeof(int))
memory_address = 0xA000 + (index × 4)
```

### 對應表

```
┌──────────────┬──────────────┬────────────────────┐
│ 陣列索引      │ 十六進制地址  │ 間接解參考         │
│ (Index)      │ (Address)    │ (*ptr)             │
├──────────────┼──────────────┼────────────────────┤
│ i = 0        │ 0xA000       │ *(nums + 0) = 1    │
├──────────────┼──────────────┼────────────────────┤
│ i = 1        │ 0xA004       │ *(nums + 1) = 3    │
├──────────────┼──────────────┼────────────────────┤
│ i = 2        │ 0xA008       │ *(nums + 2) = 5    │
├──────────────┼──────────────┼────────────────────┤
│ i = 3        │ 0xA00C       │ *(nums + 3) = 6    │
└──────────────┴──────────────┴────────────────────┘
```

**注意:** 指標算術自動以 sizeof(int) 為單位
- `nums + 1` 在記憶體上移進 4 bytes
- `nums[i]` 等同於 `*(nums + i)`

## 3. 搜尋過程追蹤

### Example 1: nums = [1,3,5,6], target = 5

**預期輸出:** 2 (找到目標值在索引 2)

#### 初始狀態

```
┌──────────────────────────────────────────────────┐
│ 陣列: [1, 3, 5, 6]                               │
│ 搜尋目標: 5                                       │
│ left = 0, right = 3 (numsSize - 1)               │
└──────────────────────────────────────────────────┘

memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
         │
         ├─ left=0   (0xA000)
         ├─ right=3  (0xA00C)
         └─ pivot=?
```

#### 迭代步驟

**步驟 1:**
```
left = 0 (0xA000)
right = 3 (0xA00C)
pivot = (0 + 3) / 2 = 1 (0xA004)
nums[1] = 3

比較: 3 < 5 (target)
決策: target 在右半部，left = pivot + 1 = 2

memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
              ▲    │
         pivot ┴─ left (新)

         舊值: left=0     → 新值: left=2
         舊值: pivot=1    → 新值: pivot=?
```

**步驟 2:**
```
left = 2 (0xA008)
right = 3 (0xA00C)
pivot = (2 + 3) / 2 = 2 (0xA008)
nums[2] = 5

比較: 5 == 5 (target)
決策: 找到！返回 pivot = 2

memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
              │   ▲
              ├──┤
           pivot (找到!)

✓ 返回值: 2
```

```
memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
```

#### 迭代步驟

**步驟 1:**
```
left = 0 (0xA000)
right = 3 (0xA00C)
pivot = (0 + 3) / 2 = 1 (0xA004)
nums[1] = 3

比較: 3 > 2 (target)
決策: target 在左半部，right = pivot - 1 = 0

memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
         │    ▲
         ├─ left
         └─ right (新)

舊值: right=3    → 新值: right=0
舊值: pivot=1    → 新值: pivot=?
```

**步驟 2:**
```
left = 0 (0xA000)
right = 0 (0xA000)
pivot = (0 + 0) / 2 = 0 (0xA000)
nums[0] = 1

比較: 1 < 2 (target)
決策: target 在右半部，left = pivot + 1 = 1

memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
         ▲    ▲
    pivot │
         └─ left (新)

舊值: left=0     → 新值: left=1
舊值: right=0    → 新值: left > right (終止)
```

**步驟 3:**
```
迴圈條件檢查: left (1) > right (0)?
YES → 迴圈終止

left = 1 代表插入位置

memory:  ┌────┬────┬────┬────┐
         │ 1  │ 3  │ 5  │ 6  │
         └────┴────┴────┴────┘
         0xA0 0xA0 0xA0 0xA0
         00   04   08   0C
              ▲
         [要插入位置]

✓ 返回值: 1 (將 2 插入在索引 1)
  結果陣列: [1, 2, 3, 5, 6]
```

## 5. 記憶體存取模式分析

### 二分搜尋的記憶體特性

#### 存取模式: 非順序式 (Non-Sequential)

```
小陣列範例 (16 bytes): [1, 3, 5, 6]

存取順序:
─────────────────────────────────────
iteration │ 存取索引 │ 十六進制位址 │ 距離上次
─────────────────────────────────────
   1      │    1    │   0xA004    │ -
   2      │    2    │   0xA008    │ 4 bytes
   3      │    3    │   0xA00C    │ 4 bytes
─────────────────────────────────────

搜尋跳躍式存取，不像線性掃描的連續存取
```

### 快取表現分析 (Cache Behavior)

#### 小陣列場景: nums = [1, 3, 5, 6] (16 bytes)

```
L1 快取通常 64 bytes 一條線

┌─────────────────────────────────────────────┐
│ L1 Cache Line (64 bytes)                    │
│                                             │
│  ┌────┬────┬────┬────┬                     │
│  │ 1  │ 3  │ 5  │ 6  │ ... (其他變數)      │
│  └────┴────┴────┴────┴                     │
│  0xA000   0xA008   0xA00C   ...            │
│                                             │
│  ✓ 整個陣列 (16 bytes) 適應於一條快取線    │
│  ✓ 首次命中後，所有迭代都是快取命中        │
│  ✓ L1 快取命中率: 100% (極佳)              │
└─────────────────────────────────────────────┘

快取效能: ★★★★★ (完美)
```

#### 大陣列場景: nums = [...100000 elements...] (400 KB)

```
陣列大小 > 2 MB (L3 快取)，跨越多條快取線

迭代 1: pivot = 50000
   存取: 0xA000 + 200000 bytes = 0xA030D40
   └─ 快取缺失: 加載快取線

迭代 2: pivot = 25000
   存取: 0xA000 + 100000 bytes = 0xA018640
   └─ 快取缺失: 加載新快取線 (原快取線已清除)

迭代 3: pivot = 75000
   存取: 0xA000 + 300000 bytes = 0xA048E40
   └─ 快取缺失: 加載另一條快取線

...

快取命中率: ≈ 20-30% (不佳)
原因: 隨機跳躍存取，空間局部性差
```

### 記憶體存取效率對比

```
┌──────────────┬────────────┬────────────┬──────────┐
│ 演算法        │ 時間複雜度 │ 空間局部性 │ 快取表現 │
├──────────────┼────────────┼────────────┼──────────┤
│ 線性掃描      │ O(n)       │ 優秀 ✓     │ 93%      │
│ (順序存取)    │            │ (連續存取) │ 命中率   │
│              │            │            │          │
│ 二分搜尋      │ O(log n)   │ 差 ✗       │ 30%      │
│ (跳躍存取)    │ ⭐ 快得多  │ (隨機存取) │ 命中率   │
│              │            │            │ (大陣列) │
│              │            │            │          │
│ 整體評估      │ 二分搜尋贏 │ 線性掃描贏 │ 平衡 ✓   │
│              │ (n>10000)  │            │          │
└──────────────┴────────────┴────────────┴──────────┘

結論: 即使快取效能較差，O(log n) 的時間優勢仍使
     二分搜尋在大多數實務應用中優於線性掃描。
```

## 6. 堆疊記憶體分析

### 函數簽名

```c
int searchInsert(int* nums, int numsSize, int target)
```

### 堆疊框架 (Stack Frame)

```
假設: x86-64 架構 (System V AMD64 ABI)
編譯器: GCC with -O2

┌────────────────────────────────────────┐
│          STACK MEMORY LAYOUT            │
│          (由高位址到低位址)              │
└────────────────────────────────────────┘

高位址  ┌──────────────────┐
        │   callers frame  │ (呼叫者的堆疊框架)
        ├──────────────────┤
 RSP-80 │  return address  │ (8 bytes, 64-bit)
        ├──────────────────┤
 RSP-76 │  saved RBP       │ (8 bytes, 基底指標)
        ├──────────────────┤
 RSP-72 │  local: left     │ (4 bytes, int)
        ├──────────────────┤
 RSP-68 │  local: right    │ (4 bytes, int)
        ├──────────────────┤
 RSP-64 │  local: pivot    │ (4 bytes, int)
        ├──────────────────┤
 RSP-60 │  alignment pad   │ (4 bytes, 64-bit對齊)
        ├──────────────────┤
 RSP-56 │  (alignment)     │
        ├──────────────────┤
 RSP    │ (current stack   │
        │  pointer)        │
低位址  └──────────────────┘

參數傳遞 (寄存器): [System V AMD64 ABI]
  rdi ──▶ nums      (int* 指標, 8 bytes)
  esi ──▶ numsSize  (int, 4 bytes)
  edx ──▶ target    (int, 4 bytes)
```

### 堆疊分配詳情

```c
// 函數區域變數
int left;      // 4 bytes @ RSP-72
int right;     // 4 bytes @ RSP-68
int pivot;     // 4 bytes @ RSP-64
// 對齊填充    // 4 bytes @ RSP-60
```

| 變數       | 位置      | 大小   | 描述                           |
|-----------|----------|--------|------------------------------|
| nums      | rdi      | 8 B   | 陣列基址指標 (參數,寄存器)      |
| numsSize  | esi      | 4 B   | 陣列大小 (參數,寄存器)        |
| target    | edx      | 4 B   | 搜尋目標 (參數,寄存器)        |
| left      | RSP-72   | 4 B   | 左邊界索引 (本地變數)         |
| right     | RSP-68   | 4 B   | 右邊界索引 (本地變數)         |
| pivot     | RSP-64   | 4 B   | 中間點索引 (本地變數)         |
| 對齊      | RSP-60   | 4 B   | 堆疊對齊填充                  |

### 堆疊使用統計

```
┌─────────────────────────────────────┐
│   堆疊空間分析                       │
├─────────────────────────────────────┤
│ 參數大小              │ 16 bytes    │
│ (通過寄存器)          │ (不占堆疊)   │
├─────────────────────────────────────┤
│ 本地變數              │ 12 bytes    │
│ (left, right, pivot)  │             │
├─────────────────────────────────────┤
│ 對齊填充              │  4 bytes    │
├─────────────────────────────────────┤
│ 總堆疊使用            │ 16 bytes    │
│ (不含 return addr)    │             │
├─────────────────────────────────────┤
│ 時間複雜度空間分析    │ O(1) ✓      │
│ (迭代實現，無遞迴)    │ 常數級空間  │
└─────────────────────────────────────┘
```

### 堆疊與陣列位址的關係圖

```
函數呼叫時:
┌────────────────────────────────────────┐
│ 呼叫者 (caller)                        │
│                                        │
│ int nums[] = {1, 3, 5, 6};             │
│             (堆上或全域)               │
│                                        │
│ int result = searchInsert(nums, 4, 5); │
│                 │      │   │   │       │
│                 ▼      ▼   ▼   ▼       │
├────────────────────────────────────────┤
│ searchInsert() 堆疊框架                │
│                                        │
│ rdi ───────▶ &nums[0] (0xA000)        │
│ esi ───────▶ 4 (numsSize)             │
│ edx ───────▶ 5 (target)               │
│                                        │
│ 本地變數:                              │
│ left @ RSP-72    ◀─ 初值: 0           │
│ right @ RSP-68   ◀─ 初值: 3           │
│ pivot @ RSP-64   ◀─ 計算值             │
│                                        │
│ 記憶體存取:                            │
│ nums + left ──▶ 堆上 0xA000           │
│ nums + pivot ──▶ 堆上 0xA004/0xA008/0xA00C
│                                        │
└────────────────────────────────────────┘
```

### 編譯後的彙編範例 (簡化)

```asm
searchInsert:
    push    rbp              ; 保存舊基底指標 (8 bytes)
    mov     rbp, rsp
    sub     rsp, 16          ; 堆疊空間 (left, right, pivot + 對齊)

    mov     dword [rbp-72], 0    ; left = 0
    mov     eax, esi
    sub     eax, 1
    mov     dword [rbp-68], eax  ; right = numsSize - 1

.L_loop:
    mov     eax, dword [rbp-72]  ; load left
    cmp     eax, dword [rbp-68]  ; compare left vs right
    jg      .L_return

    mov     eax, dword [rbp-72]
    add     eax, dword [rbp-68]
    shr     eax, 1               ; pivot = (left + right) / 2
    mov     dword [rbp-64], eax

    movsxd  rax, dword [rbp-64]  ; rax = pivot (sign-extend)
    mov     eax, dword [rdi + rax*4] ; eax = nums[pivot]
    cmp     eax, edx             ; compare nums[pivot] vs target

    ; ... 根據比較結果調整 left 或 right ...
    jmp     .L_loop

.L_return:
    mov     eax, dword [rbp-72]  ; eax = left (返回值)
    leave                        ; 恢復堆疊
    ret
```

## 總結

### 記憶體存取特性

| 特性                 | 詳情                                          |
|-------------------|-----------------------------------------------|
| **陣列基址**      | 由 `nums` 指標提供 (假設 0xA000)               |
| **元素位址計算**  | `address = 0xA000 + (index × 4)`             |
| **指標運算**      | `nums[i]` = `*(nums + i)` (自動乘以 sizeof)   |
| **堆疊變數**      | left, right, pivot (12 bytes + 4 bytes填充)  |
| **記憶體成本**    | O(1) 常數級，三個 int 區域變數                |
| **存取模式**      | 跳躍式非順序 (logarithmic narrowing)          |
| **快取表現**      | 小陣列: 完美 (100%)，大陣列: 一般 (30%)       |

### 為什麼使用二分搜尋?

```
輸入大小 n = 1,000,000
────────────────────────────────────────

線性搜尋:
  最差情況: 1,000,000 次比較
  平均: 500,000 次比較
  快取命中: 93%

二分搜尋:
  最差情況: log₂(1,000,000) ≈ 20 次比較 ✓✓✓
  平均: 10 次比較
  快取命中: 30% (但比較次數大幅減少)

結論: 即使快取效能較低，O(log n) vs O(n)
      的時間複雜度優勢壓倒性地勝出。
```

### 注意事項

- **假設位址:** 本文件中的十六進制位址 (0xA000, 0xA004 等) 為假設值，實際位址由作業系統動態分配。
- **編譯器最佳化:** 實際堆疊佈局與編譯器旗標 (-O0, -O2, -O3) 可能有所不同。
- **平台差異:** x86-64 架構的堆疊佈局可能與 ARM、MIPS 等不同。
- **執行階段:** 二分搜尋的每次迭代執行時間基本相同（無分支預測差異）。
