## 3. Kadane 演算法過程追蹤

### 演算法虛擬碼

```c
int maxSubArray(int* nums, int numsSize) {
    int max_sum = nums[0];      // 0x7FFE0024
    int current_sum = nums[0];  // 0x7FFE0020

    for (int i = 1; i < numsSize; i++) {  // i: 0x7FFE001C
        // 存取記憶體: nums + i*4
        current_sum = (nums[i] > current_sum + nums[i]) ?
                      nums[i] : current_sum + nums[i];
        max_sum = (current_sum > max_sum) ? current_sum : max_sum;
    }
    return max_sum;
}
```

### 範例 1: nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4]

```

  current_sum = nums[0] = -2    (0x7FFE0020 ← -2)

  ┌─────────────────────────────────────────────┐
  │ 陣列: -2   1  -3   4  -1   2   1  -5   4    │
  │位址: 0xA000 ▲                                 │
  │訪問: [i=0] ──────────────────────────────    │
  └─────────────────────────────────────────────┘

步驟 1: i=1, 訪問 0xA004
  ┌─────────────────────────────────────────────┐

步驟 6: i=6, 訪問 0xA018

迴圈結束 (i=9, 條件 i < numsSize 不成立)
```

## 5. 記憶體存取模式分析

### Kadane 演算法的快取友好性

#### 記憶體存取模式

```
時間軸:
  t=0  訪問 0xA000 (nums[0])
  t=1  訪問 0xA004 (nums[1])  ◄── 連續記憶體，相隔 4 bytes
  t=2  訪問 0xA008 (nums[2])  ◄── 連續記憶體，相隔 4 bytes
  t=3  訪問 0xA00C (nums[3])  ◄── 連續記憶體，相隔 4 bytes
  ...
  t=8  訪問 0xA020 (nums[8])

存取模式: 0xA000 ───▶ 0xA004 ───▶ 0xA008 ───▶ ... ───▶ 0xA020
           單一方向線性掃描 (cache-friendly!)
```

#### 快取行為分析

```
64位系統，典型快取行大小: 64 bytes

快取行 0: [0xA000 - 0xA03F]
  包含: nums[0], nums[1], ..., nums[15]

Kadane 演算法記憶體訪問:
  ✓ 空間局部性 (Spatial Locality)
    - 連續訪問 nums[i], nums[i+1], nums[i+2], ...
    - 只要 numsSize 合理，整個陣列可能在 1-2 個快取行內
    - 快取命中率極高 (~99%)

  ✓ 時間局部性 (Temporal Locality)
    - 每個陣列元素只存取一次
    - 但 max_sum 和 current_sum 在每次迴圈中重複訪問
    - 這兩個變數保持在 CPU 暫存器中 (レジスタ)
    - 不需重複載入快取
```

#### 臨時變數局部性

```
Kadane 演算法的臨時變數:
  - max_sum (4 bytes) ──┐
  - current_sum (4 bytes)──┤ 都在棧頂，相鄰記憶體位置
  - i (4 bytes) ────────┘

實際 CPU 行為:
  ┌────────────────────┐
  │ L1 快取 (32KB)     │
  │ ├─ max_sum: 寄存器 │  ◄── 立即存取
  │ ├─ current_sum: 寄存器 │  ◄── 立即存取
  │ └─ i: 寄存器       │  ◄── 立即存取
  └────────────────────┘
      每個操作 1 個 CPU 週期
```

### 與分治法 (Divide-and-Conquer) 的對比

#### 分治法記憶體存取

```
分治法虛擬碼:
int maxSubArray_DC(int* nums, int left, int right) {
    if (left == right) return nums[left];

    int mid = (left + right) / 2;
    int left_sum = maxSubArray_DC(nums, left, mid);      // 遞歸呼叫 1
    int right_sum = maxSubArray_DC(nums, mid+1, right);  // 遞歸呼叫 2
    int cross_sum = maxCrossingSubArray(nums, ...);      // 跨越計算
    ...
}

訪問模式 (nums = [-2, 1, -3, 4, -1, 2, 1, -5, 4], n=9):

第 1 層: maxSubArray_DC(nums, 0, 8)
    ├─ 呼叫 maxSubArray_DC(nums, 0, 4)  [訪問 0xA000-0xA010]
    │  ├─ 呼叫 maxSubArray_DC(nums, 0, 2)
    │  │  ├─ 呼叫 maxSubArray_DC(nums, 0, 1)
    │  │  ├─ 呼叫 maxSubArray_DC(nums, 1, 1)
    │  │  └─ maxCrossingSubArray()
    │  ├─ 呼叫 maxSubArray_DC(nums, 3, 4)
    │  └─ maxCrossingSubArray()
    └─ 呼叫 maxSubArray_DC(nums, 5, 8)  [訪問 0xA014-0xA020]
       ├─ 呼叫 maxSubArray_DC(nums, 5, 6)
       ├─ 呼叫 maxSubArray_DC(nums, 7, 8)
       └─ maxCrossingSubArray()

訪問順序不規則:
  0xA000 ──→ 0xA004 ──→ 0xA008 ───┐
                                   ├─→ 跳躍回 0xA00C
                                   │
  0xA010 ──→ 0xA014 ───┐
                       └─→ 跳躍到 0xA018

這導致:
  ✗ 空間局部性差
  ✗ 快取命中率低 (~40-60%)
  ✗ 記憶體匯流排競爭
  ✗ 指令快取失敗
```

### 分支預測分析

#### Kadane 演算法

```c
// 最常見的分支:
if (nums[i] > current_sum + nums[i])
    current_sum = nums[i];
else
    current_sum = current_sum + nums[i];
```

```
現代 CPU 分支預測器行為:

案例 1: 遞增子陣列 [5, 4, -1, 7, 8]

  i=1: 是

  i=3: 是
  i=4: 否
  i=5: 否
  i=6: 否
  i=7: 否
  i=8: 否

  模式複雜度: 中等
  預測成功率: 85-90%
  管線氣泡: 輕微

總結: Kadane 演算法分支預測性能良好 (>85%)
```

## 6. 堆疊記憶體對比

### 迭代版 Kadane 演算法 (棧空間: O(1))

```
函數呼叫時棧狀態:

呼叫前:
  堆疊頂端 ──→ [呼叫者的返回地址]

呼叫後:
  ┌────────────────────┐
  │  返回地址 (8B)     │  0x7FFE0000
  ├────────────────────┤
  │  前一個 ebp (8B)   │  0x7FFE0008
  ├────────────────────┤
  │  nums ptr (8B)     │  0x7FFE0010  ◄── 參數
  ├────────────────────┤
  │  numsSize (4B)     │  0x7FFE0018  ◄── 參數
  ├────────────────────┤
  │  i (4B)            │  0x7FFE001C  ◄── 本地變數
  ├────────────────────┤
  │  current_sum (4B)  │  0x7FFE0020  ◄── 本地變數
  ├────────────────────┤
  │  max_sum (4B)      │  0x7FFE0024  ◄── 本地變數
  └────────────────────┘
  堆疊使用: 40 bytes (含對齐) ◄── 恆定!

迭代:
  第 1 次迴圈: 使用相同棧空間，無新分配
  第 2 次迴圈: 使用相同棧空間，無新分配
  第 n 次迴圈: 使用相同棧空間，無新分配

返回:
  堆疊立即釋放 ──┐
                 ▼
  堆疊頂端回到原位置
```

#### 複雜度

```
時間複雜度: O(n)
空間複雜度: O(1) ◄── 只有固定本地變數
```


#### 棧空間分析

```
每個幀大小:
  ├─ nums (指標): 8 bytes
  ├─ left (int): 4 bytes
  ├─ right (int): 4 bytes
  ├─ mid (int): 4 bytes
  ├─ left_max (int): 4 bytes
  ├─ right_max (int): 4 bytes

  ├─ 返回地址: 8 bytes
  └─ 對齐: 8 bytes
  ─────────────────────────
  每幀總計: 56 bytes

最大棧深: O(log n)
最大棧使用: O(log n) × 56 bytes

對於 n=10,000:
  棧深: ⌈log₂(10000)⌉ = 14
  棧使用: 14 × 56 = 784 bytes

對於 n=1,000,000:
  棧深: ⌈log₂(1000000)⌉ = 20
  棧使用: 20 × 56 = 1,120 bytes

對於 n=1,000,000,000:
  棧深: ⌈log₂(1000000000)⌉ = 30
  棧使用: 30 × 56 = 1,680 bytes
```

#### 遞歸版時間複雜度

```
T(n) = 2·T(n/2) + O(n)  // 每層合併需 O(n)

主定理:
  a=2, b=2, f(n)=O(n)
  log_b(a) = log_2(2) = 1
  f(n) = O(n^1)

  ⇒ T(n) = O(n·log n)  ◄── 比 Kadane 慢!
```

## 總結

### Kadane 演算法的記憶體優勢

1. **線性記憶體存取**: 單一方向掃描陣列，快取命中率接近 100%
2. **恆定空間**: 只需 O(1) 額外空間，無遞歸呼叫開銷
3. **最優化分支**: 簡單的分支模式，分支預測器易於學習
4. **硬體友好**: 充分利用現代 CPU 特性 (快取行、指令管線、分支預測)
5. **可擴展性**: 對任意大小的 n 都能高效執行

### 記憶體位址視角的核心差異

```

分治:

          └── ┌── ┘── ┌── (回溯)
  快取行效率: ██████░░░░░░░░░░ 45%
```
