# LeetCode 238: Product of Array Except Self — 記憶體分析

## 1. 陣列記憶體佈局

### 整數陣列的記憶體結構
假定系統為 64-bit，`int` 大小為 4 bytes。

#### 輸入陣列 nums 的記憶體配置（例 1：[1,2,3,4]）

```
記憶體位址           16進制地址        陣列元素        二進制表示
┌──────────────┬──────────────┬──────────────┬──────────────┐
│   nums[0]    │   nums[1]    │   nums[2]    │   nums[3]    │
│      1       │      2       │      3       │      4       │
├──────────────┼──────────────┼──────────────┼──────────────┤
│   0xA000     │   0xA004     │   0xA008     │   0xA00C     │
└──────────────┴──────────────┴──────────────┴──────────────┘
  00000001      00000010      00000011      00000100
```

#### 輸出陣列 answer 的記憶體配置（目標：[24,12,8,6]）

```
記憶體位址           16進制地址        陣列元素        說明
┌──────────────┬──────────────┬──────────────┬──────────────┐
│  answer[0]   │  answer[1]   │  answer[2]   │  answer[3]   │
│      24      │      12      │      8       │      6       │
├──────────────┼──────────────┼──────────────┼──────────────┤
│   0xB000     │   0xB004     │   0xB008     │   0xB00C     │
└──────────────┴──────────────┴──────────────┴──────────────┘
  00011000      00001100      00001000      00000110
  (左乘積×右)   (左乘積×右)   (左乘積×右)   (左乘積×右)
```

## 2. 完整資料結構記憶體配置

### 陣列映射表

| 陣列名稱 | 基地址 | 大小 | 元素個數 | 範圍 |
|---------|--------|------|---------|------|
| `nums` | `0xA000` | 16 bytes | 4 | `0xA000` ~ `0xA00F` |
| `answer` | `0xB000` | 16 bytes | 4 | `0xB000` ~ `0xB00F` |

### ASCII 藝術記憶體圖

```
棧 (Stack)
┌─────────────────────────────────────┐
│  left_product (int)     = 1          │ esp + 0x10
│─────────────────────────────────────│
│  right_product (int)    = 1          │ esp + 0x0C
│─────────────────────────────────────│
│  i (int)                = 0          │ esp + 0x08
│─────────────────────────────────────│
│  answer (int*)  ─────────┐           │ esp + 0x04
│─────────────────────────│───────────│
│  return_size (int*)  ───┼─────┐     │ esp + 0x00
└─────────────────────────┼─────┼─────┘
                          │     │
堆 (Heap)                 │     │
                          │     └──→ [4]  (nums_size)
                          │
                          └──→ ┌─────────────────────────────┐
                               │  answer[0] │ answer[1] ...  │
                               │  0xB000    │ 0xB004  ...    │
                               └─────────────────────────────┘

全域陣列 (Global)
┌─────────────────────────────────────┐
│  nums: [1, 2, 3, 4]                 │
│  0xA000 0xA004 0xA008 0xA00C        │
└─────────────────────────────────────┘
```

### 邏輯關係圖

```
第一遍 (Pass 1): 從左到右，累計左側乘積
nums[]:     [1]  [2]  [3]  [4]
                  ↓    ↓    ↓
left_prod:   1   1    2    6   (每次乘以當前元素後儲存)
                  │    │    │
answer[]:   [1] [1]  [2]  [6]  (第一遍的結果)
                 └─────┘

第二遍 (Pass 2): 從右到左，累計右側乘積並乘以第一遍結果
right_prod:  1    4   12   24  (由右至左累計)
              ↑    ↑    ↑    ↑
answer[]:  [24] [12] [8]  [6]  (最終結果)
             ↑    ↑   ↑    ↑
         6×4=24 6×2=12 6×1=6 6×1=6
```

## 3. 陣列索引版對應

### 左→右遍歷公式（Pass 1）

```
迴圈條件: i = 0, 1, 2, ..., nums_size - 1

索引變化:
┌─────────┬───────────┬──────────────────────┐
│  步驟   │  指標位置 │  記憶體位址          │
├─────────┼───────────┼──────────────────────┤
│ i = 0   │ nums[0]   │ 0xA000, answer[0]=0xB000 │
│ i = 1   │ nums[1]   │ 0xA004, answer[1]=0xB004 │
│ i = 2   │ nums[2]   │ 0xA008, answer[2]=0xB008 │
│ i = 3   │ nums[3]   │ 0xA00C, answer[3]=0xB00C │
└─────────┴───────────┴──────────────────────┘

指標算術:
address(nums[i])    = 0xA000 + 4 * i
address(answer[i])  = 0xB000 + 4 * i
```

### 右→左遍歷公式（Pass 2）

```
迴圈條件: i = nums_size - 1, ..., 1, 0  (遞減)

索引變化:
┌─────────┬───────────┬──────────────────────┐
│  步驟   │  指標位置 │  記憶體位址          │
├─────────┼───────────┼──────────────────────┤
│ i = 3   │ nums[3]   │ 0xA00C, answer[3]=0xB00C │
│ i = 2   │ nums[2]   │ 0xA008, answer[2]=0xB008 │
│ i = 1   │ nums[1]   │ 0xA004, answer[1]=0xB004 │
│ i = 0   │ nums[0]   │ 0xA000, answer[0]=0xB000 │
└─────────┴───────────┴──────────────────────┘

指標距離:
distance = (nums_size - 1 - i) * 4 bytes
```

## 5. 指標變數追蹤總表

### 範例 1：nums = [1,2,3,4]

#### Pass 1 追蹤表

| 步驟 | 索引 i | 位址 (answer) | left_product 前 | left_product 後 | answer[i] 前 | answer[i] 後 | 操作 |
|-----|--------|--------------|-----------------|-----------------|-------------|-------------|------|
| 1   | 0      | 0xB000       | 1               | 1               | XXXX        | 1           | answer[0] = 1; left_product = 1×1 |
| 2   | 1      | 0xB004       | 1               | 2               | XXXX        | 1           | answer[1] = 1; left_product = 1×2 |
| 3   | 2      | 0xB008       | 2               | 6               | XXXX        | 2           | answer[2] = 2; left_product = 2×3 |
| 4   | 3      | 0xB00C       | 6               | 24              | XXXX        | 6           | answer[3] = 6; left_product = 6×4 |

#### Pass 2 追蹤表

| 步驟 | 索引 i | 位址 (answer) | right_product 前 | right_product 後 | answer[i] 前 | answer[i] 後 | 操作 |
|-----|--------|--------------|------------------|------------------|-------------|-------------|------|
| 1   | 3      | 0xB00C       | 1                | 4                | 6           | 6           | answer[3] = 6×1 = 6; right_product = 1×4 |
| 2   | 2      | 0xB008       | 4                | 12               | 2           | 8           | answer[2] = 2×4 = 8; right_product = 4×3 |

#### 統計摘要

```
Pass 1:
  總迭代次數: 4
  記憶體寫入: 4 次（每個 answer[i] 一次）
  陣列存取: nums 4 次讀取

Pass 2:
  總迭代次數: 4
  記憶體寫入: 4 次（每個 answer[i] 一次）
  陣列存取: nums 4 次讀取

總計:
  記憶體寫入: 8 次
  記憶體讀取: 8 次 (nums) + 4 次 (answer RMW) = 12 次
  時間複雜度: O(n) = O(4)
  空間複雜度: O(1) 額外空間（不計輸出陣列）
```

```
  記憶體寫入: 5 次（每個 answer[i] 一次）
  陣列存取: nums 5 次讀取
  零檢測: 遇到 nums[2] = 0，left_product 變為 0

Pass 2:
  總迭代次數: 5
  記憶體寫入: 5 次（每個 answer[i] 一次）
  陣列存取: nums 5 次讀取
  零檢測: 在 right_product 乘以 nums[2] = 0 時變為 0

重要觀察:
  由於陣列包含 0，除了對應位置外，其他位置的輸出都是 0
  answer[2] = 9 是唯一非零值，因為：
    answer[2] = left_product(0) × right_product(2)
              = (-1) × (-9) = 9

總計:
  記憶體寫入: 10 次
  記憶體讀取: 10 次 (nums) + 5 次 (answer RMW) = 15 次
  時間複雜度: O(n) = O(5)
  空間複雜度: O(1) 額外空間（不計輸出陣列）
```

## 6. 記憶體存取模式比較

### 方案 A：原地修改 (In-Place) 風格

```c
// 假設該方案直接修改 nums 陣列而非使用 answer
void productExceptSelfInPlace(int* nums, int nums_size) {
    int left_product = 1;
    for (int i = 0; i < nums_size; i++) {
        int temp = nums[i];
        nums[i] = left_product;           // 覆寫 nums[i]
        left_product *= temp;
    }
    int right_product = 1;
    for (int i = nums_size - 1; i >= 0; i--) {
        int temp = nums[i];               // 讀取已修改的值
        nums[i] *= right_product;
        right_product *= nums[i + 1];     // 引用原始值（須額外暫存）
    }
}
```

### 快取行為比較

| 項目 | 原地修改 (in-place) | 獨立分配 (malloc answer) |
|-----|-------------------|----------------------|
| 記憶體配置 | 修改輸入陣列 0xA000 | 新分配 answer 0xB000 |
| 快取行為 | L1/L2 快取命中率高（重複存取同區域） | 較低（兩個陣列分離） |
| 時間效率 | 更快 (快取局部性) | 稍慢 (快取未命中) |
| 指令數 | 更少（無須 malloc）| 多 (malloc + free) |
| 記憶體帶寬 | 單一陣列頻寬利用 | 雙陣列並行讀寫 |

### 空間開銷比較

```
方案 A (in-place):
┌──────────────────────────────────┐
│ nums[]: 原地修改                 │
│ 空間: 16 bytes                   │
│ 額外堆記憶體: 0 bytes            │
└──────────────────────────────────┘
  棧: i, left_product, right_product = ~12 bytes


方案 B (獨立分配):
┌──────────────────────────────────┐
│ nums[]: 讀取用                   │
│ 空間: 16 bytes                   │
│ answer[]: 儲存結果                │
│ 空間: 16 bytes                   │
│ 額外堆記憶體: 16 bytes           │
└──────────────────────────────────┘
  棧: i, left_product, right_product, answer ptr = ~20 bytes
  堆: answer 陣列 16 bytes
  malloc/free 開銷: ~64 bytes (allocator metadata)
```

### 安全性與可靠性

| 項目 | 原地修改 | 獨立分配 |
|-----|---------|--------|
| 輸入保留 | 破壞原始 nums | 保留原始 nums |
| API 清晰度 | 不明確（修改輸入） | 明確（返回新陣列） |
| 錯誤恢復 | 無法撤銷原始數據 | 可重試（nums 完好） |
| 多執行緒安全性 | 不安全（修改共享輸入） | 相對安全（新分配） |
| 記憶體洩漏風險 | 否 | 是（須 free answer） |

### 快取一致性分析

#### Pass 1 記憶體存取模式

```
方案 A (in-place):
迴圈 i=0→3:
  讀: nums[i]      @ 0xA000 + 4i
  寫: nums[i]      @ 0xA000 + 4i  (同地址，稱為 Read-Modify-Write)
  快取行 X0: [nums[0-3]] ～ 64 bytes (一個快取行)

方案 B (獨立分配):
迴圈 i=0→3:
  讀: nums[i]      @ 0xA000 + 4i
  寫: answer[i]    @ 0xB000 + 4i  (不同地址)
  快取行 X0: [nums[0-3]]    ～ 64 bytes
  快取行 Y0: [answer[0-3]]  ～ 64 bytes (需額外快取行)

方案 A 優點: 單一快取行，局部性最佳
方案 B 劣勢: 需維護兩個快取行，競爭快取資源
```

#### Pass 2 記憶體存取模式

```
方案 A (in-place):
迴圈 i=3→0:
  讀: nums[i+1]     @ 0xA004 + 4i  (記憶體讀取已修改值)
  讀-寫: nums[i]    @ 0xA000 + 4i  (RMW 快取線)
  快取行衝突: nums[0-3] (逆向存取，快取污染)

方案 B (獨立分配):
迴圈 i=3→0:
  讀: nums[i]       @ 0xA000 + 4i
  讀-寫: answer[i]  @ 0xB000 + 4i  (逆向訪問，快取污染)
  較低的快取命中率 (相同污染，但兩個陣列分離)
```

### 實務建議

```
選用 in-place (方案 A) 當:
  ✓ 輸入陣列可安全破壞
  ✓ 記憶體受限 (嵌入式系統)
  ✓ 單執行緒環境
  ✓ 快取效能至關重要

選用獨立分配 (方案 B) 當:
  ✓ 須保留原始輸入
  ✓ 多執行緒共享陣列
  ✓ API 明確性重要
  ✓ 記憶體充足，效能容許
  ✓ LeetCode 標準模式 (函數返回新陣列)
```

## 總結

### 演算法核心特性

```
LeetCode 238: Product of Array Except Self

時間複雜度: O(n)
  - Pass 1: O(n)
  - Pass 2: O(n)
  - 總計: 2n = O(n)

空間複雜度: O(1)
  - 不計輸出陣列
  - 僅額外使用常數個變數 (left_product, right_product, i)

記憶體模式: 線性存取
  - Pass 1: 順序存取 nums[0→n-1], answer[0→n-1]
  - Pass 2: 逆序存取 nums[n-1→0], answer[n-1→0]
  - 快取友善: 兩次完整陣列遍歷，最小化快取未命中

關鍵觀察:
  答案 answer[i] = (nums[0..i-1] 的乘積) × (nums[i+1..n-1] 的乘積)
                = left_product[i-1] × right_product[i+1]
```

### 記憶體佈局重點

```
輸入: nums @ 0xA000 (4 × sizeof(int) = 16 bytes)
輸出: answer @ 0xB000 (malloc'd, 16 bytes)
棧幀: 64 bytes
  · 區域變數: left_product, right_product, i
  · 指標: answer, nums (參數)

存取模式:
  Pass 1: nums 讀順序, answer 寫順序
  Pass 2: nums 讀逆序, answer 讀寫逆序

快取考量:
  單個 64-byte 快取行可容納整個 4-元素陣列
  兩趟遍歷分別汙染同一快取行 (逆向損失)
```

### 範例驗證

#### 範例 1: [1,2,3,4] → [24,12,8,6]
- left_product 軌跡: 1 → 1 → 2 → 6 → 24
- right_product 軌跡: 1 → 4 → 12 → 24 → 24
- 最終乘積驗證: answer[0] = 1 × 24 = 24 ✓

#### 範例 2: [-1,1,0,-3,3] → [0,0,9,0,0]
- 零值傳播: left_product 在 Pass 1 變 0，影響後續位置
- 獨特結果: answer[2] = (-1) × (-9) = 9 (唯一非零)
- 邊界完整性: right_product 也於零處歸零 ✓
