# 283. Move Zeroes — 記憶體位址 / 陣列索引圖

> 以下所有記憶體位址皆為假設值，僅供示意。假設 `int = 4 bytes`。

## Struct 記憶體佈局

本題使用原始 `int` 陣列，無自訂 struct。每個元素佔 4 bytes：

```
┌─────────────┐
│   int (4B)   │
│  offset: 0   │
└─────────────┘
```

陣列 `nums` 中每個元素的位址計算公式：
```
&nums[i] = base_address + i * sizeof(int)
         = base_address + i * 4
```

## 搜尋／操作過程追蹤

### 範例 1: `nums = [0, 1, 0, 3, 12]`

#### 指標變數初始狀態

```
int* nums       = 0xA000  （陣列首位址）
int  lastNonZero = 0       （慢指標，對應位址 0xA000）
int  cur         = 0       （快指標，對應位址 0xA000）
```

---

#### Step 0: cur=0, 位址 0xA000, 值=0

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  0 │  │  1 │  │  0 │  │  3 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
    ▲
    cur=0
    ▲
  lastNonZero=0
```

- 解參考 (dereference): `*(0xA000 + 0*4)` = `*(0xA000)` = **0**
- 比較: `0 != 0` → **false**
- 動作: 跳過，`cur++`

---

#### Step 1: cur=1, 位址 0xA004, 值=1

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  0 │  │  1 │  │  0 │  │  3 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
    ▲        ▲
  lastNZ=0  cur=1
```

- 解參考: `*(0xA000 + 1*4)` = `*(0xA004)` = **1**
- 比較: `1 != 0` → **true**
- 動作: swap `nums[0]` ↔ `nums[1]`

```
  交換過程：
  tmp = *(0xA000) = 0          寫入 0xA000 ← 1
  *(0xA000) = *(0xA004) = 1    寫入 0xA004 ← 0
  *(0xA004) = tmp = 0

  ┌──────────────────────┐
  │ 0xA000 ◄──── 0xA004  │   值: 0 ←→ 1
  └──────────────────────┘
```

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  1 │  │  0 │  │  0 │  │  3 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
           ▲
         lastNZ=1
```

> 指標移動: lastNonZero `0 → 1` (0xA000 → 0xA004)

---

#### Step 2: cur=2, 位址 0xA008, 值=0

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  1 │  │  0 │  │  0 │  │  3 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
           ▲        ▲
         lastNZ=1  cur=2
```

- 解參考: `*(0xA008)` = **0**
- 比較: `0 != 0` → **false**
- 動作: 跳過，`cur++`

---

#### Step 3: cur=3, 位址 0xA00C, 值=3

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  1 │  │  0 │  │  0 │  │  3 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
           ▲                 ▲
         lastNZ=1          cur=3
```

- 解參考: `*(0xA00C)` = **3**
- 比較: `3 != 0` → **true**
- 動作: swap `nums[1]` ↔ `nums[3]`

```
  交換過程：
  tmp = *(0xA004) = 0          寫入 0xA004 ← 3
  *(0xA004) = *(0xA00C) = 3    寫入 0xA00C ← 0
  *(0xA00C) = tmp = 0

  ┌──────────────────────────────┐
  │ 0xA004 ◄────────── 0xA00C   │   值: 0 ←→ 3
  └──────────────────────────────┘
```

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  1 │  │  3 │  │  0 │  │  0 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
                    ▲
                  lastNZ=2
```

> 指標移動: lastNonZero `1 → 2` (0xA004 → 0xA008)

---

#### Step 4: cur=4, 位址 0xA010, 值=12

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  1 │  │  3 │  │  0 │  │  0 │  │ 12 │
  └────┘  └────┘  └────┘  └────┘  └────┘
                    ▲                 ▲
                  lastNZ=2          cur=4
```

- 解參考: `*(0xA010)` = **12**
- 比較: `12 != 0` → **true**
- 動作: swap `nums[2]` ↔ `nums[4]`

```
  交換過程：
  tmp = *(0xA008) = 0          寫入 0xA008 ← 12
  *(0xA008) = *(0xA010) = 12   寫入 0xA010 ← 0
  *(0xA010) = tmp = 0

  ┌──────────────────────────────────────┐
  │ 0xA008 ◄──────────────── 0xA010     │   值: 0 ←→ 12
  └──────────────────────────────────────┘
```

```
  0xA000   0xA004   0xA008   0xA00C   0xA010
  ┌────┐  ┌────┐  ┌────┐  ┌────┐  ┌────┐
  │  1 │  │  3 │  │ 12 │  │  0 │  │  0 │  ← 最終結果
  └────┘  └────┘  └────┘  └────┘  └────┘
                             ▲
                           lastNZ=3
```

> 指標移動: lastNonZero `2 → 3` (0xA008 → 0xA00C)

## 指標變數追蹤總表

### 範例 1: `nums = [0, 1, 0, 3, 12]`

| Step | cur | cur 位址 | *cur 值 | lastNZ | lastNZ 位址 | 比較結果 | 動作 |
|---|---|---|---|---|---|---|---|
| 0 | 0 | 0xA000 | 0 | 0 | 0xA000 | 0==0 | 跳過 |
| 1 | 1 | 0xA004 | 1 | 0 | 0xA000 | 1!=0 | swap, lastNZ++ |
| 2 | 2 | 0xA008 | 0 | 1 | 0xA004 | 0==0 | 跳過 |
| 3 | 3 | 0xA00C | 3 | 1 | 0xA004 | 3!=0 | swap, lastNZ++ |
| 4 | 4 | 0xA010 | 12 | 2 | 0xA008 | 12!=0 | swap, lastNZ++ |

**統計**：
- 掃描步數: 5
- 交換次數: 3（等於非零元素數量）
- lastNonZero 移動次數: 3
- 陣列寫入次數: 6（每次 swap 寫 2 次）

### 範例 2: `nums = [0]`

| Step | cur | cur 位址 | *cur 值 | lastNZ | lastNZ 位址 | 比較結果 | 動作 |
|---|---|---|---|---|---|---|---|
| 0 | 0 | 0xB000 | 0 | 0 | 0xB000 | 0==0 | 跳過 |

**統計**：掃描步數: 1，交換次數: 0，寫入次數: 0

## 記憶體存取模式比較

### 陣列（連續記憶體）存取

本題直接使用 `int[]` 陣列，所有元素在連續記憶體中：

```
nums: [  0  ][  1  ][  0  ][  3  ][ 12  ]
       ^^^^   ^^^^   ^^^^   ^^^^   ^^^^
      4 bytes each, 連續排列
```

| 面向 | 分析 |
|---|---|
| Cache 行為 | 極佳。Sequential access，CPU prefetcher 可預測下一次存取位置。典型 L1 cache line = 64 bytes 可容納 16 個 int，本題陣列（5 int = 20 bytes）完全在一條 cache line 內 |
| 空間開銷 | 零額外空間。只用兩個 `int` 變數（`cur`、`lastNonZero`） |
| 存取模式 | `cur` 嚴格遞增（sequential read），`lastNonZero` 也嚴格遞增（sequential write），兩者距離單調不減 |

### 如果改用鏈結串列

假設用 `struct Node { int val; struct Node* next; }` 實作：

| 面向 | 分析 |
|---|---|
| Cache 行為 | 差。每個 node 可能在不同 cache line，pointer chasing 導致 cache miss |
| 空間開銷 | 每個元素多 8 bytes（64-bit pointer），overhead 200%（對 4-byte int） |
| 存取模式 | 無法預測，prefetcher 無效 |
| 優勢 | 「搬移」只需修改指標，不需實際複製資料；但本題 `int` 太小，搬移成本可忽略 |

### 結論

對本題而言，陣列是最佳選擇。元素小（4 bytes）、連續存取、swap 成本低，完全發揮 cache locality 優勢。

## 堆疊記憶體對比

### 迭代版（本題實作）

```
Stack Frame: moveZeroes()
┌─────────────────────────┐
│ return address    (8B)  │
├─────────────────────────┤
│ int* nums         (8B)  │  ← 參數（指向堆上陣列）
├─────────────────────────┤
│ int numsSize      (4B)  │  ← 參數
├─────────────────────────┤
│ int lastNonZero   (4B)  │  ← 區域變數
├─────────────────────────┤
│ int cur           (4B)  │  ← 迴圈變數
├─────────────────────────┤
│ int tmp           (4B)  │  ← swap 暫存
└─────────────────────────┘
總計: ~32 bytes（固定，與 n 無關）
```

### 假想遞迴版

```
moveZeroes_recursive(nums, size, cur, lastNZ):
  if cur == size: return
  if nums[cur] != 0: swap, lastNZ++
  moveZeroes_recursive(nums, size, cur+1, lastNZ)
```

每一層遞迴的 stack frame:
```
┌─────────────────────────┐
│ return address    (8B)  │
│ int* nums         (8B)  │
│ int size          (4B)  │
│ int cur           (4B)  │
│ int lastNZ        (4B)  │
└─────────────────────────┘
≈ 28 bytes per frame
```

- 遞迴深度 = n（每個元素一層）
- 最差堆疊使用: 28 × n bytes
- n = 10⁴ → 約 280 KB（可能超過預設 stack size 在嵌入式環境）

### 結論

迭代版堆疊使用 O(1)，遞迴版 O(n)。本題沒有理由使用遞迴。

## 退化情況分析

### Case 1: 全部為零 `[0, 0, 0, ..., 0]`

```
  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │ 0 │ │ 0 │ │ 0 │ │ 0 │ │ 0 │
  └───┘ └───┘ └───┘ └───┘ └───┘
    ▲                             ▲
  lastNZ=0                      cur掃到底
```

- `cur` 掃完整個陣列，`lastNonZero` 始終 = 0
- 交換次數: **0**
- 這是 Approach #3 的最佳情況（零寫入）
- Approach #2 的最差情況（需要 n 次寫零）

### Case 2: 全部非零 `[1, 2, 3, ..., n]`

```
  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │ 1 │ │ 2 │ │ 3 │ │ 4 │ │ 5 │
  └───┘ └───┘ └───┘ └───┘ └───┘
    ▲▲
  lastNZ 緊跟 cur
```

- `cur` 和 `lastNonZero` 同步前進
- 每步都做 swap（自己跟自己交換）
- 交換次數: **n**（但都是 no-op swap）
- 可以加 `if (cur != lastNonZero)` 判斷來跳過，降為 0 次實際交換

### Case 3: 只有最後一個非零 `[0, 0, ..., 0, 1]`

```
  ┌───┐ ┌───┐ ┌───┐ ┌───┐ ┌───┐
  │ 0 │ │ 0 │ │ 0 │ │ 0 │ │ 1 │
  └───┘ └───┘ └───┘ └───┘ └───┘
    ▲                       ▲
  lastNZ=0               cur=n-1
```

- 整個掃描過程只在最後一步做 1 次 swap
- 結果: `[1, 0, 0, 0, 0]`
- Approach #3 寫入次數: **2**（一次 swap = 2 writes）
- Approach #2 寫入次數: **n**（1 次寫非零 + n-1 次寫零）
- 這正是 Approach #3 顯著優於 #2 的 case
