# 162. Find Peak Element — 記憶體與陣列索引圖解

> 所有記憶體位址均為假設值，僅供說明用途。

---

## 1. 陣列記憶體佈局

`int *nums` 是一個連續的 `int` 陣列，每個元素佔 4 bytes。

### 範例 1：`nums = [1, 2, 3, 1]`（假設基底位址 0xA000）

```
位址        偏移    索引    值
┌─────────┬──────┬──────┬──────┐
│ 0xA000  │ +0   │ [0]  │   1  │
├─────────┼──────┼──────┼──────┤
│ 0xA004  │ +4   │ [1]  │   2  │
├─────────┼──────┼──────┼──────┤
│ 0xA008  │ +8   │ [2]  │   3  │  ◄── 峰值
├─────────┼──────┼──────┼──────┤
│ 0xA00C  │ +12  │ [3]  │   1  │
└─────────┴──────┴──────┴──────┘
```

位址計算公式：`&nums[i] = base + i * sizeof(int) = 0xA000 + i * 4`

### 範例 2：`nums = [1, 2, 1, 3, 5, 6, 4]`（假設基底位址 0xB000）

```
位址        偏移    索引    值
┌─────────┬──────┬──────┬──────┐
│ 0xB000  │ +0   │ [0]  │   1  │
├─────────┼──────┼──────┼──────┤
│ 0xB004  │ +4   │ [1]  │   2  │  ◄── 峰值之一
├─────────┼──────┼──────┼──────┤
│ 0xB008  │ +8   │ [2]  │   1  │
├─────────┼──────┼──────┼──────┤
│ 0xB00C  │ +12  │ [3]  │   3  │
├─────────┼──────┼──────┼──────┤
│ 0xB010  │ +16  │ [4]  │   5  │
├─────────┼──────┼──────┼──────┤
│ 0xB014  │ +20  │ [5]  │   6  │  ◄── 峰值之一
├─────────┼──────┼──────┼──────┤
│ 0xB018  │ +24  │ [6]  │   4  │
└─────────┴──────┴──────┴──────┘
```

## 2. 變數記憶體佈局

函式 `findPeakElement` 的區域變數位於堆疊上（假設堆疊起始 0xF000）：

```
堆疊位址      變數名     大小     說明
┌───────────┬─────────┬────────┬──────────────────────┐
│ 0xF000    │ l       │ 4 bytes│ 搜尋區間左端點索引     │
├───────────┼─────────┼────────┼──────────────────────┤
│ 0xF004    │ r       │ 4 bytes│ 搜尋區間右端點索引     │
├───────────┼─────────┼────────┼──────────────────────┤
│ 0xF008    │ mid     │ 4 bytes│ 中間索引（迴圈內更新） │
├───────────┼─────────┼────────┼──────────────────────┤
│ 0xF00C    │ nums    │ 8 bytes│ 指向陣列的指標         │
│ 0xF010    │         │ (ptr)  │                       │
├───────────┼─────────┼────────┼──────────────────────┤
│ 0xF014    │ numsSize│ 4 bytes│ 陣列長度               │
└───────────┴─────────┴────────┴──────────────────────┘
```

## 3. 搜尋過程追蹤

### 範例 1：`nums = [1, 2, 3, 1]`，基底位址 0xA000

**初始狀態：** `l = 0`, `r = 3`

```
搜尋空間：
  -∞   [1]  [2]  [3]  [1]   -∞
        0    1    2    3
        l              r
```

#### 第 1 步：`mid = 0 + (3 - 0) / 2 = 1`

```
比較：nums[1] vs nums[2]
位址：0xA004 → 值 2    vs    0xA008 → 值 3
結果：2 < 3 → 峰值在右半段 → l = mid + 1 = 2

搜尋空間縮小：
  -∞   [1]  [2]  [3]  [1]   -∞
        0    1    2    3
                  l    r
```

```
┌─────────────────────────────────────┐
│ 指標移動：l: 0 → 2, r: 不變 (3)    │
└─────────────────────────────────────┘
```

#### 第 2 步：`mid = 2 + (3 - 2) / 2 = 2`

```
比較：nums[2] vs nums[3]
位址：0xA008 → 值 3    vs    0xA00C → 值 1
結果：3 > 1 → 峰值在左半段（含 mid）→ r = mid = 2

搜尋空間縮小：
  -∞   [1]  [2]  [3]  [1]   -∞
        0    1    2    3
                  lr
```

```
┌─────────────────────────────────────┐
│ 指標移動：l: 不變 (2), r: 3 → 2    │
└─────────────────────────────────────┘
```

**l == r == 2 → 退出迴圈，回傳索引 2（值 3，位址 0xA008）**

---

### 範例 2：`nums = [1, 2, 1, 3, 5, 6, 4]`，基底位址 0xB000

**初始狀態：** `l = 0`, `r = 6`

```
搜尋空間：
  -∞   [1]  [2]  [1]  [3]  [5]  [6]  [4]   -∞
        0    1    2    3    4    5    6
        l                             r
```

#### 第 1 步：`mid = 0 + (6 - 0) / 2 = 3`

```
比較：nums[3] vs nums[4]
位址：0xB00C → 值 3    vs    0xB010 → 值 5
結果：3 < 5 → 峰值在右半段 → l = mid + 1 = 4

搜尋空間縮小：
  -∞   [1]  [2]  [1]  [3]  [5]  [6]  [4]   -∞
        0    1    2    3    4    5    6
                             l         r
```

```
┌─────────────────────────────────────┐
│ 指標移動：l: 0 → 4, r: 不變 (6)    │
└─────────────────────────────────────┘
```

#### 第 2 步：`mid = 4 + (6 - 4) / 2 = 5`

```
比較：nums[5] vs nums[6]
位址：0xB014 → 值 6    vs    0xB018 → 值 4
結果：6 > 4 → 峰值在左半段（含 mid）→ r = mid = 5

搜尋空間縮小：
  -∞   [1]  [2]  [1]  [3]  [5]  [6]  [4]   -∞
        0    1    2    3    4    5    6
                             l    r
```

```
┌─────────────────────────────────────┐
│ 指標移動：l: 不變 (4), r: 6 → 5    │
└─────────────────────────────────────┘
```

#### 第 3 步：`mid = 4 + (5 - 4) / 2 = 4`

```
比較：nums[4] vs nums[5]
位址：0xB010 → 值 5    vs    0xB014 → 值 6
結果：5 < 6 → 峰值在右半段 → l = mid + 1 = 5

搜尋空間縮小：
  -∞   [1]  [2]  [1]  [3]  [5]  [6]  [4]   -∞
        0    1    2    3    4    5    6
                                  lr
```

```
┌─────────────────────────────────────┐
│ 指標移動：l: 4 → 5, r: 不變 (5)    │
└─────────────────────────────────────┘
```

**l == r == 5 → 退出迴圈，回傳索引 5（值 6，位址 0xB014）**

## 4. 指標變數追蹤總表

### 範例 1：`nums = [1, 2, 3, 1]`

| 步驟 | l | r | mid | 存取位址 (mid) | 存取位址 (mid+1) | 值比較 | 動作 |
|------|---|---|-----|---------------|-----------------|--------|------|
| 初始 | 0 | 3 | — | — | — | — | — |
| 1 | 0→2 | 3 | 1 | 0xA004 (=2) | 0xA008 (=3) | 2 < 3 | l = 2 |
| 2 | 2 | 3→2 | 2 | 0xA008 (=3) | 0xA00C (=1) | 3 > 1 | r = 2 |

**統計：** 指標移動 2 次，記憶體存取 4 次（2 步 × 2 元素），搜尋深度 2

### 範例 2：`nums = [1, 2, 1, 3, 5, 6, 4]`

| 步驟 | l | r | mid | 存取位址 (mid) | 存取位址 (mid+1) | 值比較 | 動作 |
|------|---|---|-----|---------------|-----------------|--------|------|
| 初始 | 0 | 6 | — | — | — | — | — |
| 1 | 0→4 | 6 | 3 | 0xB00C (=3) | 0xB010 (=5) | 3 < 5 | l = 4 |
| 2 | 4 | 6→5 | 5 | 0xB014 (=6) | 0xB018 (=4) | 6 > 4 | r = 5 |
| 3 | 4→5 | 5 | 4 | 0xB010 (=5) | 0xB014 (=6) | 5 < 6 | l = 5 |

**統計：** 指標移動 3 次，記憶體存取 6 次（3 步 × 2 元素），搜尋深度 3

## 5. 記憶體存取模式分析

### 二分搜尋的快取行為

```
存取模式（範例 2，陣列共 7 個 int = 28 bytes）：

步驟 1：存取 索引 3, 4  → 位址 0xB00C, 0xB010  ─── 相鄰，同一 cache line
步驟 2：存取 索引 5, 6  → 位址 0xB014, 0xB018  ─── 相鄰，同一 cache line
步驟 3：存取 索引 4, 5  → 位址 0xB010, 0xB014  ─── 相鄰，同一 cache line

Cache line 大小通常為 64 bytes，此陣列（28 bytes）完全放入一條 cache line。
```

**小陣列場景（n ≤ 1000）：** 整個陣列約 4 KB，完全放入 L1 cache（通常 32–64 KB）。每次存取幾乎都是 cache hit，二分搜尋的跳躍存取模式在此不構成效能瓶頸。

**大陣列場景（理論上）：** 若陣列極大（數百萬元素），二分搜尋的跳躍存取會導致頻繁 cache miss。但因為只需 O(log n) 次存取（例如 n = 10⁶ 時約 20 次），即使每次都 miss，總代價仍遠低於線性掃描的 n 次存取。

## 6. 堆疊記憶體對比：迭代 vs. 遞迴

### 迭代版本（本解法）

```
堆疊使用量：固定

┌──────────────────────────────┐
│ findPeakElement 堆疊框       │
│   l, r, mid, nums, numsSize │
│   ≈ 24 bytes                 │
└──────────────────────────────┘

無論 n 多大，堆疊使用量恆為 O(1)。
```

### 遞迴版本

```
n = 7 時，最多遞迴 3 層：

┌──────────────────────────────┐
│ 第 3 層：l=4, r=5, mid=4    │  ≈ 24 bytes
├──────────────────────────────┤
│ 第 2 層：l=4, r=6, mid=5    │  ≈ 24 bytes
├──────────────────────────────┤
│ 第 1 層：l=0, r=6, mid=3    │  ≈ 24 bytes
├──────────────────────────────┤
│ 呼叫者                       │
└──────────────────────────────┘

堆疊總用量：≈ 72 bytes + 框架開銷 × 3
```

**最大遞迴深度：** ⌈log₂(n)⌉ = ⌈log₂(1000)⌉ = 10 層

對本題（n ≤ 1000）而言遞迴深度極淺，不構成問題。但在 Linux kernel 環境中（8 KB 堆疊），養成使用迭代的習慣是良好的防禦性程式設計。

## 7. 退化情況分析

### 最佳情況

第一次比較就確定方向，搜尋空間每次砍半。步驟數 = ⌈log₂(n)⌉。

### 最壞情況

同樣是 ⌈log₂(n)⌉ 步。二分搜尋的最壞情況和最佳情況步驟數相同（Θ(log n)），因為每一步都恰好砍掉一半搜尋空間。

### 特殊邊界：n = 1

```
nums = [42]，l = 0, r = 0
迴圈條件 l < r 不成立 → 直接回傳 0

不執行任何比較，O(1) 完成。
```

### 特殊邊界：嚴格遞增陣列

```
nums = [1, 2, 3, 4, 5]

每一步都是 nums[mid] < nums[mid+1] → l = mid + 1
最終 l == r == 4（最後一個元素）

峰值在最右邊（因為 nums[n] = -∞）。
搜尋步驟 = ⌈log₂(5)⌉ = 3 步。
```

### 特殊邊界：嚴格遞減陣列

```
nums = [5, 4, 3, 2, 1]

每一步都是 nums[mid] > nums[mid+1] → r = mid
最終 l == r == 0（第一個元素）

峰值在最左邊（因為 nums[-1] = -∞）。
搜尋步驟 = ⌈log₂(5)⌉ = 3 步。
```
