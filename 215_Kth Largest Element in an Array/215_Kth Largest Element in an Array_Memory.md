# LeetCode 215 — 第 K 大元素 (Quickselect 記憶體分析)

## 1. Struct 記憶體佈局

### 陣列元素記憶體配置 (單元素)

```
╔════════════════════════════════════════════════════════════════╗
║                   int 陣列元素記憶體佈局                        ║
╠════════════════════════════════════════════════════════════════╣
║                                                                ║
║  一個 int 元素占用 4 位元組 (32-bit)                            ║
║                                                                ║
║  ┌──────────────────────────────┐                              ║
║  │ Byte 3 │ Byte 2 │ Byte 1 │ Byte 0 │  (Little-endian)       ║
║  ├────────┼────────┼────────┼────────┤                         ║
║  │ 0xXX   │ 0xXX   │ 0xXX   │ 0xXX   │                         ║
║  └──────────────────────────────┘                              ║
║         地址: 0xAAAA (4字節對齊)                                ║
║                                                                ║
╚════════════════════════════════════════════════════════════════╝

範例: nums[0] = 3
  ┌────────────────────────┐
  │    3 (十進制)          │
  │    0x00000003 (十六進制) │
  │    地址: 0xA000        │
  └────────────────────────┘

範例: nums[1] = 2
  ┌────────────────────────┐
  │    2 (十進制)          │
  │    0x00000002 (十六進制) │
  │    地址: 0xA004        │
  └────────────────────────┘
```

## 2. 完整資料結構記憶體配置

### Example 1: nums = [3,2,1,5,6,4], k = 2

#### 地址對應表

```
┌──────────┬─────────┬────────────────────┬─────────┐
│ 索引     │ 值      │ 地址               │ 位移    │
├──────────┼─────────┼────────────────────┼─────────┤
│ [0]      │ 3       │ 0xA000             │ +0      │
│ [1]      │ 2       │ 0xA004             │ +4      │
│ [2]      │ 1       │ 0xA008             │ +8      │
│ [3]      │ 5       │ 0xA00C             │ +12     │
│ [4]      │ 6       │ 0xA010             │ +16     │
│ [5]      │ 4       │ 0xA014             │ +20     │
└──────────┴─────────┴────────────────────┴─────────┘

陣列基址: nums → 0xA000
numsSize: 6
k: 2
目標索引 target: k - 1 = 1
```

#### ASCII 記憶體圖

```
內存地址空間:
┌────────────────────────────────────────────────────────────────┐
│ 0xA000   0xA004   0xA008   0xA00C   0xA010   0xA014            │
├────────┬────────┬────────┬────────┬────────┬────────┐           │
│   3    │   2    │   1    │   5    │   6    │   4    │           │
└────────┴────────┴────────┴────────┴────────┴────────┘           │
│   [0]  │   [1]  │   [2]  │   [3]  │   [4]  │   [5]  │ (索引)    │
│        │        │        │        │        │        │           │
└────────┴────────┴────────┴────────┴────────┴────────┘           │
 ▲                                                                 │
 └─── nums (指標) = 0xA000                                         │
└────────────────────────────────────────────────────────────────┘

記憶體布局 (連續存儲):
  ┌─ 基址指標 nums
  │
  ▼
┌───────────────────────────────────────────────┐
│                  int 陣列                      │
├───┬───┬───┬───┬───┬───┐                       │
│ 3 │ 2 │ 1 │ 5 │ 6 │ 4 │ (值)                  │
├───┴───┴───┴───┴───┴───┤                       │
│ 4位元組 × 6個元素        │                       │
│ = 24位元組 (0x18)       │                       │
└───────────────────────┘                       │
│ 0xA000 ◄─────────────────────────────────────┘
  到 0xA018 (不包含)
```

## 3. 陣列索引版對應

### 索引 ↔ 記憶體地址映射公式

```
地址計算公式:
  address(nums[i]) = base_address + (i × sizeof(int))
                   = 0xA000 + (i × 4)

對應表:
┌──────────┬─────────────────┬──────────────────────────┐
│ 索引 i   │ 內存地址         │ 計算過程                 │
├──────────┼─────────────────┼──────────────────────────┤
│ i = 0    │ 0xA000          │ 0xA000 + (0 × 4) = 0xA000│
│ i = 1    │ 0xA004          │ 0xA000 + (1 × 4) = 0xA004│
│ i = 2    │ 0xA008          │ 0xA000 + (2 × 4) = 0xA008│
│ i = 3    │ 0xA00C          │ 0xA000 + (3 × 4) = 0xA00C│
│ i = 4    │ 0xA010          │ 0xA000 + (4 × 4) = 0xA010│
│ i = 5    │ 0xA014          │ 0xA000 + (5 × 4) = 0xA014│
└──────────┴─────────────────┴──────────────────────────┘

指標操作:
  left   ▶  指向目前分割左邊界 (包含)
  right  ▶  指向目前分割右邊界 (包含)
  pivot  ▶  分割軸值
  storeIndex ▶  用來放置下一個較大元素的位置
```

## 附錄: 核心程式碼回顧

```c
/* swap: 交換兩個 int 值 */
void swap(int* a, int* b) {
    int temp = *a;
    *a = *b;
    *b = temp;
}

/* partition: Lomuto 分割方案
 * 目標: 把 > pivot 的元素移到左邊
 * 返回: pivot 的最終位置 (不是分割點)
 */
int partition(int* nums, int left, int right, int pivotIndex) {
    int pivot = nums[pivotIndex];
    swap(&nums[pivotIndex], &nums[right]);    // 把 pivot 移到末尾

    int storeIndex = left;
    for (int i = left; i < right; i++) {
        if (nums[i] > pivot) {
            swap(&nums[storeIndex], &nums[i]);
            storeIndex++;
        }
    }
    swap(&nums[storeIndex], &nums[right]);     // 把 pivot 放到最終位置
    return storeIndex;
}

/* findKthLargest: 迭代 quickselect
 * target = k - 1: 0-based 索引
 */
int findKthLargest(int* nums, int numsSize, int k) {
    int target = k - 1;
    int left = 0, right = numsSize - 1;

    while (1) {
        int pivotIndex = partition(nums, left, right, left);

        if (pivotIndex == target) {
            return nums[pivotIndex];
        } else if (pivotIndex < target) {
            left = pivotIndex + 1;
        } else {
            right = pivotIndex - 1;
        }
    }
}
```
