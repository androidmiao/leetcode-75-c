# LeetCode 1431: Kids With the Greatest Number of Candies - 記憶體分析

## 問題概述
**Problem:** 1431 - Kids With the Greatest Number of Candies
**目標:** 判斷每個小孩加上額外糖果後，是否擁有最多糖果。

**C 語言簽名:**
```c
bool* kidsWithCandies(int* candies, int candiesSize, int extraCandies, int* returnSize)
```

**使用變數:**
- Input: `int* candies` (糖果陣列指標), `int candiesSize` (陣列大小), `int extraCandies` (額外糖果數)
- Output: `bool* table` (malloc'd 布林陣列)
- Local: `int greatest` (最大值), `size_t i` (迴圈計數器)

---

## 1. Struct 記憶體佈局

本問題不使用自訂結構體，但涉及基本型別陣列的記憶體配置。

### 基本型別大小（64-bit 系統）
```
sizeof(int)  = 4 bytes
sizeof(bool) = 1 byte
sizeof(int*) = 8 bytes (指標)
sizeof(bool*) = 8 bytes (指標)
```

### 輸入陣列 `candies` 的記憶體配置
```
╔════════════════════════════════════════════════╗
║     int candies[5]  (假設記憶體位址 0xA000)    ║
╟─────────┬─────────┬─────────┬─────────┬─────────╢
║ candies │ candies │ candies │ candies │ candies ║
║   [0]   │   [1]   │   [2]   │   [3]   │   [4]   ║
╟─────────┼─────────┼─────────┼─────────┼─────────╢
║ 0xA000  │ 0xA004  │ 0xA008  │ 0xA00C  │ 0xA010  ║
║  (4B)   │  (4B)   │  (4B)   │  (4B)   │  (4B)   ║
╚════════════════════════════════════════════════╝
```

### 輸出陣列 `table` 的記憶體配置
```
╔════════════════════════════════════════════════╗
║      bool table[5]  (假設記憶體位址 0xB000)    ║
╟─────────┬─────────┬─────────┬─────────┬─────────╢
║  table  │  table  │  table  │  table  │  table  ║
║  [0]    │  [1]    │  [2]    │  [3]    │  [4]    ║
╟─────────┼─────────┼─────────┼─────────┼─────────╢
║ 0xB000  │ 0xB001  │ 0xB002  │ 0xB003  │ 0xB004  ║
║  (1B)   │  (1B)   │  (1B)   │  (1B)   │  (1B)   ║
╚════════════════════════════════════════════════╝
```

### sizeof() 陷阱警示
```
注意：程式碼中常見的錯誤用法：
  bool* table = (bool*)malloc(sizeof(table) * candiesSize);

sizeof(table) = 8 bytes (指標大小，64-bit)
實際分配：8 × 5 = 40 bytes ❌ 浪費！

正確用法應為：
  bool* table = (bool*)malloc(sizeof(bool) * candiesSize);
  // 或
  bool* table = (bool*)malloc(sizeof(*table) * candiesSize);

實際需要：1 × 5 = 5 bytes ✓ 正確
```

## 5. 記憶體存取模式比較

### 存取模式分析

#### 第一遍：順序讀取 candies 陣列
```
時間軸 ──────────────────────────────────────────►

讀取模式：0xA000 ──► 0xA004 ──► 0xA008 ──► 0xA00C ──► 0xA010
           ↓        ↓        ↓        ↓        ↓
         [2]      [3]      [5]      [1]      [3]
```

**特性：**
- **連續線性存取** (Sequential Access)
- **Cache 命中率高** (High Cache Hit Rate)
- **Prefetcher 友善** (Prefetcher-Friendly)

#### 第二遍：順序讀取 candies + 順序寫入 table
```
讀取：0xA000 ◄──┐
              │ 計算      寫入：0xB000 ◄──┐
              ├──────────────────────────┤
讀取：0xA004 ◄──┤                        0xB001
              ├──────────────────────────┤
讀取：0xA008 ◄──┤                        0xB002
              ├──────────────────────────┤
讀取：0xA00C ◄──┤                        0xB003
              ├──────────────────────────┤
讀取：0xA010 ◄──┤                        0xB004
```

### Cache 行分析（假設 64 bytes 快取行）

#### int[5] 陣列的 Cache 表現
```
╔═════════════════════════════════════════════╗
║  Cache Line (64 bytes)                      ║
╟─────────────────────────────────────────────╢
║ 0xA000: [2] │ 0xA004: [3] │ 0xA008: [5] │ ║
║ 0xA00C: [1] │ 0xA010: [3] │  (padding)  │ ║
╚═════════════════════════════════════════════╝

全部 20 bytes (5×4 bytes) 可放入單一 Cache Line
Cache 命中率：100% (第一次讀取後都是命中)
```

#### bool[5] 陣列的 Cache 表現
```
╔═════════════════════════════════════════════╗
║  Cache Line (64 bytes)                      ║
╟─────────────────────────────────────────────╢
║ 0xB000:[T] 0xB001:[T] 0xB002:[T] 0xB003:[F] ║
║ 0xB004:[T] (padding×59 bytes)               ║
╚═════════════════════════════════════════════╝

全部 5 bytes 可放入單一 Cache Line
Cache 命中率：100% (第一次寫入後都是命中)
空間浪費：59 bytes (cache line 內的浪費)
```

### 效能比較表

| 指標 | candies (int[5]) | table (bool[5]) | 備註 |
|------|-----------------|-----------------|------|
| 每元素大小 | 4 bytes | 1 byte | bool 更小 |
| 陣列總大小 | 20 bytes | 5 bytes | 4:1 比例 |
| Cache Line 容納 | 1 條（全部） | 1 條（全部） | 兩者都友善 |
| 存取模式 | 循序讀 | 循序寫 | 都是最佳 |
| Prefetch 效率 | 高 | 高 | 都支援 |
| 記憶體頻寬利用 | ~20% 實用 | ~8% 實用 | int 更有效率 |

### 存取時間線（假設值）

```
Pass 1: 讀 candies
│
├─ i=0: 讀 0xA000 (Cache Miss) → ~100 ns
├─ i=1: 讀 0xA004 (Cache Hit)  → ~1 ns
├─ i=2: 讀 0xA008 (Cache Hit)  → ~1 ns
├─ i=3: 讀 0xA00C (Cache Hit)  → ~1 ns
└─ i=4: 讀 0xA010 (Cache Hit)  → ~1 ns
   小計：~104 ns

Pass 2: 讀 candies + 寫 table
│
├─ i=0: 讀 0xA000 (Hit) + 寫 0xB000 (Miss) → ~101 ns
├─ i=1: 讀 0xA004 (Hit) + 寫 0xB001 (Hit)  → ~2 ns
├─ i=2: 讀 0xA008 (Hit) + 寫 0xB002 (Hit)  → ~2 ns
├─ i=3: 讀 0xA00C (Hit) + 寫 0xB003 (Hit)  → ~2 ns
└─ i=4: 讀 0xA010 (Hit) + 寫 0xB004 (Hit)  → ~2 ns
   小計：~109 ns

總耗時估計：~213 ns (假設值，實際取決於硬體)
```

### 最佳化建議

1. **陣列大小**：輸出陣列很小（5 bytes），可以使用棧而非堆
   ```c
   bool result[5];  // 棧上，更快
   ```

2. **一次遍歷**：如果不需要最大值，可以優化為一遍
   ```c
   // 先掃描找最大值
   int max = 0;
   for (...) max = MAX(max, candies[i]);

   // 再判斷
   for (...) table[i] = ...;
   ```

3. **記憶體布局**：保持兩個陣列在不同 Cache Line 可避免 False Sharing

### 堆疊 vs 堆的使用對比

#### 當前實現（使用堆）
```c
bool* table = (bool*)malloc(sizeof(bool) * candiesSize);
```

| 方面 | 堆配置 | 評估 |
|------|--------|------|
| 靈活性 | 高（動態大小） | ✓ 支援任意大小 |
| 速度 | 較慢（malloc/free） | 耗時 ~100-1000 ns |
| 快取 | 可能不連續 | 可能 Miss 快取 |
| 記憶體管理 | 手動 | 需 free() 避免洩漏 |
| 大小限制 | 無（受可用堆大小限制） | 通常無問題 |

#### 最佳化實現（小陣列用棧）
```c
bool table[candiesSize];  // 若 candiesSize <= 某閾值
```

| 方面 | 棧配置 | 評估 |
|------|--------|------|
| 靈活性 | 受限（固定大小） | ✓ 題目限制已知 |
| 速度 | 快速（棧操作） | ✓ 無 malloc 開銷 |
| 快取 | 連續，友善 | ✓ 最佳局部性 |
| 記憶體管理 | 自動（函式返回釋放） | ✓ 無洩漏風險 |
| 大小限制 | 有（棧大小，通常 1-8 MB） | ⚠ 大陣列可能溢出 |

**LeetCode 1431 場景**：candiesSize ≤ 100，使用棧更合適！

### 指標鏈追蹤

```
function call stack:

caller → 傳遞參數 → kidsWithCandies()
                    ├─ candies* ──────▶ 0xA000 (外部陣列)
                    ├─ returnSize* ───▶ 0xC000 (外部 int)
                    └─ 內部分配
                       └─ table* ─────▶ 0xB000 (新分配)

返回時必須：
  - 將 table 指標返回給 caller
  - 將 candiesSize 寫入 *returnSize
  - 不能 free(table)，ownership 轉移給 caller
```

### 記憶體生命週期

```
時間軸 ──────────────────────────────────────────────►

Caller Stack
│ [candies*, candiesSize, extraCandies, returnSize*]
│        ▼
│   kidsWithCandies() 開始
│   ├─ 讀取參數
│   ├─ malloc(table) ◄───── Heap: 0xB000 分配
│   ├─ 第一遍：讀 candies, 計算 greatest
│   ├─ 第二遍：讀 candies, 寫 table
│   ├─ 寫入 *returnSize = 5
│   ├─ 返回 table (不 free)
│   └─ 堆疊框架銷毀 ◄───── [greatest, i, table*] 消亡
│
│ Caller 使用 table (ownership 已獲得)
│ ...
│ free(table) ◄───── Heap: 0xB000 釋放
│
呼叫結束
```

## 7. 總結與效能分析

### 演算法時間複雜度
```
Pass 1 (尋找最大值): O(n)  ─┐
Pass 2 (填充輸出):   O(n)  ├─ 總計: O(n)
                         ─┘
```

### 演算法空間複雜度
```
輸入:      O(1)   (指標，不計陣列本身)
輸出:      O(n)   (bool[n])
局部變數:  O(1)   (greatest, i)
總計:      O(n)
```

### 實際記憶體存取次數（n=5）

| 操作 | 讀取次數 | 寫入次數 | 快取機會 |
|------|---------|---------|---------|
| 初始化 greatest | 0 | 1 | N/A |
| Pass 1 循環 | 5 | 0 | 全中 |
| 初始化 table | 0 | 1 (malloc) | 第一次 Miss |
| Pass 2 循環 | 5 | 5 | 全中 |
| 清理/返回 | 0 | 1 (*returnSize) | 快取 |
| **總計** | **10** | **8** | **6/10 命中** |

### 潛在改進點

1. **sizeof() 陷阱修正**
   ```c
   // 錯誤
   bool* table = (bool*)malloc(sizeof(table) * candiesSize);  // 40 bytes!

   // 正確
   bool* table = (bool*)malloc(sizeof(bool) * candiesSize);   // 5 bytes
   bool* table = (bool*)malloc(sizeof(*table) * candiesSize); // 推薦
   ```

2. **一次遍歷優化**（如果演算法允許）
   ```c
   // 兩次遍歷（當前）
   int greatest = 0;
   for (...) greatest = MAX(...);
   for (...) table[i] = ...;

   // 若改成兩個指標，可能一次遍歷（複雜度相同，但 cache 更優）
   ```

3. **棧 vs 堆**（LeetCode 測試不涉及，但值得知道）
   ```c
   // 對於 candiesSize <= 100
   bool table[100];  // 棧上，更快，無 malloc 開銷
   ```

### 根據 LeetCode 約束的效能預測

      ▼

## 附錄：假設条件聲明

本文件中的所有位址（如 0xA000、0xB000 等）均為**假設位址**，用於演示目的。

實際執行時，記憶體位址會：
- 因 ASLR（Address Space Layout Randomization）而隨機化
- 因系統狀態而變化
- 因編譯器優化而改變

相對位址和布局邏輯保持一致，但絕對位址因環境而異。
