# Edge Cases 邊界情況速查表

## 核心概念

邊界情況（Edge Case）是測試演算法正確性的關鍵。面試時展示考慮到邊界情況能體現你的嚴謹性。

**測試策略：**
1. 最小輸入（空、單一元素）
2. 極端值（最大、最小、負數）
3. 特殊結構（重複、已排序、反向）
4. 邊界條件（溢出、無解）

---

## 按資料結構分類

### 1. 陣列 (Array)

```c
// 邊界情況檢查清單
int* solution(int* arr, int arrSize) {
    // 邊界 1: 空陣列
    if (arrSize == 0) {
        return malloc(0);  // 或返回 NULL
    }

    // 邊界 2: 單一元素
    if (arrSize == 1) {
        int *result = malloc(sizeof(int));
        result[0] = arr[0];
        return result;
    }

    // 邊界 3: 所有元素相同
    // 需考慮演算法是否能處理 arr = [5, 5, 5, 5]

    // 邊界 4: 已排序陣列
    // arr = [1, 2, 3, 4, 5]
    // 某些演算法可能有優化

    // 邊界 5: 反向排序
    // arr = [5, 4, 3, 2, 1]

    // 邊界 6: 包含負數
    // arr = [-5, 3, -2, 8]

    // 邊界 7: 元素溢出（INT_MAX, INT_MIN）
    // 避免中間計算溢出

    return NULL;
}
```

**常見邊界情況：**
- `arrSize == 0`: 空陣列
- `arrSize == 1`: 單一元素
- `arr = [1, 1, 1, ...]`: 全相同
- `arr = [1, 2, 3, ...]`: 已排序
- `arr = [INT_MAX, INT_MIN, ...]`: 極端值

### 2. 鏈表 (Linked List)

```c
typedef struct {
    int val;
    struct ListNode *next;
} ListNode;

// 邊界情況檢查清單
ListNode* solution(ListNode* head) {
    // 邊界 1: 空鏈表
    if (head == NULL) {
        return NULL;
    }

    // 邊界 2: 單一節點
    if (head->next == NULL) {
        // 只有一個節點，某些操作需特殊處理
        return head;
    }

    // 邊界 3: 兩個節點
    if (head->next->next == NULL) {
        // 可能是最小的非平凡情況
    }

    // 邊界 4: 環形鏈表
    // head -> 1 -> 2 -> 3 -> 2(環)
    // 需檢測環或避免無限迴圈

    // 邊界 5: 重複值
    // head -> 5 -> 5 -> 5 -> NULL

    // 邊界 6: 所有節點都要刪除
    // 特別是頭節點操作

    // 邊界 7: 相交鏈表（如果涉及）
    // list1: 1 -> 2 -> 3
    //             ↓
    // list2: 4 -> 5

    return head;
}
```

**常見邊界情況：**
- `head == NULL`: 空鏈表
- `head->next == NULL`: 單一節點
- 環形結構（Floyd 檢測）
- 修改頭節點（需雙層指標或虛擬節點）
- 相交但不重疊的鏈表

### 3. 二叉樹 (Binary Tree)

```c
typedef struct {
    int val;
    struct TreeNode *left;
    struct TreeNode *right;
} TreeNode;

// 邊界情況檢查清單
int solution(TreeNode* root) {
    // 邊界 1: 空樹
    if (root == NULL) {
        return 0;  // 或 -1，取決於定義
    }

    // 邊界 2: 單一節點
    if (root->left == NULL && root->right == NULL) {
        return root->val;
    }

    // 邊界 3: 完全左傾樹（退化成鏈表）
    //     1
    //    /
    //   2
    //  /
    // 3
    // 深度 = O(n)，遞迴棧開銷大

    // 邊界 4: 完全右傾樹
    // 1
    //  \
    //   2
    //    \
    //     3

    // 邊界 5: 完全平衡樹
    //       1
    //      / \
    //     2   3
    //    / \
    //   4   5

    // 邊界 6: 所有節點相同值
    //       5
    //      / \
    //     5   5
    //    /
    //   5

    // 邊界 7: 包含負數值
    // 可能影響 max/min 計算

    // 邊界 8: 只有左子樹
    // 邊界 9: 只有右子樹

    return 0;
}
```

**常見邊界情況：**
- `root == NULL`: 空樹
- 葉節點（無子樹）
- 單邊樹（只有左或右子樹）
- 完全平衡 vs. 完全不平衡
- 深度過大（棧溢出風險）

### 4. 圖 (Graph)

```c
// 邊界情況檢查清單
int solution(int** graph, int graphSize) {
    // 邊界 1: 空圖（無節點）
    if (graphSize == 0) {
        return 0;
    }

    // 邊界 2: 單一節點，無邊
    if (graphSize == 1 && graph[0] == NULL) {
        return 1;
    }

    // 邊界 3: 完全不連接的圖（多個孤立節點）
    // 每個節點都是一個連通分量

    // 邊界 4: 完全連接的圖（全連通）
    // 任意兩個節點都有邊

    // 邊界 5: 有自環（self-loop）
    // 節點 0 -> 0

    // 邊界 6: 有環
    // 0 -> 1 -> 2 -> 0

    // 邊界 7: 無環（DAG）
    // 0 -> 1 -> 2

    // 邊界 8: 有重複邊
    // 0 -> [1, 1, 2]

    // 邊界 9: 有相反邊（但未必連通）
    // 0 -> 1 且 1 -> 0 不一定表示無向

    return 0;
}
```

**常見邊界情況：**
- 空圖
- 單一節點
- 完全連接 vs. 完全不連接
- 自環
- 環形結構
- 有向 vs. 無向
- 多個連通分量

### 5. 字串 (String)

```c
// 邊界情況檢查清單
char* solution(char* s) {
    // 邊界 1: 空字串
    if (s == NULL || strlen(s) == 0) {
        return malloc(1);  // "\0"
    }

    // 邊界 2: 單一字元
    if (strlen(s) == 1) {
        char *result = malloc(2);
        strcpy(result, s);
        return result;
    }

    // 邊界 3: 所有字元相同
    // "aaaaa"

    // 邊界 4: 已排序
    // "abcdef"

    // 邊界 5: 反向排序
    // "fedcba"

    // 邊界 6: 包含特殊字元或空格
    // "a b c" 或 "a\tb\nc"

    // 邊界 7: 大小寫混合
    // "AaBbCc"

    // 邊界 8: 數字字串
    // "12345"

    // 邊界 9: 回文
    // "racecar"

    // 邊界 10: 很長的字串（可能影響效率）

    return NULL;
}
```

**常見邊界情況：**
- 空字串
- 單一字元
- 全相同字元
- 回文字串
- 已排序
- 包含空格或特殊字元
- 非常長的字串（複雜度考量）

## 按問題類型分類

### 數值計算

```c
// 邊界情況
int add(int a, int b) {
    // 邊界 1: 零值
    if (a == 0 || b == 0) { }

    // 邊界 2: 負數
    if (a < 0 || b < 0) { }

    // 邊界 3: INT_MAX / INT_MIN（溢出風險）
    // a = INT_MAX, b = 1 => 溢出

    // 邊界 4: 非常接近的值
    // a = 1000000, b = -1000000

    // 邊界 5: 相等
    // a = 5, b = 5

    return a + b;
}
```

### 搜尋問題

```c
// 邊界情況
int search(int* arr, int size, int target) {
    // 邊界 1: 目標不存在
    // 返回 -1 或 NULL

    // 邊界 2: 多個目標存在
    // 返回第一個？最後一個？個數？

    // 邊界 3: 目標在最前面
    // arr[0] == target

    // 邊界 4: 目標在最後面
    // arr[size-1] == target

    // 邊界 5: 整個陣列都是目標
    // [5, 5, 5, 5]

    return -1;
}
```

### 排序問題

```c
// 邊界情況
void sort(int* arr, int size) {
    // 邊界 1: 已排序
    // [1, 2, 3, 4, 5]

    // 邊界 2: 反向排序
    // [5, 4, 3, 2, 1]

    // 邊界 3: 部分排序
    // [1, 3, 2, 4, 5]

    // 邊界 4: 所有元素相同
    // [3, 3, 3, 3]

    // 邊界 5: 包含重複
    // [1, 2, 2, 3, 1]

    // 邊界 6: 包含負數
    // [-3, 1, -2, 4]
}
```

## 優先級測試順序

### 面試時的測試策略

1. **先測最簡單的**
   - 空輸入
   - 單一元素
   - 極少元素（2-3 個）

2. **再測常規情況**
   - 中等大小輸入
   - 無特殊結構

3. **最後測困難情況**
   - 最壞情況（如反向排序）
   - 重複元素
   - 極端值

4. **如果有時間**
   - 隨機生成測試用例
   - 壓力測試（大輸入）

## 面試話語範例

> "讓我先測試空陣列的情況...再來單一元素...然後考慮所有元素相同的情況...這樣能覆蓋大部分邊界。"

> "我注意到如果 target 不存在，應該返回 -1。同時，如果有多個 target，我的演算法返回第一個匹配的索引。"

> "我發現整數相減可能溢出，所以改用比較的方式來避免。"

> "邊界情況包括空鏈表、單節點、以及需要修改頭節點的情況，所以我使用虛擬節點來統一處理。"
