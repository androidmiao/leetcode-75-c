# Complexity Cheatsheet 複雜度速查表

## 時間複雜度概述

### 複雜度排序（從快到慢）

```
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) < O(2ⁿ) < O(n!)
```

### 直觀理解

| 複雜度        | 時間（n=1000000） | 適用場景      |
| ---------- | ------------- | --------- |
| O(1)       | < 1 ns        | 直接存取、雜湊查詢 |
| O(log n)   | ~20 us        | 二分搜尋、平衡樹  |
| O(n)       | ~1 ms         | 線性掃描      |
| O(n log n) | ~20 ms        | 合併排序、快速排序 |
| O(n²)      | ~1000 s       | 氣泡排序、嵌套迴圈 |
| O(2ⁿ)      | 無法計算          | 指數組合      |

## 按演算法模式分類

### 1. 貪心演算法 (Greedy)

| 問題                                 | 時間     | 空間          | 備註     |
| ---------------------------------- | ------ | ----------- | ------ |
| Container With Most Water          | O(n)   | O(1)        | 雙指標掃描  |
| Best Time to Buy and Sell Stock    | O(n)   | O(1)        | 追蹤最小值  |
| Best Time to Buy and Sell Stock II | O(n)   | O(1)        | 累積正差   |
| Increasing Triplet Subsequence     | O(n)   | O(1)        | 維護兩最小值 |
| Can Place Flowers                  | O(n)   | O(1)        | 局部決策   |
| GCD of Strings                     | O(m+n) | O(min(m,n)) | 驗證前綴   |
| Kids With Greatest Candies         | O(n)   | O(n)        | 追蹤最大值  |

**貪心的關鍵：** 每步選擇都最優，無須回溯

---

### 2. 前綴和 (Prefix Sum)

| 問題                           | 預處理  | 查詢  | 空間   | 備註      |
| ---------------------------- | ---- | --- | ---- | ------- |
| Product of Array Except Self | O(n) | -   | O(n) | 左右乘積陣列  |
| Find Pivot Index             | O(n) | -   | O(1) | 動態計算    |
| Find Highest Altitude        | O(n) | -   | O(1) | 累積和+最大值 |

**前綴和模式：**
```
查詢 [L, R] 的和 = prefixSum[R+1] - prefixSum[L]
```

---

### 3. 單調堆疊 (Monotonic Stack)

| 問題 | 時間 | 空間 | 堆疊類型 |
|------|------|------|----------|
| Largest Rectangle in Histogram | O(n) | O(n) | 遞增 + 哨兵 |
| Sliding Window Maximum | O(n) | O(min(n,k)) | 遞減雙端佇列 |
| Daily Temperatures | O(n) | O(n) | 遞減 + 索引 |

**單調堆疊的複雜度分析：**
- 每個元素最多 push 一次、pop 一次
- 因此總時間 O(n)，空間 O(n)

---

### 4. 回溯法 (Backtracking)

| 問題 | 時間 | 空間 | 備註 |
|------|------|------|------|
| Letter Combinations | O(4^n * n) | O(n) | n 位數字，最多 4 個字母 |
| Permutations | O(n! * n) | O(n) | n!個排列，每個 O(n) 複製 |
| Combinations | O(C(n,k) * k) | O(k) | 組合數 × 複製開銷 |

**回溯複雜度計算方法：**
```
時間 = 解的個數 × 構造每個解的時間
空間 = 遞迴深度 + 儲存結果
```

### 5. 字串操作 (String Manipulation)

| 問題 | 時間 | 空間 | 演算法 |
|------|------|------|--------|
| Reverse Words in String | O(n) | O(n) | 原地反轉 × 2 |
| GCD of Strings | O(m+n) | O(min(m,n)) | 前綴驗證 |
| Merge Strings Alternately | O(m+n) | O(m+n) | 雙指標 |

## 按資料結構分類

### 陣列/列表

| 操作 | 時間 | 空間 | 備註 |
|------|------|------|------|
| 存取元素 | O(1) | - | 直接索引 |
| 查找元素 | O(n) | - | 線性掃描 |
| 排序 | O(n log n) | O(1)-O(n) | 取決於演算法 |
| 反轉 | O(n) | O(1) | 原地操作 |

### 雜湊表（Hash Table）

| 操作 | 平均 | 最差 | 空間 |
|------|------|------|------|
| 插入 | O(1) | O(n) | O(n) |
| 刪除 | O(1) | O(n) | - |
| 查詢 | O(1) | O(n) | - |

**最差情況（all hash collision）：** O(n)

### 平衡樹（AVL, Red-Black）

| 操作 | 時間 | 空間 |
|------|------|------|
| 查找 | O(log n) | - |
| 插入 | O(log n) | O(n) |
| 刪除 | O(log n) | - |

### 堆（Heap）

| 操作 | 時間 | 空間 |
|------|------|------|
| 插入 | O(log n) | O(n) |
| 刪除最小/最大 | O(log n) | - |
| 建堆 | O(n) | O(1) |
| 堆排序 | O(n log n) | O(1) |

### 鏈表（Linked List）

| 操作 | 時間 | 空間 |
|------|------|------|
| 存取第 k 個 | O(k) | - |
| 插入/刪除 | O(1) | O(n) |
| 反轉 | O(n) | O(1) 或 O(n) |

## 空間複雜度分析

### 常見空間開銷

| 資料結構/操作 | 空間 | 備註 |
|--------------|------|------|
| 單變數 | O(1) | 常數 |
| 大小為 n 的陣列 | O(n) | 線性 |
| 大小為 n × m 的矩陣 | O(n×m) | 二維 |
| 二叉樹（n 個節點） | O(n) | 指標開銷 |
| 遞迴呼叫棧 | O(深度) | 最多 O(n) |
| 雜湊表（n 個元素） | O(n) | 包括衝突鏈 |

### 遞迴空間

```c
// 線性遞迴：O(n) 棧空間
int factorial(int n) {
    if (n <= 1) return 1;
    return n * factorial(n - 1);
}

// 二叉遞迴：O(log n) 最佳，O(n) 最差
int fib(int n) {
    if (n <= 1) return n;
    return fib(n-1) + fib(n-2);
}
```

## 漸進分析技巧

### Big O 簡化規則

1. **常數可忽略**
   ```
   O(5n + 3) = O(n)
   O(n² + n) = O(n²)
   ```

2. **取最高項**
   ```
   O(n² + n log n) = O(n²)
   O(2ⁿ + n³) = O(2ⁿ)
   ```

3. **乘積規則**
   ```
   O(n) × O(n) = O(n²)
   O(n log n) × O(n) = O(n² log n)
   ```

4. **加法規則**
   ```
   O(n) + O(m) = O(n + m)
   O(n²) + O(n) = O(n²)
   ```

## 面試中的複雜度討論

### 如何回答「複雜度是多少？」

1. **明確時間和空間**
   > "時間複雜度是 O(n log n)，空間複雜度是 O(1)。"

2. **解釋為什麼**
   > "我用雙指標掃描陣列一次（O(n)），並追蹤最大值。"

3. **討論優化空間**
   > "原本是 O(n²)，但透過前綴和優化到 O(n)。"

4. **比較替代方案**
   > "這比排序後搜尋的 O(n log n) 更優，因為只掃描一遍。"

### 常見複雜度問題

**Q: 為什麼是 O(n log n)，不是 O(n²)？**
A: 每次遞迴減半問題規模（log n 層），每層處理 n 個元素 → O(n log n)

**Q: 空間是 O(1) 還是 O(n)？**
A: 不計算輸入，只算額外空間。若使用輔助陣列，則 O(n)；原地修改則 O(1)。

**Q: 最差情況是什麼？**
A: 如快速排序最差 O(n²)（重疊度最高），但平均 O(n log n)。

## LeetCode 上常見複雜度

### 限制與可行性

```
n = 10^3      ✓ O(n²) 可行
n = 10^4      ✓ O(n²) 邊界
n = 10^5      ! O(n log n) 必須
n = 10^6      ✗ O(n log n) 或更優
n = 10^9      ✗ 只有 O(log n) 或 O(1)
```

### 根據時間限制反推複雜度

```
1 秒時間限制：
  - n = 10^8: O(1) 或 O(log n)
  - n = 10^6: O(n) 或 O(n log n)
  - n = 10^4: O(n²)
  - n = 10^2: O(n³) 或 O(n!)
```
