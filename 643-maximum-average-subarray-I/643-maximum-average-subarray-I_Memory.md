# LeetCode 643 Maximum Average Subarray I - 記憶體分析

## 問題描述
給定陣列 `nums` 和整數 `k`，找出長度為 `k` 的連續子陣列，使其平均值最大。

**時間複雜度**: O(n)
**空間複雜度**: O(1)

### 例題
- **例題1**: nums = [1,12,-5,-6,50,3], k = 4 → 輸出: 12.75
- **例題2**: nums = [5], k = 1 → 輸出: 5.0

### C函數簽名
```c
double findMaxAverage(int* nums, int numsSize, int k)
```

## 3. 陣列索引版對應

### 滑動窗口邊界公式

在第 `i` 步迭代中（i = k-1, k, ..., numsSize-1）：
- **窗口涵蓋的索引**: [i-k+1, i]
- **窗口大小**: k 個元素
- **移動規則**:
  - 初始化 (i = k-1): 將 nums[0] 到 nums[k-1] 相加
  - 滑動 (i > k-1): 加入 nums[i]，移除 nums[i-k]

### 例題1分析 (k=4)

| 步驟 | i | 窗口索引範圍 | 窗口內容 |
|-----|---|------------|---------|

| 滑動1 | 4 | [1, 4] | [12, -5, -6, 50] |
| 滑動2 | 5 | [2, 5] | [-5, -6, 50, 3] |

### 例題2分析 (k=1)

| 步驟 | i | 窗口索引範圍 | 窗口內容 |

| 初始化 | 0 | [0, 0] | [5] |

## 4. 搜尋／操作過程追蹤

### 例題1: nums = [1,12,-5,-6,50,3], k = 4

#### 步驟 0: 初始化窗口 (i = 3)

```
陣列:  [ 1 ] [12 ] [-5 ] [-6 ] [50 ] [ 3 ]
       0xA000 0xA004 0xA008 0xA00C 0xA010 0xA014
        ↓      ↓      ↓      ↓
        ◄──────────────────────────窗口 [i-k+1, i] = [0, 3]
```

**記憶體存取**:
- 0xA000 (nums[0]): 讀取 1
- 0xA004 (nums[1]): 讀取 12
- 0xA008 (nums[2]): 讀取 -5
- 0xA00C (nums[3]): 讀取 -6

**計算**:
- windowSum = 1 + 12 + (-5) + (-6) = 2
- maxSum = 2
- i = 3

#### 步驟 1: 滑動到 i = 4

```
陣列:  [ 1 ] [12 ] [-5 ] [-6 ] [50 ] [ 3 ]
       0xA000 0xA004 0xA008 0xA00C 0xA010 0xA014
                ↓      ↓      ↓      ↓
                ◄──────────────────────────窗口 [i-k+1, i] = [1, 4]
```

**記憶體操作**:
- 加入 nums[4] (0xA010): 讀取 50
- 移除 nums[0] (0xA000): 值為 1
- windowSum = 2 - 1 + 50 = 51
- maxSum = max(2, 51) = 51
- i = 4

#### 步驟 2: 滑動到 i = 5

```
陣列:  [ 1 ] [12 ] [-5 ] [-6 ] [50 ] [ 3 ]
       0xA000 0xA004 0xA008 0xA00C 0xA010 0xA014
                      ↓      ↓      ↓      ↓
                      ◄──────────────────────────窗口 [i-k+1, i] = [2, 5]
```

**記憶體操作**:
- 加入 nums[5] (0xA014): 讀取 3
- 移除 nums[1] (0xA004): 值為 12
- windowSum = 51 - 12 + 3 = 42
- maxSum = max(51, 42) = 51
- i = 5
- 迴圈結束 (i 已達 numsSize-1 = 5)

**最終結果**: maxAverage = 51 / 4 = 12.75

## 6. 記憶體存取模式比較

### 本演算法: 順序存取 (Sequential Access)

```
存取順序 (例題1):
┌─────────────────────────────────────────┐
│  0xA000 ─▶ 0xA004 ─▶ 0xA008 ─▶ 0xA00C  │ (初始化)
│  0xA010 ─▶ 0xA000                       │ (滑動1)
│  0xA014 ─▶ 0xA004                       │ (滑動2)
└─────────────────────────────────────────┘
```

**特性**:
- ✓ **快取友善** (Cache-friendly): CPU 快取可預取相鄰的記憶體行
- ✓ **空間局部性** (Spatial Locality): 訪問的地址相近
- ✓ **時間局部性** (Temporal Locality): 最近存取的資料再次使用的機率低（每個元素僅讀一次）

### 假設隨機存取演算法

```
存取順序 (隨機模式):
┌─────────────────────────────────────────┐
│  0xA000    0xA010    0xA008    0xA014   │ (隨機跳躍)
└─────────────────────────────────────────┘
```

**特性**:
- ✗ 不友善快取: 無法預測下一個記憶體位置
- ✗ 差的空間局部性: 訪問地址分散
- ✗ 快取未命中率高

### 對比分析

| 指標 | 滑動窗口（本演算法） | 隨機存取 |
|-----|-------------------|--------|
| 快取命中率 | 高 (>90%) | 低 (<20%) |
| CPU 週期延遲 | 低 (1-4週期) | 高 (200+週期) |
| 記憶體頻寬利用率 | 高 | 低 |
| 實際執行速度 | 快 | 慢 |

**結論**: 滑動窗口演算法的順序存取模式極具快取效率，是其高效能的關鍵因素。

## 總結

### 演算法優勢
1. **時間效率**: O(n) 單次掃描，無重複計算
2. **空間效率**: O(1) 常數堆疊，無額外資料結構
3. **快取友善**: 順序記憶體存取，高命中率
4. **可擴展性**: 支援任意大小的輸入無堆疊溢出風險

### 記憶體特性
- **順序存取**: 連續陣列讀取，預測性強
- **單次讀取**: 每個元素讀取一次，無重複
- **局部性強**: 空間和時間局部性均優
- **快取親善**: 適合現代CPU架構

### 注意事項
- **假設地址**: 本文所述的 0xA000, 0xB000 等為概念性假設，實際執行時地址由系統分配
- **編譯器最佳化**: 實際記憶體佈局可能由編譯器調整
- **平台差異**: 不同CPU/OS的快取行大小有所不同（通常 32-128 位元組）
