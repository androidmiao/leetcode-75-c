# LeetCode 2215 - 找出兩個陣列的差異（Find the Difference of Two Arrays）- 記憶體分析

## 問題概述
給定兩個整數陣列 `nums1` 和 `nums2`，求出只在 `nums1` 中出現的元素、和只在 `nums2` 中出現的元素。值的範圍為 [-1000, 1000]。

**函數簽名：**
```c
int** findDifference(int* nums1, int nums1Size, int* nums2, int nums2Size,
                     int* returnSize, int** returnColumnSizes)
```

---

### 索引映射規則
```
值的範圍: [-1000, 1000]
陣列索引: [0, 2000]
映射公式: index = value + 1000

例如:
  值 -1000 → index 0
  值    0  → index 1000
  值  1000 → index 2000
```

## 3. 演算法過程追蹤

### 範例 1: nums1 = [1,2,3], nums2 = [2,4,6]

#### 第一步：標記 nums1 元素
```
初始化: in1[2001] 全部設為 false
        in2[2001] 全部設為 false

Loop i = 0 to nums1Size-1:

i=0: nums1[0] = 1
     idx = 1 + 1000 = 1001
     in1[1001] ← true

i=1: nums1[1] = 2
     idx = 2 + 1000 = 1002
     in1[1002] ← true

i=2: nums1[2] = 3
     idx = 3 + 1000 = 1003
     in1[1003] ← true

結果: in1[1001]=T, in1[1002]=T, in1[1003]=T, 其他為 F
```

#### 第二步：標記 nums2 元素
```
Loop i = 0 to nums2Size-1:

i=0: nums2[0] = 2
     idx = 2 + 1000 = 1002
     in2[1002] ← true

i=1: nums2[1] = 4
     idx = 4 + 1000 = 1004
     in2[1004] ← true

i=2: nums2[2] = 6
     idx = 6 + 1000 = 1006
     in2[1006] ← true

結果: in2[1002]=T, in2[1004]=T, in2[1006]=T, 其他為 F
```

#### 第三步：尋找只在 nums1 的元素
```
count1 = 0
idx1 = 0
only1 = malloc(sizeof(int) * 2001)

Loop v = -1000 to 1000:

v=-1000: idx=0, in1[0]=F, in2[0]=F → continue
...
v=1:     idx=1001, in1[1001]=T && in2[1001]=F
         only1[0] = 1
         idx1 = 1, count1 = 1 ✓

v=2:     idx=1002, in1[1002]=T && in2[1002]=T → continue (在兩邊都有)

v=3:     idx=1003, in1[1003]=T && in2[1003]=F
         only1[1] = 3
         idx1 = 2, count1 = 2 ✓

v=4:     idx=1004, in1[1004]=F → continue
...
v=1000:  idx=2000, in1[2000]=F → continue

最終: only1 = [1, 3], count1 = 2
```

#### 第四步：尋找只在 nums2 的元素
```
count2 = 0
idx2 = 0
only2 = malloc(sizeof(int) * 2001)

Loop v = -1000 to 1000:

v=-1000: idx=0, in1[0]=F, in2[0]=F → continue
...
v=2:     idx=1002, in2[1002]=T && in1[1002]=T → continue (在兩邊都有)

v=4:     idx=1004, in2[1004]=T && in1[1004]=F
         only2[0] = 4
         idx2 = 1, count2 = 1 ✓

v=6:     idx=1006, in2[1006]=T && in1[1006]=F
         only2[1] = 6
         idx2 = 2, count2 = 2 ✓

...
v=1000:  idx=2000, in2[2000]=F → continue

最終: only2 = [4, 6], count2 = 2
```

#### 第五步：構建返回陣列
```
ans = malloc(sizeof(int*) * 2)

ans[0] = malloc(sizeof(int) * count1)
         = malloc(sizeof(int) * 2)
將 only1[0..1] 複製到 ans[0]
ans[0] = [1, 3]

ans[1] = malloc(sizeof(int) * count2)
         = malloc(sizeof(int) * 2)
將 only2[0..1] 複製到 ans[1]
ans[1] = [4, 6]

returnSize = 2
returnColumnSizes[0] = 2
returnColumnSizes[1] = 2

返回: ans = [[1, 3], [4, 6]] ✓
```

```
Loop i = 0 to 3:

i=0: nums1[0] = 1 → in1[1001] = true
i=1: nums1[1] = 2 → in1[1002] = true
i=2: nums1[2] = 3 → in1[1003] = true

i=2: nums2[2] = 2 → in2[1002] = true
i=3: nums2[3] = 2 → in2[1002] = true (已設，無變化)

結果: in2[1001]=T, in2[1002]=T
```

#### 第三步：尋找只在 nums1 的元素
```
count1 = 0
Loop v = -1000 to 1000:

v=1: idx=1001, in1[1001]=T && in2[1001]=T → continue (在兩邊都有)
v=2: idx=1002, in1[1002]=T && in2[1002]=T → continue (在兩邊都有)

最終: only2 = [], count2 = 0

ans = malloc(sizeof(int*) * 2)
ans[0] = malloc(sizeof(int) * 1) → [3]
ans[1] = malloc(sizeof(int) * 0) → []

returnColumnSizes[0] = 1
returnColumnSizes[1] = 0

返回: ans = [[3], []] ✓
```

## 5. 記憶體存取模式比較

### 本方案：布林陣列（Boolean Array）

**特性：**
```
┌──────────────────────────────────────┐
│     布林陣列方案（Boolean Array）     │
├──────────────────────────────────────┤
│ 棧使用: 8008 bytes (in1) + 8008 bytes (in2) = 16016 bytes
│ 堆使用: ~2001*4*2 + 16 bytes (only1, only2, ans)
│ 時間複雜度: O(n + m + 2001) = O(n + m)
│ 空間複雜度: O(2001) = O(1) 常數
│                                      │
│ 優點:                                │
│ ✓ 連續記憶體存取（Sequential Access) │
│ ✓ 高快取命中率（Cache Friendly）     │
│ ✓ 無碰撞（No Collisions）            │
│ ✓ 預測性記憶體模式                   │
│ ✓ 恆定額外空間 O(1)                 │
│                                      │
│ 缺點:                                │
│ ✗ 對於大值域效率低（fixed 2001 size)│
│ ✗ 棧空間較大（16KB for arrays）     │
│ ✗ 值必須在範圍內（[-1000, 1000]）   │
└──────────────────────────────────────┘

記憶體存取模式:
時間: 0     100   200   300   400   500   600
     ├─────┼─────┼─────┼─────┼─────┼─────┤
標記: │ ▮ ▮ │ ▮▮▮ │ ▮   │     │ ▮▮▮ │     │ (稀疏存取 → 密集存取)
掃描: │ ▯▯▯ │ ▯▯▯ │ ▯▯▯ │ ▯▯▯ │ ▯▯▯ │ ▯▯▯ │ (逐個掃描 → 完全線性)
     └─────┴─────┴─────┴─────┴─────┴─────┘
     ▮ = 隨機存取   ▯ = 順序存取
```

### Approach 2: 雜湊集合（Hash Set）

**特性：**
```
┌──────────────────────────────────────┐
│      雜湊集合方案（Hash Set）         │
├──────────────────────────────────────┤
│ 堆使用: capacity * sizeof(entry) bytes
│        capacity = next_pow2(n*2+1)
│        通常 capacity >= 16
│ 時間複雜度: O(n + m + max(n,m)) 平均
│ 空間複雜度: O(n) 或 O(m)
│                                      │
│ 優點:                                │
│ ✓ 適應任意值域                        │
│ ✓ 空間與資料量成正比                 │
│ ✓ 存在性檢查 O(1) 平均               │
│                                      │
│ 缺點:                                │
│ ✗ 雜湊函數計算開銷                   │
│ ✗ 開放尋址法中的碰撞                 │
│ ✗ 快取未命中（Cache Misses）        │
│ ✗ 不規則記憶體跳躍                   │
│ ✗ 堆分配/釋放開銷                   │
└──────────────────────────────────────┘

記憶體存取模式:
時間: 0     100   200   300   400   500   600
     ├─────┼─────┼─────┼─────┼─────┼─────┤
標記: │ ▲   │ ▲▲▲ │ ▲   │ ▲▲  │ ▲   │     │ (隨機跳躍 + 碰撞探測)
掃描: │ ▲▲▲ │ ▲▲▲ │ ▲▲▲ │ ▲▲▲ │ ▲   │     │ (仍為隨機存取)
     └─────┴─────┴─────┴─────┴─────┴─────┘
     ▲ = 隨機存取 (非連續)
```

### Approach 1: 暴力法（Brute Force）

**特性：**
```
┌──────────────────────────────────────┐
│        暴力法（Brute Force）          │
├──────────────────────────────────────┤
│ 堆使用: 0 bytes (無額外空間)
│ 時間複雜度: O(n*m) 最壞情況
│ 空間複雜度: O(1) 不計輸出
│                                      │
│ 優點:                                │
│ ✓ 無額外記憶體分配                   │
│ ✓ 實現簡單                           │
│                                      │
│ 缺點:                                │
│ ✗ O(n*m) 時間複雜度 (3*3=9 vs O(n+m)=6)
│ ✗ 重複掃描陣列多次                   │
│ ✗ 快取效率極低（循環嵌套）          │

─────────────────────────────────────

─────────────────────────────────────

```

## 6. 棧 vs 堆記憶體對比

### 記憶體配置對比

```
┌───────────────────────────────────────────────────────────────┐
│                  棧 (Stack) vs 堆 (Heap) 對比                   │
├───────────────────────────────────────────────────────────────┤
│                                                                 │
│  棧 (Stack) - 線性增長                                          │
│  ┌─────────────────────────────────────────────────┐           │
│  │ 0x7FF0_0000 │ in1[2001]    8008 bytes        │ │           │
│  │ 0x7FF0_1F48 │ in2[2001]    8008 bytes        │ │ esp ─┐   │
│  │ 0x7FF0_3E90 │ locals, ptrs   ~100 bytes      │ │     │   │
│  │ 0x7FF0_3EF4 │ 返回位址         8 bytes       │ │     │   │
│  └─────────────────────────────────────────────────┘ │   │   │
│                                                      │   │   │
│                            棧使用: 16,124 bytes    ◄───┘   │
│                            (16.1 KB)                       │
│                                                            │
│  堆 (Heap) - 動態配置                                      │
│  ┌─────────────────────────────────────────────────┐       │
│  │ 0xA000_0000 │ [未使用]                         │       │
│  │ 0xA000_0008 │ ans (int** → 8 bytes)            │       │
│  │ 0xA000_0010 │ [padding]                        │       │
│  │ 0xA000_2000 │ only1 (int[2001] → 8004 bytes)  │       │
│  │ 0xA000_4000 │ only2 (int[2001] → 8004 bytes)  │       │
│  │ 0xA000_6000 │ ans[0] (int[count1] → 8 bytes)  │       │
│  │ 0xA000_6008 │ ans[1] (int[count2] → 8 bytes)  │       │
│  │ 0xA000_6010 │ [空閒空間]                       │       │
│  └─────────────────────────────────────────────────┘       │
│                                                            │
│                堆使用: ~16,032 bytes (最大)                │
│                (15.6 KB)                                  │
│                但根據 count1/count2 變動                   │
│                                                            │
└───────────────────────────────────────────────────────────────┘

成本分析:

布林陣列方案:
  ┌──────────────────────────────────────────┐
  │ 棧      16,124 bytes  ← 函數內生命週期    │
  │ 堆      ~8,024 bytes  ← 函數返回後存在   │
  │ 總計    ~24,148 bytes                    │
  │                                          │
  │ 棧分配: 一次 push (O(1))                 │
  │ 堆分配: 4 次 malloc (O(n))               │
  │ 釋放:   僅 only1, only2 (在函數內)      │
  │       ans, ans[0], ans[1] 返回給呼叫者   │
  └──────────────────────────────────────────┘

雜湊集方案:
  ┌──────────────────────────────────────────┐
  │ 棧      ~200 bytes                        │
  │ 堆      ~16,000 bytes (capacity ~ 2^14)  │
  │ 總計    ~16,200 bytes                    │
  │                                          │
  │ 棧分配: 一次 push                        │
  │ 堆分配: 2-3 次 malloc                    │
  │ 釋放:   hash set + ans 系列              │
  └──────────────────────────────────────────┘

成本總結:
  布林陣列: 棧使用大，但快取友善，無碰撞
  雜湊集:   棧使用小，但記憶體跳躍，碰撞風險

  對於本題 (範圍固定 [-1000,1000]), 布林陣列優越。
```

### 記憶體初始化成本

```
初始化比較 (n=1000, m=1000):

布林陣列:
  memset(in1, 0, 2001) → 2001 次寫入 (連續)
  memset(in2, 0, 2001) → 2001 次寫入 (連續)
  效率: 快 (單次初始化，高快取命中)
  成本: ~4002 CPU 週期

雜湊集:
  calloc(capacity, sizeof(entry)) → 分配並初始化
  capacity = next_pow2(n*2+1) ≈ 2048
  每個 entry ~32 bytes (key + value + metadata)
  初始化: 2048 * 32 = 65,536 bytes (~16K 快取 miss)
  成本: ~65,000 CPU 週期

暴力法:
  無初始化
  成本: 0

結論: 布林陣列的初始化成本被掃描效率大幅補償。
```

## 附錄：記憶體假設說明

本文件中的十六進制地址均為**假設值**，用於示例和說明。實際執行時的地址取決於：

1. **棧位置** — 由作業系統決定，通常在 0x7FFF_XXXX (Linux) 或 0x0000_XXXX (嵌入式)
2. **堆位置** — 由 malloc 的實現決定，通常在 0xXXXX_0000 以上
3. **地址空間配置隨機化 (ASLR)** — 現代系統啟用時，每次執行地址不同
4. **編譯器最佳化** — 可能改變實際記憶體佈局

**重點是相對位置和存取模式，而非絕對地址。**

### 時間複雜度分析
```
標記 nums1:         O(nums1Size)
標記 nums2:         O(nums2Size)

──────────────────────────────────

```

### 空間複雜度分析
```
in1, in2:           O(2001) = O(1)
only1, only2:       O(n + m) (臨時)
ans 結構:           O(1) (固定 2 個指標)
ans[0], ans[1]:     O(count1 + count2) ≤ O(n+m)
──────────────────────────────────
總計:               O(1) + O(n+m) = O(n+m) (含輸出)
                    O(1) (不計輸出)
```
