# LeetCode 2130 - 連結串列最大孿生和 記憶體配置圖解

**問題：** 給定一個連結串列，計算最大孿生和（第 i 個節點和第 n-1-i 個節點的值之和的最大值）
**解法：** 使用 reverseList() + 快慢指標 + 孿生比較

---

## 1. Struct 記憶體佈局

### ListNode 結構定義
```c
struct ListNode {
    int val;              // 4 bytes, offset 0
    struct ListNode *next; // 8 bytes, offset 8 (64-bit system)
};
```

### 64位系統中的記憶體配置
- `val` (int)：4 位元組，偏移量 0
- **填充** (padding)：4 位元組，偏移量 4
- `next` (pointer)：8 位元組，偏移量 8
- **總計**：16 位元組（包括對齐填充）

### 單一節點記憶體圖（假設位址 0xA000）
```
┌──────────────────────────────────┐
│ Address: 0xA000                  │
├──────────────────────────────────┤
│ 0xA000: val = 5       (int)      │ ◄─── 4 bytes
│ 0xA004: [padding]     (4 bytes)  │ ◄─── alignment
│ 0xA008: next = 0xB000 (ptr)      │ ◄─── 8 bytes
│ 0xA010: (next node)              │
└──────────────────────────────────┘
```

記憶體視圖（逐位元組）：
```
Offset  |  Content (Hex)
--------|────────────────
0xA000  |  0x00000005  (val=5)
0xA004  |  0x00000000  (padding)
0xA008  |  0x0000B000  (next pointer)
0xA010  |  [next node starts here]
```

## 3. 陣列索引版對應

### 等價陣列表示
```
┌───────┬────┬────┬────┬────┐
│ Index │ 0  │ 1  │ 2  │ 3  │
├───────┼────┼────┼────┼────┤
│ Value │ 5  │ 4  │ 2  │ 1  │
└───────┴────┴────┴────┴────┘
```

### 孿生配對規則
**公式：** `twin(i) = n - 1 - i`，其中 n = 4

| 索引 i | 孿生索引 twin(i) | 節點值 arr[i] | 孿生值 arr[twin(i)] | 孿生和 |
|--------|-----------------|---------------|--------------------|--------|
| 0      | 3               | 5             | 1                  | 6      |
| 1      | 2               | 4             | 2                  | 6      |
| 2      | 1               | 2             | 4                  | 6 (重複) |
| 3      | 0               | 1             | 5                  | 6 (重複) |

**最大孿生和：** 6

## 4. 搜尋／操作過程追蹤

### Example 1: [5, 4, 2, 1] → 預期輸出 6

#### Phase 1: 快慢指標尋找中點

**初始狀態：**
```
head: 0xA000 (val=5)
slow: 0xA000 (val=5)  ◄─── 初始指向 head
fast: 0xA000 (val=5)  ◄─── 初始指向 head
```

**執行邏輯：**
```c
while (fast && fast->next) {
    slow = slow->next;
    fast = fast->next->next;
}
```

**追蹤步驟：**

```
【Step 0】初始：
┌─────────────────────┬─────────────────────┐
│ slow: 0xA000 [val=5]│ fast: 0xA000 [val=5]│
│     ▼               │      ▼              │
│ [5|→0xB000]         │ [5|→0xB000]         │
│  0xA000             │  0xA000             │
└─────────────────────┴─────────────────────┘
檢查條件：fast(0xA000) && fast->next(0xB000) ? ✓ true
  ├─ 執行：slow = slow->next = 0xB000
  └─ 執行：fast = fast->next->next = 0xC000

【Step 1】迭代 1：
┌─────────────────────┬─────────────────────┐
│ slow: 0xB000 [val=4]│ fast: 0xC000 [val=2]│
│     ▼               │      ▼              │
│ [4|→0xC000]         │ [2|→0xD000]         │
│  0xB000             │  0xC000             │
└─────────────────────┴─────────────────────┘
檢查條件：fast(0xC000) && fast->next(0xD000) ? ✓ true
  ├─ 執行：slow = slow->next = 0xC000
  └─ 執行：fast = fast->next->next = NULL

【Step 2】迭代 2：
┌─────────────────────┬─────────────────────┐
│ slow: 0xC000 [val=2]│ fast: NULL          │
│     ▼               │                     │
│ [2|→0xD000]         │ ✗ (指標為空)        │
│  0xC000             │                     │
└─────────────────────┴─────────────────────┘
檢查條件：fast(NULL) ? ✗ false → 迴圈終止

【結果】慢指標停在中點：slow = 0xC000 (val=2)
```

**中點分析：**
- 偶數長度串列 (n=4) 時，slow 指向第 n/2 = 2 個節點（0 索引起）
- slow 位置：第 2 個節點（val=2）
- 第二半部分的開始：slow->next = 0xD000 (val=1)

#### Phase 2: 反轉第二半部分

**呼叫：** `secondHalf = reverseList(slow->next)` 其中 `slow->next = 0xC000`

實際呼叫：`secondHalf = reverseList(0xC000)` （因為 slow 已在 0xC000）

**反轉前狀態：**
```
第二半：[2|→0xD000] ──▶ [1|NULL]
        0xC000           0xD000
```

**reverseList() 逐步執行（假設反轉整個剩餘鏈）：**

```
【Step 0】reverseList(0xC000) 初始：
變數初始化：
  prev = NULL
  curr = 0xC000 (val=2)
  next_temp = undefined

狀態：
[2|→0xD000]   [1|NULL]
 0xC000        0xD000

【Step 1】迭代 1（curr = 0xC000, val=2）：
動作：
  1. next_temp = curr->next = 0xD000  (保存下一個)
  2. curr->next = prev = NULL         (反轉指標)
  3. prev = curr = 0xC000             (前進)
  4. curr = next_temp = 0xD000        (前進)

記憶體變化：
  ┌─ 0xC008 欄位由 0xD000 改為 0x0000 ─┐
  ▼                                      ▼
[2|0x0]   ◄──  [1|→NULL]
0xC000         0xD000

狀態視圖：
prev = 0xC000 (val=2)
curr = 0xD000 (val=1)

【Step 2】迭代 2（curr = 0xD000, val=1）：
動作：
  1. next_temp = curr->next = NULL     (保存下一個)
  2. curr->next = prev = 0xC000        (反轉指標)
  3. prev = curr = 0xD000              (前進)
  4. curr = next_temp = NULL           (前進)

記憶體變化：
  ┌─ 0xD008 欄位由 NULL 改為 0xC000 ─┐
  ▼                                     ▼
[2|0x0]   ◄──  [1|→0xC000]
0xC000         0xD000

狀態視圖：
prev = 0xD000 (val=1)
curr = NULL

【Step 3】檢查條件：curr == NULL ? ✓ yes → 迴圈終止
傳回：curr 前的節點實際應傳回 prev = 0xD000
```

**反轉後狀態：**
```
新的第二半鏈：
[1|→0xC000]  ──▶  [2|NULL]
 0xD000             0xC000

原始頭部鏈（未改變）：
[5|→0xB000]  ──▶  [4|→NULL]
 0xA000             0xB000
```

**內存狀態比較：**

| 地址 | 欄位 | 反轉前 | 反轉後 |
|------|------|--------|--------|
| 0xC008 (Node 2 next) | next | 0xD000 | 0x0000 (NULL) |
| 0xD008 (Node 3 next) | next | 0x0000 (NULL) | 0xC000 |

#### Phase 3: 孿生配對比較

**設定指標：**
```
p1 = head = 0xA000 (第一半開始)
p2 = secondHalf = 0xD000 (反轉後第二半開始)
maxSum = 0
```

**逐步比較：**

```
【步驟 1】
p1 = 0xA000 (val=5)
p2 = 0xD000 (val=1)
孿生和 = 5 + 1 = 6
maxSum = max(0, 6) = 6
前進：p1 = p1->next = 0xB000, p2 = p2->next = 0xC000

【步驟 2】
p1 = 0xB000 (val=4)
p2 = 0xC000 (val=2)
孿生和 = 4 + 2 = 6
maxSum = max(6, 6) = 6
前進：p1 = p1->next = 0xC000, p2 = p2->next = NULL

【步驟 3】
p1 = 0xC000 (val=2)
p2 = NULL
檢查終止條件：p2 == NULL ? ✓ yes → 停止迴圈

傳回：maxSum = 6 ✓
```

**比較過程視覺化：**
```
配對 1：        配對 2：
[5]  +  [1]   [4]  +  [2]
0xA000  0xD000 0xB000 0xC000
  │       │      │       │
  └───6───┘      └───6───┘

最大孿生和 = 6
```

### Example 2: [4, 2, 2, 3] → 預期輸出 7

#### 記憶體配置
```
Node 0 (val=4): 0xA000, next → 0xB000
Node 1 (val=2): 0xB000, next → 0xC000
Node 2 (val=2): 0xC000, next → 0xD000
Node 3 (val=3): 0xD000, next → NULL
```

#### 快慢指標追蹤
```
【Step 0】初始
slow=0xA000, fast=0xA000

【Step 1】迭代 1：fast && fast->next ✓
slow→0xB000, fast→0xC000

【Step 2】迭代 2：fast && fast->next ✓
slow→0xC000, fast→NULL

檢查條件：fast(NULL) ? ✗ → 終止
結果：slow = 0xC000 (val=2)
```

#### 反轉第二半 [2, 3]
```
反轉前：[2|→0xD000] ──▶ [3|NULL]  (0xC000→0xD000)
反轉後：[3|→0xC000] ◄── [2|NULL]  (0xD000→0xC000)
```

#### 孿生比較
```
【步驟 1】
val(0xA000)=4 + val(0xD000)=3 = 7 ✓
maxSum = 7

【步驟 2】
val(0xB000)=2 + val(0xC000)=2 = 4
maxSum = max(7, 4) = 7

【步驟 3】p2=NULL → 終止

傳回：maxSum = 7 ✓
```

## 6. 記憶體存取模式比較

### 連結串列 vs 陣列 的記憶體特性

#### 1. 快取表現（Cache Behavior）

**連結串列（鏈式存取）：**
```
存取序列：
0xA000 ──▶ 0xB000 ──▶ 0xC000 ──▶ 0xD000
  │          │          │          │
  └─▶ 跳躍式 ◀──────────────────────┘
      記憶體位置不連續

快取效能：
┌─────────────────────┬──────────┬──────────┐
│ 存取方式            │ L1快取率 │ 效率評級 │
├─────────────────────┼──────────┼──────────┤
│ 連結串列指標追蹤    │ 較低 ~40%│ 低       │
│ 隨機位址跳躍        │ 低 ~20%  │ 低 ◄──┐  │
│ 快取缺失           │ 高 ~60%  │ 差  │  │
└─────────────────────┴──────────┴──────────┘
         ┌─────────────────────────────┘
         └─ 原因：無時間/空間局部性
```

**陣列（連續存取）：**
```
存取序列：
arr[0] → arr[1] → arr[2] → arr[3]
0xA000   0xA010   0xA020   0xA030
  │       │       │       │
  └──────── 連續線性記憶體空間 ─────────┘

快取效能：
┌─────────────────────┬──────────┬──────────┐
│ 存取方式            │ L1快取率 │ 效率評級 │
├─────────────────────┼──────────┼──────────┤
│ 順序讀取（順向）    │ 高 ~95%  │ 優       │
│ 預取命中            │ 高 ~90%  │ 優 ◄──┐  │
│ 快取缺失           │ 低 ~5%   │ 優  │  │
└─────────────────────┴──────────┴──────────┘
         ┌─────────────────────────────┘
         └─ 原因：空間局部性強
```

#### 2. 空間開銷（Space Overhead）

**陣列：**
```
每個元素佔用：
┌────────────┐
│ val (4B)   │ ◄─── 純資料，無額外開銷
└────────────┘
4 位元組/元素

節點 4 個時總大小：4 × 4 = 16 位元組
```

**連結串列：**
```
每個節點佔用：
┌────────────┐
│ val (4B)   │ ◄─── 資料
├────────────┤
│ padding(4B)│ ◄─── 結構對齐開銷
├────────────┤
│ next (8B)  │ ◄─── 指標開銷 ◄─── 2 倍於陣列！
└────────────┘
16 位元組/節點

節點 4 個時總大小：4 × 16 = 64 位元組
記憶體開銷率：(64-16)/16 = 300% 額外
```

**比較表：**
| 資料結構 | 單位大小 | n=4 總大小 | 開銷 |
|---------|---------|-----------|------|
| 陣列 | 4B | 16B | 0 |
| 連結串列 | 16B | 64B | +300% |

#### 3. 反轉操作的複雜性

**陣列反轉：**
```
操作：使用索引運算，無需修改資料
┌──────┬──────┬──────┬──────┐
│ [0]  │ [1]  │ [2]  │ [3]  │  原始
│  5   │  4   │  2   │  1   │
└──────┴──────┴──────┴──────┘
 i=0    i=1    i=2    i=3

邏輯反轉（透過索引計算，不移動資料）：
temp = arr[0]; arr[0] = arr[3]; arr[3] = temp; ...
┌──────┬──────┬──────┬──────┐
│ [3]  │ [2]  │ [1]  │ [0]  │  邏輯反轉視圖
│  1   │  2   │  4   │  5   │  (無記憶體修改)
└──────┴──────┴──────┴──────┘
 i=0    i=1    i=2    i=3

記憶體改動：0 個指標改動（純邏輯）
```

**連結串列反轉：**
```
操作：必須物理改變指標，修改記憶體

反轉前：
[2|→0xD000]  →  [1|NULL]
 0xC000          0xD000

逐步指標改動：
指標欄位 0xC008：0xD000 → NULL (改動 1)
指標欄位 0xD008：NULL → 0xC000 (改動 2)

反轉後：
[2|NULL]  ←  [1|→0xC000]
0xC000       0xD000

記憶體改動：2 個指標欄位被重寫
```

#### 4. 插入/刪除操作對比

**陣列：** O(n) 複雜度
```
插入位置 arr[1]：
┌────┬────┬────┬────┐
│ 5  │ 4  │ 2  │ 1  │ 原始
└────┴────┴────┴────┘

需要移位：
  ↓ ↓ ↓
┌────┬────┬────┬────┬────┐
│ 5  │ X  │ 4  │ 2  │ 1  │ 插入後 (需移動 3 個元素)
└────┴────┴────┴────┴────┘

記憶體複製：n-1 次 (大量資料移動)
```

**連結串列：** O(1) 複雜度
```
插入位置 Node 1 後：
[5]→[4]→[2]→[1]

插入 Node X 後：
[5]→[X]→[4]→[2]→[1]
   └─只改 2 個指標欄位
    (Node 5 的 next, 新 Node 的 next)

記憶體改動：2 個指標欄位改寫 ✓
無資料移動
```

#### 5. 存取時間複雜度

## 7. 指標反轉的記憶體狀態變化

### reverseList() 函式的完整記憶體追蹤

#### 函式定義
```c
struct ListNode* reverseList(struct ListNode* head) {
    struct ListNode *prev = NULL;
    struct ListNode *curr = head;

    while (curr) {
        struct ListNode *next_temp = curr->next;  // 【Line 1】
        curr->next = prev;                         // 【Line 2】
        prev = curr;                               // 【Line 3】
        curr = next_temp;                          // 【Line 4】
    }
    return prev;
}
```

## 8. 演算法複雜度分析（含記憶體視角）

### 時間複雜度

```
┌──────────────────────────────────────────┐
│ Phase 1: 快慢指標尋找中點                 │
├──────────────────────────────────────────┤
│ while (fast && fast->next) {             │
│    slow = slow->next;    ──▶ O(1)        │
│    fast = fast->next->next; ▶ O(1)        │
│ }                                        │
│ 迴圈次數：⌈n/2⌉ 次                       │
│ 小計：O(n)                               │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│ Phase 2: 反轉第二半                      │
├──────────────────────────────────────────┤
│ reverseList(slow->next) {                │
│    while (curr) {                        │
│        next_temp = curr->next;  ──▶ O(1) │
│        curr->next = prev;       ──▶ O(1) │
│        prev = curr;              ▶ O(1) │
│        curr = next_temp;        ──▶ O(1) │
│    }                                     │
│ }                                        │
│ 迴圈次數：⌊n/2⌋ 次                       │
│ 小計：O(n)                               │
└──────────────────────────────────────────┘

┌──────────────────────────────────────────┐
│ Phase 3: 孿生配對比較                     │
├──────────────────────────────────────────┤
│ while (p2) {                             │
│    maxSum = max(maxSum, p1->val + p2->val)│
│    p1 = p1->next;               ──▶ O(1) │
│    p2 = p2->next;               ──▶ O(1) │
│ }                                        │
│ 迴圈次數：⌊n/2⌋ 次                       │
│ 小計：O(n)                               │
└──────────────────────────────────────────┘

【總時間複雜度】
  O(n) + O(n) + O(n) = O(3n) = O(n) ✓

  n=4 時實際操作數：
    Phase 1: 2 次迴圈 × 2 個操作 = 4 次指標移動
    Phase 2: 2 次迴圈 × 4 個操作 = 8 次指標操作
    Phase 3: 2 次迴圈 × 3 個操作 = 6 次指標操作
    ─────────────────────────────────
    總計：18 個基本操作
```

## 資料注記

- 所有記憶體位址均為 **假設位址**，僅用於圖解說明
- 實際執行時位址由作業系統決定
- 結構體填充（padding）取決於編譯器和目標平台
- 64 位元系統上指標大小統一為 8 位元組
- 反轉操作不複製節點資料，僅改寫指標欄位
