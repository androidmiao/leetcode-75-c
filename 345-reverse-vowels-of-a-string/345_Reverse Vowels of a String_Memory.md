# 345. Reverse Vowels of a String — 記憶體位址 / 陣列索引圖解

> 以下所有十六進位位址皆為假設值，用於說明指標操作過程。

## 1. 字串記憶體佈局

`char* str` 指向 malloc 配置的連續記憶體區塊。以 `s = "IceCreAm"` 為例（長度 8 + '\0' = 9 bytes）：

```
假設 str = 0xA000

位址        內容      索引
┌─────────┬──────┬──────┐
│ 0xA000  │  'I' │  [0] │  ← s_lptr 初始位置
├─────────┼──────┼──────┤
│ 0xA001  │  'c' │  [1] │
├─────────┼──────┼──────┤
│ 0xA002  │  'e' │  [2] │
├─────────┼──────┼──────┤
│ 0xA003  │  'C' │  [3] │
├─────────┼──────┼──────┤
│ 0xA004  │  'r' │  [4] │
├─────────┼──────┼──────┤
│ 0xA005  │  'e' │  [5] │
├─────────┼──────┼──────┤
│ 0xA006  │  'A' │  [6] │
├─────────┼──────┼──────┤
│ 0xA007  │  'm' │  [7] │  ← s_rptr 初始位置
├─────────┼──────┼──────┤
│ 0xA008  │ '\0' │  [8] │  ← null terminator
└─────────┴──────┴──────┘
```

### 指標變數本身的記憶體

```
變數名稱      儲存位置(假設)    值(位址)      解引用值
┌────────────┬──────────────┬────────────┬──────────┐
│ str        │ 0xF000       │ 0xA000     │ 'I'      │
├────────────┼──────────────┼────────────┼──────────┤
│ s_lptr     │ 0xF008       │ 0xA000     │ 'I'      │
├────────────┼──────────────┼────────────┼──────────┤
│ s_rptr     │ 0xF010       │ 0xA007     │ 'm'      │
└────────────┴──────────────┴────────────┴──────────┘
```

## 2. 搜尋 / 操作過程追蹤

### Example 1: `s = "IceCreAm"`

```
初始狀態:
  s_lptr = 0xA000  s_rptr = 0xA007
  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
  │  I  │  c  │  e  │  C  │  r  │  e  │  A  │  m  │
  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
    ▲L                                          ▲R
```

**步驟 1：左指標掃描**
```
  s_lptr = 0xA000, *s_lptr = 'I' → isVowel('I') = true ✓
  → 左指標已在母音上，開始從右邊找母音
```

**步驟 1：右指標掃描**
```
  s_rptr = 0xA007, *s_rptr = 'm' → isVowel('m') = false
  s_rptr-- → s_rptr = 0xA006, *s_rptr = 'A' → isVowel('A') = true ✓
```

**步驟 1：交換**
```
  swap_xor(0xA000, 0xA006)
  XOR 過程:
    *0xA000 = 'I'(0x49) ^ 'A'(0x41) = 0x08
    *0xA006 = 0x08 ^ 'A'(0x41) = 'I'(0x49)
    *0xA000 = 0x08 ^ 'I'(0x49) = 'A'(0x41)

  s_rptr-- → 0xA005    s_lptr++ → 0xA001

  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
  │  A  │  c  │  e  │  C  │  r  │  e  │  I  │  m  │
  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
           ▲L                      ▲R
```

**步驟 2：左指標掃描**
```
  s_lptr = 0xA001, *s_lptr = 'c' → isVowel('c') = false → s_lptr++
  s_lptr = 0xA002, *s_lptr = 'e' → isVowel('e') = true ✓
```

**步驟 2：右指標掃描**
```
  s_rptr = 0xA005, *s_rptr = 'e' → isVowel('e') = true ✓
```

**步驟 2：交換**
```
  swap_xor(0xA002, 0xA005)
  'e'(0x65) ^ 'e'(0x65) = 0x00... 但 0xA002 ≠ 0xA005 所以不走 early return
  XOR 過程:
    *0xA002 = 0x65 ^ 0x65 = 0x00
    *0xA005 = 0x00 ^ 0x65 = 0x65 ('e')
    *0xA002 = 0x00 ^ 0x65 = 0x65 ('e')
  → 值相同，交換後結果不變

  s_rptr-- → 0xA004    s_lptr++ → 0xA003

  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
  │  A  │  c  │  e  │  C  │  r  │  e  │  I  │  m  │
  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
                       ▲L    ▲R
```

**步驟 3：左指標掃描**
```
  s_lptr = 0xA003, *s_lptr = 'C' → isVowel('C') = false → s_lptr++
  s_lptr = 0xA004, *s_lptr = 'r' → isVowel('r') = false → s_lptr++
  s_lptr = 0xA005 ≥ s_rptr = 0xA004 → 外迴圈條件 s_lptr < s_rptr 不成立 → 結束
```

**最終結果：** `"AceCreIm"`

### Example 2: `s = "leetcode"`

```
假設 str = 0xB000

初始:
  s_lptr = 0xB000  s_rptr = 0xB007
  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
  │  l  │  e  │  e  │  t  │  c  │  o  │  d  │  e  │
  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
    ▲L                                          ▲R
```

**步驟 1：**
```
  L: 0xB000 'l' → 不是母音 → L++
  L: 0xB001 'e' → 母音 ✓
  R: 0xB007 'e' → 母音 ✓
  swap(0xB001, 0xB007): 'e' ↔ 'e' → 值不變
  R-- → 0xB006, L++ → 0xB002

  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
  │  l  │  e  │  e  │  t  │  c  │  o  │  d  │  e  │
  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
                 ▲L                      ▲R
```

**步驟 2：**
```
  L: 0xB002 'e' → 母音 ✓
  R: 0xB006 'd' → 不是母音 → R--
  R: 0xB005 'o' → 母音 ✓
  swap(0xB002, 0xB005): 'e' ↔ 'o'
  R-- → 0xB004, L++ → 0xB003

  ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
  │  l  │  e  │  o  │  t  │  c  │  e  │  d  │  e  │
  └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
                       ▲L    ▲R
```

**步驟 3：**
```
  L: 0xB003 't' → 不是母音 → L++
  L: 0xB004 'c' → 不是母音 → L++
  L = 0xB005 ≥ R = 0xB004 → 結束
```

**最終結果：** `"leotcede"`

## 4. 記憶體存取模式分析

### 字串的連續記憶體特性

字串在記憶體中是連續的 `char` 陣列，這對 cache 非常友好：

```
Cache line (假設 64 bytes):
┌──────────────────────────────────────────────────────────────────┐
│ 0xA000: I  c  e  C  r  e  A  m \0  ...  (剩餘 55 bytes 其他資料) │
└──────────────────────────────────────────────────────────────────┘
```

整個字串（9 bytes）很可能在同一條 cache line 中，因此：
- 左指標的順序存取 → **cache 命中率極高**
- 右指標的逆序存取 → **同一條 cache line，同樣命中率極高**
- 這也是為什麼雙指標法對字串問題非常高效的原因之一

### malloc 開銷分析

```
malloc(strlen(s) + 1) 的實際記憶體佈局:

                    malloc metadata
                    ┌─────────────────┐
                    │ chunk size      │  ← 8-16 bytes（平台相關）
                    │ flags           │
                    ├─────────────────┤
  str ───────────▶  │ I c e C r e A m │  ← 使用者可見的 9 bytes
                    │ \0              │
                    ├─────────────────┤
                    │ alignment pad   │  ← 對齊填充
                    └─────────────────┘

實際消耗 ≈ 9 bytes + 16~32 bytes metadata = 25~41 bytes
```

若改為原地修改（in-place），可省去這些額外開銷。

## 5. XOR 交換的位元層級追蹤

以交換 'I'(0x49) 和 'A'(0x41) 為例：

```
步驟 1: *a = *a ^ *b
  0x49 = 0100 1001
  0x41 = 0100 0001
  XOR  = 0000 1000 = 0x08
  → *a = 0x08

步驟 2: *b = *a ^ *b
  0x08 = 0000 1000
  0x41 = 0100 0001
  XOR  = 0100 1001 = 0x49 = 'I'
  → *b = 'I' ✓（原來的 *a 值）

步驟 3: *a = *a ^ *b
  0x08 = 0000 1000
  0x49 = 0100 1001
  XOR  = 0100 0001 = 0x41 = 'A'
  → *a = 'A' ✓（原來的 *b 值）
```

### 危險情境：當 *a == *b 且 a != b

以交換兩個 'e'(0x65) 為例（位址不同但值相同）：

```
步驟 1: *a = 0x65 ^ 0x65 = 0x00  ← 暫時歸零！
步驟 2: *b = 0x00 ^ 0x65 = 0x65  ← 正確還原
步驟 3: *a = 0x00 ^ 0x65 = 0x65  ← 正確還原

→ 最終結果正確，但中間狀態 *a 為 0（在多線程環境可能被觀察到）
```
