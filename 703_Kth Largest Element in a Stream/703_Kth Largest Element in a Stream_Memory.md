# 703. Kth Largest Element in a Stream — 記憶體分析

## 1. Struct 記憶體佈局

### KthLargest 結構定義
```c
typedef struct {
    int* heap;      // pointer to heap array (8 bytes, 64-bit)
    int size;       // current number of elements (4 bytes)
    int capacity;   // max capacity = k+1 (4 bytes)
    int k;          // the k value (4 bytes)
} KthLargest;     // Total: 24 bytes (with alignment padding)
```

假設 KthLargest 結構位於 **0xA000**，heap 陣列位於 **0xB000**，k=3（capacity=4）：

## 3. 陣列索引版對應關係

### 堆積樹的索引公式

```
┌─────────────────────────────────────────────┐
│ 節點 i 的親子關係                           │
├─────────────────────────────────────────────┤
│ 父節點: parent(i) = (i-1) / 2               │
│        (整數除法)                           │
│                                              │
│ 左子節點: left(i) = 2*i + 1                 │
│                                              │
│ 右子節點: right(i) = 2*i + 2                │
└─────────────────────────────────────────────┘
```

### k=3 堆積的索引對應表

```
┌─────────┬───────────────────────┬──────────────────┐
│ Index i │ Parent Index          │ Children Indices │
├─────────┼───────────────────────┼──────────────────┤
│ 0       │ —（root）             │ left: 1, right:2 │
├─────────┼───────────────────────┼──────────────────┤
│ 1       │ (1-1)/2 = 0           │ left: 3, right:4 │
│         │                       │ (3 exists, 4不存 │
├─────────┼───────────────────────┼──────────────────┤
│ 2       │ (2-1)/2 = 0           │ left: 5, right:6 │
│         │                       │ (都不存在)        │
├─────────┼───────────────────────┼──────────────────┤
│ 3       │ (3-1)/2 = 1           │ left: 7, right:8 │
│         │ (不存在)              │ (都不存在)        │
└─────────┴───────────────────────┴──────────────────┘
```

## 4. 搜尋／操作過程追蹤

### Initialization Phase: k=3, nums=[4,5,8,2]

#### Step 1: 插入 4
```
操作: Insert 4
┌──────────────────────┐
│ size: 0 → 1          │
│ size < k, 直接插入   │
└──────────────────────┘

heap[0] = 4, size = 1

Heap 狀態：
┌──────┬──────┬──────┬──────┐
│ [4]  │ [ ]  │ [ ]  │ [ ]  │
└──────┴──────┴──────┴──────┘
 i=0

樹狀圖：
    4
```

#### Step 2: 插入 5
```
操作: Insert 5
┌──────────────────────────────────┐
│ size: 1 → 2                      │
│ size < k, 直接插入至末尾         │
│ heap[1] = 5                      │
└──────────────────────────────────┘

插入前：heap = [4, ?, ?, ?]
插入後：heap = [4, 5, ?, ?]
調用 heapifyUp(1):
  ├─ index = 1, parent = (1-1)/2 = 0
  ├─ heap[1]=5 vs heap[0]=4
  ├─ 5 > 4 ✓ (min-heap，不需交換)
  └─ 停止

最終 heap = [4, 5, ?, ?], size = 2

樹狀圖：
      4 (0xB000)
     /
    5 (0xB004)
```

#### Step 3: 插入 8
```
操作: Insert 8
┌──────────────────────────────────┐
│ size: 2 → 3                      │
│ size == k, 已滿                  │
│ heap[2] = 8                      │
└──────────────────────────────────┘

插入前：heap = [4, 5, ?, ?]
插入後：heap = [4, 5, 8, ?]
調用 heapifyUp(2):
  ├─ index = 2, parent = (2-1)/2 = 0
  ├─ heap[2]=8 vs heap[0]=4
  ├─ 8 > 4 ✓ (不需交換)
  └─ 停止

最終 heap = [4, 5, 8, ?], size = 3

樹狀圖：
        4 (0xB000)
       / \
      5   8
  (0xB004)(0xB008)
```

#### Step 4: 處理 2 (skip，因為 2 < 4)
```
操作: Consider inserting 2
┌──────────────────────────────────┐
│ size == k (3)                    │
│ 2 < heap[0]=4 ? TRUE             │
│ 跳過插入                         │
└──────────────────────────────────┘

heap 不變：[4, 5, 8, ?]
size 不變：3
```

### add() 操作序列追蹤

#### Operation 1: add(3)

```
輸入: 3
┌────────────────────────────────────────────┐
│ 條件檢查                                   │
├────────────────────────────────────────────┤
│ 3 < heap[0]=4 ? TRUE                       │
│ ═══════════════════════════════════════    │
│ 決策: 跳過插入，直接返回最小值              │
└────────────────────────────────────────────┘

┌─────────────────────────┐
│ 操作前：heap = [4,5,8]  │
├─────────────────────────┤
│ 操作後：heap = [4,5,8]  │ (無變化)
│ 返回值：4               │
└─────────────────────────┘

樹狀圖 (無變化)：
        4 ◄─── (最小元素，返回給使用者)
       / \
      5   8
```

#### Operation 2: add(5)

```
輸入: 5
┌────────────────────────────────────────────┐
│ 條件檢查                                   │
├────────────────────────────────────────────┤
│ 5 < heap[0]=4 ? FALSE                      │
│ ═══════════════════════════════════════    │
│ 決策: 替換根節點，然後向下調整              │
└────────────────────────────────────────────┘

替換根節點：
  heap[0] = 5

┌──────────────────────────────────────────────┐
│ 替換前：[4, 5, 8]                            │
│ 替換後：[5, 5, 8]                            │
└──────────────────────────────────────────────┘

樹狀圖 (替換後)：
        5 (新根)
       / \
      5   8

調用 heapifyDown(0):
  ├─ index = 0
  ├─ left_child = 2*0+1 = 1, left_val = 5
  ├─ right_child = 2*0+2 = 2, right_val = 8
  ├─ 最小子節點是 left(idx=1, val=5)
  ├─ 5 <= 5 ? TRUE (相等，不交換)
  └─ 停止

最終 heap = [5, 5, 8]

樹狀圖 (最終)：
        5 (0xB000)
       / \
      5   8
  (0xB004)(0xB008)

返回值：5
```

```
輸入: 10
┌────────────────────────────────────────────┐

│ 決策: 替換根節點，向下調整                  │
└────────────────────────────────────────────┘

替換根節點：

  │  └─ 結束

  └─ 停止
```

## 6. 記憶體存取模式比較

### 陣列型堆積 vs. 樹形堆積 (Linked Priority Queue)

```
┌──────────────────────────────────────────────────────────────┐
│ 特性              │ 陣列堆積        │ 鏈表堆積               │
├──────────────────┼─────────────────┼───────────────────────┤
│ 存儲佈局          │ 連續記憶體       │ 分散記憶體            │
│                  │ [4][5][8][?]    │ 4 → 5 → 8 (指標鏈)   │
├──────────────────┼─────────────────┼───────────────────────┤
│ 快取局部性        │ 優秀 ★★★★★   │ 差 ★★ (亂序訪問)    │
│ (Cache Locality)  │ 連續記憶體       │ 指標追蹤很難預測      │
├──────────────────┼─────────────────┼───────────────────────┤
│ 索引計算          │ O(1) 算術運算   │ O(1) 指標解參考       │
│                  │ parent(i)       │ curr→left, curr→right │
│                  │ = (i-1)/2       │                       │
├──────────────────┼─────────────────┼───────────────────────┤
│ 空間開銷          │ 零指標開銷       │ 每節點 2×8B = 16B    │
│                  │ (純陣列)        │ (left, right 指標)    │
│                  │ k+1 個 int      │ k+1 個 Node 結構      │
├──────────────────┼─────────────────┼───────────────────────┤
│ 空間複雜度        │ O(k)            │ O(k) + O(k)指標       │
│                  │ = 4*(k+1) bytes │ = 4*(k+1) + 16*(k+1)  │
├──────────────────┼─────────────────┼───────────────────────┤
│ 記憶體訪問次數    │ ~5-6 次/操作    │ ~8-10 次/操作         │
│ (Example 1)       │ (+指標)         │ (多次指標解參)        │
├──────────────────┼─────────────────┼───────────────────────┤
│ CPU 快取效能      │ 極佳 L1/L2 命中 │ 低 L3/主記憶體命中    │
├──────────────────┼─────────────────┼───────────────────────┤
│ 記憶體頻寬        │ 32B/操作        │ 48B+/操作             │
│                  │ (8 個 int)      │ (含指標追蹤)          │
└──────────────────┴─────────────────┴───────────────────────┘
```

### 快取行為分析

#### 陣列型堆積快取圖

```
L1 快取行 (64 bytes)：
┌────────────────────────────────────────────────────┐
│ 0xB000 ┃ [4] [5] [8] [?] [padding...]             │
└────────────────────────────────────────────────────┘
 ▲       ▲       ▲       ▲
 └───────└───────└───────┘
 單次快取行載入，覆蓋整個堆積

快取命中率：~95% (k 通常很小 ≤ 100)
```

#### 鏈表型堆積快取圖

```
L1 快取行 (64 bytes)：

載入節點 1：
┌────────────────────────────────────────────────────┐
│ Node1: [val][left*][right*] [padding...]           │
└────────────────────────────────────────────────────┘

載入節點 2 (追蹤 left* 指標)：
┌────────────────────────────────────────────────────┐
│ Node2: [val][left*][right*] [padding...]           │
└────────────────────────────────────────────────────┘
   ▲
   │ 指標地址可能跨越多個快取行
   │ (隨機分散的堆積記憶體)

快取命中率：~30-40% (多次快取行缺失)
```

## 7. 堆疊記憶體對比：迭代 vs. 遞迴

### 操作上下文：heapifyDown 實現方式

#### 迭代版本（Current Implementation）

```c
void heapifyDown(KthLargest* obj, int i) {
    while (1) {
        int smallest = i;
        int left = 2 * i + 1;
        int right = 2 * i + 2;

        if (left < obj->size && obj->heap[left] < obj->heap[smallest])
            smallest = left;
        if (right < obj->size && obj->heap[right] < obj->heap[smallest])
            smallest = right;

        if (smallest == i) break;

        // swap
        int tmp = obj->heap[i];
        obj->heap[i] = obj->heap[smallest];
        obj->heap[smallest] = tmp;

        i = smallest;
    }
}
```

#### 遞迴版本（Alternative）

```c
void heapifyDownRecursive(KthLargest* obj, int i) {
    int smallest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < obj->size && obj->heap[left] < obj->heap[smallest])
        smallest = left;
    if (right < obj->size && obj->heap[right] < obj->heap[smallest])
        smallest = right;

    if (smallest != i) {
        // swap
        int tmp = obj->heap[i];
        obj->heap[i] = obj->heap[smallest];
        obj->heap[smallest] = tmp;

        heapifyDownRecursive(obj, smallest);
    }
}
```

```

│ ✓ 使用迭代版本 heapifyDown()                        │
│                                                     │
│ 理由：                                              │

│ • 代碼可讀性優先                                    │
│ • k 值非常小 (< 10，堆疊開銷可忽略)                 │
```
