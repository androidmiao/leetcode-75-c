## 一、TreeNode struct 在記憶體中的佈局

```
struct TreeNode {
    int val;                /* 4 bytes（通常）     偏移量 +0  */
    struct TreeNode *left;  /* 8 bytes（64-bit）  偏移量 +8  */
    struct TreeNode *right; /* 8 bytes（64-bit）  偏移量 +16 */
};
/* 每個節點 ≈ 24 bytes（含 padding）*/
```

### 單一節點記憶體圖（以 val=4 為例，假設位址 0xA000）

```
              地址          內容
         ┌──────────────────────────────┐
 0xA000  │  val  = 4                    │  ← int（4 bytes）
         ├──────────────────────────────┤
 0xA008  │  left = 0xB000  ──────────┐  │  ← struct TreeNode*
         ├──────────────────────────────┤  │
 0xA010  │  right= 0xC000  ─────┐   │  │  ← struct TreeNode*
         └──────────────────────┼───┼──┘
                                │   │
                  指向 val=7 ◄──┘   └──► 指向 val=2
```

## 三、陣列索引版 — 層序 (Level-order) 對應

BST 常以陣列表示（如 LeetCode 輸入格式 `[4, 2, 7, 1, 3]`），
對於索引 `i` 的節點：
- 左子節點索引 = `2*i + 1`
- 右子節點索引 = `2*i + 2`
- 父節點索引 = `(i - 1) / 2`

```
陣列索引：   [0]    [1]    [2]    [3]    [4]
儲存的值：    4      2      7      1      3
位址對照：  0xA000 0xB000 0xC000 0xD000 0xE000

           index 0
             (4)
            /     \
     index 1       index 2
       (2)           (7)
      /    \
 index 3   index 4
   (1)       (3)

索引關係對照表：
┌───────┬─────┬─────────────┬──────────────┬──────────┐
│ index │ val │ left child  │ right child  │  parent  │
├───────┼─────┼─────────────┼──────────────┼──────────┤
│   0   │  4  │ 2*0+1 = [1] │ 2*0+2 = [2] │   ---    │
│   1   │  2  │ 2*1+1 = [3] │ 2*1+2 = [4] │ (1-1)/2=0│
│   2   │  7  │ 2*2+1 = [5] │ 2*2+2 = [6] │ (2-1)/2=0│
│       │     │  (不存在)    │  (不存在)    │          │
│   3   │  1  │ 2*3+1 = [7] │ 2*3+2 = [8] │ (3-1)/2=1│
│       │     │  (不存在)    │  (不存在)    │          │
│   4   │  3  │ 2*4+1 = [9] │ 2*4+2 =[10] │ (4-1)/2=1│
│       │     │  (不存在)    │  (不存在)    │          │
└───────┴─────┴─────────────┴──────────────┴──────────┘
```

## 四、搜尋過程 — 記憶體位址版追蹤

### Example 1：val = 2（找到）

```
searchBST(root=0xA000, val=2)

═══════════════════════════════════════════════════════
 Step 1：root = 0xA000, root->val = 4
═══════════════════════════════════════════════════════

  root 指標
   │
   ▼
  [0xA000] val=4    ← 當前節點
     /       \
  [0xB000]  [0xC000]
    (2)       (7)

  比較：val(2) < root->val(4)
  決策：往左走 → root = root->left = 0xB000

  指標移動：
  ┌──────────────────────────────────────┐
  │  root:  0xA000  ──→  0xB000         │
  │         ^^^^          ^^^^           │
  │         舊值          新值（左子節點）│
  └──────────────────────────────────────┘


═══════════════════════════════════════════════════════
 Step 2：root = 0xB000, root->val = 2
═══════════════════════════════════════════════════════

              root 指標
               │
               ▼
  [0xA000]   [0xB000] val=2    ← 當前節點
    (4)         /       \
           [0xD000]  [0xE000]
             (1)       (3)

  比較：val(2) == root->val(2)  ✓ 找到！
  決策：while 條件不滿足，離開迴圈

  return root = 0xB000（指向 val=2 的節點）

═══════════════════════════════════════════════════════
 結果：回傳以 0xB000 為根的子樹
═══════════════════════════════════════════════════════

  回傳的子樹（root = 0xB000）：

        [0xB000]
          (2)
         /   \
   [0xD000] [0xE000]
     (1)      (3)

  對應 LeetCode 輸出：[2, 1, 3]
```

### 陣列索引版追蹤（Example 1）

```
陣列：[4, 2, 7, 1, 3]
       [0] [1] [2] [3] [4]

Step 1：看 index 0 → val=4
        val(2) < 4 → 往左子節點 → index = 2*0+1 = 1

Step 2：看 index 1 → val=2
        val(2) == 2 → 找到！

        回傳 index 1 為根的子樹：
        index 1 → val=2
        index 3 → val=1（左子）
        index 4 → val=3（右子）
        輸出 [2, 1, 3]

搜尋路徑：
  [4]  [2]  [7]  [1]  [3]
  [0]  [1]  [2]  [3]  [4]
   ↓    ✓
   └──→─┘
   比較    找到
```

searchBST(root=0xA000, val=5)

═══════════════════════════════════════════════════════

   │
   ▼

     /       \
  [0xB000]  [0xC000]
    (2)       (7)

  比較：val(5) > root->val(4)

  │  root:  0xA000  ──→  0xC000         │
  │         ^^^^          ^^^^           │
  │         舊值          新值（右子節點）│
  └──────────────────────────────────────┘

 Step 2：root = 0xC000, root->val = 7
═══════════════════════════════════════════════════════

                      │
                      ▼
  [0xA000]         [0xC000] val=7    ← 當前節點
    (4)              /       \
              left=NULL   right=NULL

  比較：val(5) < root->val(7)
  決策：往左走 → root = root->left = NULL

  指標移動：
  ┌──────────────────────────────────────┐

  離開迴圈

═══════════════════════════════════════════════════════

### 陣列索引版追蹤（Example 2）

```
陣列：[4, 2, 7, 1, 3]
       [0] [1] [2] [3] [4]

Step 1：看 index 0 → val=4
        val(5) > 4 → 往右子節點 → index = 2*0+2 = 2

Step 2：看 index 2 → val=7
        val(5) < 7 → 往左子節點 → index = 2*2+1 = 5

Step 3：index 5 超出陣列範圍（陣列長度=5）→ 不存在

  [4]  [2]  [7]  [1]  [3]   [ ]
  [0]  [1]  [2]  [3]  [4]   [5]
   ↓          ↓               ✗
   └────→─────┘──────→────────┘
   比較        比較          不存在
```

## 六、BST 搜尋 vs. 陣列索引 — 記憶體存取模式比較

```
【指標版（linked structure）】

  程式碼：root = root->left;
  實際動作：
    1. 從 root（0xA000）讀取 left 欄位 → 得到 0xB000
    2. 將 root 指標更新為 0xB000
    3. 下次存取 *root 時，CPU 去取 0xB000 的記憶體

  特點：
  ┌─────────────────────────────────────────────────┐
  │  ● 每個節點動態分配（malloc），位址不連續       │
  │  ● 0xA000 → 0xB000 → 0xD000 跳躍存取           │
  │  ● Cache miss 機率較高                           │
  │  ● 但結構靈活，插入/刪除 O(1) 指標操作         │
  └─────────────────────────────────────────────────┘

【陣列版（implicit array）】

  程式碼：index = 2 * index + 1;  // 左子
  實際動作：
    1. 計算新索引（純算術，不需讀記憶體）
    2. 存取 arr[new_index]

  特點：
  ┌─────────────────────────────────────────────────┐
  │  ● 節點在連續記憶體中，Cache 友好               │
  │  ● 不需額外指標欄位，省空間                     │
  │  ● 但插入/刪除可能需要搬移大量元素               │
  │  ● 稀疏樹浪費空間（空位也要佔陣列位置）         │
  └─────────────────────────────────────────────────┘

記憶體存取對比：
  指標版：root(0xA000) ──deref──▶ left=0xB000 ──deref──▶ val=2  ✓
  陣列版：arr[0]=4     ──計算──▶  arr[1]=2                      ✓
                         └─ 2*0+1=1（無記憶體讀取，純 ALU 運算）
```

## 七、迭代法 vs. 遞迴法 — 堆疊記憶體對比

```
【迭代法（本題解法）】— 空間 O(1)

  只有一個 root 指標變數在 stack frame 上
  ┌──────────────────────────────┐
  │  searchBST stack frame       │
  │  ┌────────────────────────┐  │
  │  │  root: 8 bytes         │  │  ← 唯一的區域變數
  │  │  val:  4 bytes         │  │  ← 函式參數
  │  └────────────────────────┘  │
  │  total ≈ 12 bytes（固定）    │
  └──────────────────────────────┘


【遞迴法】— 空間 O(H)

  每次遞迴呼叫都會推一個新 stack frame
  ┌──────────────────────────────┐
  │  searchBST(0xA000, 2)       │  frame 1
  │  ┌────────────────────────┐  │
  │  │  root=0xA000, val=2    │  │
  │  └────────────────────────┘  │
  ├──────────────────────────────┤
  │  searchBST(0xB000, 2)       │  frame 2
  │  ┌────────────────────────┐  │
  │  │  root=0xB000, val=2    │  │  ← 找到，開始返回
  │  └────────────────────────┘  │
  └──────────────────────────────┘

  最壞情況（退化為鏈表時）：
  ┌─────────────────┐
  │  frame 1        │
  ├─────────────────┤
  │  frame 2        │
  ├─────────────────┤
  │  frame 3        │
  ├─────────────────┤
  │  ...            │
  ├─────────────────┤
  │  frame N        │  ← N 層深，堆疊可能溢出
  └─────────────────┘
  total ≈ N × ~16 bytes
```

## 八、退化情況 — 最壞 BST（單鏈形狀）

```
如果 BST 退化為單邊鏈（例如 [1, 2, 3, 4, 5] 依序插入）：

  root = 0xA000
   │
   ▼
  [0xA000] val=1
     \
   [0xB000] val=2
       \
     [0xC000] val=3
         \
       [0xD000] val=4
           \
         [0xE000] val=5

陣列表示（大量空洞）：
  [0]  [1]  [2]  [3]  [4]  [5]  [6]  [7]  ... [14]
   1   __    2   __   __    __    6   __   ...   5
       空        空   空    空         空         ↑
                                           index=2^4-2

  指標版：搜尋 val=5 需要走 4 步 → O(N)
  陣列版：index 不斷 2*i+2，到 index 14 才找到 → 浪費空間

  結論：BST 性能取決於樹的平衡度
        平衡時 O(log N)，退化時 O(N)
```
