## 6. 環形緩衝區 vs 線性陣列比較

### 6.1 環形緩衝區方案 (Circular Buffer)

**特點:**
- 固定大小 `k`
- 使用 `front` 和 `size` 追蹤區間
- 寫入位置: `(front + size) % k`

```
優點:
  ✓ 固定記憶體大小 (k 個整數)
  ✓ 無需動態擴展
  ✓ 快速移除頭元素 (只需 front++, size--)
  ✓ 避免陣列移位 O(k) 開銷
  ✓ 適合空間受限環境 (嵌入式系統)

缺點:
  ✗ 模運算開銷 (% 操作 CPU 昂貴)
  ✗ 邊界條件處理複雜
  ✗ 圓形索引追蹤困難，易出錯
  ✗ 快取位置性較差 (記憶體分散)

時間複雜度:
  • 單次迴圈: O(1) (模運算)
  • 總體: O(n)

空間複雜度:
  • Deque: O(k)
  • Result: O(n - k + 1)
  • 總計: O(n + k)
```

**記憶體配置圖 (Step 4 時刻):**

```
初始配置 (Step 0):
  front=0, size=1
  ┌────┬────┬────┐
  │ 0  │    │    │
  └────┴────┴────┘
   B00 B04 B08

中間 (Step 2):
  front=0, size=2
  ┌────┬────┬────┐
  │ 1  │ 2  │    │
  └────┴────┴────┘
   B00 B04 B08

繞圈 (Step 4):
  front=1, size=1
  ┌────┬────┬────┐
  │ 1  │ 4  │ 3  │  (邏輯: [4])
  └────┴────┴────┘
   B00 B04 B08
        ▲
      front=1
```

### 6.2 線性陣列方案 (Linear Array)

**特點:**
- 動態大小為 `n`
- 使用 `front` 和 `rear` 指標

```
優點:
  ✓ 不需模運算
  ✓ 邊界邏輯簡單
  ✓ 記憶體訪問連續 (快取友善)
  ✓ 易於調試和理解

缺點:
  ✗ O(n) 額外空間 (vs O(k))
  ✗ 移除頭元素時需複製 O(k) 操作
  ✗ 或使用雙指標但仍需空間洗牌
  ✗ 不適合 k << n 的情況

時間複雜度:
  • 移除: O(k)  (如果實現為複製)
  • 總體: O(n*k) 或 O(n) (使用雙指標)
```

## 8. 複雜度分析

### 時間複雜度

```
對每個元素 i (i = 0 to n-1):

  1. 檢查 front 是否超出窗口: O(1)
     if (front < i - k + 1) front++

  2. 移除比 nums[i] 小的元素: 均攤 O(1)
     while (size > 0 && nums[deque[(front + size - 1) % k]] < nums[i])
         size--

     關鍵: 每個元素最多進出 deque 一次
     進: 一次 (加入 deque)
     出: 一次 (被比它大的元素移除)
     總計: O(n) 次進出操作

  3. 加入當前元素: O(1)
     deque[(front + size) % k] = i
     size++

  4. 輸出窗口最大值: O(1)
     result[resultIndex++] = nums[deque[front]]

總時間: O(n)
```

### 空間複雜度

```
1. deque 陣列: O(k)
   - 最多存儲 k 個元素索引

2. result 陣列: O(n - k + 1)
   - 最多輸出 n - k + 1 個值
   - 在 k = 1 時最大 (為 n)
   - 在 k = n 時最小 (為 1)

3. 其他變數: O(1)
   - front, size, resultIndex

總空間: O(n + k)
         = O(n) 在 k ≤ n 時

優化注: 若不計算 result (即輸出直接寫入流)，可將空間降至 O(k)
```

## 9. 程式碼與記憶體映射

### 9.1 完整函數結構

```c
int* maxSlidingWindow(int* nums, int numsSize, int k, int* returnSize) {
    // 記憶體配置
    int* deque = (int*)malloc(k * sizeof(int));      // @0xB000
    int* result = (int*)malloc((numsSize - k + 1) * sizeof(int));  // @0xC000

    int front = 0;           // 堆疊 -20(RBP)
    int size = 0;            // 堆疊 -24(RBP)
    int resultIndex = 0;     // 堆疊 -28(RBP)

    for (int i = 0; i < numsSize; i++) {  // 堆疊 -32(RBP)
        // 移除窗口外的元素 (位於 deque 前端)
        if (front < i - k + 1) {
            front++;  // front = (front + 1) % k 的簡化形式
        }

        // 移除比當前元素小的元素 (位於 deque 後端)
        while (size > 0 && nums[deque[(front + size - 1) % k]] < nums[i]) {
            size--;
        }

        // 加入當前元素
        deque[(front + size) % k] = i;
        size++;

        // 當窗口滿時，輸出當前最大值
        if (i >= k - 1) {
            result[resultIndex++] = nums[deque[front]];
        }
    }

    *returnSize = numsSize - k + 1;
    free(deque);
    free(nums);  // 注: 實際代碼不應 free 輸入的 nums

    return result;  // 返回 @0xC000
}
```

### 9.2 關鍵語句的記憶體效應

#### 語句 1: 移除過期的 front

```c
if (front < i - k + 1) {
    front++;
}

記憶體變化 (Step 4 時):
  之前: front = 0, i = 4, k = 3
        條件: 0 < 4 - 3 + 1 = 2 ✓ (true)
  執行: front = 1
  之後: front @堆疊(-20) = 1

對 deque 的邏輯影響:
  舊狀態: deque[0]=1, deque[1]=2, deque[2]=3
         front=0, size=3
         邏輯內容: [deque[0], deque[1], deque[2]]

  新狀態: deque[0]=1, deque[1]=2, deque[2]=3 (物理不變)
         front=1, size=3
         邏輯內容: [deque[1], deque[2], ?]
         有效元素變為: deque[1]=2, deque[2]=3
```

#### 語句 2: 移除後端小元素

```c
while (size > 0 && nums[deque[(front + size - 1) % k]] < nums[i]) {
    size--;
}

循環 (Step 4 時, i=4, nums[4]=5):

迭代 1:
  條件檢查: size=3 > 0 ✓, nums[deque[(0+3-1)%3]] = nums[deque[2]] = nums[3] = -3
           -3 < 5 ✓
  執行: size = 2

迭代 2:
  條件檢查: size=2 > 0 ✓, nums[deque[(0+2-1)%3]] = nums[deque[1]] = nums[2] = -1
           -1 < 5 ✓
  執行: size = 1

迭代 3:
  條件檢查: size=1 > 0 ✓, nums[deque[(0+1-1)%3]] = nums[deque[0]] = nums[1] = 3
           3 < 5 ✓
  執行: size = 0

迭代 4:
  條件檢查: size=0 > 0 ✗ (迴圈終止)

最終: front=1, size=0, deque 邏輯內容: (空)
```

#### 語句 3: 加入新元素

```c
deque[(front + size) % k] = i;
size++;

執行 (Step 4 後, 加入 i=4):
  位置計算: (front + size) % k = (1 + 0) % 3 = 1
  寫入: deque[1] = 4
  遞增: size = 1

  deque @0xB000 狀態:
  ┌────┬────┬────┐
  │ 1  │ 4  │ 3  │ (新值在 [1])
  └────┴────┴────┘
       ▲
    front=1

  邏輯內容: [deque[1]] = [4]
  最大值: nums[4] = 5
```

#### 語句 4: 輸出結果

```c
if (i >= k - 1) {
    result[resultIndex++] = nums[deque[front]];
}

執行 (Step 2, i=2, k=3):
  條件: 2 >= 3 - 1 = 2 ✓
  執行: result[0] = nums[deque[0]] = nums[1] = 3
       resultIndex = 1

  result @0xC000:
  ┌────┬────┬────┬────┬────┬────┐
  │ 3  │ ?  │ ?  │ ?  │ ?  │ ?  │
```

## 10. 常見陷阱與除錯

### 10.1 圓形索引計算錯誤

```
常見錯誤 ✗:
  idx = (front + size) % k  (前端 + 大小)
  vs
  idx = (front - 1 + k) % k  (試圖反向)

正確做法 ✓:
  後端索引 = (front + size - 1) % k
  下一個寫入位置 = (front + size) % k

範例 (front=1, size=2, k=3):
  ┌────┬────┬────┐
  │ 1  │ 4  │ 5  │
  └────┴────┴────┘
   [0]  [1]  [2]
       ▲    ▲    ▲
     front curr next

  前端: deque[1] = 4
  後端: deque[(1+2-1)%3] = deque[2] = 5
  下一個: (1+2)%3 = 0 ◀── 會覆蓋 deque[0]
```

### 10.2 Window Validity 檢查

```
容易遺漏 ✗:
  for (int i = 0; i < numsSize; i++) {
      // ... deque 操作 ...
      result[resultIndex++] = nums[deque[front]];  // 提前輸出!
  }

正確做法 ✓:
  for (int i = 0; i < numsSize; i++) {
      // ... deque 操作 ...
      if (i >= k - 1) {  // 只在窗口滿時輸出
          result[resultIndex++] = nums[deque[front]];
      }
  }

邏輯:
  第一個完整窗口在 i = k - 1 時形成
  因為窗口涵蓋索引 [0, k-1]，共 k 個元素
```

### 10.3 off-by-one 錯誤

```
邊界條件:

i = 0:     窗口 [0, ..., k-1]，i >= k-1? 0 >= 2? ✗ (k=3)
i = 1:     窗口 [1, ..., k]，i >= k-1? 1 >= 2? ✗
i = 2:     窗口 [2-2, ..., 2] = [0,1,2]，i >= k-1? 2 >= 2? ✓ ◀── 第一次輸出
i = 3:     窗口 [3-2, ..., 3] = [1,2,3]，i >= k-1? 3 >= 2? ✓
...
i = n-1:   窗口 [n-1-2, ..., n-1] = [n-3,n-2,n-1]，i >= k-1? ✓ ◀── 最後一次輸出

總輸出次數: n - k + 1 = 8 - 3 + 1 = 6
```

### 10.4 front 移動的邏輯

```
錯誤 ✗:
  if (front < i - k + 1) {
      front++;  // 只移動一次，可能不夠!
  }

原因: 可能連續多個索引都超出窗口

正確做法 ✓:
  if (front < i - k + 1) {
      front++;
  }

  註: 在此問題中，front 最多超出一次，
      因為每次迴圈 i 只增加 1

但通用做法應為:
  while (front < i - k + 1) {
      front++;
  }
```

## 12. 效能優化建議

### 12.1 模運算優化

```c
// 原始做法 (有模運算開銷):
int idx = (front + offset) % k;

// 優化做法 1: 預先檢查避免模運算
int idx;
if (offset < k - front) {
    idx = front + offset;
} else {
    idx = offset - (k - front);
}

// 優化做法 2: 使用位運算 (k 為 2 的次方時):
if ((k & (k - 1)) == 0) {  // k 是 2 的次方?
    int idx = (front + offset) & (k - 1);  // 用 & 取代 %
}

// 優化做法 3: 避免重複計算
const int rear = (front + size) % k;
deque[rear] = i;
```

### 12.2 快取最佳化

```
避免 ✗:
  nums[deque[(front + size - 1 + k) % k]] < nums[i]

建議 ✓:
  預先計算並快取:
  int rearIdx = (front + size - 1) % k;
  if (nums[deque[rearIdx]] < nums[i]) {
      size--;
  }
```

### 12.3 邊界檢查內聯

```c
// 編譯器友善版本
if (i >= k - 1) {  // 條件簡單，易於分支預測
    result[resultIndex++] = nums[deque[front]];
}
```
