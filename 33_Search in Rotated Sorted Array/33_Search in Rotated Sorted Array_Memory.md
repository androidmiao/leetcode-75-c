## 1. 陣列記憶體佈局 (Array Memory Layout)

**基址** — 0xA000（32位元整數系統，每個int佔4個位元組）

| 索引 | 元素值 | 基址位移 | 記憶體位址 |

| [0]  | 4      | 0        | 0xA000    |
| [1]  | 5      | 4        | 0xA004    |
| [2]  | 6      | 8        | 0xA008    |
| [3]  | 7      | 12       | 0xA00C    |
| [4]  | 0      | 16       | 0xA010    |
| [5]  | 1      | 20       | 0xA014    |

```
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤

      ▲                   ▲
      │ Sorted Region 1   │ Sorted Region 2
      │ (4,5,6,7)         │ (0,1,2)
      │                   └── Rotation Point (Value 0)
      └── Left Boundary
```

**旋轉點分析** — 數組在索引4處旋轉，分成兩個已排序的片段：
- 左片段 [0,3]：4 ≤ nums[i] ≤ 7（已排序）
- 右片段 [4,6]：0 ≤ nums[i] ≤ 2（已排序）
- 旋轉點特性：nums[4] = 0 < nums[3] = 7（旋轉發生在此）

## 3. 陣列索引對應 (Array Index Correspondence)

### 原始排序數組 vs 旋轉後的數組

```
Original Sorted Array:  [0, 1, 2, 4, 5, 6, 7]

                         │  │  │  │  │  │  │

                         ↓  ↓  ↓  ↓  ↓  ↓  ↓

                         ↑  ↑  ↑  ↑  ↑  ↑  ↑
Index in Rotated:       0  1  2  3  4  5  6
```

```
┌──────────────────────┬──────────────────────┐
│ Original Index       │ Rotated Index        │
├──────────────────────┼──────────────────────┤
│ 0 (value 0)          │ 4                    │
│ 1 (value 1)          │ 5                    │
│ 2 (value 2)          │ 6                    │
│ 3 (value 4)          │ 0                    │
│ 4 (value 5)          │ 1                    │

│ 6 (value 7)          │ 3                    │

旋轉公式 (Rotation Formula):
  rotated_index = (original_index + 4) mod 7
```

## 4. 搜尋過程追蹤 (Search Process Tracing)

### 例題 1: nums = [4,5,6,7,0,1,2], target = 0 → 預期輸出: 4

#### 初始狀態

```
┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  4  │  5  │  6  │  7  │  0  │  1  │  2  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│[0]  │[1]  │[2]  │[3]  │[4]  │[5]  │[6]  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│A000 │A004 │A008 │A00C │A010 │A014 │A018 │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┘
▲                                           ▲
l=0                                         r=6
        Target: 0
```

#### 第1次迭代 (Iteration 1)

```
計算:
  l = 0, r = 6
  mid = 0 + (6 - 0) / 2 = 3
  nums[mid] = nums[3] = 7 (Address: 0xA00C)
  target = 0

比較:
  nums[mid] (7) ≠ target (0)
  nums[l] (4) ≤ nums[mid] (7) → TRUE (左片段已排序)

判斷目標位置:
  nums[l] (4) ≤ target (0) → FALSE
  不在左片段中，必在右片段

┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  4  │  5  │  6  │  7  │  0  │  1  │  2  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│     │     │     │ mid │     │     │     │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┘
▲                   ▲                       ▲
l                   mid(3)                  r

決策: 目標在右片段 → l = mid + 1 = 4
```

**指標變動** — `l: 0 → 4, r: 6 → 6`

#### 第2次迭代 (Iteration 2)

```
計算:
  l = 4, r = 6
  mid = 4 + (6 - 4) / 2 = 5
  nums[mid] = nums[5] = 1 (Address: 0xA014)
  target = 0

比較:
  nums[mid] (1) ≠ target (0)
  nums[l] (0) ≤ nums[mid] (1) → TRUE (右片段已排序)

判斷目標位置:
  nums[l] (0) ≤ target (0) → TRUE ✓
  target (0) < nums[mid] (1) → TRUE ✓
  目標在左部份

┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  4  │  5  │  6  │  7  │  0  │  1  │  2  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│     │     │     │     │  l  │ mid │     │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┘
                          ▲       ▲       ▲

決策: 目標在mid左邊 → r = mid - 1 = 4
```

**指標變動** — `l: 4 → 4, r: 6 → 4`

#### 第3次迭代 (Iteration 3)

```
計算:
  l = 4, r = 4
  mid = 4 + (4 - 4) / 2 = 4
  nums[mid] = nums[4] = 0 (Address: 0xA010)

比較:
  nums[mid] (0) == target (0) → ✓ 找到目標！

┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  4  │  5  │  6  │  7  │  0  │  1  │  2  │
```

## 6. 記憶體存取模式分析 (Memory Access Pattern Analysis)

### 二分搜尋的空間局部性 (Spatial Locality)

```
典型陣列配置 (Typical Array Layout):

┌─────┬─────┬─────┬─────┬─────┬─────┬─────┐
│  4  │  5  │  6  │  7  │  0  │  1  │  2  │
├─────┼─────┼─────┼─────┼─────┼─────┼─────┤
│A000 │A004 │A008 │A00C │A010 │A014 │A018 │
└─────┴─────┴─────┴─────┴─────┴─────┴─────┘
└───────────────── 單一CPU快取行 (64 bytes) ─────────────────┘

快取行大小: 64 bytes
整數大小: 4 bytes per element
該陣列: 7 * 4 = 28 bytes (完全適應單一快取行)
```

### 存取模式與快取行為

```
第1次存取 (首次):
  位址: 0xA00C (nums[3])
  狀態: L1 MISS ✗ (冷啟動)
  動作: 快取行載入 (0xA000 - 0xA03F)

  ┌──────────────────────── Cache Line (64B) ──────────────────────┐
  │ A000  A004  A008  A00C  A010  A014  A018 │ ...未使用... │
  │  4     5     6     7     0     1     2   │            │
  └──────────────────────────────────────────────────────────────────┘
  已載入        ▲
             nums[3]

第2次存取:
  位址: 0xA014 (nums[5])
  狀態: L1 HIT ✓ (已在快取行中)
  延遲: ~4 cycles (vs. ~200 cycles for miss)

第3次存取:
  位址: 0xA010 (nums[4])
  狀態: L1 HIT ✓ (已在快取行中)
  延遲: ~4 cycles
```

### 快取效能對比

```
┌──────────────────────────────────────────────────────────────┐
│              Binary Search on Different Data Structures     │
├──────────────────────────────────────────────────────────────┤
│                                                              │
│  Contiguous Array (C):                                      │
│    存取1: [0xA00C] MISS ─── 載入快取行 ───┐                │
│    存取2: [0xA014]  HIT ◄──┤               │               │
│    存取3: [0xA010]  HIT ◄──┤ 所有後續      │               │
│    ⟹ 高度局部化 ✓           │ 存取都命中    │               │
│                             └───────────────┘               │
│                                                              │
│  Linked List Based BST (Worst Case):                        │
│    存取1: [0xB000] MISS ─── 載入快取行 ───┐                │
│    存取2: [0xE008] MISS ─── 新快取行 ──┐  │               │
│    存取3: [0xC1F0] MISS ─── 新快取行 ──┼─┐│               │
│    ⟹ 無局部化 ✗             └─────────┘ ││               │
│                                         ││               │
│    平均延遲對比:                        ││               │
│    Array:  (1 × 200 + n-1 × 4)  cycles││               │
│    BST:    n × 200  cycles             ││               │
│    優勢:   ~50倍 (針對n≥8時)           ││               │
│                                        ▼▼               │
└──────────────────────────────────────────────────────────────┘
```

### 我們的例題分析

```
Example 1 (target = 0) - 3 次迭代:

  迭代 1: 存取 0xA00C ───────────────┐
  迭代 2: 存取 0xA014 (same line) ───┤ 1 × MISS + 2 × HIT
  迭代 3: 存取 0xA010 (same line) ───┘

  預估時間: 1 × 200 + 2 × 4 = 208 cycles

  vs. 無優化線性搜尋:
  最差: 7 × 200 = 1400 cycles

  加速比: 1400 / 208 ≈ 6.7×

Example 2 (target = 3) - 3 次迭代:

  迭代 1: 存取 0xA00C ───────────────┐
  迭代 2: 存取 0xA014 (same line) ───┤ 1 × MISS + 2 × HIT
  迭代 3: 存取 0xA018 (same line) ───┘

  預估時間: 1 × 200 + 2 × 4 = 208 cycles
  加速比: 1400 / 208 ≈ 6.7×
```

### 結論

- **優勢**: 數組的連續記憶體佈局提供優秀的空間局部性
- **首次存取**: 初始快取未命中，但快速載入整個快取行
- **後續存取**: 絕大多數快取命中，因為相鄰元素在同一快取行
- **陣列規模n≤64/4=16時**: 整個數組適應單一快取行，效能最優
- **分散式結構**: 如果使用指標型數據結構（鏈表、樹），每次存取都會錯過快取，性能下降50-100倍

## 程式碼解析 (Code Analysis)

### 完整C實現

```c
int search(int* nums, int numsSize, int target) {
    int l = 0;                              // 左指標初始化
    int r = numsSize - 1;                   // 右指標初始化

    while (l <= r) {                        // 當搜尋範圍有效時
        int mid = l + (r - l) / 2;          // 計算中點 (避免溢出)

        if (nums[mid] == target)             // 直接命中
            return mid;

        // 判斷哪一側已排序
        if (nums[l] <= nums[mid]) {          // 左側已排序
            // 檢查目標是否在左側已排序區間
            if (nums[l] <= target && target < nums[mid])
                r = mid - 1;                 // 搜尋左側
            else
                l = mid + 1;                 // 搜尋右側
        } else {                             // 右側已排序
            // 檢查目標是否在右側已排序區間
            if (nums[mid] < target && target <= nums[r])
                l = mid + 1;                 // 搜尋右側
            else
                r = mid - 1;                 // 搜尋左側
        }
    }

    return -1;                               // 未找到目標
}
```

### 時間複雜度分析

```
最壞情況: O(log n)
  • 每次迭代減半搜尋空間
  • n=7: log₂(7) ≤ 3 次迭代 (我們例題確認)
  • n=10⁶: log₂(10⁶) ≈ 20 次迭代

最佳情況: O(1)
  • 目標正好在 mid 位置
  • 首次迭代即命中

平均情況: O(log n)
  • 期望值仍為對數級別
```

### 空間複雜度分析

```
堆疊空間: O(1)
  • 只使用固定數量的局部變數 (l, r, mid)
  • 無遞迴調用

堆空間: O(0)
  • 無動態記憶體分配

總計: O(1) ✓ (所需的最優空間)
```

## 常見錯誤與陷阱 (Common Pitfalls)

### 1. 溢出風險

```
危險寫法:
  int mid = (l + r) / 2;  ✗
  問題: 當 l 和 r 很大時, l + r 可能整數溢位

安全寫法:
  int mid = l + (r - l) / 2;  ✓
  原因: (r - l) 的範圍是 [0, numsSize-1], 不會溢位
```

### 2. 邊界條件

```
常見錯誤:
  if (l < r)  ✗  會遺漏單一元素的情況 [1], target=1

正確寫法:
  if (l <= r)  ✓  包含 l==r 的情況
```

### 3. 旋轉點判斷

```
需要檢查:
  if (nums[l] <= nums[mid])  ✓ 左側已排序

不能假設:
  if (nums[mid] < nums[r])   ✗ 不一定表示右側已排序
  (在旋轉數組中可能兩側都有相同值)
```

## 記憶體優化建議 (Memory Optimization Tips)

```
✓ 當前方案: 已達最優
  - O(1) 堆疊空間
  - O(1) 堆空間
  - 無額外分配

如需進一步優化:
  1. 內嵌編譯: 編譯器可內嵌此函數 (更少堆疊開銷)
  2. SIMD: 對非常大陣列可用 SIMD 指令 (但對本問題無益)
  3. 分支預測: 現代CPU可預測比較邏輯流 (編譯器通常自動處理)
```
