# 217. Contains Duplicate — 記憶體位址 / 陣列索引圖解

> 以下位址皆為假設值，用於說明演算法執行過程中的記憶體存取模式。

## 陣列記憶體佈局

### 單一 `int` 元素

每個 `int` 佔 4 bytes（32 bits）。陣列元素在記憶體中連續排列。

```
       地址          內容
      ┌──────────┬──────────┐
0xA000│  nums[0] │  4 bytes │
      ├──────────┼──────────┤
0xA004│  nums[1] │  4 bytes │
      ├──────────┼──────────┤
0xA008│  nums[2] │  4 bytes │
      ├──────────┼──────────┤
0xA00C│  nums[3] │  4 bytes │
      └──────────┴──────────┘
```

### 函式參數

```
containsDuplicate(int* nums, int numsSize)

  nums     ───▶ 0xA000（陣列起始位址）
  numsSize  =   4（元素個數）
```

├────────┼──────┼─────┼──────┤
│ 0xA004 │ +4   │  2  │ [1]  │

陣列總大小：4 × sizeof(int) = 16 bytes

## 排序過程追蹤 — 範例 1：nums = [1, 2, 3, 1]

### 排序前

```
索引：   0     1     2     3
       ┌─────┬─────┬─────┬─────┐
  值： │  1  │  2  │  3  │  1  │
       └─────┴─────┴─────┴─────┘
  位址：A000  A004  A008  A00C
```

### qsort 排序後（原地修改）

```
索引：   0     1     2     3
       ┌─────┬─────┬─────┬─────┐
  值： │  1  │  1  │  2  │  3  │
       └─────┴─────┴─────┴─────┘
  位址：A000  A004  A008  A00C

注意：位址不變，是值被交換了。
原本 nums[1]=2, nums[3]=1 → 排序後 nums[1]=1, nums[3]=3
```

### 線性掃描過程

```
Step 1: i = 0
  讀取 *(0xA000) = 1   與   *(0xA004) = 1
  比較結果：相等 → 回傳 true ✓

  ┌─────┬─────┬─────┬─────┐
  │ [1] │ [1] │  2  │  3  │
  └─────┴─────┴─────┴─────┘
    ▲     ▲
    i    i+1
   比較 → 相等！
```

## 排序過程追蹤 — 範例 2：nums = [1, 2, 3, 4]

### qsort 排序後（已有序，無交換）

```
索引：   0     1     2     3
       ┌─────┬─────┬─────┬─────┐
  值： │  1  │  2  │  3  │  4  │
       └─────┴─────┴─────┴─────┘
  位址：B000  B004  B008  B00C
```

### 線性掃描過程

```
Step 1: i = 0
  讀取 *(0xB000) = 1   與   *(0xB004) = 2
  比較：1 ≠ 2 → 繼續

  ┌─────┬─────┬─────┬─────┐
  │ [1] │ [2] │  3  │  4  │
  └─────┴─────┴─────┴─────┘
    ▲     ▲
    i    i+1      → 不等

Step 2: i = 1
  讀取 *(0xB004) = 2   與   *(0xB008) = 3
  比較：2 ≠ 3 → 繼續

  ┌─────┬─────┬─────┬─────┐
  │  1  │ [2] │ [3] │  4  │
  └─────┴─────┴─────┴─────┘
          ▲     ▲
          i    i+1    → 不等

Step 3: i = 2
  讀取 *(0xB008) = 3   與   *(0xB00C) = 4
  比較：3 ≠ 4 → 繼續

  ┌─────┬─────┬─────┬─────┐
  │  1  │  2  │ [3] │ [4] │
  └─────┴─────┴─────┴─────┘
                ▲     ▲
                i    i+1  → 不等

迴圈結束（i = 3 不滿足 i < numsSize - 1 = 3）
→ 回傳 false ✓
```

## 指標變數追蹤總表

### 範例 1：nums = [1, 2, 3, 1]（排序後 [1, 1, 2, 3]）

| Step | i | 位址 nums[i] | 值 nums[i] | 位址 nums[i+1] | 值 nums[i+1] | 比較結果 | 動作 |
|------|---|-------------|------------|----------------|--------------|---------|------|
| 1 | 0 | 0xA000 | 1 | 0xA004 | 1 | 相等 | 回傳 true |

- 記憶體讀取次數：2（一對相鄰元素）
- 比較次數：1

### 範例 2：nums = [1, 2, 3, 4]（排序後不變）

| Step | i | 位址 nums[i] | 值 nums[i] | 位址 nums[i+1] | 值 nums[i+1] | 比較結果 | 動作 |
|------|---|-------------|------------|----------------|--------------|---------|------|
| 1 | 0 | 0xB000 | 1 | 0xB004 | 2 | 不等 | 繼續 |
| 2 | 1 | 0xB004 | 2 | 0xB008 | 3 | 不等 | 繼續 |
| 3 | 2 | 0xB008 | 3 | 0xB00C | 4 | 不等 | 繼續 |

- 記憶體讀取次數：6（3 對相鄰元素 × 2）
- 比較次數：3（= numsSize - 1）

## 記憶體存取模式分析

### 排序法（本解法）

```
排序階段：qsort 的比較 + 交換
  - 比較：隨機存取陣列中任意兩元素（視 quicksort/heapsort partition 而定）
  - 交換：memcpy 或直接賦值，涉及同一 cache line 或跨 cache line

掃描階段：嚴格順序存取
  ┌─────┬─────┬─────┬─────┐
  │ → → │ → → │ → → │     │  順序讀取，cache 友好
  └─────┴─────┴─────┴─────┘
  每次讀取 nums[i] 和 nums[i+1]，步幅固定 = sizeof(int) = 4 bytes
  CPU prefetcher 可有效預取
```

### 雜湊集合法（對照）

```
  插入/查找：依雜湊值跳躍存取 bucket
  ┌─────┐     ┌─────┐
  │ b0  │     │ b7  │    隨機存取，cache 不友好
  └──┬──┘     └──┬──┘
     │           │
     ▼           ▼
  ┌─────┐     ┌─────┐
  │node │     │node │    鏈式結構，指標追蹤（pointer chasing）
  └─────┘     └─────┘

  每次查找/插入涉及：
  1. 計算雜湊值
  2. 存取 bucket 陣列（可能 cache miss）
  3. 遍歷鏈結串列（pointer chasing，幾乎必定 cache miss）
```

### 比較

| 面向 | 排序法 | 雜湊集合法 |
|------|--------|-----------|
| 掃描階段 cache 行為 | 順序存取，非常友好 | 隨機存取 + pointer chasing |
| 總記憶體使用 | O(1) 額外（原地排序） | O(n) 額外（bucket 陣列 + 節點） |
| 排序階段 cache 行為 | 較多隨機交換，但仍在同一陣列 | N/A |
| 最佳情境 | 小 n、cache 受限環境 | 大 n、需要 O(n) 最佳時間 |

## 堆疊記憶體分析

### 迭代版本（本解法）

```
containsDuplicate() 的堆疊框架：

  ┌───────────────────────────┐  ← RSP (Stack Pointer)
  │ int i           (4 bytes) │  迴圈變數
  ├───────────────────────────┤
  │ int numsSize    (4 bytes) │  參數（或暫存器）
  ├───────────────────────────┤
  │ int* nums       (8 bytes) │  參數（64-bit 指標）
  ├───────────────────────────┤
  │ return address  (8 bytes) │
  └───────────────────────────┘

  固定堆疊用量：~24 bytes（不計 qsort 內部）
  qsort 內部遞迴深度：O(log n)，每層 ~32-64 bytes
  最壞總堆疊用量：~24 + 64 × log₂(10⁵) ≈ 24 + 64 × 17 ≈ 1.1 KB
```

### 若改用遞迴暴力法（對比）

```
// 假想遞迴版本（不推薦）
bool hasDup(int* nums, int n, int start) {
    if (start >= n) return false;
    for (int j = start+1; j < n; j++)
        if (nums[start] == nums[j]) return true;
    return hasDup(nums, n, start+1);
}

  最差遞迴深度：O(n)
  每層堆疊框架：~32 bytes
  n = 10⁵ → 堆疊用量 ≈ 3.2 MB → 可能 stack overflow！

  ┌─────────────┐
  │ hasDup(0)   │
  ├─────────────┤
  │ hasDup(1)   │
  ├─────────────┤
  │ hasDup(2)   │
  ├─────────────┤
  │    ...       │  ← n 層深
  ├─────────────┤
  │ hasDup(n-1) │
  └─────────────┘
```

## 退化情況分析

### 最佳情況

所有元素相同：`[1, 1, 1, ..., 1]`
- 排序後仍為 `[1, 1, 1, ..., 1]`
- 掃描 Step 1 即找到重複
- 線性掃描：O(1)（但排序仍為 O(n log n)）

### 最差情況

所有元素不同：`[1, 2, 3, ..., n]`
- 排序後不變（或做了無用功）
- 需掃描所有 n-1 對
- 總時間：O(n log n) + O(n) = O(n log n)

### qsort 退化情況

若 qsort 內部使用 quicksort 且無 introsort 保護：
- 已排序或逆序陣列可能觸發最差 O(n²)
- glibc 的 `qsort` 使用 merge sort（穩定）或 introsort（混合策略），有保護機制
- 實務上不太需要擔心，但面試中值得提及
