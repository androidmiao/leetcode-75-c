# 392 Is Subsequence — 記憶體分析

## 1. 字串記憶體佈局

### 例子: s="abc", t="ahbgdc"

**s 字串的記憶體佈局** (假設起點地址 0xA000)

```
地址         內容      ASCII 值
────────────────────────────────
0xA000       'a'       97
0xA001       'b'       98
0xA002       'c'       99
0xA003       '\0'      0 (null terminator)
```

**t 字串的記憶體佈局** (假設起點地址 0xB000)

```
地址         內容      ASCII 值
────────────────────────────────
0xB000       'a'       97
0xB001       'h'       104
0xB002       'b'       98
0xB003       'g'       103
0xB004       'd'       100
0xB005       'c'       99
0xB006       '\0'      0 (null terminator)
```

## 2. 完整資料結構記憶體配置

### 記憶體映射表

| 字串 | 變數 | 地址   | 內容 | ASCII | 用途 |
|------|------|--------|------|-------|------|
| s    | s[0] | 0xA000 | 'a'  | 97    | 第一個待匹配字符 |
| s    | s[1] | 0xA001 | 'b'  | 98    | 第二個待匹配字符 |
| s    | s[2] | 0xA002 | 'c'  | 99    | 第三個待匹配字符 |
| s    | s[3] | 0xA003 | '\0' | 0     | null 終止符 |
| t    | t[0] | 0xB000 | 'a'  | 97    | 第一個掃描字符 |
| t    | t[1] | 0xB001 | 'h'  | 104   | 第二個掃描字符 |
| t    | t[2] | 0xB002 | 'b'  | 98    | 第三個掃描字符 |
| t    | t[3] | 0xB003 | 'g'  | 103   | 第四個掃描字符 |
| t    | t[4] | 0xB004 | 'd'  | 100   | 第五個掃描字符 |
| t    | t[5] | 0xB005 | 'c'  | 99    | 第六個掃描字符 |
| t    | t[6] | 0xB006 | '\0' | 0     | null 終止符 |

### 記憶體視覺化圖

```
堆疊: s 指標         堆疊: t 指標       堆疊: s_idx        堆疊: t_idx
┌──────────┐        ┌──────────┐      ┌──────────┐      ┌──────────┐
│ s        │        │ t        │      │ s_idx    │      │ t_idx    │
│0xA000    │        │0xB000    │      │ 0        │      │ 0        │
└────┬─────┘        └────┬─────┘      └──────────┘      └──────────┘
     │                   │
     │                   │
     ▼                   ▼
堆記憶體 (s):        堆記憶體 (t):

s+0  ┌───┐           t+0  ┌───┐
     │'a'│                │'a'│
0xA000└───┘           0xB000└───┘
s+1  ┌───┐           t+1  ┌───┐
     │'b'│                │'h'│
0xA001└───┘           0xB001└───┘
s+2  ┌───┐           t+2  ┌───┐
     │'c'│                │'b'│
0xA002└───┘           0xB002└───┘
s+3  ┌───┐           t+3  ┌───┐
     │\0 │                │'g'│
0xA003└───┘           0xB003└───┘
                      t+4  ┌───┐
                           │'d'│
                      0xB004└───┘
                      t+5  ┌───┐
                           │'c'│
                      0xB005└───┘
                      t+6  ┌───┐
                           │\0 │
                      0xB006└───┘
```

### 索引與地址對應

- **s[i] 的記憶體地址** = 0xA000 + i
- **t[j] 的記憶體地址** = 0xB000 + j
- **s_idx** 是 s 中的偏移量 (初始值 0，範圍 [0, strlen(s)])
- **t_idx** 是 t 中的偏移量 (初始值 0，範圍 [0, strlen(t)])

## 3. 陣列索引版對應

### 索引公式推導

```c
// 字串指標與索引的對應關係
char *s_ptr = s;          // 初始指向 s[0]
char *t_ptr = t;          // 初始指向 t[0]

// 在本解法中，我們使用索引值 s_idx 和 t_idx
// 對應的字符存取方式:
s[s_idx]   ← 地址為 (s + s_idx) 或 (0xA000 + s_idx)
t[t_idx]   ← 地址為 (t + t_idx) 或 (0xB000 + t_idx)

// 指標移動等價於索引遞增
s_idx++    等價於 s_ptr++ (假設 s_ptr 初始為 s)
t_idx++    等價於 t_ptr++ (假設 t_ptr 初始為 t)
```

### 索引映射表 (s="abc", t="ahbgdc")

| s_idx | 地址   | 字符 | 含義 |
|-------|--------|------|------|
| 0     | 0xA000 | 'a'  | 等待匹配第一個字符 |
| 1     | 0xA001 | 'b'  | 等待匹配第二個字符 |
| 2     | 0xA002 | 'c'  | 等待匹配第三個字符 |
| 3     | 0xA003 | '\0' | 達到終點，返回 true |

| t_idx | 地址   | 字符 | 含義 |
|-------|--------|------|------|
| 0     | 0xB000 | 'a'  | 掃描開始 |
| 1     | 0xB001 | 'h'  | 繼續掃描 |
| 2     | 0xB002 | 'b'  | 繼續掃描 |
| 3     | 0xB003 | 'g'  | 繼續掃描 |
| 4     | 0xB004 | 'd'  | 繼續掃描 |
| 5     | 0xB005 | 'c'  | 繼續掃描 |
| 6     | 0xB006 | '\0' | 掃描結束 |

## 4. 搜尋／操作過程追蹤

### Example 1: s="abc", t="ahbgdc" → **true**

```
初始狀態:
  s_idx = 0 (指向 s[0]='a')
  t_idx = 0 (指向 t[0]='a')
  s_len = 3
  t_len = 6

記憶體狀態:
  s: [0xA000]┌───┬───┬───┬───┐
             │'a'│'b'│'c'│\0│
             └───┴───┴───┴───┘
  t: [0xB000]┌───┬───┬───┬───┬───┬───┬───┐
             │'a'│'h'│'b'│'g'│'d'│'c'│\0│
             └───┴───┴───┴───┴───┴───┴───┘
      s_idx↑
      t_idx↑
```

#### 步驟逐詞詞

| 步驟 | t_idx | t[t_idx] | s_idx | s[s_idx] | 位址讀取 | 比較結果 | s_idx 進位 | 動作 |
|------|-------|----------|-------|----------|---------|----------|-----------|------|
| 1    | 0     | 'a'      | 0     | 'a'      | 0xB000  | ✓ 相符   | 是 → 1    | s_idx++ |
| 2    | 1     | 'h'      | 1     | 'b'      | 0xB001  | ✗ 不符   | 否 → 1    | 無動作  |
| 3    | 2     | 'b'      | 1     | 'b'      | 0xB002  | ✓ 相符   | 是 → 2    | s_idx++ |
| 4    | 3     | 'g'      | 2     | 'c'      | 0xB003  | ✗ 不符   | 否 → 2    | 無動作  |
| 5    | 4     | 'd'      | 2     | 'c'      | 0xB004  | ✗ 不符   | 否 → 2    | 無動作  |
| 6    | 5     | 'c'      | 2     | 'c'      | 0xB005  | ✓ 相符   | 是 → 3    | s_idx++, **s_idx==s_len** |

**結果**: 因為 s_idx (3) == s_len (3)，立即返回 **true**

## 6. 記憶體存取模式比較

### 本解法: 二指標線性掃描 (Sequential Scan)

**記憶體存取特性**

```
存取模式: 單向順序掃描
  t 的存取順序: t[0] → t[1] → t[2] → ... → t[t_len-1]
  地址增長: 0xB000 → 0xB001 → 0xB002 → ...

時間複雜度: O(|t|)
空間複雜度: O(1) (只用兩個索引變數)
```

**快取友善度 (Cache Locality)**

```
優點:
  ✓ 順序記憶體存取 (Sequential Access)
  ✓ CPU 快取效能極佳
  ✓ 預測性記憶體讀取 (Prefetch-friendly)
  ✓ 記憶體頻寬充分利用
  ✓ 分支預測效能好 (for 迴圈)

記憶體存取圖:
  堆記憶體 t: [0xB000] ┌───┬───┬───┬───┬───┬───┐
                      │   │   │   │   │   │   │
                      └───┴───┴───┴───┴───┴───┘
              掃描頭→  ▶───▶───▶───▶───▶───▶
              (連續遞進)

CPU 快取行 (假設 64 字節):
  快取行 1: [0xB000 ~ 0xB03F] 一次載入 64 個字符
  利用率: 極高 (順序存取所有已載入字符)
```

**對比: 雜湊表方法 (HashMap Approach)**

```
存取模式: 隨機存取 + 雜湊計算

實現範例 (假設 s 中所有字符預先建立雜湊表):
  1. 遍歷 s，建立 {字符 → 是否在 s 中} 的雜湊表
  2. 遍歷 t，只檢查字符是否在雜湊表中 (常數時間查詢)

時間複雜度: O(|s| + |t|) (雜湊表建立 + 查詢)
空間複雜度: O(|s|) (儲存雜湊表)

記憶體存取圖:
  雜湊表: [0xC000] ┌────┬────┬────┐
                   │hash│hash│hash│ (非連續記憶體)
                   └────┴────┴────┘
           隨機跳躍 ▶──┐
                    └─▶──┐
                      └─▶

快取效能: 較差
  ✗ 隨機記憶體存取 (Random Access)
  ✗ 快取失誤率 (Cache Miss) 高
  ✗ CPU 預取無法優化
  ✗ 記憶體頻寬利用率低
```

**性能結論**

當 |s| << |t| 時（例如 s="abc", t="ahbgdc"）:
- **本解法更優**: O(1) 空間，順序存取
- 雜湊表方法: O(|s|) 空間，但實際快取友善度更低

當 |s| ≈ |t| 且字符不匹配時:
- 本解法: 必須掃描整個 t
- 雜湊表: 因為 s 中不存在 'x' (Example 2)，可以快速判斷不匹配

## 7. 堆疊記憶體對比

### 本解法: 迭代版 (Iterative)

**函數呼叫堆疊 (Call Stack)**

```c
bool isSubsequence(char* s, char* t) {
    // 堆疊フレーム配置
    // [RBP+16]  → 返回地址
    // [RBP+8]   → t (指標)
    // [RBP]     → s (指標)
    // [RBP-8]   → s_len (size_t, 8 字節)
    // [RBP-16]  → t_len (size_t, 8 字節)
    // [RBP-24]  → s_idx (size_t, 8 字節)
    // [RBP-32]  → t_idx (size_t, 8 字節)
    // [RBP-40]  → 區域變數結束

    // 總堆疊使用: ~40 字節 (含對齐)
}
```

**堆疊狀態圖**

```
高地址 [RBP+16] ┌─────────────────┐
               │  返回地址       │
```

### 記憶體複雜度對比

| 指標 | 迭代版 (本解法) | 遞迴版 |
|------|-----------------|--------|
| **時間複雜度** | O(\|t\|) | O(\|t\|) |
| **堆疊空間** | O(1) | O(\|t\|) (最壞情況) |
| **堆疊幀數** | 1 | \|t\| (每個字符一層) |
| **每幀大小** | ~40 字節 | ~32 字節 |
| **最大堆疊用量** | ~40 字節 | \|t\| × 32 字節 |

**結論**

- **迭代版 (本解法)**: 常數堆疊空間，最適合競賽編程和嵌入式系統
- **遞迴版**: 程式碼簡潔但堆疊用量線性成長，不適合超大輸入

## 附錄: 完整解法代碼

```c
#include <string.h>
#include <stdbool.h>

bool isSubsequence(char* s, char* t) {
    size_t s_len = strlen(s);
    size_t t_len = strlen(t);

    // 邊界案例: 空字串
    if (s_len == 0 && t_len == 0) return true;

    size_t s_idx = 0;

    // 線性掃描 t，嘗試在 t 中找到 s 的所有字符 (保持順序)
    for (size_t t_idx = 0; t_idx < t_len; t_idx++) {
        // 比較當前的 t[t_idx] 與 s[s_idx]
        if (s[s_idx] == t[t_idx]) {
            s_idx++;  // 成功匹配，進入 s 的下一個字符
        }

        // 提前終止: s 的所有字符都已找到
        if (s_idx == s_len) return true;
    }

    // 迴圈結束，s_idx 未達到 s_len (s 中存在 t 中沒有的字符)
    return false;
}
```

**時間複雜度分析**
- 最好情況 (Best Case): O(1)
  - 當 s="" 或 t="" 時立即返回
- 平均情況 (Average Case): O(\|t\|)
  - 需要掃描大部分 t 的字符
- 最壞情況 (Worst Case): O(\|t\|)
  - s 的全部字符在 t 的末尾，必須掃描所有 t

**空間複雜度分析**
- O(1) 常數空間
- 只使用固定數量的變數 (s_idx, t_idx, s_len, t_len)
- 不依賴輸入大小的額外記憶體分配
