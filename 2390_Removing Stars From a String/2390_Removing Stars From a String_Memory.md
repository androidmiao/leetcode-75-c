# LeetCode 2390 - 移除字串中的星號 (Removing Stars From a String) - 記憶體分析

## 1. 字串記憶體佈局

### 代碼概況
```c
char * removeStars(char * s) {
    int w = 0;
    for (int r = 0; s[r] != '\0'; r++) {
        if (s[r] == '*') {
            if (w > 0) w--;
        } else {
            s[w++] = s[r];
        }
    }
    s[w] = '\0';
    return s;
}
```

**演算法原理**：使用雙指標原地編輯，w為寫入指標，r為讀取指標。遇到星號時，若w>0則回退一位（刪除前一字元），否則直接略過。

### 範例 1：`s = "leet**cod*e"`

**字元陣列佈局**：
- 輸入字串：`"leet**cod*e"` （11個字元 + 1個null terminator）
- 記憶體起址：`0xA000`
- 字元大小：1 byte


## 6. 記憶體存取模式分析

### 存取模式特性

#### **順序掃描 + 回溯寫入**

該演算法採用以下存取模式：

```
讀取指標 r：總是向前進行（0 → 1 → 2 → ... → n）
            ↓
            單向掃描，無回溯

寫入指標 w：通常向前，但遇到 '*' 時可回溯
            ↓
            雙向移動

實際行為：
  - 大多數步驟：r 和 w 同時推進
  - 遇到星號：w 回退，r 繼續前進
```

#### **快取友好性分析**

```
┌───────────────────────────────────────────────────────┐
│ 快取局部性（Cache Locality）評估                      │
├───────────────────────────────────────────────────────┤
│ 【優點】                                               │
│ ✓ 線性掃描：讀取指標 r 單向遍歷，預取友好             │
│ ✓ 低記憶體占用：僅 2 個 int 變數，常駐暫存器           │
│ ✓ 原地編輯：無額外堆積分配，無 cache miss              │
│ ✓ 時間局部性：同一字串反覆訪問，易命中快取             │
│ ✓ 空間局部性：w 和 r 都在同一陣列內，相鄰位置          │
│                                                       │
│ 【缺點】                                               │
│ ✗ 寫入點分散：w 位置不連續，可能導致 cache invalidate │
│ ✗ 回溯成本：雖然回溯範圍小，但仍需重新存取舊位址      │
│                                                       │
│ 【整體評價】                                           │
│ 🟢 快取友好度：HIGH（8/10）                           │
│   理由：順序讀取優於隨機存取，足以抵消回溯成本         │
└───────────────────────────────────────────────────────┘
```

### 與其他方法的比較

#### **方法 1（本題解）：原地編輯 + 雙指標**

```
記憶體結構：
  s (char*) ────┐
                ▼
            ┌─────────────────────┐
            │ l e e t * * c o d * e \0 │
            └─────────────────────┘
               ▲                   ▲
               w                   r

優點：
  • 時間複雜度：O(n)
  • 空間複雜度：O(1)（僅使用 2 個 int）
  • 無額外分配，堆積壓力小

缺點：
  • 需理解指標邏輯，實現稍複雜
```

#### **方法 2：使用棧**

```
記憶體結構：
  s (char*)     ────┐
  stack (char[])    │

            ┌──────────┐
            │ l e e t * * c o d * e \0 │
            └──────────┘

   ┌───────────────────────────────┐
   │ 棧記憶體（Stack）              │
   ├───────────────────────────────┤
   │ char stack[size]              │
   │  ─┬─ 'l'                       │
   │   ├─ 'e'                       │
   │   ├─ 'c'                       │
   │   ├─ 'o'                       │
   │   └─ 'e'                       │
   │     (top=4)                    │
   └───────────────────────────────┘

優點：
  • 邏輯直觀，利用棧特性（LIFO）
  • 星號直接 pop，易於理解

缺點：
  • 需額外棧空間：O(n)
  • 需兩次複製（棧→原陣列）
  • 快取局部性較差
```

#### **方法 3：使用第二陣列**

```
記憶體結構：
  s (char*)        ────┐
  result (char[])      │

            ┌──────────────────┐
            │ l e e t * * c o d * e \0 │
            └──────────────────┘

            ┌──────────────────┐
            │ l e c o e \0     │  ◄── 新陣列
            └──────────────────┘

優點：
  • 邏輯清晰，避免複雜指標操作

缺點：
  • 空間複雜度：O(n)（需新陣列）
  • 堆積分配成本
  • 最後需複製回原陣列（或返回新陣列）
  • 快取表現：訪問兩個不同陣列，cache miss 增加
```

### 性能比較表

| 方法 | 時間複雜度 | 空間複雜度 | 快取友好度 | 實現難度 |
|------|-----------|-----------|-----------|---------|
| 原地雙指標 | O(n) | O(1) | 🟢 HIGH | 中等 |
| 棧 | O(n) | O(n) | 🟡 MEDIUM | 容易 |
| 第二陣列 | O(n) | O(n) | 🟠 LOW | 容易 |

### 棧幀詳細解析（X86-64 調用約定）

#### **System V AMD64 ABI 調用約定**

```
進入 removeStars() 時：

┌─────────────────────────────────┐
│ 暫存器狀態                        │
├─────────────────────────────────┤
│ RDI = s (char*)                 │ ◄── 第1參數
│ RSI = ?                         │
│ RDX = ?                         │
│ ...                             │
│ RSP = 棧指標（指向返回位址）      │
│ RBP = 基指標（函數的底部）        │
└─────────────────────────────────┘

函數序言（Function Prologue）：
  push rbp              ▶ 保存舊 RBP
  mov rbp, rsp          ▶ 建立新棧幀
  sub rsp, 8            ▶ 配置局部變數空間
                          (r, w 共 8 bytes)

實行中：
  [RBP-4]  ◀─ int r = 0  (寄存器 RAX 或記憶體)
  [RBP-8]  ◀─ int w = 0  (寄存器 RCX 或記憶體)
  [RBP+8]  ◀─ char* s    (參數 RDI)

函數尾聲（Function Epilogue）：
  mov rax, ?            ▶ 準備返回值（字串位址）
  mov rsp, rbp          ▶ 復原棧指標
  pop rbp               ▶ 復原舊 RBP
  ret                   ▶ 返回
```
